用户是该 SRPG 独立游戏项目的创意总监，直接指派完整任务（包含上下文文件路径、设计约束、输出范围、硬约束清单）。

**协作偏好：**
- 给出完整任务说明后期望直接产出，不需要逐步确认每个小决策
- 对 GDD 类任务：提供 skeleton + 填写指定节数即可，其余节标 TODO
- 硬约束明确列出时严格遵守（行数限制、不改 src/、不 commit 等）
- 完成后只需简短回报（文件路径 + 行数），不需要总结内容

**Why:** 用户在 CLAUDE.md 中明确"极简输出——直接给结果，不重复、不总结、不废话"。

**How to apply:** 收到带硬约束的设计任务时，优先读取所有指定上下文文件，
然后一次性产出，最后一行只报"已写入 [路径] (Nx 行)"。
不在正文末尾重复设计决策，不生成额外 README/分析文件。
