# Quick Specs

> **Purpose**: Small, bounded design decisions that do not justify a full GDD rewrite or new ADR.
> **Created**: 2026-05-02

Use this directory for tuning or interaction decisions that are:

- narrower than a full system design;
- larger than a one-line changelog note;
- expected to affect tests, balance, UX, or content authoring.

Each quick spec should include:

| Field | Required |
|---|---|
| Decision | What will change |
| Context | Why the decision exists |
| Scope | What is included and excluded |
| Acceptance | How the decision is verified |
| Owner | System or sprint owner |
