## 1. Overview

本文档定义 SRPG 所有 HUD 模式的统一设计规范，包括组件复用规则、视觉常量、状态过渡和响应式适配。各屏幕的详细 UX spec 见对应的 per-screen 文档。

---

## 2. Rendering Modes

游戏使用双轨 HUD 渲染策略（art-bible 7.1）:

| 模式 | 适用场景 | 实现方式 |
|------|----------|----------|
| **固有型 HUD** | 战斗状态标牌、回合立牌、范围投影 | 3D 场景内节点，参与深度排序 |
| **屏幕空间 HUD** | 面板、技能栏、菜单、飘字 | CanvasLayer 叠加 |

**切换规则**: 战斗中同时使用两种模式；菜单/暂停仅使用屏幕空间。

## 3. Game States & HUD Configurations

### 3.1 State Map

```
启动 → 主菜单 → 探索 → 战斗 → 结算 → 探索
                ↓              ↓
             暂停菜单       暂停菜单
                ↓              ↓
             设置/存档      设置(禁存档)
                ↓
             标题画面
```

### 3.2 HUD Per State

| 游戏状态 | 固有型 HUD | 屏幕空间 HUD | 交互模式 |
|----------|-----------|-------------|----------|
| 主菜单 | 无 | 标题+菜单选项 | 键盘/手柄导航 |
| 探索 | 角色标牌 | 资源条+小地图 | 鼠标点击+键盘移动 |
| 战斗（玩家回合） | 标牌+立牌+范围 | 行动面板+技能栏+控制栏 | 键盘快捷键+鼠标 |
| 战斗（敌方回合） | 标牌+立牌 | 敌方行动提示 | 仅观察 |
| 战斗（自动） | 标牌+立牌 | 技能栏(只读)+控制栏 | 随时可中断 |
| 暂停 | 无 | 暂停面板 | 键盘/手柄导航 |
| 结算 | 无 | 结算面板 | 键盘/手柄导航 |
| 过场演出 | 无 | 对话框 | 点击/Enter 继续 |

## 4. Shared Components

以下组件在多个游戏状态中复用，定义统一规格。

### 4.1 角色状态标牌 (Status Plaque)

| 规格项 | 值 | 使用场景 |
|--------|-----|----------|
| 位置 | 角色上方 1.5 单位 | 探索、战斗 |
| 宽度 | 64px | — |
| 背景 | 宣纸 #F5F2EB + 1px 焦墨边框 | — |
| HP 条 | 56×6px，橙红 #E85D3B | — |
| 职业图标 | 16×16px | 战斗时显示 |
| 元素图标 | 12×12px | 战斗时显示 |
| buff/debuff | 12×12px，最多 6 个 | 战斗时显示 |

**信息过滤**:

| 条件 | 显示级别 |
|------|----------|
| ≤6 同屏单位 | 完整（HP+职业+元素+buff） |
| >6 同屏单位 | 核心（HP+职业） |
| >10 同屏单位 | 最小（仅 HP 条） |
| 选中单位 | 强制完整 |
| HP<25% | 强制完整+闪烁 |

### 4.2 资源条 (Resource Bar)

| 规格项 | 值 |
|--------|-----|
| 位置 | 屏幕左上角 |
| 内容 | 金币数值 + 关卡/区域名 |
| 字体 | 等宽印刷体 14px |
| 颜色 | 焦墨文字，金铜色数值 |

**可见性**: 主菜单隐藏，其余状态可见。

### 4.3 确认对话框 (Confirm Dialog)

| 规格项 | 值 |
|--------|-----|
| 位置 | 屏幕居中 |
| 尺寸 | 400×200px (1080p) |
| 背景 | 宣纸 + 2px 浓墨边框 |
| 危险操作 | 橙红边框 #E85D3B + 警告图标 |

**使用场景**: 覆盖存档、退出游戏、返回标题、放弃战斗。

### 4.4 存档槽卡片 (Save Slot Card)

| 规格项 | 值 |
|--------|-----|
| 尺寸 | 280×60px |
| 空状态 | 虚线边框 + "空" 文字 |
| 有数据 | 宣纸底 + 角色名/Lv/章节/时间 |
| 焦点 | 金铜色边框 2px |

## 5. Visual Constants

所有 HUD 元素共享的视觉常量，源自 art-bible Chapter 4 & 7。

### 5.1 色彩

| Token | 色值 | 用途 |
|-------|------|------|
| `--ink-deep` | #1A1A1A | 主文字、描边 |
| `--ink-medium` | #3D3D3D | 边框、分隔线 |
| `--ink-light` | #8A8A8A | 次文字、提示 |
| `--ink-faded` | #BEBEBE | 禁用状态 |
| `--paper` | #F5F2EB | 面板背景 |
| `--gold` | #B8860B | 焦点边框、选中强调 |
| `--gold-light` | #D4A843 | 悬停文字 |
| `--danger` | #E85D3B | HP 条、危险操作、暴击 |
| `--heal` | #3D8B5F | 治疗数值 |
| `--range-move` | #2D4A4A | 移动范围 |
| `--range-attack` | #5C3D2E | 攻击范围 |

### 5.2 字体

| Token | 字体 | 字号(1080p) | 用途 |
|-------|------|------------|------|
| `--font-title` | 仿古印刷体 | 48-64px | 游戏标题 |
| `--font-heading` | 仿古印刷体 | 24-32px | 面板标题 |
| `--font-body` | 楷书 | 14-18px | 正文、选项 |
| `--font-data` | 等宽印刷体 | 12-16px | 数值、时间 |

### 5.3 动画

| Token | 时长 | 曲线 | 用途 |
|-------|------|------|------|
| `--ease-in` | 0.2s | ease-in | 元素出现 |
| `--ease-out` | 0.3s | ease-out | 元素消失 |
| `--ease-in-out` | 0.5s | ease-in-out | 色彩/大小变化 |
| `--scroll-open` | 0.3s | ease-out | 卷轴展开 |
| `--scroll-close` | 0.2s | ease-in | 卷轴收起 |
| `--pulse` | 0.5s 循环 | sine | 范围投影脉动 |

### 5.4 间距

| Token | 值(1080p) | 用途 |
|-------|----------|------|
| `--gap-xs` | 4px | 图标间距 |
| `--gap-sm` | 8px | 同组元素 |
| `--gap-md` | 16px | 不同组 |
| `--gap-lg` | 24px | 面板内边距 |
| `--panel-radius` | 0px | 无圆角（宣纸风格） |

## 6. State Transitions

### 6.1 Transition Table

| From → To | 时长 | 序列 |
|-----------|------|------|
| 主菜单 → 探索 | 1.0s | 卷轴收起 0.3s → 淡出 0.5s → 场景加载 |
| 探索 → 战斗 | 0.8s | 叙事 UI 淡出 0.3s → 色调赭墨 0.3s → HUD 凝聚 0.2s |
| 战斗 → 探索 | 0.8s | HUD 淡出 0.2s → 色调青墨 0.3s → 探索 UI 展开 0.3s |
| 战斗 → 结算 | 0.5s | HUD 淡出 0.2s → 结算面板展开 0.3s |
| 任意 → 暂停 | 0.3s | 场景 blur + 遮罩 + 面板展开 |
| 暂停 → 任意 | 0.3s | 面板收起 + 遮罩消散 |

### 6.2 Transition Rules

- 过渡期间不接受输入（防误触）
- 粒子效果在过渡期间保持连续性
- 音频按交叉淡入淡出处理

## 7. Responsive Rules

### 7.1 Resolution Scaling

| 分辨率 | 缩放系数 | 标题字号 | 正文字号 | 数值字号 |
|--------|----------|----------|----------|----------|
| 1080p | 1.0x | 48px | 14px | 12px |
| 1440p | 1.5x | 32px | 18px | 16px |
| 4K | 2.0x | 48px | 24px | 20px |

> 4K 与 1080p 相同物理字号，由缩放系数自动放大。

### 7.2 Aspect Ratio Handling

| 比例 | 策略 |
|------|------|
| 16:9 | 默认布局，无调整 |
| 16:10 | 面板垂直空间增加 10%，内容不裁切 |
| 21:9 | 战场可见范围加宽，面板位置不变 |
| 4:3 | 面板水平压缩 10%，字号不变 |

### 7.3 Screen-Safe Area

所有屏幕空间 HUD 元素距离屏幕边缘 ≥ 32px（1080p），避免被过扫描裁切。
