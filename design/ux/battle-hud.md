## 1. Screen Overview

Battle HUD 是 SRPG 核心战斗界面的唯一交互层。采用 **固有型 HUD**（Intrinsic HUD）设计——UI 元素根植于 3D 场景空间，参与透视和深度排序，而非悬浮于屏幕上方的独立图层。

仅暂停菜单、技能详情、道具详情使用屏幕空间 UI。

**玩家目标**: 同时感知多系统状态（HP/MP、回合顺序、地形、buff），并在行动窗口内做出移动/攻击/技能/待机决策。

## 2. Screen Layout

```
┌─────────────────────────────────────────────────────────────────┐
│                                                                 │
│  [资源条]                    3D 战场场景                    [回] │
│  金币 / 关卡名                                               │
│                                                          ┌────┐│
│                                                          │ ▼1 ││
│    ┌──────────┐                                          │主角││
│    │ [头像]   │            ┌──────┐                      │ ▼2 ││
│    │ HP ████░ │           │[移动]│                      │敌A ││
│    │ MP ███░░ │           │[攻击]│                      │ ▼3 ││
│    │ 职业 剑士│           │[技能]│                      │敌B ││
│    │ [火][土] │           │[道具]│                      │ ▼4 ││
│    └──────────┘           │[待机]│                      │友C ││
│   当前行动单位面板          └──────┘                      └────┘│
│                            行动选项面板               回合顺序立牌│
│                                                                 │
│  ┌─────────────────────────────────────────────────────┐        │
│  │  [技能1 8MP] [技能2 12MP] [技能3 禁用] [技能4 5MP]  │ [自动]│
│  └─────────────────────────────────────────────────────┘ [1x2x] │
│                      技能栏                            控制栏    │
└─────────────────────────────────────────────────────────────────┘

场景内元素（非屏幕空间）：
  - 角色头顶：卷轴式状态标牌（HP/职业/元素/buff）
  - 地面投影：移动范围（青墨30%）、攻击范围（赭墨40%脉动）
  - 场景边缘：回合顺序立牌（木质底座+宣纸面）
```

## 3. Component Inventory

### 3.1 当前行动单位面板 (Current Unit Panel)

| 属性 | 值 |
|------|-----|
| 位置 | 屏幕左侧，垂直居中 |
| 渲染 | 屏幕空间叠加 |
| 尺寸 | 200×280px (1080p) |

**子组件**:

| 组件 | 规格 | 数据来源 |
|------|------|----------|
| 头像 | 64×64px，像素化+墨线描边 | 角色管理 |
| 名称 | 楷书 16px，焦墨 #1A1A1A | 角色管理 |
| HP 条 | 160×8px，留白底+橙红 #E85D3B 填充+浓墨描边 | 属性系统 |
| MP 条 | 160×8px，留白底+青墨 #2D4A4A 填充 | 属性系统 |
| HP 数值 | 等宽印刷体 14px，金铜色 #B8860B | 属性系统 |
| MP 数值 | 等宽印刷体 14px，金铜色 #B8860B | 属性系统 |
| 职业标签 | 仿古印刷体 12px + 16×16 职业图标 | 职业系统 |
| 元素属性 | 2×12×12px 五行色图标 | 战术机制 |
| buff/debuff | 最多 6 个 12×12px 图标 | 战斗系统 |

**状态变化**:

| 状态 | 视觉表现 |
|------|----------|
| 玩家回合（活跃） | 金铜色边框 #B8860B，面板完全不透明 |
| 玩家回合（非活跃） | 淡墨边框，面板 70% 透明度 |
| 敌方回合 | 隐藏面板，仅显示选中敌人信息 |
| HP<25% | HP 条闪烁，面板边框变橙红 #E85D3B |
| MP 不足技能消耗 | 对应技能栏位灰化 |

### 3.2 行动选项面板 (Action Panel)

| 属性 | 值 |
|------|-----|
| 位置 | 屏幕左下方 |
| 渲染 | 屏幕空间叠加 |
| 尺寸 | 120×200px |

**按钮列表**:

| 按钮 | 图标 | 条件 | 触发 |
|------|------|------|------|
| 移动 | 脚印 | 移动力 > 0 | 显示地面移动范围 |
| 攻击 | 剑 | 有可攻击目标 | 显示地面攻击范围 |
| 技能 | 星 | 有可用技能 | 展开技能栏 |
| 道具 | 袋 | 背包非空 | 展开道具列表 |
| 待机 | 钟 | 始终可用 | 结束本回合行动 |

**交互规则**:
- 移动消耗移动力后，移动按钮灰化
- 攻击/技能消耗 MP 后，MP 不足的选项灰化显示 "MP不足"
- 道具耗尽时灰化
- 键盘: 1-5 快捷键对应 5 个按钮

### 3.3 技能栏 (Skill Bar)

| 属性 | 值 |
|------|-----|
| 位置 | 屏幕底部居中 |
| 渲染 | 屏幕空间叠加 |
| 尺寸 | 4×100×48px（4 个槽位） |

**槽位布局**:

```
┌─────────────┬─────────────┬─────────────┬─────────────┐
│ [图标]      │ [图标]      │ [图标]      │ [图标]      │
│ 技能名      │ 技能名      │ 技能名      │ 技能名      │
│ 8 MP   Q   │ 12 MP  W   │ 锁定   E   │ 5 MP   R   │
└─────────────┴─────────────┴─────────────┴─────────────┘
```

| 组件 | 规格 |
|------|------|
| 技能图标 | 32×32px，像素化+墨线描边 |
| 技能名 | 楷书 12px，焦墨 |
| MP 消耗 | 等宽 12px，金铜色 |
| 快捷键 | 仿古印刷体 10px，淡墨 |
| 冷却覆盖 | 半透明浓墨遮罩 + 回合数 |

**状态**:

| 状态 | 视觉 |
|------|------|
| 可用 | 宣纸底色 #F5F2EB，1px 浓墨边框 |
| MP 不足 | 灰墨覆盖 #BEBEBE，文字变淡 |
| 冷却中 | 浓墨覆盖 70% 透明度 + 剩余回合数 |
| 选中 | 金铜色边框 #B8860B，轻微放大 105% |

### 3.4 回合顺序立牌 (Turn Order Plaques)

| 属性 | 值 |
|------|-----|
| 位置 | 屏幕右侧，场景边缘 3D 立牌 |
| 渲染 | 固有型 HUD（场景内嵌） |
| 单个尺寸 | 48×48×80px（3D 木质底座+宣纸面） |
| 最大显示 | 8 个单位 |

**立牌结构**:

```
  [1] ← 当前行动指示（金铜色箭头）
  ▼
┌──────┐
│ 主角  │ ← 金铜色边框 + 放大 120%
└──────┘
┌──────┐
│ 敌A   │ ← 淡墨边框，正常尺寸
└──────┘
```

**状态变化**:

| 状态 | 视觉 |
|------|------|
| 当前行动 | 顶部放大 + 金铜色边框 #B8860B + 脉动动画 |
| 待命 | 正常尺寸，淡墨边框 |
| 已行动 | 浓墨覆盖 50% 透明度 + "完" 标记 |
| 死亡 | 移除立牌，0.3s 消散动画 |

### 3.5 角色状态标牌 (Character Status Plaques)

| 属性 | 值 |
|------|-----|
| 位置 | 角色正上方 1.5 单位（3D 场景内） |
| 渲染 | 固有型 HUD，参与深度排序 |
| 尺寸 | 宽 64px，高度可变 |

```
┌─────────────────────┐
│  [职业图标] 角色名    │
├─────────────────────┤
│  ████████░░ HP 85   │
├─────────────────────┤
│  [火] [土]  ⚔强化   │
└─────────────────────┘
```

| 组件 | 规格 |
|------|------|
| 标牌整体 | 宣纸质感底 #F5F2EB，1px 焦墨边框 |
| 职业图标 | 16×16px |
| HP 条 | 宽 56px × 高 6px，橙红填充 |
| HP 数值 | 等宽 12px |
| 元素图标 | 12×12px 五行色底+对应形状 |
| buff/debuff | 12×12px，最多 6 个 |

**信息过滤规则**（art-bible 7.3.2）:

| 条件 | 显示内容 |
|------|----------|
| 同屏 ≤6 | 完整状态（HP+职业+元素+buff） |
| 同屏 >6 | 仅核心状态（HP+职业） |
| 同屏 >10 | 仅 HP 条 |
| 选中单位 | 强制显示完整状态 |
| HP<25% | 强制显示完整状态 + 闪烁警示 |

### 3.6 范围指示 (Range Indicators)

| 属性 | 值 |
|------|-----|
| 位置 | 地面投影 |
| 渲染 | 固有型 HUD（3D 地面 overlay） |

| 范围类型 | 颜色 | 透明度 | 动画 |
|----------|------|--------|------|
| 移动范围 | 青墨 #2D4A4A | 30% | 无脉动 |
| 攻击范围 | 赭墨 #5C3D2E | 40% | 缓慢脉动 0.5s |
| 技能范围 | 按技能属性色 | 35-50% | 按技能设计 |
| 危险区域 | 橙红 #E85D3D | 50% | 快速闪烁 0.25s |

### 3.7 自动战斗/加速控制 (Auto/Speed Controls)

| 属性 | 值 |
|------|-----|
| 位置 | 屏幕右下角 |
| 渲染 | 屏幕空间叠加 |
| 尺寸 | 160×60px |

```
┌──────────────────┐
│ [自动: OFF] [1x] │
│          [2x][3x]│
└──────────────────┘
```

| 按钮 | 行为 |
|------|------|
| 自动开关 | 切换自动战斗 ON/OFF |
| 1x | 正常速度，AI 决策延迟 0.5-1.5s |
| 2x | 2 倍速，AI 决策延迟 0.2-0.5s |
| 3x | 3 倍速，AI 决策延迟 0s |

**状态联动**:
- 自动 ON + 手动干预 → 自动暂停，显示提示 "手动干预已暂停自动战斗"
- 加速模式下伤害飘字仍显示，动画按倍率加速

| 属性 | 值 |
|------|-----|
| 位置 | 屏幕左上角 |
| 渲染 | 屏幕空间叠加 |

| 显示项 | 格式 | 数据来源 |
|--------|------|----------|
| 金币 | 图标 + 数值 | 资源经济 |
| 关卡名 | 楷书 14px | 战斗系统 |
| 暂停按钮 | 24×24px 图标 | 系统 |

## 4. Floating Feedback Elements

### 4.1 伤害/治疗飘字

| 类型 | 颜色 | 动画 |
|------|------|------|
| 伤害 | 焦墨 #1A1A1A → 渐隐 | 从受伤位置上飘 1.5s |
| 暴击 | 橙红 #E85D3B + 放大 130% | 同上 + 屏幕微震 |
| 治疗 | 青绿 #3D8B5F | 同上 |
| MP 消耗 | 淡墨 #8A8A8A | 同上 |

### 4.2 buff/debuff 图标

- 出现时: 0.2s 从透明→不透明 + 缩放弹性
- 消失时: 0.3s 淡出
- 最大同屏: 每角色 6 个图标，超出时最旧的被替换

## 5. Screen States & Transitions

### 5.1 战斗状态机

```
战斗开始 → 玩家回合 → [选择行动] → 执行动画 → 下一单位
                         ↑                         │
                         └── 手动干预 ← 自动战斗 ←─┘
                                                    │
                                              战斗结束 → 结算界面
```

### 5.2 关键过渡

| 过渡 | 时长 | 视觉 |
|------|------|------|
| 进入战斗 | 1.0s | 场景色调→赭墨 + HUD 从场景"凝聚"显现 |
| 玩家回合开始 | 0.3s | 当前面板金铜色边框亮起 + 确认音 |
| 行动执行 | 按技能 | 范围投影消散 + 伤害飘字 + 音效 |
| 敌方回合 | 0.5-1.5s | 玩家面板淡出 + 敌方面板显示 |
| 退出战斗 | 0.8s | HUD 淡出 + 场景色调→青墨 + 结算界面展开 |

### 5.3 叙事↔战斗过渡（art-bible 7.4.5）

| 方向 | 时长 | 序列 |
|------|------|------|
| 叙事→战斗 | 0.8s | 叙事 UI 淡出 0.3s → 场景变赭墨 0.3s → HUD 凝聚显现 0.2s |
| 战斗→叙事 | 0.8s | HUD 淡出 0.2s → 场景变青墨 0.3s → 叙事 UI 展开 0.3s |

## 6. Input Mapping

### 6.1 键盘/鼠标

| 操作 | 输入 | 上下文 |
|------|------|--------|
| 移动光标 | 鼠标移动 / 方向键 | 战场 |
| 选中单位/格子 | 左键 / Enter | 通用 |
| 取消 | 右键 / Esc | 通用 |
| 行动: 移动 | 1 | 行动选项面板 |
| 行动: 攻击 | 2 | 行动选项面板 |
| 行动: 技能 | 3 | 行动选项面板 |
| 行动: 道具 | 4 | 行动选项面板 |
| 行动: 待机 | 5 | 行动选项面板 |
| 技能快捷键 | Q/W/E/R | 技能栏 |
| 自动战斗 | Tab | 通用 |
| 加速切换 | Space | 通用 |
| 暂停 | Esc（无上下文时） | 通用 |

### 6.2 游戏手柄

| 操作 | 输入 |
|------|------|
| 移动光标 | 左摇杆 / 方向键 |
| 选中 | A |
| 取消 | B |
| 切换行动选项 | LB/RB |
| 技能栏 | X（展开）+ 方向键选择 + A 确认 |

## 7. Accessibility

遵循 `design/ux/accessibility-requirements.md` Basic 级别:

| 要求 | 实现方式 |
|------|----------|
| 焦点可见 | 金铜色边框 #B8860B，2px |
| 键盘全导航 | 所有操作可通过键盘完成 |
| 色彩备份 | 所有颜色信号配合形状（art-bible 7.6.2） |
| 文字缩放 | 50%-200% 范围，最小 14px (1080p) |
| HP 低值警示 | HP<25% 时边框闪烁 + 音效，不仅依赖颜色 |

## 8. Performance Notes

| 预算项 | 目标 | 来源 |
|--------|------|------|
| 战斗场景 Draw Calls | < 150 | art-bible 8.4 |
| 含 UI 场景 Draw Calls | < 200 | art-bible 8.4 |
| HUD 元素帧预算 | < 2ms | 16.6ms 总帧预算的 12% |
| 飘字同时活跃数 | ≤ 20 | 避免文字重叠和性能问题 |
