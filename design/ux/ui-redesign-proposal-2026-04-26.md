# UI 重设计提案

> 版本: v0.1 | 日期: 2026-04-26 | 状态: 提案草稿
> 作者: ux-designer
> 依据: playtest-2026-04-25-fun-validation-rerun.md、playtest-2026-04-24-visual-signoff.md
> 对齐: interaction-patterns.md v0.1、accessibility-requirements.md v0.1

---

## 背景与问题定性

两轮测试均定性为 "PASS WITH NOTES"，核心问题归纳为三类：
1. **视觉简陋** — 无美术方向感，玩家感知不到武侠 HD-2D 气质
2. **交互突兀** — 按钮状态变化缺乏过渡，操作反馈薄弱
3. **信息层级缺失** — 屏幕上所有元素视觉权重相近，玩家无法快速定位关键信息

本提案不改动任何代码或场景文件，只提供可移交给 ui-programmer 实施的交互规格。

## 屏 1：主菜单（Main Menu）

### 当前问题清单

1. **无视觉引导层次** — 标题、菜单选项、版本号三者字号/颜色差异不足，玩家视线无自然落点
2. **首次启动与再次启动体验无差异** — 有存档时"继续游戏"未明显置顶或突出，玩家需逐行扫描
3. **菜单项悬停反馈缺失** — 当前实现无焦点高亮，键盘/手柄导航时玩家不知自己"在哪"
4. **过渡动画缺席** — 进入游戏直接切换场景，无卷轴收起等武侠意象过渡，体验生硬
5. **底部按键提示信息过于简陋** — 仅有版本号，玩家不知道手柄如何操作

### 新交互目标

- 玩家应能：在 3 秒内识别出标题并移动到"继续/开始"
- 玩家应能：通过键盘或手柄完成全部操作，无需鼠标
- 玩家不必再：在进入游戏前看到任何令人困惑的无焦点状态
- 玩家不必再：通过试错发现哪个键可以确认

### 键鼠 + 手柄输入映射

| 操作 | 键鼠 | 手柄 |
|------|------|------|
| 上下导航 | ↑↓ / 鼠标悬停 | 十字键 / 左摇杆 |
| 确认 | Enter / 左键点击 | A |
| 退出确认取消 | Esc / 右键 | B |
| 跳过开场动画 | 任意键 | 任意键 |

### 视觉层级（眼球前 3 落点）

1. **游戏标题书法字** — 尺寸最大（64px），对比度最高，位于屏幕上 1/3
2. **当前焦点菜单项** — 金铜色箭头 + 边框，与其他选项有明显色差
3. **存档摘要行**（继续游戏时）— 等宽小字显示角色名/进度，辅助决策

### 与 interaction-patterns.md 的对齐

- 采用 §2.1 菜单导航模式：Tab 跳转、方向键选择、Enter 确认、Esc 取消
- 退出采用 §4.1 二次确认对话框
- 焦点边框使用 §6.1 定义的 `#4A9EFF` 升级为武侠主题 `--gold #B8860B`（视觉常量保持一致）

## 屏 2：战斗 HUD（Battle HUD）

### 当前问题清单

1. **回合顺序条信息量不足** — 仅显示单位名，无 HP 状态，玩家无法预判威胁
2. **行动选项面板与技能栏职能重叠** — "技能"按钮展开技能栏，但技能栏已常驻底部，造成双重入口困惑
3. **Auto/手动切换按钮太小且无状态反馈** — 玩家不知道 Auto 当前是开还是关
4. **地形提示无触发引导** — 玩家不知道悬停格子可以看地形信息
5. **当前行动单位面板在敌方回合消失过于突然** — 缺乏淡出过渡，屏幕布局骤变产生跳跃感

### 新交互目标

- 玩家应能：一眼判断回合顺序中每个单位的大致血量状态
- 玩家应能：通过键盘数字键 1-5 完成全部行动，无需鼠标点击面板
- 玩家不必再：通过"技能"按钮二次展开才能看到技能——技能栏应始终可见
- 玩家不必再：猜测 Auto 是否已激活

| 操作 | 键鼠 | 手柄 |

### 视觉层级（眼球前 3 落点）

1. **当前行动单位面板** — 左侧固定，金铜色边框激活，与场景拉开层次
2. **回合顺序立牌（顶部单位）** — 右侧最顶位 120% 放大 + 金铜色脉动
3. **地面范围投影** — 移动/攻击范围覆盖面积大，天然吸引视线到决策区域

### 与 interaction-patterns.md 的对齐

- §2.2 战斗目标选择：青绿高亮悬停、白色描边选中、深红半透明攻击范围
- §5.1/5.2 反馈模式：操作成功绿色粒子 0.5s；操作失败红色闪烁 + 100ms 震动
- Auto 状态变化触发 §5.1 成功反馈（开启）/ §5.2 失败反馈音（关闭）

## 屏 3：战斗结算（Settlement）

### 当前问题清单

1. **结算屏不存在** — 当前版本战斗结束后直接返回菜单，无任何奖励展示或进度感
2. **无 EXP/装备获得展示** — 玩家无法感知战斗带来的成长，缺乏正反馈回路
3. **"回营"入口不清晰** — 玩家不知道下一步去哪，容易误操作退出主菜单

### 新交互目标

- 玩家应能：在 10 秒内完成结算确认并进入下一步，不需要阅读大量文字
- 玩家应能：清晰看到本场战斗获得了什么（EXP、装备、金币）
- 玩家不必再：手动返回菜单寻找"回营"入口——结算屏直接提供推荐下一步

### 视觉层级（眼球前 3 落点）

1. **战斗胜利标题** — 卷轴展开动画 + 最大字号，第一眼确认胜负
2. **升级/成长区域** — 进度条动画天然吸引视线，传递成长感
3. **[继续] 按钮** — 金铜色高亮焦点，引导下一步操作

### 与 interaction-patterns.md 的对齐

- §5.1 操作成功反馈：升级触发绿色粒子爆发 + 上升音阶
- §4.1 确认模式：[返回大营] 触发二次确认（防误触中断连续作战）
- 布局采用 §3.1 属性面板的宣纸底色 + 浓墨边框视觉语言

## 屏 4：管理屏（Camp / Party / Equipment）

### 当前问题清单

1. **管理入口只有"推荐动作"** — 当前仅显示系统建议，玩家无法手动编队或自主决策
2. **装备切换无比较界面** — 玩家无法在切换装备前对比新旧属性差异
3. **技能加点入口不存在** — 无法为角色分配技能点，成长感缺失
4. **角色列表无快速状态一览** — 进入管理屏前玩家不知道谁需要治疗/谁可以升级

### 新交互目标

- 玩家应能：在 30 秒内完成一次编队调整（换人 + 换装备）
- 玩家应能：对比装备属性差异后再决定是否切换
- 玩家不必再：只能接受系统推荐，无法自主决定出战阵容

### 键鼠 + 手柄输入映射

| 确认编队并退出 | Enter（底部按钮） | A |

### 视觉层级（眼球前 3 落点）

1. **出战阵容槽** — 左侧最大区域，出战/空位对比度高，玩家最先关注战斗准备状态
2. **角色详情面板** — 右侧 HP/MP 条 + 大头像，当前角色状态一目了然
3. **装备差值提示** — 绿色▲/红色▼差值行，属性对比结论化，减少认知负担

### 与 interaction-patterns.md 的对齐

- §7.1 拖放模式：出战槽支持拖拽（鼠标）；手柄用方向键 + A 确认替代拖拽
- §3.1 属性面板：复用现有属性展示 wireframe（力量/敏捷/体力/智力/魅力 + 进度条）
- §4.1 二次确认：切换装备、移出出战角色不需要二次确认（低风险操作，遵循渐进式披露）
- §4.2 危险操作确认：放弃全部编队更改时（Esc 有未保存变更）触发警告对话框

## 实施分期建议

### Alpha 优先（当前 Sprint 内）

| 屏幕 | 优先任务 | 理由 |
|------|----------|------|
| 战斗 HUD | Auto 状态按钮可读性修复 | 测试已报告直接阻碍操作理解 |
| 战斗 HUD | 回合顺序立牌加迷你 HP 条 | 战术决策核心信息缺失 |
| 主菜单 | 焦点状态高亮（金铜色边框） | 无焦点=键盘/手柄不可用，违反 accessibility §2.1 |
| 主菜单 | 底部按键提示补全（手柄操作说明） | 玩家第一个看到的屏幕，教育成本最低 |

### Beta 目标（下一个大里程碑前）

| 屏幕 | 任务 | 理由 |
|------|------|------|
| 战斗结算 | 新建结算屏全量实施 | 当前无结算屏，正反馈回路断裂 |
| 管理屏 | 手动编队 + 装备切换对比 | 核心深度玩法，Pre-Production 阶段已明确为欠债 |
| 战斗 HUD | 地形提示触发引导（T 键提示文字） | 教学成本低，信息密度提升明显 |
| 主菜单 | 入场/离场卷轴动画 | 武侠视觉基调，非阻塞性润色 |
