## 摘要

本文档定义 MVP 玩家从点击"开始修炼"到完成首次离线回归的完整引导流程，覆盖约 0:00 至首次退出+回归的完整体验。核心方法论对标 `player-journey-map.md` §5 七大 FTUE 原则：**先行为后界面**、**数字跳动是第一语言**、**每次操作有即时反馈**、**"即将解锁"制造期望**、**离线回归是 FTUE 下半场**、**境界突破有仪式感**、**内容边界是蓄力阶段**。

引导哲学对标 **A Dark Room / Candy Box 2**：不出现教程弹窗、不出现箭头指示、不出现"请点击这里"的系统提示。玩家通过"做"而非"读"来学习——灵气数字跳动教会"修炼在进行"、战斗日志滚动教会"战斗在自动运行"、灵石从 0 跳到 1 教会"战斗有收益"。

## 1. 引导设计哲学

### 1.1 MVP 引导的五条核心原则

对标 `player-journey-map.md` §5 七大 FTUE 原则，提炼出 MVP 引导层面直接落地的五条核心原则：

#### 原则 A：不教，只暗示

> 对标 FTUE 原则 1（先行为后界面）、原则 2（数字跳动是第一语言）
> 来源：`player-journey-map.md` §5.1

玩家不需要知道"灵力值是做什么的"才能被它吸引。引导的设计语言是：**让正确行为变得显而易见，让错误行为变得不可能**，而非"告诉玩家该做什么"。

- 灵气数字每秒跳动 → 暗示"有东西在自动增长"
- 手动修炼按钮呼吸光晕 → 暗示"这里可以点"
- 灵气接近 100 时数字颜色从白渐变浅金 → 暗示"快达到某阈值了"
- 战斗日志自动滚动 → 暗示"战斗在自动运行"

**落地点**：`cultivation-screen.md` §States.Empty 中定义的新存档手动修炼按钮呼吸光晕（1.5s 周期，触发一次后永久消失）。

#### 原则 B：每个阶段只给一个核心行为

> 对标 FTUE 原则 1（先行为后界面）、原则 3（即时可见反馈）
> 来源：`player-journey-map.md` §1.1、§1.2

引导流程不允许同一时间出现多个新系统或新按钮。每个阶段只引入一个核心行为，让玩家充分适应后再引入下一个。

| 阶段 | 核心行为 | 仅有的交互 |
|------|---------|-----------|
| Stage 0 | 感受灵气跳动 | 观察灵气数字 + 点击手动修炼按钮 |
| Stage 1 | 发现战斗存在 | 观察战斗日志（纯观察，无操作） |
| Stage 2 | 理解自动循环 | 继续观察 + 注意到灵石/经验在涨 |
| Stage 3 | 做出第一个决策 | 选择"是否换区" |

#### 原则 C：每一个解锁都是一次叙事事件

> 对标 FTUE 原则 1（先行为后界面）、原则 4（"即将解锁"制造期望）
> 来源：`world-skeleton.md` §6.3 "一个系统 = 一个叙事理由"、`screen-flow.md` §3.4

任何 UI 解锁（新 Tab、新面板、新按钮）都必须有对应的世界内叙事文本给出"为什么现在才出现"的解释。解锁是叙事事件，不是 UI toggle。

| 解锁 | 叙事理由 | 来源 |
|------|---------|------|
| 战斗屏出现 | "体内灵气似乎冲破了什么关窍……你隐约感知到山中有异动。" | `world-skeleton.md` §6.1 |
| 资源屏出现 | "你捡起了第一片灵草——这山里的东西，或许比想象的值钱。" | `screen-flow.md` §3.1 #2 |
| 离线结算屏出现 | "闭关归来，洞府中灵气自行汇聚。" | `screen-flow.md` §3.1 #3 |
| Zone 2 解锁 | "道基铸就，你感受到谷中传来的灵气波动——那是更深处的地方。" | `screen-flow.md` §3.1 #4 |

**禁止模式**：绝对不在左上角弹出系统通知"新功能已解锁！战斗系统现已开放"——那是菜单地狱的做法，对标 `screen-flow.md` §3.4 明确反例。

#### 原则 D：首战必胜、首破必成、首离必获

> 对标 FTUE 原则 3（即时可见反馈）、原则 5（离线回归是 FTUE 下半场）
> 来源：`player-journey-map.md` §4.1 流失点 #1、#2、#3

玩家在引导期间的每一次"尝试"都必须获得成功反馈。首次战斗不能失败（这会传递"我太弱了"的错误信号）、首次突破不能被数值卡住、首次离线回归必须有"可观收益"的感觉。

具体护航机制见 §4。

#### 原则 E：用灰度预告替代完全隐藏

> 对标 FTUE 原则 4（"即将解锁"制造期望）
> 来源：`screen-flow.md` §3.4 "灰度预告 > 完全隐藏"、`world-skeleton.md` §6.3 "用限制制造渴望"

玩家在 Zone 1 期间，Zone 2 和 Zone 3 始终可见但为锁定态（🔒 + 灰度 + hover tooltip 解锁条件）。这给玩家一个明确的视觉锚点——"前方还有内容，我快到了"。完全隐藏未解锁内容会制造"这游戏就这么点东西？"的错觉。

**落地点**：`combat-screen.md` §States.Zone Locked —— Zone tabs 显示 🔒 图标 + 饱和度 -60% + hover 显示解锁条件 tooltip。

### 1.2 MVP 引导明确不做什么

| 禁止项 | 理由 | 替代方案 |
|--------|------|---------|
| **不出现教程弹窗** | 破坏沉浸感，对标 A Dark Room/Candy Box 2。弹窗教程让玩家从"我在修仙"切换到"我在学软件"。 | 用数字跳动、呼吸光晕、叙事文本替代所有教程。 |
| **不出现箭头指示** | 箭头是"游戏在命令你"的信号，与放置 RPG 的自主探索精神冲突。 | 用颜色渐变（灵气 100 时白→浅金）暗示阈值，用呼吸光晕暗示可点击。 |
| **不强制点击路径** | 玩家必须先点 A 才能看到 B 的强制线性引导违反"放置 = 低频高价值决策"（pillar 4.2）。 | 系统静默解锁，LEFT NAV Tab 淡入出现，玩家自己决定何时去探索。 |
| **不出现信息轰炸** | 开局同时展示 12 个 HUD 元素。对标 `hud.md` §HUD Philosophy 的渐进披露。 | 开局 HUD 仅 4-5 项（灵气/修为/当前区域/战斗状态/设置入口），随 §3 阶段逐项解锁。 |
| **不在前 30 秒展示任何菜单** | 对标 `world-skeleton.md` §6.1 "开局只有修炼"。菜单是"系统界面"，不是"修仙世界"。 | 修炼屏作为唯一可见内容，LEFT NAV 只有 Tab 1（修炼）激活。 |
| **不惩罚失败的探索** | 玩家切换到"太难"的区域后死亡造成资源损失（MVP 无死亡惩罚机制）。 | 战斗失败不丢资源、不掉经验、不进负面状态——只有 5s cooldown + 推荐换区 chip。 |
| **不用系统通知宣告解锁** | 对标 `screen-flow.md` §3.4 "解锁 = 叙事事件，不是 UI toggle"。 | LEFT NAV Tab 淡入 + 对应屏首次进入时展示叙事文本，不弹出系统 toast。 |

### 1.3 "发现式学习" vs "教程式学习"的边界定义

| 维度 | 发现式学习（本游戏） | 教程式学习（传统游戏） |
|------|-------------------|---------------------|
| **知识来源** | 玩家从系统反馈中自行推断 | 系统通过文字/箭头/高亮直接告诉玩家 |
| **学习动机** | 好奇心驱动（"这个数字为什么变金色了？"） | 完成任务驱动（"请点击高亮按钮以继续"） |
| **失败成本** | 零或极低（失败后 5s cooldown，无资源损失） | 有成本（生命、货币、进度） |
| **信息节奏** | 系统行为 → 玩家好奇 → 玩家尝试 → 系统确认 | 系统告知 → 玩家执行 → 系统检查 → 下一教程 |
| **适用场景** | 核心循环（修炼/战斗/掉落）、低频决策（换区/换姿态） | MVP 不适用；未来复杂系统（阵法、Build 配置）可能需要轻度教程 |
| **MVP 占比** | 100%（MVP 所有引导走发现式学习） | 0% |

**边界规则**：
- 当系统行为本身足够直观（数字在涨 = 有东西在产出），用发现式学习
- 当系统涉及不可逆决策且玩家可能不理解后果（如删除存档），用 P-INP-02 Confirm-Critical Modal + 明确警告文字 —— 这是 MVP 中唯一需要"告诉玩家"而非"让玩家发现"的场景
- 当系统操作复杂度超过 3 步且无直观反馈（如未来阵法配置），可考虑在 Phase 2+ 引入轻量引导，但仍优先走"可发现"模式

## 2. 分阶段引导流程

### Stage 0：标题 → 首次修炼（0:00–0:02）

#### 触发条件
玩家启动游戏，从标题画面进入。

#### 持续时长
约 2-3 分钟（灵气从 0 累积至 100 所需时间，按自动 +1/s + 玩家手动点击加速）。

#### 玩家所见

**标题画面**（`world-skeleton.md` §4.1）：
```
天地不仁，万物刍狗。
唯修者，逆天而行。
```
- 背景：青丘山林静态远景（水墨风格）
- 按钮：仅"开始修炼"一个可交互元素
- 无菜单、无设置入口（设置通过游戏内 Esc 访问，不在标题画面占位）

**点击"开始修炼"后**（120ms cross-fade 过渡）：
- 进入修炼屏（`cultivation-screen.md` §Layout），默认首屏
- HERO ZONE：主角立绘 + idle_sheet 微动画，stance icon = meditate，Label 显示 "Lv.1 凡人"
- DECISION ZONE：4 资源行（灵气/修为/灵石/药材）收起态，灵气初始显示 "0"，约 1 秒后开始每秒 +1
- ACTION ZONE：手动修炼按钮**呼吸光晕**（1.5s 周期 scale 100↔105 + alpha 100↔70，`cultivation-screen.md` §States.Empty）
- LEFT NAV：仅 Tab 1（修炼）可见 + 激活；Tab 2-5 为 HIDDEN 状态（`screen-flow.md` §3.2、§4.3）
- RIGHT PANEL：战斗日志空状态（无内容）
- 修炼界面显示沉浸式引导文本（`world-skeleton.md` §4.2）：

> 你盘膝坐定，闭目凝神。
> 山风穿过窗棂，带来第一缕若有若无的天地灵气。

- HUD TOP STRIP 初始元素：灵气数、修为数、当前区域（显示"青丘山林"）、设置入口 ⚙。共 4 项（`hud.md` §Information Architecture — "开局新存档 HUD 仅显示 4 项"）

#### 玩家行为

| 时间 | 行为 | 系统反馈 |
|------|------|---------|
| 0-3s | 阅读引导文本，观察灵气数字从 0 跳到 1 | 灵气每秒 +1（auto），数字微微跳动 |
| 3-8s | 可能注意到手动修炼按钮在"呼吸"并点击 | 每次点击获得 +5 灵气（`cultivation-system.md` §Tuning Knobs: `base_click_gain=5`），按钮触发 `manual_click_pulse` VFX 220ms，冷却 0.5s 开始倒计时 |
| 8-30s | 混合观察 + 点击，灵气累积至 30-80 | 灵气数字持续跳动，颜色保持 `text_secondary`（白色） |
| 30-90s | 继续观察/点击，灵气接近 100 | 灵气数字颜色从白色**渐变为浅金色**（暗示阈值接近，`player-journey-map.md` §4.1 流失点 #1 缓解策略②） |

#### 叙事文本交付时机

| 时机 | 文本 | 位置 | 来源 |
|------|------|------|------|
| 进入修炼屏立即 | "你盘膝坐定，闭目凝神。山风穿过窗棂，带来第一缕若有若无的天地灵气。" | 修炼屏 HERO ZONE 下方 | `world-skeleton.md` §4.2 |
| 灵气首次达到 10 | 无额外文本（数字本身即为语言） | — | FTUE 原则 2 |
| 灵气首次达到 50 | 无额外文本（颜色渐变开始） | — | FTUE 原则 A |

#### 数据门槛

| 参数 | 值 | 来源 |
|------|-----|------|
| 灵气阈值（解锁战斗） | ≥ 100 | `player-journey-map.md` §1.1 |
| 自动灵气产出 | +1/s（默认 meditate 姿态） | `cultivation-system.md`、`mvp-content-progression.md` |
| 手动修炼产出 | +5/次（`base_click_gain`） | `cultivation-system.md` §Tuning Knobs |
| 手动修炼冷却 | 0.5s | `cultivation-system.md` §Tuning Knobs |
| 最快解锁时间（纯点击） | ~10 次点击 = ~5 秒（含冷却等待） | 计算：100 ÷ 5 = 20 次 → 20 × 0.5s = 10s |
| 自然解锁时间（纯自动） | ~100 秒 ≈ 1 分 40 秒 | 100 ÷ 1 = 100s |
| 典型解锁时间（混合） | ~40-60 秒 | 先观察 20s（+20 auto），再点击若干次加速 |
| 首 30 秒无操作降速 | 灵气阈值降低 50%（降至 50）提前触发解锁 | `player-journey-map.md` §4.1 流失点 #1 缓解策略③ |

#### 完成标志
灵气 ≥ 100 → 触发战斗系统解锁，Stage 0 结束。

#### 下一步引导
Stage 1 自动开始——叙事文本浮现，LEFT NAV Tab 2 淡入，战斗系统静默解锁。

### Stage 1：灵气积累 → 战斗解锁（0:02–0:05）

#### 触发条件
`lingqi >= 100` 首次达标。

#### 持续时长
约 2-3 分钟（从战斗解锁到玩家理解"战斗是自动的"）。

#### 玩家所见与系统反馈

**叙事文本淡入**（`world-skeleton.md` §6.1）：
> "体内灵气似乎冲破了什么关窍……你隐约感知到山中有异动。"

- 文本在修炼屏 HERO ZONE 下方以 fade-in 200ms 出现，持续显示 8 秒后渐隐（玩家可 hover 回看）

**LEFT NAV 变化**（`screen-flow.md` §4.2 Tab 2）：
- Tab 2（⚔战斗）从 HIDDEN → VISIBLE+ACTIVE，200ms fade-in 出现在 Tab 1 下方
- LEFT NAV 从 48px 宽变至 96px 宽（2 tab × 48px）

**战斗系统状态**：
- `SemiAutoCombatSystem` 从 hidden → unlocked
- 战斗自动在默认区域"青丘山林"（Zone 1, `zone_starter`）开始 Seeking
- 区域选择器显示"区域：青丘山林 ▼"（仅 1 个可用 zone），Zone 2-3 标签灰度 🔒 显示（`screen-flow.md` §3.4 "灰度预告 > 完全隐藏"）
- RIGHT PANEL 战斗日志开始活动（`combat-screen.md` §Layout —— 战斗日志默认展开）

**战斗自动开始**：玩家无需点击任何按钮，战斗在解锁后 ~1-2 秒内自动从 Seeking 进入 Combat Active。

**首次战斗的敌人**（`mvp-content-progression.md` §2.2.1）：
- 敌人：石傀儡（`golem_stone`），Lv.1
- HP：30 / ATK：4 / DEF：2 / SPD：7
- 经验掉落：8 exp

**战斗日志首次行**（`semi-auto-combat-system.md` §Visual/Audio Requirements）：
```
[时间戳] 遭遇 石傀儡 Lv.1
[时间戳] 普攻 — 石傀儡 -8（剩余 22/30）
[时间戳] 普攻 — 石傀儡 -9（剩余 13/30）
[时间戳] 普攻 — 石傀儡 -11 击杀!
[时间戳] 战斗胜利 +8 经验 +1 灵石
```
- 每行 fade-in 220ms（`hud.md` §长 Session 视觉漂移防护）
- 击杀行显示 `victory_burst_gold` 颜色
- 日志底端 auto-scroll 确保最新行始终可见

**HUD TOP STRIP 变化**：
- 战斗状态点由 ○（待机）变为 ●（战斗中）—— `combat-screen.md` §ZONE SELECTOR
- 灵石行解锁（HUD 顶栏资源行新增灵石数字，首次从 0 → 1 时微微 highlight）

#### 玩家行为

| 时间 | 可能行为 | 系统反馈 |
|------|---------|---------|
| 0-10s | 注意到灵气数字变金色、叙事文本浮现、LEFT NAV 多了一个 Tab | 文本淡入 + Tab 淡入 —— 三个视觉信号汇聚暗示"有变化" |
| 10-30s | 主动点击 LEFT NAV "⚔战斗" Tab 或按数字键 `2` 切换到战斗屏 | 120ms cross-fade 切换到战斗屏（`screen-flow.md` §6.1），看到敌人 portrait + health bar 递减 + 战斗日志滚动 |
| 30-60s | 观看战斗日志 2-3 轮战斗（约 20-30 秒可看完 2-3 场石傀儡战） | 战斗日志持续刷新，每场战斗 3-5 行（普攻×3-4 + 击杀 + 奖励），约 8-10 秒一轮 |
| 60-90s | 首次击败石傀儡获得 8 exp + 可能 1 灵石 | `mvp-content-progression.md` §2.2.2：石傀儡 15% 概率掉落 1-2 灵石；EXP 条首次推进（Lv.1 需 10 exp 升级，一场战斗即推进 80%） |
| 90-120s | 可能切换回修炼屏（`1` 键或 LEFT NAV）、可能继续观察、可能注意到灵石数字从 0→1 | 切换后战斗继续后台运行（`combat-screen.md` §Derived Design "战斗继续后台运行"） |

#### 叙事钩子文本（首次战斗，`world-skeleton.md` §4.3）

当玩家首次切换到战斗屏时，战斗屏 ENEMY ZONE 上方显示：
> "村中猎户警告过你：山中有妖。此刻，灌木丛后的那双猩红眼睛，终于让你明白了这句话的分量。"

- 仅在新玩家首次进入战斗屏时显示
- fade-in 200ms，持续 6s 后渐隐
- 回归玩家加载存档后不显示此文本

#### 数据门槛

| 参数 | 值 | 来源 |
|------|-----|------|
| 石傀儡 HP | 30 | `mvp-content-progression.md` §2.2.1 |

#### 护航保障：首战必胜

`player-journey-map.md` §4.1 流失点 #2 要求首次战斗不能失败。

**实现方式**：首次战斗的敌人固定为石傀儡（Lv.1）。Lv.1 石傀儡 ATK=4 vs Lv.1 玩家 DEF=5 → 伤害 = `max(1, 4 - 1.5) = max(1, 2.5) = 2.5` → 约 3/击（向上取整）。玩家 HP=100 vs 石傀儡每击 ~3 → 石傀儡需要 ~34 次普攻才能击败玩家；而玩家 3-4 次普攻即可击杀石傀儡。**数学上不可能失败**——无需特殊"无敌"逻辑，数值本身保证首战必胜。

### Stage 5：首次离线回归（首次退出 + 回归）

#### 触发条件
玩家首次退出游戏（任何时长），再次打开并加载存档。

#### 退出时机的自然设计

MVP 不做"提示你该休息了"的推送。玩家自然退出的时机由游戏节奏本身引导：

| 自然退出窗口 | 玩家状态 | 退出体感 |

不设强制退出提示或"健康提醒"弹窗。退出是玩家的自由选择，回归时的离线收益是对"回来"的唯一正向激励。

**抽屉关闭后的牵引**（`player-journey-map.md` §4.1 流失点 #3 缓解策略①）：

抽屉关闭后，修炼屏 DECISION ZONE 下方出现 soft nudge 文字：
> "距下一级还需 XXX 经验（约 N 分钟）" / "距突破至筑基期还需 X 级"

- 这不是强制引导，而是"回来之后有个小目标"的温和牵引
- 如果离线战斗触发了等级升级（经验累积达标），TOAST "离线突破！升至 Lv.X" 弹出（`screen-flow.md` §2.4 Toast）
- 如果离线战斗击败了 Zone 2 守关（噬石兽），TOAST "离线期间击败噬石兽！" 弹出并伴随掉落展示

#### 护航保障：首次离线的收益下限

对标 FTUE 原则 5（离线回归是 FTUE 下半场），首次离线回归的收益必须让玩家感到"可感知的正向反馈"。

**实现方式**：

| 离线时长 | 收益倍率 | 收益下限保证 | 来源 |
|---------|---------|------------|------|
| ≤ 1h | 100% | 自然收益：1h × online_rate。例如 Lv.10 在线 Zone 2：灵气 3,600、经验 45,000、灵石 3,600 | `mvp-content-progression.md` §5.3 |
| 1-4h | 100% | 4h × online_rate。收益可观 | 同上 |
| 4-8h | 100%→90% 线性 | 不设额外保底——4h 内 100% 本身就保证"睡一觉回来至少有 4h 收益" | 同上 |
| 首次离线（<5 min） | 100% | **特殊处理**：如果首次离线时长 < 5 分钟（玩家只是退出测试一下），系统仍然按标准公式结算，但抽屉展示中追加一句："灵气在无人的洞府中会自行汇聚" —— 让极短离线也有叙事反馈 | 本设计新增 |

**不做的保证**：不在首次离线额外赠送"欢迎回来礼包"或暗改数值（这会让玩家对后续离线的真实收益产生错误预期）。真实收益足够好，不需要虚假兜底。

#### 数据门槛

| 参数 | 值 | 来源 |
|------|-----|------|
| 离线倍率（≤4h） | 100% | `mvp-content-progression.md` §5.3 |
| 离线倍率（4-8h） | 100%→90% 线性 | 同上 |
| 离线倍率（>24h） | 50% 地板 | 同上 |
| 离线抽屉 slide-in 动画 | 150ms | `screen-flow.md` §6.1 |
| 数字递增动画 | 1s | `screen-flow.md` §2.4 |
| 抽屉宽度 | 480px | `screen-flow.md` §1.5 |

## 4. 护航机制

### 4.1 首次战斗的必胜保证

**保证方式：数值设计层面保证，非特殊逻辑**。

| 要素 | 值 | 分析 |
|------|-----|------|
| 首战敌人 | 石傀儡 Lv.1（HP=30, ATK=4, DEF=2） | Zone 1 最弱敌人 |
| Lv.1 玩家 ATK | 10 | 来源：`mvp-content-progression.md` §3.2 |
| 玩家单次伤害 | ~8.5 | `max(1, 10 - 2*0.3) = 8.5` |
| 击杀所需普攻 | 3-4 次 | 30 ÷ 8.5 ≈ 3.5 |
| 敌人单次伤害 | ~2.5 | `max(1, 4 - 5*0.3) = 2.5`（Lv.1 玩家 DEF=5） |
| 击败玩家所需普攻 | ~40 次 | 100 ÷ 2.5 = 40 |
| **结论** | **玩家在敌人击败自己之前可击杀敌人 ~10 次** | 数学上不可能失败 |

不实现"首战无敌锁定"或"首次战斗伤害 ×10"等丑陋的特殊逻辑。数值本身就保证首战必胜。

### 4.2 首次突破的不可失败保证

MVP 中境界突破 **0% 失败概率**。突破是纯粹的仪式化正向体验——只要经验达到门槛，确认突破后 100% 成功。渡劫/突破失败机制（`world-skeleton.md` §5.2 天劫系统 181）为 Phase 4+ 内容。

### 4.3 首次离线的收益下限保证

详见 §Stage 5 护航保障。核心保证：离线 ≤4h 收益为 100% 在线等效，**不做衰减**。玩家午餐/通勤/睡觉等自然间隔的收益不打折扣。即使离线仅 5 分钟，系统也按标准公式结算 + 给予叙事反馈。

### 4.4 玩家"迷路"时的隐式引导

不出现箭头、弹窗或"提示：你应该去 XXX"的系统消息。以下为隐式引导策略：

| 迷路场景 | 隐式引导 | 玩家感受到的 |
|---------|---------|------------|
| 修炼屏看了 30s 不知道能干嘛 | 手动修炼按钮呼吸光晕引导视线 + 灵气数字颜色在接近 100 时渐变浅金 | "这个按钮在暗示我点它" / "这个数字变金色了，是不是快到什么了" |
| 战斗解锁后不知道去哪看 | LEFT NAV Tab 2 淡入动画引导视线 + 战斗状态点从 ○→● 的视觉变化 | "左边多了个按钮" / "顶上那个点变亮了" |
| 切换到战斗屏但战斗日志没滚动 | Seeking 指示器旋转墨点 + "搜寻中..." 文字 | "哦还在找敌人" |
| 战斗日志看了 5 分钟不知道还能干嘛 | Zone Selector 中 Zone 2/3 锁定 🔒 标签始终可见 + hover tooltip 显示解锁条件 | "前面还有两个区域，需要 Lv.5 才能去" |
| Lv.5 达到后不知道该不该换区 | Zone 2 从灰度变亮 + Zone 1 的蛇妖在 Lv.6+ 后明显"不够打"（秒杀） | "好像可以去那个新地方了" + "这边打起来太容易了" |
| 退出后忘记回来 | **无强制提醒**（单机游戏无推送）。首次离线回归时的 offline drawer 数字递增动画提供"回来看看"的记忆锚点 | "每次回来都有数字蹦一下，挺爽的" |
| 打开资源屏但背包是空的 | 空状态文案"暂无物品 —— 前往战斗获取掉落" + 附文"战斗击败敌人可获取灵草、矿石等材料"（`screen-flow.md` §7.4 新玩家空库存） | "哦要去战斗才能拿到东西" |

### 4.5 连败自动暂停（非引导特有，但涉及新玩家保护）

`combat-screen.md` §States.Consecutive Failure Threshold：连败 ≥5 时，战斗自动暂停 + TEAM STATUS ZONE 旁出 P-FBK-02 chip "建议换区或提升属性"。

- 新玩家在引导期基本不会触发（Zone 1/2 的敌人在对应等级下都是安全 farm）
- 如果玩家提前跳到 Zone 2 且实力不足，连败 5 次后的自动暂停 + 建议 chip 是"温和的系统介入"，不指责玩家、不弹窗、只提供方向

## 5. 反模式与禁忌

### 5.1 明确禁止的设计模式

| # | 禁止模式 | 理由 | 替代方案 |
|---|---------|------|---------|
| 1 | **弹窗教程**（"欢迎来到修仙世界！请点击灵气按钮以开始修炼 →"） | 破坏沉浸感。对标 A Dark Room/Candy Box 2 的极简开局。FTUE 原则 A "不教，只暗示"。 | 手动修炼按钮呼吸光晕 + 灵气数字自然跳动。 |
| 2 | **强制点击路径**（灰色遮罩 + "请点击高亮区域以继续" + 其他区域 disabled） | 剥夺玩家的自主探索感。与 pillar 4.2 "放置 = 低频高价值决策" 冲突——玩家的第一个"高价值决策"不应该是"在提示下点击"。 | 系统静默解锁，玩家自己选择何时去探索。 |
| 3 | **信息轰炸**（开局同时展示 12 个 HUD 元素 + 5 个 LEFT NAV Tab + 3 个区域 tab 全部亮起） | 认知负荷爆炸。FTUE 原则 B "每个阶段只给一个核心行为"。 | 渐进解锁：HUD 从 4 项逐步到 12 项，LEFT NAV 从 1 个 Tab 到 5 个 Tab。 |
| 4 | **系统通知宣告解锁**（弹 toast "新系统已解锁：战斗！点击左方 Tab 2 查看"） | 解锁是叙事事件不是 UI toggle。对标 `screen-flow.md` §3.4 明确反例。 | LEFT NAV Tab 淡入 + 对应屏首次显示叙事文本。 |
| 5 | **首次战斗安排"教程敌人"**（特殊敌人 HP=9999 + ATK=0 + "请观察战斗日志了解自动战斗系统"） | 教程敌人制造"这不真实"的感觉。玩家会怀疑之后的战斗是否也是假的。 | 数值保证首战必胜（见 §4.1），但战斗本身是真实系统运行。 |
| 6 | **强制引导箭头/高亮圈/手指图标** | 来自移动游戏和浅度休闲游戏的"过度引导"模式，在深度放置 RPG 中被视为侮辱玩家智商。 | 颜色渐变（白→浅金暗示阈值）+ 呼吸光晕（暗示可点击）+ 淡入动画（暗示新东西出现了）。 |
| 7 | **前 N 次操作给额外奖励**（"首 10 次修炼额外 +100% 灵气!"） | 制造虚假的初期加速，玩家后续发现"怎么变慢了"会产生被骗感。 | 真实数值本身够好（Lv.1-3 连升在 ~2 min 内完成），不需要虚假 buff。 |
| 8 | **"你知道吗"加载提示** | 对标放置游戏的"沉浸式等待"——标题画面只有"开始修炼"一个按钮和氛围文案，不需要破坏沉浸的 loading tip。 | `world-skeleton.md` §4.1 标题画面文字本身即是氛围载体。 |

### 5.2 "不做"的深层理由：放置 RPG 的信任模型

放置/Idle RPG 的核心心理契约是：**世界在运转，数字是真的**。每一个"虚假"的引导（特殊 buff、教程敌人、强制路径）都在侵蚀这个信任。玩家在放置游戏中投入的是"等待"——如果等待的结果是预先编排好的人工体验，他们会离开且不再回来。

引导的目标不是"教会玩家所有功能"，而是"让玩家在第一小时内自行验证：这个世界值得我每天回来看一眼"。

## 6. 度量与验证

### 6.1 引导成功的关键行为信号

以下不是数据打点/telemetry 指标，而是 **设计层面的可观察行为**——在 playtest 中可被直接观察到的玩家行为模式：

| # | 成功信号 | 观察方法 | 对应阶段 |
|---|---------|---------|---------|
| S1 | 玩家在首 30 秒内主动点击手动修炼按钮（未被提示） | playtest 观察 → 玩家是否被呼吸光晕吸引 | Stage 0 |
| S2 | 灵气达 100 后，玩家在被提示前主动切换到战斗屏（通过 LEFT NAV 或数字键 2） | playtest 观察 → 玩家是否对"新出现的 Tab"产生了好奇心 | Stage 1 |
| S3 | 玩家在首次战斗后继续观看战斗日志 ≥ 2 场（而非立即切换回修炼屏或关闭游戏） | playtest 观察 + session 记录 | Stage 1-2 |
| S4 | 玩家在 Lv.5 后 5 分钟内主动切换到 Zone 2（未被任何提示推动） | playtest 观察 → Zone 2 的"解锁亮起"是否触发了自然的探索行为 | Stage 3 |
| S5 | 玩家首次退出后，在 24h 内再次打开游戏 | 回归率——这是最关键的单一指标 | Stage 5 |
| S6 | 玩家回归后主动打开离线抽屉（或点击离线结算屏）查看收益详情 | playtest 观察 → 抽屉的数字递增动画是否激发了"我想知道更多"的好奇心 | Stage 5 |
| S7 | 玩家在首次 session 中至少完成以下操作之一：切换修炼姿态 / 在资源屏查看物品 / 手动存档 | playtest 观察 → 玩家是否在"发现"而非"被告诉"更多系统 | Stage 3-4 |

### 6.2 引导失败的症状列表

| # | 失败症状 | 最可能的根本原因 | 对应调优参数 |
|---|---------|---------------|------------|
| F1 | 玩家在标题画面停留 >10s 或在进入修炼屏后 30s 内无任何操作 | 不知道"该做什么"。呼吸光晕不够明显，或灵气跳动太慢以至于玩家未注意到数字在变 | `click_cooldown_seconds`（适当缩短以加速初期反馈）、呼吸光晕对比度 |
| F2 | 玩家在战斗解锁后 >2 分钟未切换到战斗屏 | LEFT NAV Tab 2 的淡入动画可能不够引起注意，或玩家不知道 LEFT NAV 可交互 | LEFT NAV 淡入动画时长增加到 300ms + 加一个微妙的 1 次性 bounce |
| F3 | 玩家在观看战斗日志 5 分钟后关闭游戏 | "只有自动战斗？"的流失——战斗日志差异化不足（击杀文本重复），或玩家期望操作性战斗 | 增加击杀文本变体、首次击败每种敌人的 Toast 更显著、Zone 2 解锁条件更早可见（让玩家看到"前方有内容"） |
| F4 | 玩家在 Lv.5 后不切换 Zone 2，在 Zone 1 继续挂机 >20 分钟 | Zone 2 的解锁不够明显，或 tooltip 没有明确表达"现在可进入" | Zone 2 解锁时增加一个微弱的视觉 flash（border highlight 1s）、tooltip 增加"当前可进入"字样 |
| F5 | 玩家首次退出后 >48h 未回归 | 离线收益不够有吸引力、或游戏没有在退出时给玩家留下"期待"的锚点 | 退出时考虑一个极简的"离开前"提示（非弹窗——仅在修炼屏底部显示"灵气会在无人的洞府中自行汇聚……"） |
| F6 | 玩家回归后直接关闭离线抽屉（<2s 未阅读）然后什么也不做就退出 | 离线抽屉信息设计失败——数字不突出、没有递增动画、或玩家对"离线收益"的概念不感兴趣 | 抽屉数字递增动画速度调整、增加"你离开期间击败了 X 个敌人"的战斗叙事行 |
| F7 | 玩家在达到 Lv.10 时没有主动点击"突破"按钮 | 突破按钮不够醒目，或突破效果预览不够有吸引力 | 突破按钮增加微弱的 border glow（`burst_gold` 低强度脉冲）、突破预览增加具体数值对比 |

### 6.3 可调整的调优参数

以下参数可在 playtest 后调整而不影响核心玩法设计：

| 优先级 | 参数 | 当前值 | 调整范围 | 影响的行为 |
|--------|------|--------|---------|-----------|
| P0 | 灵气解锁战斗的阈值 | 100 | 50-150 | F1（无操作）——降低阈值可加速初期节奏 |
| P0 | 首 30 秒无操作降速触发 | 阈值降 50% | 30%-70% | F1——加大降幅可让"发呆玩家"更快触发解锁 |
| P0 | 手动修炼按钮呼吸光晕强度 | alpha 100↔70 | alpha 100↔50（更强）/ 100↔85（更弱） | F1——更强的呼吸吸引更多点击 |
| P1 | 手动修炼 `base_click_gain` | 5 | 3-10 | F1 + 初期灵气节奏 |
| P1 | 手动修炼 `click_cooldown_seconds` | 0.5s | 0.3-1.0s | F1——更短的冷却让初期"点击加速"体感更强 |
| P1 | 战斗日志 fade-in 时长 | 220ms | 150-300ms | F3——更慢的 fade-in 让日志更"可读" |
| P1 | 首次击败敌人 Toast 持续时长 | 3s | 2-5s | F3——更长的 Toast 强化收集感 |
| P2 | 离线抽屉数字递增动画时长 | 1s | 0.5-2s | F6——更长的动画增加仪式感但可能让急性子玩家不耐烦 |
| P2 | 离线抽屉自动弹出延迟 | 500ms（加载完成后） | 200-1000ms | F6——太早弹出可能被玩家忽略 |
| P2 | 境界突破 `burst_gold` 印章持续时长 | 3s | 2-5s | F7——更长的印章强化仪式感 |
| P2 | LEFT NAV Tab 淡入动画时长 | 200ms | 150-400ms | F2——更长的动画更容易被注意但可能显得拖沓 |
| P3 | 灵气颜色渐变触发点（白→浅金） | 80/100 | 60-90/100 | F1——更早的渐变暗示给玩家更多"准备时间" |
| P3 | Zone 2 解锁条件提示文案 | "需 Lv.5 + atk ≥ 180" | 可加"当前 Lv.X / atk XXX"对比 | F4——对比数字直接显示进度，减少玩家犹豫 |
