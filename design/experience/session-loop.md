# 会话循环设计 (Session Loop Design)

> 版本：v1.0
> 最后更新：2026-05-05
> 范围：MVP（阶段 1）—— 3 区域、单角色、5 境界
> 关联文档：`design/experience/player-journey-map.md` · `design/experience/screen-flow.md` · `design/balance/mvp-content-progression.md` · `design/narrative/world-skeleton.md` · `design/gdd/game-concept.md` · `design/gdd/semi-auto-combat-system.md` · `design/gdd/cultivation-system.md` · `design/gdd/offline-reward-settlement-system.md`

## 摘要

本文档定义 MVP 玩家从"打开游戏"到"关闭游戏"的完整会话闭环——每次回来看到什么、做什么、得到什么。设计锚定 game-concept.md §15 的核心目标：**"让玩家每天回来都有三件事可做：1. 看见挂机带来的成长；2. 做出几个有价值的配置决策；3. 解锁或接近一个新的系统、境界、装备、区域或故事。"** Sprint 12 中"装备"只作为 Phase 2+ 路线图牵引，不作为当前会话内可操作系统。

本文将会话按时长和意图分为 4 种类型，为每种会话定义入口体验（离线回归/在线切回/冷启动）、核心决策流（去哪、怎么配、何时推进、何时退出）、产出与奖励节奏（从灵气跳动到境界突破），并设计每日节律和回归牵引机制。MVP 无每日目标系统（系统 64，225 蓝图 Phase 4），但通过离线收益预览、进度预估和"期待锚点"实现等效的回归牵引力。

## 1. 会话定义与分类

### 1.1 什么是"一次 play session"

一次 play session 定义为：玩家从**打开游戏/切回游戏**到**关闭游戏/切出游戏**之间的连续交互周期。MVP 是单机放置游戏——没有登录、没有匹配、没有社交。会话的唯一判据是玩家的注意力在场。

### 1.2 四类会话

| 会话类型 | 时长 | 典型场景 | 玩家意图 | 设计目标 |
|----------|------|----------|----------|----------|
| **微会话** | < 1 分钟 | 看一眼灵气涨了多少、收离线收益、确认挂机区域是否正确 | 快速满足好奇心，建立"世界在运转"的信任 | 30 秒内让玩家看到至少一个数字显著增长，看完即走无压力 |
| **短会话** | 1–5 分钟 | 收离线收益 + 调整姿态/区域 + 查看进度条 + 检查下一里程碑 | 低投入高回报——用极少时间做出有价值的微调 | 每次短会话至少有 1 件"做了有效果"的事（换了更高效的区、升级了、接近新区域/突破） |
| **标准会话** | 5–20 分钟 | 检查离线收益 → 推图/突破 → 评估要不要换区 → 配置姿态 → 设置退出前状态 | 有目标的深度投入——"今天我要推进 Zone 3"或"今晚试试挑战守护灵" | 覆盖完整决策闭环：去哪 → 怎么配 → 何时推 → 规划退出 |
| **长会话** | 20 分钟+ | 新区域首次探索、守护灵挑战、终局资源储备和路线图查看 | 沉浸式体验——玩家有完整的连续时间投入 | 内容深度足够支撑玩家沉浸，但保持"随时可停"的放置友好性 |

### 1.3 会话分类的玩家画像覆盖

| 玩家画像 | 主要会话类型 | 设计支撑 |
|----------|-------------|---------|
| 上班族碎片时间 | 微/短会话（早晨收菜） + 标准会话（晚间） | 离线结算 4h 内 100% 收益覆盖工作间隔（参见 mvp-content-progression.md §5.3） |
| 学生/自由职业 | 标准/长会话 | 所有系统在 session 内完整可用，screen-flow.md §3 的渐进解锁在首次长 session 中即可走完 |
| 多任务并行玩家 | 微/短会话（挂机为主） | 战斗独立于 UI 运行，切屏不暂停，RIGHT PANEL 始终可见战斗日志缩影（screen-flow.md §5.3） |
| 硬核刷宝玩家 | 标准/长会话 | Zone 3 守护灵挑战 + Phase 2+ 装备刷宝路线图提供长尾期待 |

## 2. 会话入口体验（回来看到什么）

### 2.1 入口体验总则

所有三种回归类型共享同一设计原则（对标 player-journey-map.md §5.1 FTUE 原则 5："首次离线回归是 FTUE 的下半场"）：

1. **数字先行**：回归后 0.5 秒内必须有数字变化可见——无论是灵气从 8.2K 跳到 12.5K，还是离线结算抽屉的收益数字正在计数。
2. **收益 = 奖励，不是通知**：离线收益的展示不是"提醒你有东西"，而是"你看你没玩也赚了——快看看赚了啥"。
3. **回归可中断、可延后**：离线结算抽屉（P-NAV-04）不阻塞主屏交互（screen-flow.md §7.2），玩家可以先修炼、先战斗、再回来查看。

### 2.2 类型 A：离线后首次回归

**触发条件**：`offline.settled` 事件触发，离线时长 > 0

**展示序列**（严格按此顺序，参见 offline-reward-settlement-system.md §Detailed Design 规则 7）：

```
第 1 层（0-1s）：TOAST 通知 — "闭关归来"
  └─ burst_gold toast（screen-flow.md §6.3 OFFLINE 优先级）持续 8s
  └─ LEFT NAV Tab 5 "离线"点亮 🔴 角标（screen-flow.md §4.2 Tab 5 状态转换）

第 2 层（1-3s）：离线沉淀抽屉（P-NAV-04）自动从右侧滑入 480px
  └─ 头：离线时长（"你闭关了 3 小时 42 分钟"）
  └─ 核心收益条（数字递增动画 ease-out cubic 1.5s）：
      灵气 +12,847 / 战斗经验 +34,520 / 灵石 +1,820 / 药材 +328
  └─ 事件高亮（如有）："离线期间突破至炼气期！"、"击败噬石兽（首次）"
  └─ 底：2 个 action — [查看详情]（→ 离线结算屏全屏） / [延后查看]（关闭抽屉）

第 3 层（玩家点击"查看详情"后）：离线结算屏全屏（ScreenStack 切换）
  └─ 资源拆解（每个资源的 gross → claimed → lost，与 overflow 逻辑一致）
  └─ 战利品网格（exp / 灵石 / 药材的来源拆解；装备碎片为 Phase 2+ 不显示）
  └─ 战斗摘要（击败敌人计数、胜率、连败次数）
  └─ 资源溢出警告（如有满仓损失，红色高亮并提示"下次回来前升级存储可减少损失"）
```

**收益数字的呈现技巧**（对标 game-concept.md §4.1 "数字增长就是快乐"）：

| 技巧 | 实现 | 目的 |
|------|------|------|
| **数字递增动画** | ease-out cubic 1.5s 从旧值计数到新值 | 让"增长"可见、可感受，不是瞬间跳变 |
| **对比基线** | 每个资源行附带"这相当于在线挂机 X 分钟的收益" | 锚定离线收益的价值感——玩家知道"没白关" |
| **大数格式化** | 使用 NumberFormatter 缩略（12.8K / 1.2M），不展示原始整数 | 减少视觉噪音，聚焦变化幅度 |
| **高亮首次** | 首次离线回归时抽屉底部附一句："这是你第一次闭关——以后每次回来，世界都会带着礼物等你。" | 建立"回归 = 有收获"的心理锚定（FTUE 原则 5） |

**从结算屏到"继续玩"的过渡路径**：

```
抽屉/结算屏关闭后
  → 如果离线触发了升级或境界突破 → 自动切换到修炼屏展示新境界
  → 如果离线期间首次击败关键敌人 → toast "首次击败" + 角标指向战斗屏/区域选择
  → 如果离线期间资源接近上限 → 资源屏角标提示"容量压力"
  → 如果以上都没有 → 鼠标/焦点停留在当前屏，玩家自行选择下一步
```

### 2.3 类型 B：在线挂机后切回游戏

**触发条件**：玩家在另一窗口/应用，游戏在后台持续运行（在线挂机状态未中断）

**展示序列**：

```
0:00-0:02 — 即时状态刷新：
  └─ HUD 所有数字刷新为当前值（无动画，瞬间跳变——数据已在后台持续更新）
  └─ TOP STRIP 灵气/修为/灵石 翻新

0:02-0:05 — 战斗日志摘要（RIGHT PANEL 最后 5 行可见）：
  └─ 最近击杀的敌人列表
  └─ 关键事件高亮（附带时间戳）：
      · "[3 分钟前] 击败噬石兽——首次击败!" 
      · "[12 分钟前] 获得药材 ×3"
      · "[28 分钟前] 灵气储量接近上限，灵草溢出 ×15 损失"

0:05+: 关键事件 TOAST 重放（仅在切换回来时）：
  └─ 如果离开期间触发了首次击败/境界变化 → burst_gold toast
  └─ 如果离开期间触发了灵气满仓 → bottleneck_red toast
  └─ 最多重放 3 条（避免 toast 风暴）
```

**设计意图**：在线挂机期间玩家可能错过了关键事件。切回时的摘要不是"全部重放"，而是"最重要的事先告诉你"——让玩家几秒内就能判断"我离开期间有没有发生什么值得注意的事"。

### 2.4 类型 C：冷启动（每天第一次打开）

**触发条件**：游戏进程全新启动（不是从后台切回），`SaveManager` 加载存档

**与新玩家首次 session 的差异**（参见 screen-flow.md §7.4 "首次进入 vs 回归玩家的不同展示"）：

| 维度 | 新玩家（首次 session） | 回归玩家（加载存档后） |
|------|----------------------|----------------------|
| **首屏** | 修炼屏，手动修炼按钮带呼吸光晕（onboarding 暗示） | 修炼屏，上次保存的姿态，按钮无呼吸光晕 |
| **LEFT NAV** | 仅 Tab 1（修炼），其余按 screen-flow.md §3 序列逐步淡入 | 所有已解锁 tab 全部可见 |
| **离线结算** | 不存在（尚无离线经历） | 如有离线收益，LEFT NAV Tab 5 点亮角标 + 抽屉自动弹出 |
| **叙事感知** | 沉浸式引导文本："你盘膝坐定，闭目凝神……" | 无引导文本，直接进入功能界面 |

**"今天有什么不同"的感知营造**：

冷启动回归时，除了离线结算抽屉，额外展示"自上次离开以来的变化摘要"（仅在离线时长 > 4h 且发生了显著事件时）：

```
[抽屉底部 / 关闭后 toast]：
  "自昨日起：灵气产出 +12%（修炼姿态优化）· 灵石 +85,000 · 击败幽墟灵谷噬石兽（首次）"
```

这条摘要的设计要点：
- **不重复抽屉已有信息**——它是"一句话概括变化"，不是数据表
- **只在有"变化"时才出现**——如果离线期间什么都没发生，不说"自昨日起：无变化"
- **强调"每天的累积"**——让玩家感知到"每天回来都在变得更强"，对标 game-concept.md §3 "我回来时，角色更强、仓库更多"

## 3. 会话核心决策流（做什么）

以**标准会话（5-20 分钟）**为模板，定义玩家在单次会话中经历的完整决策序列。四个决策构成闭环：去哪挂机 → 怎么配置 → 何时推进 → 何时退出。

### 3.1 决策 1：去哪里挂机

这是玩家回归后的第一个高价值决策（对标 game-concept.md §4.2 "低频高价值决策"）。

**决策窗口的 UI 呈现**：

信息显示在区域选择器（combat-screen.md Zone Selector 组件）：

| 展示信息 | 数据来源 | 格式 | 示例 |
|---------|---------|------|------|
| **当前区域名称** | ZoneSystem | 正常文本 | "幽墟灵谷" |
| **区域状态** | ZoneSystem + MapProgressionSystem | 🔒 锁定 / ✅ 解锁 / ⭐ 推荐 / ⚠️ 危险 | "✅ 已解锁" |
| **推荐条件** | MapProgressionSystem | 文本 | "建议 Lv.5+ / atk ≥ 180" |
| **exp/min 预估** | 基于当前战斗力 + 区域敌人池计算 | 数字 + 趋势箭头 | "~750 exp/min ↑ vs 当前" |
| **灵石/min 预估** | 同上 | 数字 + 趋势箭头 | "~60 灵石/min ↑ vs 当前" |
| **战力匹配度** | 玩家战力评分 vs 区域敌人评分范围 | 三档：安全 / 适中 / 危险 | "战力匹配：安全"（绿色） |
| **收益倾向** | LootSystem zone_loot_mult + 资源标签 | 标签 | "经验多 · 灵石多 · 药材多" |

**决策因素优先级**（玩家通常在以下三者间权衡）：

| 因素 | 设计如何支撑 | 来源 |
|------|------------|------|
| **离线产出 vs 在线效率** | 离线收益按关游戏时的当前区域计算。如果玩家上次关游戏在 Zone 1，离线 8h 后回来——在线推图时切换到 Zone 2 能获得更高在线效率，但离线收益已按 Zone 1 结算。这制造一个自然决策："要不要换区再下线？" | mvp-content-progression.md §5.3 离线倍率 |
| **战力匹配度** | 显示三档评估帮助玩家避免"进入打不过的区域"。但故意不做"你必须 X 战力才能进"的硬锁——grace period 让进阶玩家自主承担风险。 | player-journey-map.md §3.1 决策点 2、3 |
| **资源需求** | 当前 MVP 主要追踪 exp / lingshi / herb。若玩家想快速升级或储备 Phase 2+ 资源，应选择收益更高但风险更高的区域。 | mvp-content-progression.md §2.2.2-2.4.2 掉落表 |

**决策辅助微提示**（hover tooltip 方式，不弹窗）：

- 如果当前区域已"毕业"（毕业条件满足，参见 mvp-content-progression.md §2.1 毕业条件列）→ selector 显示绿色"已毕业"标签 + tooltip "推荐切换至更高区域以获得更高收益"
- 如果玩家战力远高于当前区域敌人最高评分（战力比 > 10:1）→ tooltip "此区域对你已无挑战，可切换至更高区域"
- 如果玩家战力低于目标区域最低敌人评分 → tooltip "战力不足，建议在此区域继续挂机至 Lv.X"

### 3.2 决策 2：如何配置

**MVP 可配置项**：

| 配置项 | 界面位置 | 选项 | 决策影响 | 默认值 |
|--------|---------|------|---------|--------|
| **修炼姿态** | 修炼屏 → 姿态选择 Modal（P-NAV-03） | `Meditate`（灵气产出为主）/ `Condense`（灵气转修为） | 灵气产出速率 vs 修为累积速率 | `Meditate`（新玩家首次进入修炼屏默认） |
| **战斗区域** | 战斗屏 → Zone Selector | Zone 1 / Zone 2 / Zone 3（按进度解锁） | 战斗经验、灵石、掉落种类和品质 | Zone 1（青丘山林）——默认选中唯已解锁区域 |
| **战法策略** | Phase 2+ 预留（MVP 不展示 UI） | MVP 未实装（系统 90，225 蓝图 Phase 5） | — | 普攻自动（`skill_multiplier = 1.0`） |

**配置界面的信息架构**（遵循 screen-flow.md P1 "先行为后界面"原则）：

```
修炼屏（首屏）
  ├─ DECISION ZONE（当前姿态 + 每秒产出 + 切换按钮）
  │     └─ 点击"切换姿态" → Modal 浮层（底层修炼屏内容保持可见，灵气数字仍在跳动）
  │           ├─ Meditate 行（当前产出预览 + 选中态高亮）
  │           ├─ Condense 行（预期修为产出预览 + 点击选中）
  │           └─ Closed Door 行（灰度显示 + 🔒 + tooltip "即将解锁"）

战斗屏
  ├─ ZONE SELECTOR（横向 tabs：zone_starter / zone_forest / zone_mine）
  │     ├─ 已解锁 zone tab 正常色 → 点击切换
  │     ├─ 未解锁 zone tab 灰度 + 🔒 + hover tooltip 显示解锁条件
  │     └─ 当前选中 zone 高亮 + 右侧显示敌人数和预期效率
  └─ MVP 不展示战法配置 UI；路线图中预告系统 90
```

**默认值策略**（让新玩家不配置也能玩）：

| 配置项 | 初始默认值 | 理由 |
|--------|----------|------|
| 修炼姿态 | `Meditate` | 开局只需要灵气积累——让玩家先感受"灵气的自然增长"（cultivation-system.md §Player Fantasy） |
| 战斗区域 | 自动选中当前唯一已解锁区域 | 区域 1 开局默认解锁，无需玩家选择。"只有一个选项时，连选择器都不需要"（screen-flow.md §3.4 灰度预告原则） |
| 战法 | 无（普攻自动） | MVP 无需配置，所有战斗使用默认普攻（semi-auto-combat-system.md §Detailed Design 规则 6） |

**配置后的即时反馈**：

- 切换修炼姿态 → `cultivation.stance_changed` 事件 → 修炼屏 DECISION ZONE 每秒产出数字立即更新
- 切换战斗区域 → `zone.changed` 事件 → 战斗屏 ENEMY ZONE 立即显示新区域的敌人预览 + RIGHT PANEL 战斗日志隔行 "——切换至 幽墟灵谷 ——"
- 所有配置变更通过 EventBus 广播，其他屏刷新对应数据（screen-flow.md §5.2 EventBus + 共享 Service 双轨模式）

### 3.3 决策 3：何时推进

"推进"在 MVP 中有两层含义：区域推进（A）和境界突破（B）。

#### A. 区域推进的信号

| 信号 | 触发条件 | 界面呈现 | 来源 |
|------|---------|---------|------|
| **毕业条件满足** | 当前区域毕业条件达成（如 Zone 1: atk ≥ 180 + Lv.5） | Zone Selector 中当前 zone 标签变为"已毕业"绿色 | mvp-content-progression.md §2.1 |
| **下级区域解锁** | 解锁条件满足（如 Zone 2: Lv.5 + atk ≥ 180） | Zone Selector 中已锁定的 zone tab 从 🔒 变为正常色 + toast "新区域已解锁" | map-progression-system.md |
| **效率对比** | 当前区域 exp/min 低于目标区域 | Zone Selector hover 对比数据 | 战斗系统 + 区域系统 |
| **安全度** | 玩家战力评分 vs 目标区域敌人最高评分，战力比 > 2:1 | tooltip 显示"战力匹配：安全" | combat-calculator + zone-system |

**决策辅助**（game-concept.md §4.2 低频高价值决策）：

- 区域 1 毕业时，Zone 2 的解锁条件（Lv.5 + atk ≥ 180）和当前玩家状态之间的差距通过进度条直观展示——不是"你还不够格"，而是"你即将可以进入幽墟灵谷（atk 差距：只需再升 1 级即可）"。
- 对标 player-journey-map.md §5.1 FTUE 原则 4："用"即将解锁"制造期望，不用"锁定"制造挫折"。

#### B. 境界突破的信号

MVP 境界突破触发点（cultivation-system.md + level-system.md）：

| 境界 | 触发等级 | 全属性倍率变化 | 解锁内容 |
|------|---------|-------------|---------|
| 凡人 → 炼气 | Lv.10 | realm_pool MULT 0→0.20（全属性 +20%） | 修炼姿态 Condense 解锁 |
| 炼气 → 筑基 | Lv.30 | realm_pool MULT 0.20→0.50（全属性 +25% 跃迁） | MVP 终局仪式 |

**突破前的信息支持**：

| 展示信息 | 界面位置 | 内容 |
|---------|---------|------|
| 距下一境界的进度 | HUD Top Strip 境界徽章下方进度条 | "距筑基期还需 5,305 修为"（以修为为单位） |
| 预计时间 | 进度条 hover tooltip | "按当前修炼速率（Condense 姿态），约需 X 小时 Y 分钟" |
| 突破后预览 | 境界徽章 hover tooltip | "筑基期：全属性 +25%，Phase 2+ 将继续开启金丹目标" |

**"再刷一会 vs 现在就去"的决策辅助**：

突破在 MVP 中是等级驱动（Lv.10 炼气、Lv.30 筑基），不是材料收集驱动。玩家看到的进度是等级经验条——不需要额外的材料/仪式。"再刷一会"的决策由自然升级节奏驱动——如果经验条显示"距下一级还需 300 exp（在当前区域约 2 分钟）"，玩家大概率会选择等 2 分钟后看突破演绎，而不是关游戏。

### 3.4 决策 4：何时退出

放置游戏的退出决策与动作游戏截然不同——"退出"不是失败的承认，而是策略的一部分（"挂机让离线收益生效"）。

**自然退出点**（设计上刻意创建的"此任务完成"时刻）：

| 退出点 | 触发条件 | 感知 | 设计如何让退出有方向感 |
|--------|---------|------|---------------------|
| **刚完成一个成就** | 突破境界 / 击败守护关 / 解锁新区域 | "我做到了——接下来挂机让它累积" | 退出前展示离线收益预览——"下次回来时你将有 X 灵气、Y 灵石" |
| **刚设置好挂机状态** | 切换姿态为 Condense、切回最优区域 | "我已为离线做好配置" | 退出前显示挂机预估——"明早回来时预计 Lv.X，距突破还需 Y" |
| **刚查看完离线收益** | 微/短会话自然结尾——收菜完毕、无新决策 | "今天没什么要做的，明天再来" | Left NAV Tab 5 角标熄灭，状态"已确认" |
| **战力卡点前** | 距离突破/推进还需挂机 X 小时 | "现在不做任何事——等离线收益帮我冲过去" | 退出前展示"预计 X 小时后可突破至下一境界" |

**离线收益预期展示**：

对标 player-journey-map.md §5.1 FTUE 原则 5——退出是"FTUE 下半场"的开始，不是结束。

**退出时的展示方案**：

```
MVP 方案（无退出确认弹窗——理由见下方）：
  退出前在 HUD TOP STRIP 常驻一条预测条（关闭前最后一眼可见）：
    "若现在退出，明早 8:00 回来时预计：
     灵气 +~47,000 · 灵石 +~8,200 · 经验 +~36,000
     （按当前姿态 Meditate + Zone 2 计算，离线 10h 衰减至 ~70%）"

  预测条的数值计算公式：
    predicted = online_rate_per_min × 60 × offline_hours_until_8am × multiplier(offline_hours)
    其中 multiplier 取自 mvp-content-progression.md §5.3 离线倍率表
```

**MVP 不做退出确认弹窗的理由**：

- 放置游戏频繁进出（一天可能 5-6 次），每次退出弹"确定退出？"是高摩擦。对标 screen-flow.md §7.2 决策 D-2——离线抽屉不阻塞主屏，退出也不应被弹窗打断。
- 退出 = 开始离线收益，不是"失去什么"。
- 存档自动保存后台持续运行（SaveManager 每 N 秒自动保存），退出时自动触发最终保存——无需玩家手动确认"是否保存"。

**例外**：如果玩家在设置中禁用离线模拟（debug 功能），退出前 toast "离线收益已禁用——退出后不累积收益"。这是设置选择的结果，不是系统要求确认。

### 4.2 微会话产出（< 1 分钟，核心牵引力）

| 产出 | 具体内容 | 满足的核心动机 | 对应 Pillar |
|------|---------|--------------|-----------|
| **离线收益** | 灵气 +N、战斗经验 +N、灵石 +N、材料 +N、战利品列表（数字递增动画） | "我没玩也赚了"——放置核心承诺兑现。SDT Competence（角色在变强，即使我不在） | 4.1 数字增长就是快乐 |
| **离线收益对比** | "这相当于在线挂机 X 分钟的收益" | 锚定价值感——不是简单的数字变化 | 4.2 放置不是无操作 |
| **关键事件（如有）** | "离线期间击败了噬石兽（首次）"、"升至 Lv.12" | 意外惊喜——不操作也可能有成就感事件 | 4.3 刷宝提供惊喜 |

**微会话的 30 秒验证标准**：玩家打开游戏 → 离线抽屉滑出 → 看完前三个数字（灵气/经验/灵石增幅）→ 关闭。全程 30 秒内完成。如果玩家被某个数字激起兴趣（"差一点就能升级/解锁新区域了"），自然转化为短会话。

### 4.3 短会话产出（1–5 分钟，低投入高回报）

| 产出 | 具体内容 | 触发条件 | 对应 Pillar |
|------|---------|---------|-----------|
| **等级提升** | 离线经验+在线继续挂机，触发升级 → 等级徽章变化 + 属性面板全量重算 + toast"升至 Lv.X" | 离线经验 + 在线继续累积达到下一级阈值（mvp-content-progression.md §3.1） | 4.1 数字增长就是快乐 |
| **灵石累积** | 灵石数字每 30s 可见跳动（从 TOP STRIP 或资源屏查看） | 战斗掉落 + 修炼产出（mvp-content-progression.md §5.1） | 4.1 |
| **关键资源接近目标** | 经验、灵石、药材或区域条件接近下一里程碑 → 对应屏幕角标点亮"接近完成" | 离线收益 + 在线挂机累积 | 4.3 回归提供惊喜 |
| **微调配置** | 切换修炼姿态、检查区域选择是否正确 | 玩家自主决策 | 4.2 低频高价值决策 |

**短会话的设计节奏**：

```
0:00 — 离线结算抽屉（自动弹出）
0:30 — 关闭抽屉，查看资源屏 → 发现距 Lv.11 只差一小段经验
1:00 — 决定留在当前区域继续挂机（Zone 2 的 exp/min 更合适）
2:00 — 查看等级进度："距 Lv.11 还需 800 exp（约 1 分钟）"
2:30 — 等待最后一分钟，升级到 Lv.11 → toast "升至 Lv.11!"
3:00 — 查看新属性（atk +X, def +Y），设置 Meditate 姿态 → 退出
       → 退出前看到预测条："明早回来时预计 Lv.12，距突破还需 15,000 exp"
```

### 4.4 标准会话产出（5–20 分钟，有目标的深度投入）

|------|---------|---------|------------|
| **境界突破** | 炼气期（Lv.10）→ 全属性 +20%（atk 524→755，HP 6,352→8,708），金色印章 3s burst，Condense 姿态解锁 | **峰值体验**——MVP 第二大跃迁感（player-journey-map.md §1.4） | Sensation + Challenge + Fantasy |

**标准会话的典型决策路径**（与 §3 四个决策对应）：

```

  ↓

  ↓
决策 2：怎么配（检查修炼姿态是否最优；如果修为差距大，切换到 Condense）
  ↓
决策 3：何时推（查看等级进度 "距 Lv.X 还需 Y exp"、查看区域条件 "atk 还差 Z"）
  ├→ 条件满足 → 推进（切区域 / 挑战守护灵）
  └→ 条件不足 → 继续挂机 → 决策 4
  ↓
决策 4：何时退出（达到一个自然退出点，设置挂机状态 → 退出前看预估）
```

以自然日为周期，设计玩家从早到晚的典型行为模式。此节奏基于"规律回访的上班族"画像绘制——MVP 最普适的目标玩家（对标 mvp-content-progression.md §5.3 离线倍率覆盖的自然离线间隔）。

### 5.1 一日典型节奏表

| 时段 | 时间窗 | 典型行为 | 会话类型 | 设计支持 |
|------|--------|----------|----------|----------|
| **早晨** | 7:00–9:00 | 起床检查手机/PC——收离线收益（8-10h 离线积累）、查看昨晚挂机成果、快速换到最优区域 | 微/短会话（1-3 min） | 离线结算抽屉摘要 + "自昨日起"变化综述 + 快速换区（Zone Selector 一键切换）。离线衰减 90%→70%（mvp-content-progression.md §5.3 8-16h 区间），收益仍可观 |

| 维度 | 早晨 | 午间 | 晚间 | 睡前 |

| **Toast 优先级** | OFFLINE > BREAKTHROUGH > WARNING | WARNING（满仓溢出等）> RARE > INFO | RARE（Phase 2+ 高价值事件）> BREAKTHROUGH > WARNING | INFO（保存成功等） |

## 6. 回归牵引机制

本节定义"玩家关闭游戏后为什么会再打开"的所有设计机制。MVP 无每日目标系统（系统 64），回归牵引力由以下四层机制共同提供。

### 6.1 第一层：离线收益预览（退出时展示）

**机制**：退出前在 HUD TOP STRIP 常驻预测条，展示下一次回归时的预期收益。

**预测条内容**：
```
"若现在退出，下次回来时（按当前区域 Zone X + 姿态 Meditate 估算）：
 灵气 +~XX,XXX · 灵石 +~X,XXX · 经验 +~XX,XXX
（离线 X 小时，收益倍率 Y%）"
```

**牵引力来源**：具体化的数字预期——"明早回来就能有 47,000 灵气"比"你不玩也能涨"更有牵引力。这利用了心理学中的**蔡格尼克效应**——"我已经启动了一个过程，但我还没看到结果"。

### 6.2 第二层：挂机进度预估（"距离下次突破还需约 X 小时"）

**机制**：修炼屏境界徽章下方的进度条 hover tooltip 展示时间估计。

**内容**：
```
"距筑基期还需 5,305 修为
按当前修炼速率（Condense 姿态，~180 修为/min）：
  在线挂机约需 30 分钟
  离线 (8h 衰减至 70%) 约需 25 小时"
```

**牵引力来源**：玩家知道"大概什么时候会突破"——形成自然的回归时间锚点。"明天晚上差不多能突破"比"总有一天能突破"更能触发回访。

### 6.3 第三层：未完成目标的记忆（"你距离 Zone 3 毕业还差 atk +150"）

**机制**：系统追踪玩家的"进行中目标"，在退出时和回归时展示。

**MVP 的进行中目标追踪**：

| 目标类型 | 检测方式 | 展示位置 | 示例 |
|---------|---------|---------|------|
| **区域推进** | 检查当前区域毕业条件 vs 玩家状态（如 atk=1,550 距毕业条件 1,700 差 150） | 退出前预测条下方 + Zone Selector tooltip | "距 Zone 2 毕业还需要 atk +150（约还需升 1 级）" |
| **资源储备** | 检查灵石/药材/经验的当前值 vs 下一里程碑建议值 | 资源屏 + 退出前预测条 | "当前灵石储备足够支撑下一阶段，建议继续推进 Zone 3" |
| **境界突破** | 检查当前等级 vs 下一境界触发等级 | 修炼屏境界徽章 hover | "距炼气期突破还需升至 Lv.10（当前 Lv.8，还需 ~13,000 exp）" |
| **守护灵首杀目标** | 守护灵击败状态 | Zone Selector tooltip（仅在 Zone 3） | "矿脉守护灵尚未击败——推荐 Lv.28+ 挑战" |

**牵引力来源**：对标 player-journey-map.md §5.1 FTUE 原则 4——"用"即将解锁"制造期望，不用"锁定"制造挫折"。玩家关闭游戏时脑中带着一个具体的 mini-goal（"还差 atk +150"、"距 Lv.15 还需约 18 分钟"），回归的动机比"看看涨了多少"强得多。

### 6.4 第四层：新内容暗示（"Phase 2+ 将开放金丹与装备刷宝"）

**机制**：在未解锁的系统/区域/境界上始终展示"即将到来"的预览信息。

**MVP 中的新内容暗示实例**：

| 位置 | 暗示内容 | 玩家看到的 |
|------|---------|-----------|
| **Zone Selector** | 未解锁区域的名称 + 解锁条件 tooltip | Zone 2 标签灰度 + 锁定图标 + hover:"需 Lv.5 + atk ≥ 180" |
| **修炼屏姿态 Modal** | Closed Door 姿态灰度 + 🔒 + tooltip:"闭关——即将解锁" | 让玩家感知到"修炼不止打坐一种姿态，后面还有更高级的" |
| **境界徽章** | 当前境界下方一行小字 | "下一境界：筑基期——道基铸就，肉身脱胎换骨"（world-skeleton.md §2.1） |
| **资源屏路线图卡片** | Phase 2+ 装备刷宝预告 + 不可交互说明 | "装备品质、词条与套装将在后续阶段开放" |

**牵引力来源**：对标 game-concept.md §4.6 "渐进叙事展开"——世界逐步展开的承诺。玩家不是看到"你已解锁全部内容"，而是始终看到"前方还有更大的世界"。

### 6.5 回归牵引与流失风险对照

回归牵引机制与 player-journey-map.md §4.1 五大流失点的交互：

| 流失点 | 回归牵引如何缓解 | 对应的机制层 |
|--------|----------------|------------|
| #1: 不知道做什么（0:01） | 修炼界面的呼吸光晕按钮 → 灵气数字跳动 → 灵气接近 100 时颜色变金（暗示阈值） | （FTUE 层，不在回归机制中。参见 player-journey-map.md §5.1 原则 1） |
| #2: "只有自动战斗？"（0:15） | 首次击败每种敌人有 toast + 区域 2 锁定状态始终可见 → 制造"前方还有内容"的期待 | 第四层：新内容暗示 |
| #3: 离线后无牵引（D1-2） | 离线收益抽屉 + "下一次里程碑预告" + "距突破至筑基期还需 X" + 退出前预测条 | 第一层 + 第二层 + 第三层 |
| #4: 中后期 grind 感（D4-6） | "距离守护灵还需 X 级 / atk +Y"的进度感知 + Zone 3 首杀与毕业目标 | 第三层：未完成目标记忆 |
| #5: 终局空虚（D7+） | "等待 Phase 2"信息设计——开发路线图（系统 225）+ 稀有收集 sub-goal + 灵石储备提示 | 第四层：新内容暗示（Phase 2 预告） |

## 7. 会话结束体验

### 7.1 退出确认策略

**MVP 方案：无退出确认弹窗，直接退出。**

玩家关闭窗口 / Alt+F4 / 切出游戏 → 触发自动保存 → 进程退出。无弹窗、无确认、无"你确定要退出吗？"。

**设计理由**：

| 理由 | 详细说明 |
|------|---------|
| **放置游戏的进出频率** | 玩家的日常使用模式是一天打开 5-8 次（早晨/午间/下午茶/晚餐后/睡前）。每次退出都弹确认 = 每天至少多出 5 次无意义的交互摩擦 |
| **退出 = 开始离线收益** | 玩家退出不是因为"不想玩了"，而是因为"让离线收益生效"——退出是游戏循环的正常环节，不是异常中断 |
| **自动保存保证安全** | SaveManager 以固定间隔（如 60s）自动保存 + 退出时触发最终保存（save-screen.md §7.3）。玩家不需要手动确认"是否保存"，进度始终安全 |
| **逆向操作始终可用** | 玩家关游戏 5 秒后反悔 → 重新打开 → 离线收益 5 秒，无任何损失。退出和回来一样轻松 |

**例外场景**：

- 如果自动保存损坏/磁盘满 → toast "保存失败，建议稍后重试"（bottleneck_red 8s），不阻止退出
- 如果玩家长时间未操作（AFK > 30 分钟）→ 游戏本身不自动退出或锁屏（避免打断离线感知）

### 7.2 存档状态的可信感

玩家退出时不需要看到"保存成功"的确认（MVP 自动保存为静默后台操作，screen-flow.md §7.3），但存档状态在玩家回归时通过存档屏展示：

- 存档屏始终可访问（LEFT NAV Tab 4 始终 VISIBLE + ACTIVE，screen-flow.md §4.2 Tab 4 状态转换）
- 存档屏展示最近自动保存时间戳（如"上次自动保存：23:15:42"）——玩家随时可以确认"我的进度被保护着"
- 读取存档成功时 toast "存档读取成功"（burst_gold 4s）

**长期信任构建**（通过多次 session 的连贯体验）：

- 玩家每次回归看到离线收益数字与退出前的预测条一致 → "游戏在我不在时确实在运转"
- 玩家回归后数字与离开时连续（不是重置或跳跃） → "我的进度是连续且真实的"
- 存档屏始终可见且可手动保存 → "我随时可以加固我的进度安全"

### 7.3 退出前的"期待锚点"

"期待锚点"是玩家退出前最后一眼看到的信息——它的作用是让玩家在关闭游戏后记住"下次回来我要做什么"。

**MVP 期待锚点层次**：

```
优先级 1（若有未完成目标）：最具体的 mini-goal
  "距 Zone 2 毕业还需要 atk +150
   当前区域预计再升 1 级即可尝试"

优先级 2（若接近突破）：自然的时间锚点
  "距炼气期突破还需约 1,200 exp
   按当前速率，下次回来时即可突破"

优先级 3（日常节奏）：数字预期的承诺
  "明早回来时预计：
   灵气 +~47,000 · 灵石 +~8,200 · 经验 +~36,000"

优先级 4（无特殊状态）：基础的离线承诺
  "若现在退出，下次回来时仍会获得离线收益"
```

**展示位置**：退出前最后一眼——修炼屏底部或 TOP STRIP 预测条（始终可见，不需要玩家主动点击任何按钮才能看到）。

**设计原则**：

- "期待锚点"不是弹窗——不给最后一秒增加交互摩擦
- "期待锚点"必须是具体的（数字/时间/进度），不是模糊的（"你会变得更强"）
- "期待锚点"在实现时通过系统状态动态生成（不是硬编码文案）

## 8. 为 225 系统终局蓝图保留的扩展标注

MVP 会话循环的设计基于"单角色散修"的最小系统集。当 225 系统蓝图中的后续系统陆续实装时，会话循环将发生根本变化。以下标注哪些系统会改变哪些会话环节。

### 8.1 对"决策流"的扩展

| 未来系统（系统编号） | 影响决策 | 变化 |
|-------------------|---------|------|
| **队伍系统 (84)** | 决策 2：如何配置 | 从"单角色姿态选择"扩展为"队伍阵容 + 站位 + 阵法配置"——配置决策量增加 5-10×。需要队伍配置屏 + Build 评分系统 (87) 辅助决策 |
| **战法策略系统 (90)** | 决策 2：如何配置 | 从"普攻自动"扩展为"条件规则 + 技能冷却 + 消耗管理"——战斗从纯观察变为"配置后观察结果反馈循环"，大幅降低流失点 #2（"只有自动战斗？"） |
| **宗门系统 (147-165)** | 决策 1：去哪里 | 宗门建筑/生产链/科技树解锁后，"去哪"不再只是 zone 选择——需要同时管理洞府/宗门资源流 |
| **远征系统 (172-173)** | 决策 1：去哪里 | 增加"远征路线规划"作为"去哪"的第三选项（zone / 挂机 / 远征） |
| **飞升系统 (193)** | 决策 3：何时推进 | 突破不再只是等级触发——需要收集飞升材料 + 选择保留项 + 确认进入新世界层。成为 MVP 中"退出"决策之上的大层级决策 |
| **轮回系统 (194)** | 决策 3 + 决策 4 | 引入"什么时候轮回"的最高级退出决策——退出前不止是设置挂机区域，而是决定"我要不要抛弃一切重新开始" |

### 8.2 对"奖励节奏"的扩展

| 未来系统（系统编号） | 变化的奖励层 | 新产出 |
|-------------------|------------|--------|
| **装备词条系统 (110) + 套装系统 (112)** | 短/标准会话 | 装备从"凡品/精良/稀有"的品质层级扩展为"品质 + 词条随机 + 套装效果"——刷宝惊喜从 2 级跳（品质跳）变为多层嵌套惊喜 |
| **灵宠系统 (131-136)** | 标准/长会话 | 新增"灵宠孵化/进化"作为新的奖励维度——不局限于装备和数字 |
| **天劫系统 (181, 192)** | 标准/长会话 | 境界突破从"等级触发 + 演绎"变为"天劫挑战 + 风险收益决策"——突破本身成为一个玩法事件而非被动触发 |
| **图鉴系统 (128, 200)** | 全会话 | 新增收集型奖励——击败 N 次某种敌人、收集齐全区掉落、图鉴全亮——为 Explorer/Achiever 提供长尾目标 |
| **每日目标系统 (64)** | 微/短会话 | 每日回归后第一个看到的是"今日目标：击败 20 只晶石傀儡"——为微/短会话提供 immediate mini-goal，替代当前依赖的"玩家自驱查看" |

### 8.3 对"每日节奏"的扩展

| 未来系统（系统编号） | 节奏变化 |
|-------------------|---------|
| **生产队列 (54) + 在线生产模拟系统 (63)** | 离线不只是结算战斗——生产队列在离线期间完成（丹药炼制、装备锻造、材料加工）。早晨回归时看到的不仅是"战斗收益"，还有"生产车间完成了 3 批丹药"——深化"我离开时世界在运转"的体感 |
| **宗门任务 (160)** | 引入"周期性任务"——不是 dailies 的强迫性设计，而是宗门居民自动接取并完成，玩家回归时查看完成报告。让"有人在帮我干活"的感受强化 Relatedness |
| **兽潮/秘境 (175-176, 171)** | 晚间深度 session 增加"挑战模式"——不只是挂机观察，而是需要主动参与的高风险高回报内容（对标 game-concept.md §8 中操作频率"中高"的模式） |

### 8.4 会话循环与阶段路线图的对应

对标 game-concept.md §12 阶段路线图，不同阶段对会话循环的影响：

| 阶段 | 新增系统族 | 会话循环核心变化 |
|------|----------|----------------|
| **Phase 2（资源链与建筑）** | 洞府、生产队列、生活技能、NPC 市场 | 短会话新增"查看生产线状态"，退出前新增"设置生产队列" |
| **Phase 3（宠物与宗门）** | 灵宠、宗门、科技树、居民 | 标准会话决策流新增"宗门管理"，回归后不止看离线收益还看宗门报告 |
| **Phase 4（技能与天劫）** | 技能系统、战法策略、天劫、突破仪式 | 决策 2（怎么配）从"单选项"扩展为"技能选择 + 战法规则"，战斗观察层深度剧增 |
| **Phase 5（秘境与远征）** | 秘境、远征、阵盘、Deckbuilder | 长会话新增"远征路线规划"和"秘境探索"——从"挂机/切换"扩展为有策略选择的主动内容 |
| **Phase 6+（飞升与诸天）** | 飞升、轮回、合道、世界层级 | 会话循环出现大层级"退出决策"——轮回/飞升不是一次 session 内的决策，而是跨多日/周的大周期 |

## 9. 附录 — 关键设计决策

| # | 决策 | 选择 | 理由 |
|---|------|------|------|
| D-1 | 退出确认弹窗 | **不做，直接退出** | 放置游戏频繁进出，弹窗是摩擦。退出 = 开始离线收益，不是异常中断 |
| D-2 | 离线收益展示顺序 | **先总览（抽屉）→ 再详情（全屏）** | 分两层：速览满足 30s 微会话，全屏满足深度查看需求。对标 screen-flow.md 决策 D-2 |
| D-3 | "期待锚点"的展示时机 | **退出前静态展示，不做弹窗** | 最后一秒不加交互摩擦。玩家自己看到就好 |
| D-4 | 预测条数值 | **基于当前配置 + 离线倍率曲线实时计算** | 数值必须与玩家实际回归时结算的值一致——否则损耗信任 |
| D-5 | 在线切回时的日志重放 | **摘要（后 5 行）+ 最多 3 条关键事件 toast** | 不全量重放——玩家不需要看到每一条战斗日志。只告诉"最重要的事" |
| D-6 | MVP 无每日目标系统 | **通过期待锚点 + 未完成目标追踪实现等效回归牵引** | 每日目标系统（系统 64）是 Phase 4 的内容——MVP 通过更轻量的机制达到相似效果 |
| D-7 | 会话类型分类 | **按时长和意图 4 分类（微/短/标准/长）** | 基于放置游戏的现实使用模式——不是所有玩家都是"坐下来深度玩 30 分钟" |
