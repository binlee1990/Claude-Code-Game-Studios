# 玩家体验路径图 (Player Journey Map)

> 版本：v1.0
> 创建日期：2026-05-05
> 范围：MVP（阶段 1）—— 3 区域、单角色、5 境界、30 系统
> 关联文档：`design/gdd/game-concept.md`（§3、§4、§7、§11）· `design/narrative/world-skeleton.md` · `design/balance/mvp-content-progression.md` · `design/gdd/cultivation-system.md` · `design/gdd/map-progression-system.md` · `design/ux/hud.md`

---

## 摘要

本文档描述 MVP 玩家从首次打开游戏到完成首周回归的完整体验路径。核心路径分为 8 个时间节点，覆盖从 0:00（首次修炼）到 Day 7+（MVP 终局与内容边界）。每个节点从触发条件、玩家所见、所做、所感、系统状态变化五个维度展开。本文同时包含情感曲线分析、关键决策点、流失风险评估，以及标对 game-concept.md §4 设计支柱的 FTUE 核心原则。

> **核心设计假设**：MVP 不包含宗门/宠物/秘境/多人/付费系统。3 个区域、单角色，Lv.1–30 只把凡人→炼气→筑基写成可实现闭环；金丹/元婴、装备品质、词条、强化、套装与战法配置均为 Phase 2+ 牵引。战斗为半自动普攻循环，玩家的主要决策是修炼姿态、挂机区域和回归时机。

---

## 1. 体验路径时间线

### 1.1 0:00–0:05 首次打开游戏：从标题到首次修炼

> **节点定位**：onboarding 第一步，建立世界感和"数字在跳"的初始欲望。

| 维度 | 描述 |
|------|------|
| **触发条件** | 玩家启动游戏，点击"开始修炼" |
| **玩家看到的** | 标题画面展示水墨风格"天地不仁，万物刍狗。唯修者，逆天而行。"（参见 world-skeleton.md §4.1），点击后进入修炼界面。HUD 仅显示 5 项（灵气/修为/当前区域/战斗状态点/设置入口，参见 hud.md §Dynamic Behaviors 渐进解锁），修炼界面展示沉浸式引导文本："你盘膝坐定，闭目凝神。山风穿过窗棂，带来第一缕若有若无的天地灵气。"（参见 world-skeleton.md §4.2）。灵气数字开始缓慢跳动（每秒+1），修为条为空。 |
| **玩家做的** | 观察灵气跳动，可能在 3-5 秒后注意到手动修炼按钮并点击（每次获得 5 灵气，0.5s 冷却，参见 cultivation-system.md §Tuning Knobs），点击若干次后灵气突破 100。 |
| **玩家感受到的** | 初始好奇心 → "数字在变"的基本满足（Pillar 4.1 数字增长就是快乐）→ 触碰到阈值后的轻微惊喜——"体内灵气似乎冲破了什么关窍……你隐约感知到山中有异动。"（world-skeleton.md §6.1）战斗系统解锁。 |
| **系统状态变化** | 修炼系统：默认进入 `Meditate` 姿态（cultivation-system.md §Detailed Design 规则 2）。灵气累积至 ≥100。HUD 解锁"区域选择器"元素（仅显示区域 1：青丘山林）。战斗系统由 hidden → unlocked。 |

**此阶段的设计意图**：不出现任何教程弹窗、箭头提示或文字说明"你应该做什么"。玩家通过灵气数字自然跳动建立因果直觉——"我什么都不做，灵气就在涨；我点了，灵气涨更快。" 这是渐进叙事展开的首次落地（Pillar 4.6，参见 game-concept.md §4.6、world-skeleton.md §6.1）。

### 1.2 0:05–0:30 第一个发现：从修炼到首次战斗

> **节点定位**：首次行为扩展，验证"修炼不是终点"。

| 维度 | 描述 |
|------|------|
| **触发条件** | 灵气 ≥ 100，解锁战斗系统 |
| **玩家看到的** | HUD 顶栏出现战斗状态点（○ 待机）。区域选择器显示"区域：青丘山林 ▼"。战斗日志面板在右侧激活（hud.md §RIGHT PANEL）。首次进入战斗时，叙事钩子文本："村中猎户警告过你：山中有妖。此刻，灌木丛后的那双猩红眼睛，终于让你明白了这句话的分量。"（world-skeleton.md §4.3）。战斗日志开始刷新：普攻、击杀、经验+8、灵石+1（参见 mvp-content-progression.md §2.2.2，石傀儡首次击败）。 |
| **玩家做的** | 观看战斗日志自动刷新（纯观察，无需操作）。可能在了解系统后手动点击区域名称尝试切换区域（此时只有青丘山林可选）。首次击败石傀儡获得 8 exp，在约 2 分钟内升至 Lv.2。 |
| **玩家感受到的** | "原来我还能走出这片林子？"（world-skeleton.md §6.2）——世界扩展的初次惊喜。秒杀石傀儡（Lv.1 玩家 atk=10 vs 石傀儡 HP=30，约 3 次普攻击杀，参见 mvp-content-progression.md §2.2.1）建立早期胜任感。"自动战斗"的发现感——不需要我操作就打赢了。 |
| **系统状态变化** | 战斗系统：状态切换为 ●（战斗中）。自动战斗在青丘山林开始循环。玩家 Lv.1→Lv.3 连升（每级 ≤2 分钟，参见 mvp-content-progression.md §6.4 关键节奏落点）。首次灵石入账（HUD 灵石 row 解锁）。经验条在等级徽章下方出现（hud.md §TOP STRIP）。 |

**此阶段的设计意图**：前 30 分钟必须有可感知的"变强"——3 级连升、弱敌秒杀、灵石从无到有。战斗不要求玩家理解策略，只要求它"看得见、打得赢、有东西掉"。

---

### 1.3 0:30–2:00 第一个循环：区域 1 熟悉 → 等级提升 → 首次资源掉落

> **节点定位**：玩家跑通第一个完整的"挂机→战斗→掉落→升级"闭环。

| 维度 | 描述 |
|------|------|
| **触发条件** | 持续在青丘山林自动战斗，Lv.3+ |
| **玩家看到的** | 战斗日志稳定刷新，击败敌人种类从石傀儡扩展至野兔妖、枯木精、毒雾蝠。灵石数字累积至 50+。首次掉落药材（`herb`）时资源屏解锁（参见 mvp-content-progression.md §2.2.2）。装备碎片不进入 Sprint 12 reward bundle，只作为 Phase 2+ 路线图预告。 |
| **玩家做的** | 持续的被动观察——战斗日志如溪流般滚动（game-concept.md §7.1 10 秒循环）。可能在 Lv.5 附近发现"初生蛇妖"略微变强（atk=38 vs Lv.4 玩家 def=50，造成少量但可见的伤害），注意到"不是所有敌人都一样"。查看灵石数量并思考"这东西能干嘛"。 |
| **玩家感受到的** | "我在变强"的稳态满足感。从 Lv.3 到 Lv.5 的等级增长在 ~12 分钟内完成（累计 ~340 exp，参见 mvp-content-progression.md §3.1），节奏快但不过于仓促。蛇妖的出现制造第一个微小的不对称——"这个怪好像难一点，但奖励也更多"（蛇妖 exp=52 vs 石傀儡 exp=8）。 |
| **系统状态变化** | Lv.5 达成。atk ≥ 180 达成。Zone 2（幽墟灵谷）从灰色锁定变为亮起状态："当前可进入，建议 Lv.5+ / atk ≥ 180"（map-progression-system.md §Detailed Design 规则 2）。区域选择器中出现可切换的第二选项。 |

**此阶段的设计意图**：第一个 2 小时内出现可感知的卡点（Lv.4→5 蛇妖略难），但玩家有明确的突破路径（继续挂机升级即可，参见 mvp-content-progression.md §6.2 卡点 1）。区域 2 的解锁条件是"恰好在你准备好时呈现"，制造"我该换地方了"的自然决策点。

---

### 1.4 2:00–6:00 首次突破：炼气期突破 → 新姿态解锁 → 区域 2 发现

> **节点定位**：MVP 第二大跃迁感，境界突破带来的质变体验。

| 维度 | 描述 |
|------|------|
| **触发条件** | Lv.10 达成（凡人→炼气突破），累计 exp ~3,791（参见 mvp-content-progression.md §3.1） |
| **玩家看到的** | 等级徽章文字变化："Lv10 · 炼气"。HUD 全屏 "burst_gold" 印章 3s 收缩（hud.md §CENTER CONTENT 状态切换）。属性面板全体 +20%（realm_pool MULT 0→0.20，参见 mvp-content-progression.md §3.2）：HP 6,352→8,708、ATK 524→755、SPD 86→284。战斗速度明显加快。修炼姿态新增 `Condense` 选项（将灵气转为修为），修炼系统解锁更多姿态。区域 2（幽墟灵谷）选择亮起，推荐进入。 |
| **玩家做的** | 体验突破带来的"特效反馈"后，可能尝试以下顺序之一：① 停留在青丘山林继续挂机（已形成习惯）② 手动切换到幽墟灵谷（"看看新地方什么样"）③ 查看新增的姿态选项尝试 Condense。大概率选择切换区域。 |
| **玩家感受到的** | **峰值体验 1：境界突破的快感**——炼气突破是玩家第一次明确体会到"我变强了"的仪式化瞬间。数字跳变（atk 从 524 跳到 755，+44%）、视觉反馈（金色印章）、新选项（姿态+新区域）——三个层面的"变强证据"叠加，制造强成就感。区域 2 的视觉/氛围差异（参见 world-skeleton.md §3.2：幽墟灵谷"灵气浓度骤升，却混杂着一丝诡异的寒意"）提供新鲜感。 |
| **系统状态变化** | 境界标签：凡人→炼气。全属性 ×1.20。Zone 2 状态 locked→unlocked。修炼系统姿态选项扩展（新增 Condense）。Lv.10→Lv.15 的升级需求开始陡峭（Lv.10 需 1,514 exp，Lv.15 需 4,309 exp，约 3× 增幅）。 |

**此阶段的设计意图**：炼气突破是 MVP 第一个真正的"仪式化时刻"——境界突破应有视觉、数值和选项三方面的反馈，缺一不可。突破后玩家立即有"我该做什么"的新目标（新区域、新姿态），避免突破后空虚。

---

### 1.5 6:00–24:00 首个回归日：首次离线 → 离线结算 → 回归收益体感

> **节点定位**：验证"离线不是惩罚是回报"的核心设计假设。

| 维度 | 描述 |
|------|------|
| **触发条件** | 玩家首次退出游戏（任何时长），再次打开游戏 |
| **玩家看到的** | 标题画面 → 进入游戏。HUD TOP STRIP 状态点旁出现"📦 N"角标（hud.md §OFFLINE DRAWER）。点击后从右侧滑入 480px 的离线结算 Drawer（P-NAV-04），展示：灵气 +X、战斗经验 +Y、灵石 +Z、药材 +W、战利品列表（参见 offline-reward-settlement 系统）。如果离线 ≤4h，收益为 100% 在线等效（mvp-content-progression.md §5.3）。如果离线 4-8h，收益标注"衰减 90%"。 |
| **玩家做的** | ① 查看离线收益面板，感受"我没玩也赚了多少" ② 检查角色是否升级（离线战斗经验可能触发升级）③ 查看灵石/药材是否明显增长。可能继续挂机或切换到下一个游戏环节。 |
| **玩家感受到的** | **峰值体验 2：离线的惊喜**——"我离开时世界仍在运转，我回来时角色更强、仓库更满"（game-concept.md §3 玩家幻想）。首次离线回来的体感是验证"放置"核心承诺的关键时刻。如果收益太少（玩家体感"根本没涨"），会严重冲击留存信心。如果收益太多（"不用玩也能通关"），会削弱在线动力。 |
| **系统状态变化** | 离线结算完成。`offline.settled` 事件触发。灵气/经验/灵石/药材各资源增量写入。可能触发等级升级。离线收益入口角标消失（玩家查看后）。 |

**此阶段的设计意图**：首次离线回归是"留存的第一道坎"。设计上必须让玩家看到**有意义的但不替代在线游戏的**收益。4h 内 100% 无衰减的策略覆盖午休/通勤/睡觉等自然间隔，确保"每次回来都有获得感"。离线结算 Drawer 的展示设计至关重要——必须有"战利品展开"的仪式感，不是简单几行数字。

### 1.6 Day 2–3 推进节奏：区域 2 farm → Zone 3 准备 → Phase 2+ 期待

> **节点定位**：进入稳定的日常回访节奏。

| 维度 | 描述 |
|------|------|
| **触发条件** | 玩家多次回访（自然日 Day 2-3），Lv.10-20 区间 |
| **玩家看到的** | 幽墟灵谷的敌人种类形成熟悉感（影狼→绞杀藤→林地怨灵→毒牙蛛→噬石兽）。灵石每分钟产出约 60（Lv.10，参见 mvp-content-progression.md §2.3.3）。掉落仍以 exp / lingshi / herb 为主，但敌人名、区域文本和路线图开始暗示妖兽骨、精良碎片等 Phase 2+ 刷宝内容。Lv.15 左右遭遇噬石兽——atk=340 对防御不足的玩家造成可见压力（卡点 3，参见 mvp-content-progression.md §6.2）。 |
| **玩家做的** | 每日回访的核心动作：① 查看离线收益 ② 检查是否升级 ③ 手动推图（尝试新解锁区域）④ 查看下一里程碑提示。装备合成不在 Sprint 12 MVP 内，资源屏只提示"装备刷宝将在后续阶段开放"。 |
| **玩家感受到的** | 从"发现新东西"过渡到"稳定 routine"的舒适感。噬石兽的轻微卡点不让人沮丧（升到 Lv.15 即可碾压），而是制造"我还差一点点"的挂机动力。离线收益成为每日期望——"明天回来就能推进荒殒战场了吧"。 |
| **系统状态变化** | 稳定在区域 2 farm。灵石累积速度加快（~60/min）。Lv.15 解锁 Zone 3（荒殒战场）。丹药/炼丹/装备系统尚未解锁（Phase 2+），但药材与灵石持续累积暗示"将来会有用"。 |

**此阶段的设计意图**：Day 2-3 是留存验证的关键窗口。玩家从"新鲜感"过渡到"习惯性回访"——这个过渡需要足够的期望牵引（Lv.15 不远了 + Zone 3 在望 + Phase 2+ 刷宝预告）。如果这个阶段玩家打开游戏发现"没什么可做的"，流失概率极高。

---

### 1.7 Day 4–7 中程目标：区域 3 解锁 → 筑基终局 → MVP 守护灵挑战

> **节点定位**：MVP 最深内容层，最高难度的验收。

| 维度 | 描述 |
|------|------|
| **触发条件** | 玩家稳定回访，Lv.20-30 区间，Zone 3（荒殒战场）farm |
| **玩家看到的** | 荒殒战场的视觉氛围剧变——"焦黑的古战场，破碎的法宝碎片散落荒原，巨大的剑痕切开了山脊"（world-skeleton.md §3.3）。敌人不再是"野兔妖"级别——晶石傀儡、晶蝠、矿脉精魄、晶牙龙蜥，每个都有完整战斗数值。玄晶碎片和精良装备只作为未来刷宝预告，不进入当前掉落包。Lv.28→30 遭遇矿脉守护灵（MVP 终局 Boss，战力评分参见 mvp-content-progression.md §2.4.1）。 |
| **玩家做的** | 核心决策循环加剧（参见 game-concept.md §4.2）：① 选择留在 Zone 3 高风险高收益还是退回 Zone 2 安全 farm ② 主动挑战守护灵（手动推图）vs 等待离线自然升级 ③ 查看 Phase 2+ 路线图，理解当前灵石/药材储备的未来用途。首次击败守护灵触发 `zone_mine.first_clear` 事件。 |
| **玩家感受到的** | **峰值体验 3：击败守护灵的终局成就感**——"千年前就有人在这里死战。我得比他们更强，或者更聪明。"（world-skeleton.md §6.2）。守护灵的战力压制（elite 标签 + 高 ATK/HP）迫使玩家认真对待 MVP 唯一的"真挑战"。击败后的高额 exp / 灵石 / 药材反馈提供强奖励；Lv.29→30 的筑基突破（全属性 +25% 跃迁）作为 MVP 最终仪式。 |
| **系统状态变化** | Zone 3 farmable 建立。Lv.30 筑基境界突破（realm_pool MULT 0.20→0.50）。守护灵首杀完成。资源存储上限大幅提升（灵气上限达 1,000,000）。金丹/元婴保持为 Phase 2+ 目标，不写作 MVP 已达状态。 |

**此阶段的设计意图**：MVP 的"内容终点"需要被设计为一个自然的山峰，而非突然的断崖。守护灵是刻意设置的"需要积累才过"的卡点（卡点 5，参见 mvp-content-progression.md §6.2），让 MVP 通关有达成感。当前阶段不依赖装备合成；不同 playstyle 的差异来自区域选择、离线节奏和是否提前挑战守护灵。

---

### 1.8 Day 7+ 终局与等待：筑基达成 → MVP 内容边界 → 为 Phase 2 留期待

> **节点定位**：MVP 内容消耗完毕后的"等待台"。

| 维度 | 描述 |
|------|------|
| **触发条件** | Lv.30 达成 + 守护灵击败 + 三区域全清 |
| **玩家看到的** | 区域选择界面三区域全部 farmable（绿色标记）。修炼姿态全部解锁（meditate/condense/closed_door，后者 MVP 预留，见 cultivation-system.md §Detailed Design 规则 2）。灵石累积加速（Zone 3 约 300/min），但灵石消费渠道有限。可能注意到一些灰色/锁定的系统入口（宗门、秘境、装备刷宝、金丹境界）作为 Phase 2+ 路线图预告。 |
| **玩家做的** | 可能的三个行为分支：① **优化党**：继续在 Zone 3 挂机，追求更高资源储备 ② **等待党**：每日回访收割离线收益，查看灵石数量接近百万级 ③ **倦怠党**：感觉"好像没什么新东西了"，降低回访频率。 |
| **玩家感受到的** | **低谷风险：MVP 内容边界暴露**——如果玩家在 Day 7 没有任何新的系统解锁或目标指引，可能产生"就这样？"的失落感。关键变量是：① 大数字增长是否还在提供快感（Pillar 4.1）② 守护灵首杀是否有足够仪式感 ③ 游戏是否给了明确的 Phase 2+ 预告以制造期待。 |
| **系统状态变化** | 全系统 MVP 状态：灵气产出最高（Zone 3 300+ lingqi/sec），经验累积（Lv.30 硬上限），境界=筑基，灵石=几百万级，离线结算稳定运行。开发路线图系统（系统 225，225 蓝图预留）可在此阶段以"即将解锁"的形式展示后续计划。 |

**此阶段的设计意图**：MVP 不应该是"内容耗尽 → 流失"的断崖。设计上需要三条缓冲线：① 大数字的继续增长（即使等级封顶，灵石/药材仍在累积，为 Phase 2 升级留"存货"的期望）② 守护灵首杀后的资源储备目标 ③ "等待 Phase 2"的信息设计——不是空白的"敬请期待"，而是展示具体的下一阶段系统名称和模糊预览（如"宗门系统：收弟子、建洞府、开宗立派"），将"内容耗尽"转化为"蓄力等待"。

## 4. 流失风险评估

### 4.1 五大流失点

#### 流失点 1：0:01–0:02 —— 进入游戏后不知道做什么

| 维度 | 分析 |
|------|------|
| **流失时机** | 标题画面 → 点击"开始修炼"后的前 30 秒 |
| **流失画像** | 缺少耐心、期望"动作感"的玩家；被其他游戏训练为"先弹出三个教程箭头"的玩家 |
| **失败模式** | 玩家看到灵气数字跳动但不知道它意味着什么，不知道"接下来干嘛"，也没有明确的引导按钮（设计上刻意没有教程箭头），产生"这游戏没做完？"的困惑 |
| **根本原因** | 渐进叙事展开（Pillar 4.6）的"极简开局"设计刻意省略了教程和显性引导。对于习惯了强制引导的玩家，沉默 = 不完整 |
| **缓解策略** | ① 修炼界面必须有一个**弱但明确的**视觉焦点：手动修炼按钮（"吐纳"）应有微弱的呼吸动画，引导视线 ② 灵气数字的颜色在接近 100 时从白色转为浅金色（暗示"快达到某阈值"）③ 首 30 秒内若玩家无任何操作（无点击、无退出），灵气阈值降低 50% 提前触发解锁加速节奏 ④ 标题画面文案"唯修者，逆天而行"暗示了"修炼 = 第一个动作"。不做弹窗和箭头，但做"温和的眼球引导" |

#### 流失点 2：0:15–0:45 —— "只有自动战斗？就这？"

| 维度 | 分析 |
|------|------|
| **流失时机** | 首次解锁战斗后，观看战斗日志约 10-15 分钟 |
| **流失画像** | 期待操作性战斗的玩家（ARPG 或动作游戏背景）；期待"战棋策略"但发现是纯自动的玩家 |
| **失败模式** | 玩家发现战斗完全不需要操作，只看日志刷新。从修炼的"手动点击"到战斗的"纯观察"，操作频率断崖式下降。产生"这不就是个屏保？"的负面判断 |
| **根本原因** | MVP 战斗为半自动（普攻自动），但 MVP 尚未实现技能系统和战法策略（game-concept.md §11 明确不含复杂装备词条海和战法），导致战斗层缺少可调配置——玩家没有"配置后观察结果"的反馈循环，只有"观察" |
| **缓解策略** | ① 战斗日志的差异化至关重要——不同敌人有不同的击杀文本（"石傀儡——粉碎"、"野兔妖——斩杀"、"枯木精——驱散"），让日志不是单调重复 ② 首次击败每种敌人应有短暂的 toast 提示（"首次击败：石傀儡"）激活收集感 ③ 区域选择器上始终可见 Zone 2 的锁定状态和下方条件提示——让玩家看到"前方还有内容" ④ Lv.5 解锁 Zone 2 的节奏应在 ~15-18 min 内达成（参见 mvp-content-progression.md §6.1），足够快让玩家在"视觉疲劳"前获得新目标 |

#### 流失点 3：D1–D2 —— 首次离线后无"必须回来"的牵引

| 维度 | 分析 |
|------|------|
| **流失时机** | 第一次离线后（Day 1 晚上或 Day 2 上午），返回游戏后 5-10 分钟 |
| **流失画像** | 轻度玩家、尝鲜型玩家——安装 → 玩一次 → 关闭 → 忘记 |
| **失败模式** | 离线收益面板关闭后，玩家做了 2-3 个操作（看升级、换区域、看资源储备），然后发现"好像没什么新东西要做的"。关闭游戏后，由于游戏没有推送通知（单机游戏），Day 2 忘记打开 |
| **根本原因** | MVP 缺少"每日目标系统"（系统 64，game-concept.md 系统列表）和"离线回归的二次钩子"——离线收益是"回来"的理由，但不是"回来之后再做点什么"的理由 |
| **缓解策略** | ① 离线回归后的第一屏不是主界面，而是**离线结算摘要 Drawer**自动弹出（已经在 hud.md §OFFLINE DRAWER 设计中），关闭 Drawer 后应有"下一次里程碑预告"："距突破至筑基期还需 X 修为"作为 soft nudge ② 如果离线战斗触发了升级或击败了守护关（Zone 2 噬石兽），回到游戏时 toast 强调"离线突破!" ③ 离线收益 ≥ 一定阈值时，存在"分享"钩子（截图分享按钮），将外部社交作为回来的理由 ④（远期）当每日目标系统实现后，离线回归后的第一件事是"今日目标"查看 |

#### 流失点 4：D4–D6 —— Zone 3 mid-late game grind 感

| 维度 | 分析 |
|------|------|
| **流失时机** | Lv.20-28 区间（Zone 3 中段），约 4-10 小时在线累计 |
| **流失画像** | 中度投入型玩家——已经投入了一定时间，但对"重复刷"没有耐心的玩家 |
| **失败模式** | 从 Lv.20 到 Lv.28 期间，Zone 3 已经解锁但守护灵尚未稳定击败。对于每天只回访 15 分钟的玩家，这可能变成 2-3 天的重复挂机。 |
| **根本原因** | MVP 内容在 Lv.20-28 区间缺少中频刺激点——没有新敌人种类（Zone 3 的 5 种敌人自 Lv.16 解锁后不变）、没有新系统解锁，且装备/图鉴/合成尚未进入 Sprint 12。 |
| **缓解策略** | ① Zone 3 的每种敌人首次击败触发不同日志文本和一次性 toast，提供轻量收集感 ② 晶牙龙蜥（Lv.27）和守护灵（Lv.30）之间的 Lv.28-29 是最长的空窗，应插入"即将接近终局"提示（"战场深处传来不祥的震动……守护灵在等待"）③ 路线图明确说明 Phase 2+ 将打开玄晶/装备刷宝，让当前药材和灵石储备有未来意义 |

#### 流失点 5：D7+ —— MVP 内容边界暴露后的"空虚期"

| 维度 | 分析 |
|------|------|
| **流失时机** | Lv.30、守护灵击败、三区域 farmable 后 |
| **流失画像** | 硬核/成就型玩家——快速通关消耗内容的玩家 |
| **失败模式** | 玩家完成所有 MVP 内容后发现：无新区域解锁、无新境界可突破、灵石无处消费。体感从"修仙世界在扩张"变为"就到这了"。 |
| **根本原因** | MVP 设计边界限制了内容深度——这是阶段 1 的正常终点，但如果没有缓冲带就会直接变成流失点 |
| **缓解策略** | ① **系统 225：开发路线图系统**应在 MVP 实装——不是简单的"Coming Soon"，而是可交互的路线图：展示下一阶段的系统名、概念美术草稿、预计解锁顺序。将"内容耗尽"重帧为"蓄力等待 Phase 2 更精彩内容" ② 三个轻量子目标：灵石千万级 milestone、Zone 3 每种敌人击败 100 次计数、守护灵首杀回看。这些不消耗大量开发资源但提供 long-tail engagement ③ MVP 终局应有明确的"等待 Phase 2 的最佳状态"提示：当前累积的灵石/材料在 Phase 2 开始后会直接转化为初期优势（让玩家感觉现在的 grinding 不是浪费） |

### 4.2 流失风险矩阵

| 流失点 | 时间窗 | 风险等级 | 受影响玩家画像 | 核心问题 | 缓解优先级 |
|--------|--------|---------|-------------|---------|----------|
| #1 不知道做什么 | 0:01–0:02 | 高 | 低耐心、高期望动作感 | 渐进叙事展开的极简开局 vs 玩家预期待 | P0 |
| #2 "只有自动战斗？" | 0:15–0:45 | 中高 | ARPG/动作玩家、策略玩家 | MVP 战斗缺少操作层（技能/战法未实装） | P1 |
| #3 离线后无牵引 | D1–D2 | 高 | 轻度/尝鲜玩家 | 缺少"回来的理由"的二次钩子 | P0 |
| #4 中后期 grind 感 | D4–D6 | 中 | 中度投入型 | Lv.20-28 无中频刺激 | P2 |
| #5 终局空虚 | D7+ | 中低（MVP 正常边界） | 硬核/成就型 | MVP 内容消耗完毕的正常终点 | P2 |

## 5. 首次用户体验（FTUE）核心原则

### 5.1 七大 FTUE 原则

对标 game-concept.md §4 设计支柱，结合上述旅程分析和流失风险评估，定义 MVP 新手引导的 7 条核心原则。

#### 原则 1：先行为，后界面

> **对标 Pillar**：4.6 渐进叙事展开
> **来源**：流失点 #1（不知道做什么）

玩家先用最简方式体验系统核心行为，再获得完整 UI。开局不展示任何菜单、标签页、系统列表——只给修炼界面。战斗解锁后先默认在青丘山林自动战斗，达到 Lv.5 + atk ≥ 180 后再让幽墟灵谷成为可选目标（参见 world-skeleton.md §6.3 "一个系统 = 一个叙事理由"原则）。

**落地点**：修炼界面 → 战斗自动开始 → 区域选择器渐进出现。HUD 的 12 个 must-show 元素遵循 hud.md §Dynamic Behaviors 的渐进解锁顺序。

#### 原则 2：数字跳动是第一语言

> **对标 Pillar**：4.1 数字增长就是快乐
> **来源**：全旅程各节点的共同驱动

第一个 10 秒内，玩家必须看到至少一个数字在变化（灵气）。在 FTUE 中，数字的增长方向和频率比数字的具体意义更重要。玩家先感受"有东西在涨"，再被告知"涨的是什么"。任何需要文字解释的系统，在解释文字出现前，应已有至少 3 次数字跳动作铺垫。

**落地点**：灵气每秒跳动 → 修为累积反馈 → 灵石从 0→1 的首次入户 → 经验条填充。数字颜色变化（接近阈值时白→浅金）替代箭头引导。

#### 原则 3：每次操作都有即时可见的反馈

> **对标 Pillar**：4.2 放置不是无操作
> **来源**：流失点 #2（纯观察战斗无反馈）

放置游戏的"低频操作"不等于"无操作"——每一次操作应该有及时的、可感知的反馈。手动修炼按钮的每次点击应有：数字跳动、按钮缩放、音效（轻击 chime，参见 cultivation-system.md §Visual/Audio Requirements）。战斗日志的每条新行应有 fade-in 220ms 动画（hud.md §长 Session 视觉漂移防护）。离线回归后 Drawer 的展开应包含数字递增动画。

**落地点**：修炼点击反馈 → 战斗日志 fade-in → 离线 Drawer 展开动画 → 突破印章 burst 3s。

#### 原则 4：用"即将解锁"制造期望，不用"锁定"制造挫折

> **对标 Pillar**：4.6 渐进叙事展开
> **来源**：流失点 #3（离线后无牵引）+ 流失点 #4（grind 感）

锁定的区域/系统/境界在 UI 中始终可见但灰显，附带解锁条件提示。玩家看到的不是"你不能去"，而是"你还没到，但快了"（参见 world-skeleton.md §6.3 "用限制制造渴望"）。Zone 2 在 Zone 1 期间可见并显示"需 Lv.5 + atk ≥ 180"——当玩家 Lv.4 时，经验条会额外高亮以暗示"快到了"。

**落地点**：区域选择器的三区锁定/解锁状态 → 境界徽章下方的"距下一境界"进度 → Phase 2+ 路线图显示"装备刷宝将在后续阶段开放"。

#### 原则 5：首次离线回归是 FTUE 的下半场，不是终点

> **对标 Pillar**：4.2 放置不是无操作
> **来源**：流失点 #3（离线后无牵引）+ 情感曲线 V3（回访倦怠）

FTUE 不应在"玩家学会操作"时结束，而应延伸至首次离线回归后。玩家在离线回归后的体验决定了他们是否形成"每天回来看看"的习惯。首次离线回归的结算面板设计应包含：收益的全景摘要、今日的预期成长（'若保持此节奏，明日可达 Lv.X'）、以及归来后的第一个 mini-goal。

**落地点**：离线结算 Drawer（P-NAV-04）→ 自动弹出 → 数字递增动画 → 关闭后的"下一次里程碑预告"soft nudge。

#### 原则 6：境界突破是 FTUE 的高潮，必须有仪式感

> **对标 Pillar**：4.1 数字增长就是快乐 + 4.6 渐进叙事展开
> **来源**：情感曲线峰值 P2/P4

境界突破是 MVP 中最强的情感峰值。它必须包含三重反馈：① 视觉（HUD 全屏 burst_gold 印章 3s，参见 hud.md §CENTER CONTENT）② 数值（全属性跳变 20-25%，跃迁效应）③ 选项（新姿态、新区域、新系统解锁）。突破不是"经验够了自动弹窗"，而是"需要玩家确认→然后演绎"。

**落地点**：炼气突破（Lv.10）→ 筑基突破（Lv.30）→ 首次突破额外包含弱引导确认（"是否尝试突破至炼气期？"→ 是 → 演绎）。

#### 原则 7：MVP 的内容边界应被设计为"蓄力阶段"，不是"到此为止"

> **对标 Pillar**：4.9 单机经济优先 + 4.5 多层重置制造长期目标
> **来源**：流失点 #5（终局空虚）

MVP 的内容终点需要有意识的"缓冲带"设计：玩家通关后不是面对空白，而是看到明确的下一阶段预告和可继续从事的 long-tail 活动（数字积累、材料储备、守护灵首杀回看）。开发路线图（系统 225）在 MVP 终局出现，将"内容耗尽"的负面情绪转化为"我已经准备好了"的蓄力感。对标 game-concept.md §12 阶段路线图——让玩家知道 Phase 2 会带来资源链与建筑体系，当前的灵石和材料不会浪费。

**落地点**：开发路线图界面 → 守护灵首杀回看 → 材料/灵石储备提示（"待宗门系统开放后，这些灵石将成为你的开宗立派之资"）。

### 5.2 原则对实现系统的映射

| 原则 | 落地系统 | 对应 HUD 元素 | 优先级 |
|------|---------|-------------|--------|
| 1. 先行为后界面 | 修炼系统、战斗系统、map-progression-system | HUD 渐进解锁（5 → 12 元素） | P0 |
| 2. 数字跳动 = 第一语言 | 资源系统、LevelSystem | TOP STRIP 资源 rows | P0 |
| 3. 即时可见反馈 | 修炼系统 (manual click)、semi-auto-combat (battle log)、offline-reward-settlement (drawer) | 按钮动画、战斗日志 fade-in、Drawer 数字递增 | P0 |
| 4. "即将解锁"期望 | map-progression-system、cultivation-system | 区域选择器锁定态、境界徽章进度、Phase 2+ 路线图预览 | P1 |
| 5. 离线 = FTUE 下半场 | 离线模拟内核、offline-reward-settlement | P-NAV-04 Drawer + 里程碑预告 | P1 |
| 6. 突破 = 仪式 | LevelSystem、cultivation-system、UI 动效 | burst_gold 印章 3s + 属性面板变化 | P0 |
| 7. 终局 = 蓄力阶段 | AllMVPSystems + 开发路线图系统 (225) | 路线图界面 + 守护灵首杀回看 + 储备提示 | P2 |
