## 一、总体重设计方向（≤ 200 字）

以"**水墨山水 × 像素工笔**"为统一基调，拒绝"东方装饰品堆砌"式的伪中国风。
视觉语言从两条轴线出发：**克制**（大面积留白 + 低饱和度底色）与**精准**（关键信息用高对比度朱红/金色点睛）。
占位素材的"简陋感"源于色彩体系缺失和字体不一致，而非精度不足——因此重设计优先级为：
① 统一字体 → ② 建立调色板 → ③ 规范视觉层级 → ④ 逐步替换精灵/Tileset。
所有免费素材在接入前必须经过"水墨五色化"处理（统一到下列调色板），确保替换后风格一致而非"拼贴感"。

## 二、调色板

art-bible v0.3 第四章已定义完整色彩系统，本节从中提取**重设计工作色板**（Sprint-002 操作基准）。

### 2.1 基础五色（直接沿用 art-bible 4.1）

| 色名 | Hex | 用途 |
|------|-----|------|
| 焦墨 | `#1A1A1A` | 轮廓线、UI 描边、主文字 |
| 浓墨 | `#3D3D3D` | 次级阴影、分割线 |
| 淡墨 | `#8A8A8A` | 远景、禁用状态 |
| 灰墨 | `#BEBEBE` | 装饰纹理、次要信息 |
| 留白 | `#F5F2EB` | 背景底色、高光 |

### 2.2 叙事/语义强调色（直接沿用 art-bible 4.2 / 4.4）

| 色名 | Hex | 用途 |
|------|-----|------|
| 朱红 | `#D4501A` | 路线标识、选中强调、主菜单装饰 |
| 金黄 | `#D4A843` | 悬停强调、重要 UI 高光 |
| 危险橙 | `#E85D3B` | 警告、低血量提示 |
| 青墨 | `#2D4A4A` | 探索场景基调色 |
| 赭墨 | `#5C3D2E` | 战斗场景叠加基调 |

### 2.3 素材重染操作规则

接入免费素材时按以下映射关系重染，确保风格统一：

| 原色范围 | 替换为 | 说明 |
|----------|--------|------|
| 纯黑 #000000 | 焦墨 #1A1A1A | 描边统一 |
| 纯白 #FFFFFF | 留白 #F5F2EB | 背景统一 |
| 鲜红类 | 朱红 #D4501A | 危险/路线统一 |
| 蓝紫类（非属性）| 浓墨 #3D3D3D | 去除西式配色影响 |
| 饱和度 > 60% 的杂色 | 降饱和度 -30%，色相偏向青墨或赭墨 | 防止素材"跳色" |

## 三、字体选型

### 3.1 中文标题字体

| 字段 | 内容 |
|------|------|
| 字体名 | ZCOOL XiaoWei（站酷小薇 LOGO 体） |
| License | OFL 1.1（开源免费商用） |
| 来源 | https://fonts.google.com/specimen/ZCOOL+XiaoWei |
| 适用场景 | 主菜单游戏标题、章节名称、Boss 名称、重大选择提示文本 |
| 字号基准 | 主标题 48pt（1080p 基准），章节名 32pt，Boss 名 24pt |
| 颜色规则 | 常态：焦墨 #1A1A1A；强调：朱红 #D4501A；禁用：淡墨 #8A8A8A |
| 设计理由 | 楷体骨架有力，具古典气质，笔画粗细对比适中——在 pixel 场景中不会显得"太现代" |

### 3.2 中文正文字体

| 字段 | 内容 |
|------|------|
| 字体名 | Zhuque Fangsong（朱雀仿宋） |
| License | OFL 1.1 |
| 来源 | https://github.com/TrionesType/zhuque-fangsong |
| 适用场景 | 对话气泡、技能描述、物品说明、战斗 Log、菜单列表文字 |
| 字号基准 | 正文 14pt（1080p），说明文字 12pt，最小不低于 11pt（保证可读性） |
| 颜色规则 | 主文字：焦墨 #1A1A1A；次文字：浓墨 #3D3D3D；提示文字：淡墨 #8A8A8A |
| 设计理由 | 宋仿混合体横细竖粗，在低分辨率下比纯宋体更清晰；仿宋笔意与水墨气韵呼应 |

### 3.3 字体组合层级

```
主菜单标题     ZCOOL XiaoWei 48pt   焦墨/朱红
章节名/Boss名  ZCOOL XiaoWei 24-32pt 焦墨
对话/描述      朱雀仿宋 14pt         焦墨
HUD 数字       朱雀仿宋 12pt（加粗） 语义色
战斗 Log       朱雀仿宋 11pt         浓墨
```

## 四、四屏视觉层级建议

### 4.1 主菜单

**核心目标**：建立第一印象，传递"水墨武侠"基调

| 层级 | 内容 | 视觉处理 |
|------|------|----------|
| 层 1（最前） | 游戏标题 | ZCOOL XiaoWei 大字，朱红 #D4501A，纵向居中偏上 |
| 层 2 | 菜单选项（开始/继续/设置/退出） | 朱雀仿宋 18pt，焦墨，左对齐或居中；悬停时金黄 #D4A843 描边 |
| 层 3（中景） | 主视觉插图/角色剪影 | 水墨剪影风格，轮廓清晰，填色不超过 2 种（焦墨 + 朱红） |
| 层 4（远景） | 山水背景 | 低饱和度青墨 #2D4A4A，留白 20-30%，体积光 15% |
| 层 5（最后） | 版本号/版权 | 淡墨 #8A8A8A，8pt，右下角，视觉权重最低 |

**视觉重心**：标题 > 菜单选项 > 主视觉 > 背景（从上到下、从前到后）

**占位素材替换优先级**：字体 → BGM → 背景色调（三步即可大幅提升第一印象）

---

### 4.2 战斗 HUD

**核心目标**：策略信息清晰可读，不干扰战场判断

| 层级 | 内容 | 视觉处理 |
|------|------|----------|
| 层 1（最高优先级） | 行动顺序条（Turn Order） | 顶部水平排列，角色头像 32×32，当前行动者金黄 #D4A843 描边 |
| 层 2 | 选中单位状态（HP/MP/Buff） | 底部左侧面板，宣纸白背景 95% 不透明，焦墨描边 1px |
| 层 3 | 技能/行动菜单 | 底部右侧，按钮用 art-bible 4.4.3 状态色，技能名朱雀仿宋 14pt |
| 层 4 | 地形格高亮 | 移动范围：青绿 #4CAF7D 15% 叠加；攻击范围：危险橙 #E85D3B 15% 叠加 |
| 层 5（最低优先级） | 战场背景 | 赭墨 #5C3D2E 基调叠加 10%，保持战斗氛围但不抢 HUD 可读性 |

**视觉层级原则**：HUD 元素的不透明度 > 场景元素；关键数字（HP 低于 30%）字色切换为危险橙 #E85D3B

---

### 4.3 战斗结算屏

**核心目标**：传递成就感，快速传达关键信息（胜/负/奖励）

| 层级 | 内容 | 视觉处理 |
|------|------|----------|
| 层 1 | 胜负标语 | ZCOOL XiaoWei 48pt，胜利用金黄 #D4A843，失败用浓墨 #3D3D3D |
| 层 2 | 经验/奖励列表 | 朱雀仿宋 14pt，每行对齐，新获得条目朱红 #D4501A 标注 |
| 层 3 | 角色成长提示（升级/技能解锁） | 独立卡片，金黄 #D4A843 边框，在列表上方弹出 |
| 层 4 | 按钮区（继续/重试） | art-bible 4.4.3 按钮样式，"继续"为主按钮（朱红描边），"重试"为次按钮 |
| 层 5（背景） | 结算背景 | 胜利：留白 #F5F2EB + 轻微金黄粒子；失败：浓墨 #3D3D3D 暗化 |

**设计原则**：结算屏留白 > 40%（让成就感"呼吸"），避免数字堆砌

---

### 4.4 管理屏（角色 / 装备 / 技能）

**核心目标**：信息密度高但不压迫，支持键盘/手柄导航

| 层级 | 内容 | 视觉处理 |
|------|------|----------|
| 层 1 | 当前选中角色/装备 | 左侧大图区，角色精灵 128×128（2x 放大显示），金黄描边当前焦点 |
| 层 2 | 属性面板 | 右侧列表，朱雀仿宋 13pt，分隔线用淡墨 #8A8A8A 1px |
| 层 3 | Tab 导航（角色/装备/技能） | 顶部 Tab，当前 Tab 朱红 #D4501A 下划线，ZCOOL XiaoWei 16pt |
| 层 4 | 对比提示（装备属性差值） | 绿色 +n 用安全绿 #4CAF7D，红色 -n 用危险橙 #E85D3B |
| 层 5（背景） | 面板底色 | 淡墨灰 #E8E5DE，与游戏世界保持"翻开书页"的质感 |

**键盘导航规则**：焦点元素必须有金黄 #D4A843 2px 描边，焦点切换时有 0.1 秒缓入；确保 Tab 顺序与视觉顺序一致

## 五、与既有 art-bible 规范的兼容/取舍

### 5.1 完全兼容项（直接沿用，无需修改）

| 规范 | 出处 | 说明 |
|------|------|------|
| 水墨五色色板 | art-bible 4.1 | 本文档调色板完全基于此 |
| 语义色系统 | art-bible 4.2 | 危险/安全/增益色未做任何改动 |
| 按钮状态色彩 | art-bible 4.4.3 | 四屏 HUD/管理屏均直接引用 |
| 角色精灵规格 | art-bible 8.2（经 character-visual-profiles.md 引用） | 32×32 / 128×128 标准不变 |
| 色盲形状备份 | art-bible 4.5 | 所有语义色均保留形状备份要求 |

### 5.2 本文档补充项（art-bible 未覆盖）

| 补充内容 | 理由 |
|----------|------|
| 字体选型（标题 + 正文） | art-bible 未指定具体字体，本文档首次确定 |
| 四屏视觉层级量化描述 | art-bible 给出原则，本文档落地为可操作的层级表 |
| 素材重染操作规则（2.3） | 针对免费素材接入的实操指引，art-bible 不涉及外部素材处理 |

### 5.3 轻微取舍项

| 取舍点 | art-bible 原规范 | 本文档调整 | 理由 |
|--------|-----------------|------------|------|
| 主菜单视觉 | art-bible 未专门描述 | 新增：层级表 + 留白 20-30% | Sprint-002 主菜单是玩家第一印象，需明确操作规范 |
| 战斗结算屏留白 | art-bible 未规定 | 新增：留白 > 40% | 结算屏堆砌数字是"简陋感"的主要来源之一 |
| 字体最小字号 | art-bible 未规定 | 新增：正文最小 11pt | 可访问性保障，与 art-bible 4.5.4 对比度要求配套 |

### 5.4 不兼容 / 需后续对齐项

| 问题 | 说明 | 处理方式 |
|------|------|----------|
| art-bible 角色视觉档案（CV-OQ-1/2/3）仍为 Open Items | 四屏建议中的"角色剪影"层暂用占位符 | 待 worldbuilding-narrative OQ-2 确认后更新 |
| 免费素材重染后需验证 art-bible 4.5 色盲测试 | 重染操作可能改变对比度 | 每批素材接入后用 colorblindly 插件做快速验证 |
