## 1. Visual Identity Statement

> **最高视觉法则**：水墨为骨，深面板为肉——题材识别度依赖墨韵笔触与留白，功能可读性依赖深色低对比面板；两者冲突时，功能可读性绝对优先。

1. **墨痕承载题材**
   所有装饰性笔触、边框、印章仅出现在非数据区域（背景、分割线、标题装饰），不得侵入数据列表或词条展示区。
   *设计测试*：当边框样式模糊时，本原则要求选"笔触仅压背景层，数据层保持纯色底板"。
   *支柱锚定*：柱 6（渐进叙事）——水墨装饰随系统解锁逐步出现，强化"世界逐步展开"的节奏感；柱 2（低频高价值决策）——题材氛围需在玩家规划阅读时保持沉浸而不干扰数据。

2. **信息层级即视觉层级**
   面板内所有元素按"决策优先级"映射亮度：当前可操作项最亮，次级信息次之，背景装饰最暗，绝不反转。
   *设计测试*：当按钮与标注的对比度模糊时，本原则要求选"可交互元素对比度 ≥ 4.5:1，背景装饰 ≤ 1.5:1"。
   *支柱锚定*：柱 1（数字增长就是快乐）——关键战力 / 掉落数字须即时可读；柱 3（刷宝惊喜）——稀有词条须在密集列表中被瞬间识别。

3. **写意图标，非写实图标**
   技能、装备、材料图标采用墨笔勾线加单色填充，保留笔触感；避免渐变仿真贴图，保持与深色面板的融合度。
   *设计测试*：当图标风格模糊时，本原则要求选"2 色墨线勾勒 + 单一主色，不用超过 3 色渐变"（具体配色由 Section 4 Color System 定）。
   *支柱锚定*：柱 10（数据驱动可扩展）——统一的写意图标规范使批量生成新词条 / 装备图标成本最低；柱 8（自动化是成长奖励）——解锁的新自动化功能图标须与已有图标语言无缝衔接。

**原则冲突裁决**：可读性 > 信息层级 > 墨痕装饰。任何原则之间的歧义由最高视觉法则一票否决。

## 3. Shape Language

### 3.1 角色剪影哲学

角色缩略图的目标尺寸为 64×64，职业辨认必须在该尺寸下零歧义。每个职业拥有一个**独占轮廓特征**，禁止两个职业共用同一剪影策略。

| 角色层 | 剪影策略 | 标志性特征 |
|--------|----------|------------|
| 主角 | 最大轮廓占比 ≥ 70% 正方形内切圆 | 唯一可拥有对称双侧延伸（翅膀 / 披风） |
| 弟子 | 中等；单侧特征主导 | 职业工具道具形成单侧突出（剑脊 / 法器 / 丹炉） |
| 妖灵 | 非人体比例；重心低且宽 | 至少一个非人体轮廓破点（兽耳 / 尾 / 爪） |
| 器灵 | 几何感强；可无腿 | 轮廓中必须出现明确几何硬边（圆形 / 菱形 / 方形器核） |

*设计测试*：将图标去色并缩至 64px，任意两职业剪影叠加若有超过 50% 的像素重合则判定失败。
*支柱锚定*：柱 3（刷宝惊喜）——掉落弟子卡片须在 0.5 秒内让玩家判断职业稀缺度；柱 6（渐进叙事）——职业解锁顺序须在剪影层面传递角色成长感。

### 3.2 环境 / 建筑几何

建筑图示以 **Angular-Geometric（硬边几何）为主导**，有机曲线仅用于屋檐翘角与瓦顶弧线，占比不超过轮廓线总长 20%。理由：在 64×64 缩略图中，硬边是功能可识别度最高的编码方式；曲线在小尺寸下退化为噪声。

| 建筑类型 | 主识别形 | 禁用形 |
|----------|----------|--------|
| 洞府 / 藏经阁 | 尖顶三角 + 门洞圆拱 | 全弧屋顶 |
| 丹房 / 炼器室 | 圆鼎 / 方炉居中 | 细线装饰 |
| 灵田 / 矿洞 | 梯形地基 + 斜坡格栅 | 对称正方 |
| 符箓阁 / 阵盘 | 八角外框或六边形 | 无边框平铺 |
| 灵兽园 / 拍卖行 | 旗帜三角形突出顶部 | 与洞府同顶型 |

建筑图示与水墨笔触的兼容方式：硬边轮廓线宽为 2px，外侧叠加 1px 半透明墨晕（opacity 30%）模拟线稿感，内部填色保持纯色无笔触。
*支柱锚定*：柱 10（数据驱动可扩展）——新建筑用主识别形组合即可生成，外包美术无需接受全量 brief。

### 3.3 UI 形状语法

面板与卡片采用**直角 + 单侧切角**基底（右上或左上切 6px 斜角），按钮为纯直角，列表行无任何圆角。理由：微圆角在 Godot 高 DPI 渲染下容易产生次像素锯齿，且与水墨硬边笔触风格冲突。

UI 形状定性为 **screen-space 独立 HUD 语言**，非 diegetic（不模拟卷轴 / 竹简物理形态）。理由：游戏 95% 视觉是 UI，diegetic 方案的渲染成本和扩展成本过高；水墨身份通过印章、笔触纹理、切角装饰线在面板边缘体现，而非面板本身异形化。

*设计测试*：任意截取一个数据列表截图，若美术风格需要用形状而非文字 / 颜色来传达信息，判定违反本原则。
*支柱锚定*：柱 2（低频高价值决策）——面板形状统一可减少玩家认知负荷，决策精力集中于数据本身；柱 1（数字增长就是快乐）——4.5:1 对比由深色纯色底板保证，不因异形面板牺牲。

### 3.4 Hero shapes vs. supporting shapes

**Hero rule（一句话）**：面积最大、亮度最高、边缘最清晰的形状拥有视觉优先权——所有其他形状须在这三项中至少让出两项。

稀有度 8 阶分阶机制以**边框线宽 + 外发光半径**双轴编码，形状本身不变（保持一致的直角切角语法），避免异形边框在网格布局中撑破行高。

> **Forward-note → Section 4**：下表中颜色名（灰 / 白 / 蓝 / 紫 / 金 / 青白 / 多色）为初拟，遵循行业稀有度通用直觉。**Section 4 Color System 须以 hex 六进制锁定**每阶具体色值与饱和度，并复核与"瓶颈红 / 失败红"等其它语义色的对比安全。

| 品阶 | 边框宽度 | 外发光半径 | 附加形状标记 |
|------|----------|------------|-------------|
| 凡品 | 1px 灰 | 无 | 无 |
| 精良 | 1px 白 | 无 | 无 |
| 稀有 | 2px 蓝 | 4px | 无 |
| 史诗 | 2px 紫 | 6px | 无 |
| 传说 | 2px 金 | 8px | 无 |

*支柱锚定*：柱 3（刷宝惊喜）——玩家在 0.3 秒内须能从外发光尺寸判断品阶而无需阅读文字。

### 3.5 形状禁区

1. **禁止全圆角面板（`border-radius` ≥ 50%）**：气泡感与水墨硬笔文化冲突，且在列表行密铺时产生视觉碎片。
2. **禁止异形轮廓窗口（非矩形裁剪容器）**：Godot `ClipChildren` 的非矩形裁剪在高频刷新列表中有 GPU 性能陷阱（技术锚定：ADR-0011 UI 屏幕管理架构）。
3. **禁止超过 3 条装饰线重叠在同一边框**：线叠线导致小字号文字对比度损失，违反功能可读性首条原则（Section 1 冲突裁决）。
4. **禁止在图标内使用透视 / 三点透视变形**：写意图标须保持正面 / 四分之三俯视统一视角；透视差异在图标海（1000+）中引发视觉抖动。
5. **禁止用形状动画表达数据变化**（如面板拉伸、卡片翻转）：数据高频变更（挂机数值刷新），形状动画每帧触发会导致视觉噪声并掩盖真实数值；数值变化只能用颜色 / 文字动画表达。

**"瓶颈红" vs. "失败红"三轴区分**：

- **色相**：`bottleneck_red #B04040` 偏棕褐（Hue ≈ 0°，desaturated）；`failure_red #CC2222` 纯正暖红（Hue ≈ 5°，saturated）
- **饱和度**：瓶颈红 S ≈ 55%，失败红 S ≈ 82%——相差 27%，即使在低饱和显示器上仍可区分
- **亮度**：瓶颈红 L ≈ 47%（偏暗、非紧急）；失败红 L ≈ 46% 但配合 60% 灰墨覆层，整体场景亮度骤降使红色显得鲜明

此三轴差异保证不依赖颜色名称也可直觉读取："暗沉旧红 = 提醒"，"鲜艳正红 + 全灰 = 失败"。

*支柱锚定（柱 3）*：爆发金与肃杀紫色温相距 ≥ 120°（暖 vs. 冷），掉落惊喜与危机恐惧不会视觉混淆。

> **Hex 重叠备注**：`rarity_chaos_3` 与 `burst_gold` 同为 `#F5C842` 是**有意的语义独立、共享色值**，不是设计不一致。在 `res://assets/ui/theme.tres` 中分别定义两个 token 指向同一 `Color`；后续若任一语义色需调整，只动各自 token，互不影响。Theme alias 优先于 hardcode hex。

### 4.5 UI 色板 vs. 世界色板

**分离方案**：hero 立绘 / 区域插画使用**水墨写意色板**（高黑色笔墨、低饱和自然色、留白为设计元素）；UI 数据面板使用 4.1 深色功能色板。两套色板在引擎层完全独立。

**渡口规则**（hero 区 → UI 区过渡）：

1. hero 插画在 `panel_bg_secondary #242430` 底板上以 **1px `ink_stroke` 硬边** 与数据区割断，不做渐变融合。
2. hero 区域最亮点（高光留白）亮度不超过 `text_primary #E8E0D0`，避免抢夺视觉焦点。
3. 插画色板不得引入 4.2 语义色的任何纯色形态——语义色只存在于 UI 数据层，防止玩家误读"插画里有金色 = 有稀有掉落"。

*支柱锚定（柱 6）*：分离方案使插画随叙事升级可独立替换（换插画不动 UI token），渐进叙事成本最低。
