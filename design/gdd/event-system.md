# 事件系统

> **Status**: Designed (pending review)
> **Author**: binlee1990 + agents
> **Last Updated**: 2026-04-22
> **Implements Pillar**: 系统互锁（事件连接叙事系统与角色管理，驱动世界观动态变化）

## Overview

事件系统管理游戏内叙事事件的触发、执行和记录。事件包括剧情事件（主线推进）、羁绊事件（角色发展）、环境事件（世界观碎片）和随机事件（重玩性）。事件由条件触发（章节进度、信念值阈值、角色状态），执行时产生叙事内容和游戏效果（属性变化、物品获取、路线偏移），完成后记录到事件日志供后续事件引用。

## Player Fantasy

- **世界在回应我的选择**：之前的选择在后续事件中产生回响——NPC 记得玩家做过什么
- **发现隐藏故事**：环境叙事和隐藏事件让探索者感到"我发现了别人可能错过的东西"
- **角色真正在成长**：羁绊事件不是数值加成的附庸，而是角色人格的展开

## Detailed Design

### Core Rules

**规则 1 — 事件分类**

| 类型 | 触发方式 | 不可跳过 | 示例 |
|------|---------|---------|------|
| 主线事件 | 章节推进自动触发 | 是 | Boss 战前对话、路线分流、结局 |
| 羁绊事件 | 羁绊等级提升 | 否 | 角色背景故事、羁绊发展 |
| 支线事件 | 与 NPC 交互/探索 | 否 | 帮助村民、调查悬案 |
| 环境事件 | 进入特定区域 | 否 | 壁画描述、古籍碎片、NPC 闲聊 |
| 随机事件 | 战斗间歇概率触发 | 否 | 商队遭遇、天气变化、突发事件 |

**规则 2 — 事件触发条件**

事件触发由条件组合决定：

| 条件类型 | 格式 | 示例 |
|----------|------|------|
| 章节进度 | `chapter >= N` | 第 5 章后可用 |
| 信念值 | `belief[dimension] >= V` | 果断值 ≥ 40 |
| 羁绊等级 | `bond[charA][charB] >= level` | A 和 B 羁绊 ≥ B 级 |
| 角色状态 | `party.contains(charID)` | 角色在队伍中 |
| 事件历史 | `event_completed(eventID)` | 事件 X 已完成 |
| 路线 | `route == routeID` | 当前处于某路线 |

**规则 3 — 事件执行流程**

```
条件检查 → 触发事件 → 播放事件内容 → 玩家选择(如有) → 执行效果 → 记录到事件日志
```

事件效果：
- 属性变化（临时或永久）
- 物品获取/消耗
- 信念值偏移
- 羁绊值变化
- 解锁/锁定新事件
- NPC 状态变化（友好/敌对/离开/加入）

**规则 4 — 事件引用系统**

已完成的事件记录在事件日志中，后续事件可引用：
- "如果你之前帮了铁匠，铁匠会在这场战斗中援助你"
- "如果你之前杀了俘虏，俘虏的同伴会在后续关卡伏击你"

引用机制：`event_log.contains(eventID)` 作为触发条件的一部分。

### States and Transitions

```
Locked (条件未满足) → Available (条件满足) → Active (播放中) → Completed (已完成)
                                                    ↓
                                              Skipped (玩家选择跳过，仅非必须事件)
```

### Interactions with Other Systems

| 系统 | 接口 | 方向 | 说明 |
|------|------|------|------|
| 叙事系统 | 事件内容定义 | 叙事→事件 | 世界观和剧情内容 |
| 角色管理 | 角色状态查询 | 角色→事件 | 角色是否在队伍中 |
| 羁绊系统 | 羁绊等级查询/修改 | 双向 | 羁绊事件触发和效果 |
| 战斗结算 | 战斗后事件触发 | 战斗→事件 | 战斗结果可能触发后续事件 |
| 存档系统 | 事件日志持久化 | 事件→存档 | 保存已完成事件列表 |
| 多周目系统 | 事件历史参考 | 多周目→事件 | 已完成的路线影响后续周目事件 |

## Formulas

**F1 — 随机事件触发概率**

```
trigger_chance = base_probability × exploration_modifier × curiosity_modifier
```

| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| base_probability | P_base | float | 0.05 ~ 0.15 | 基础触发概率（5-15%） |
| exploration_modifier | M_exp | float | 1.0 ~ 1.5 | 探索维度信念值加成 |
| curiosity_modifier | M_cur | float | 0.8 ~ 1.3 | 探索维度信念值修正 |

**F2 — 环境事件发现概率**

```
discovery_chance = base_discovery × (1 + perception_bonus)
```

perception_bonus 来自角色属性（智力影响文本发现，敏捷影响隐藏区域发现）。

## Edge Cases

- **If 必须事件（主线）被跳过**：主线事件不可跳过，必须完成后才能推进章节。
- **If 事件触发条件在事件执行中发生变化**（如事件中角色离队）：事件继续执行但不产生涉及该角色的效果。
- **If 事件日志在多周目间保留**：主线事件日志重置，但全局成就和路线完成记录保留。
- **If 两个事件同时满足触发条件**：按优先级排序（主线 > 羁绊 > 支线 > 环境 > 随机），依次触发。

## Dependencies

### 上游
- **世界观/叙事**：事件内容由叙事系统定义
- **角色管理**：角色状态影响事件触发

### 下游
- **羁绊系统**：羁绊事件的触发和效果
- **存档系统**：事件日志持久化
- **多周目系统**：跨周目事件历史

## Tuning Knobs

| Knob ID | 名称 | 默认值 | 安全范围 | 极端行为 |
|---------|------|--------|----------|----------|
| `TK-EVT-01` | 随机事件基础概率 | 10% | 5~20% | 过低=无变化；过高=干扰节奏 |
| `TK-EVT-02` | 环境事件发现基础概率 | 30% | 10~50% | 过低=探索无回报；过高=太容易 |

## Visual/Audio Requirements

- 主线事件：全屏对话 + 角色立绘 + 背景变化
- 羁绊事件：双人对话界面 + 羁绊类型色彩提示
- 环境事件：浮动文本框，不中断游戏
- 随机事件：弹出式提示，可选择参与或忽略

## UI Requirements

- 事件日志：按时间线显示已完成事件（可回看）
- 事件选择：多选项事件使用色彩编码暗示信念值倾向
- 事件跳过：非必须事件显示"跳过"按钮

## Acceptance Criteria

**GIVEN** 主线事件"章节 5 分支"，**WHEN** 玩家到达章节 5 且信念值满足条件，**THEN** 事件自动触发，不可跳过。

**GIVEN** 羁绊事件要求 A 和 B 羁绊 ≥ B 级，**WHEN** A 和 B 羁绊达到 B 级，**THEN** 事件出现在酒馆可用事件列表中。

**GIVEN** 随机事件在战斗间歇期，**WHEN** 概率检定通过（10% 基础概率），**THEN** 弹出随机事件选项，玩家可选择参与或忽略。

**GIVEN** 事件日志记录了"帮助铁匠"，**WHEN** 后续事件检查 `event_log.contains("help_blacksmith")`，**THEN** 铁匠援助战斗事件触发。
