# 音效/音乐系统

> **Status**: Designed (pending review)
> **Author**: binlee1990 + agents
> **Last Updated**: 2026-04-22
> **Implements Pillar**: 系统互锁（音效反馈战斗行为，音乐驱动叙事氛围，与环境系统互锁）

## Overview

音效/音乐系统管理游戏内所有音频体验，包括背景音乐（BGM）、战斗音效（SFX）、环境音（Ambient）、角色语音（Voice）和 UI 音效。系统根据游戏状态（探索/战斗/叙事/基地）自动切换音乐和音效层，与美术风格的氛围状态机同步。音乐和音效的设计语言与水墨山水视觉风格一致——东方乐器为主，留白为韵。

## Player Fantasy

- **听觉沉浸**：音乐不只是背景——每首曲子都在说"你现在在哪里，你在感受什么"。战斗音乐紧张但不嘈杂，基地音乐安宁但不无聊
- **音效即信息**：打击音效的重量感传达伤害大小，元素音效的音色传达属性类型，玩家闭上眼睛也能"听懂"战场
- **东方美学的听觉延伸**：古琴的余韵、笛声的悠远、鼓点的沉稳——听觉与视觉的水墨风格统一

## Detailed Design

### Core Rules

**规则 1 — 音频层级**

| 层级 | 类型 | 优先级 | 同时播放数 | 说明 |
|------|------|--------|-----------|------|
| BGM | 背景音乐 | 1（最低） | 1 | 始终存在，按场景切换 |
| Ambient | 环境音 | 2 | 3 | 风声/水声/鸟鸣等 |
| SFX | 音效 | 3 | 8 | 武器打击/技能施放/移动等 |
| Voice | 角色语音 | 4 | 1 | 战斗喊声/技能台词 |
| UI | 界面音效 | 5（最高） | 2 | 按钮点击/弹窗/确认 |

高优先级音频可以打断低优先级（如 Voice 打断 SFX），但不打断 BGM。

**规则 2 — BGM 分类与切换**

| 场景类型 | 音乐风格 | 乐器组合 | 循环方式 |
|----------|---------|---------|---------|
| 主菜单 | 庄严/悠远 | 古琴 + 箫 + 弦乐 | 无限循环 |
| 探索 | 宁静/开阔 | 笛 + 古琴 + 轻打击 | 无限循环 |
| 普通战斗 | 紧张/有节奏 | 鼓 + 琵琶 + 弦乐 | 战斗循环 |
| Boss 战 | 史诗/压迫 | 太鼓 + 编钟 + 全编制 | 阶段切换 |
| 叙事抉择 | 沉思/聚焦 | 古琴独奏 + 氛围合成 | 单次播放 + 淡出 |
| 基地 | 温暖/安宁 | 琵琶 + 笛 + 轻弦乐 | 无限循环 |
| 胜利 | 振奋/释放 | 全编制欢快曲调 | 单次播放 |
| Game Over | 沉重/余韵 | 古琴 + 风声 | 单次播放 |

BGM 切换使用 2 秒交叉淡入淡出（crossfade），不使用硬切。

**规则 3 — Boss 战音乐阶段**

Boss 战 BGM 分 2-3 阶段，与 Boss 血量阶段同步切换：

| 阶段 | 血量 | 音乐变化 |
|------|------|---------|
| Phase 1 | >50% | 基础战斗主题 |
| Phase 2 | 25-50% | 加入紧迫鼓点 + 升调 |
| Phase 3 | <25% | 全编制最强段落 + 合唱 |

阶段切换使用 1 秒渐变过渡（在当前音乐上叠加新乐器层）。

**规则 4 — 音效分类**

| 类别 | 数量（估算） | 示例 | 设计原则 |
|------|-------------|------|---------|
| 武器打击 | 20~30 | 剑/枪/弓/法杖命中 | 不同武器有不同音色，命中/暴击/格挡区分 |
| 元素效果 | 15~20 | 火球/冰冻/雷电/风刃 | 每种元素有独特音色标识 |
| UI 操作 | 15~20 | 按钮/确认/取消/弹窗 | 统一使用清脆的东方乐器音色（如木鱼/磬） |
| 环境音 | 10~15 | 森林/洞穴/城镇/水域 | 持续循环的背景层 |
| 角色语音 | 50~80 | 战斗喊声/技能台词 | 每个主要角色 5-10 条 |

**规则 5 — 音量控制**

| 通道 | 默认音量 | 范围 | 保存 |
|------|---------|------|------|
| Master | 80% | 0-100% | 存档 |
| BGM | 70% | 0-100% | 存档 |
| SFX | 80% | 0-100% | 存档 |
| Voice | 90% | 0-100% | 存档 |
| Ambient | 50% | 0-100% | 存档 |
| UI | 60% | 0-100% | 存档 |

### States and Transitions

BGM 状态与美术风格氛围状态机同步：

```
Explore_BGM ←2s crossfade→ Combat_BGM ←1s layer add→ Boss_Phase_2
     ↑                              ↓
     └──── Narrative_BGM ←──────────┘
```

### Interactions with Other Systems

| 系统 | 接口 | 方向 | 说明 |
|------|------|------|------|
| 回合制模式 | 战斗开始/结束事件 | 回合→音频 | 触发战斗 BGM 切换 |
| Boss 战 | 阶段切换事件 | Boss→音频 | Boss 阶段切换同步音乐 |
| 美术风格 | 氛围状态同步 | 美术→音频 | BGM 与视觉氛围同步 |
| 叙事系统 | 叙事事件触发 | 叙事→音频 | 对话/分支点音乐 |
| 战术机制 | 元素效果 | 战术→音频 | 元素技能音效 |
| UI 系统 | UI 交互事件 | UI→音频 | 按钮/弹窗音效 |

## Formulas

**F1 — 3D 空间音效衰减**

```
volume = base_volume × (1.0 - (distance / max_distance))
```

| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| base_volume | V_base | float | 0.8 ~ 1.0 | 音效基础音量 |
| distance | D | float | 0 ~ max_distance | 听众到声源的距离 |
| max_distance | D_max | float | 10 格 | 最大可听距离 |

**F2 — Crossfade 插值**

```
bgm_volume[A] = 1.0 - (t / crossfade_duration)
bgm_volume[B] = t / crossfade_duration
```

crossfade_duration = 2.0s（普通切换），1.0s（Boss 阶段切换）。

## Edge Cases

- **If 音频文件加载失败**：静默跳过，不影响游戏逻辑。在输出日志中记录警告。
- **If 同时触发 8+ 个音效**：按优先级截断，只保留最新 8 个。UI 音效永不截断。
- **If Boss 音乐阶段切换与普通 BGM 切换冲突**：Boss 音乐优先。退出 Boss 战后 2 秒内不切换其他 BGM。
- **If 玩家将 BGM 音量设为 0**：仅静音 BGM，环境音和音效不受影响。

## Dependencies

### 上游
- **回合制模式**：战斗开始/结束触发 BGM 切换
- **Boss 战**：阶段切换触发音乐变化

### 下游
- **UI 系统**：音量设置界面
- **存档系统**：音量设置持久化

## Tuning Knobs

| Knob ID | 名称 | 默认值 | 安全范围 | 极端行为 |
|---------|------|--------|----------|----------|
| `TK-AUD-01` | BGM 切换时长 | 2.0s | 0.5~4.0s | 过短=突兀；过长=拖沓 |
| `TK-AUD-02` | 最大同时音效数 | 8 | 4~16 | 过少=丢失信息；过多=嘈杂 |
| `TK-AUD-03` | 3D 音效最大距离 | 10格 | 5~20 | 过小=听不到远处；过大=混乱 |
| `TK-AUD-04` | 环境音默认音量 | 50% | 20~80% | 过高=干扰；过低=无氛围 |

## Visual/Audio Requirements

本系统即是音频需求本身。补充说明：
- 所有音频文件使用 OGG Vorbis 格式（Godot 原生支持）
- BGM 比特率 ≥128kbps；SFX ≥96kbps
- 音乐使用中国民族乐器录音或高质量 MIDI 音源

## UI Requirements

- 音频设置面板：6 个通道独立音量滑块 + Master 总控
- 音效预览：点击滑块旁的播放按钮试听
- 音频设置保存到游戏设置（存档系统持久化）

## Acceptance Criteria

**GIVEN** 游戏从探索进入战斗，**WHEN** 氛围状态切换，**THEN** BGM 在 2 秒内完成交叉淡入淡出，从探索曲切换到战斗曲。

**GIVEN** Boss 血量降至 50% 以下，**WHEN** 进入 Phase 2，**THEN** 音乐在 1 秒内叠加紧迫鼓点和升调层。

**GIVEN** 玩家触发火属性技能，**WHEN** 技能生效，**THEN** 播放火焰专属音效，音量随施放者与镜头距离衰减。

**GIVEN** 同时触发 10 个音效，**WHEN** 系统处理音效队列，**THEN** 按优先级保留最新 8 个，UI 音效不被截断。

**GIVEN** 玩家将 BGM 音量设为 0，**WHEN** 保存设置，**THEN** BGM 静音，SFX/Voice/Ambient 不受影响，设置持久化到存档。

## Open Questions

- **OQ-1**：角色语音是否纳入 MVP？当前估算 50-80 条语音的工作量对独立开发者较大。
- **OQ-2**：音乐是使用原创作曲还是免版权素材改编？影响预算和独特性。
- **OQ-3**：是否支持动态音乐系统（基于战斗强度实时调整乐器层）？实现复杂度较高。
