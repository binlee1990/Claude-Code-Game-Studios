# AI系统

> **Status**: In Design
> **Author**: user + agents
> **Last Updated**: 2026-04-16
> **Implements Pillar**: 系统互锁——AI敌人行为让战斗体验完整

## Overview

AI系统是游戏战斗体验的对手基础——它决定了敌人如何决策、攻击和配合。

游戏AI采用**分层决策架构**：
- **战术层**：决定"打什么目标"（基于威胁评估、仇恨系统）
- **策略层**：决定"用什么方式打"（基于技能冷却、资源状况）
- **执行层**：决定"具体位置和时机"（基于战场位置、高低差利用）

AI行为由**行为树驱动**，每个敌人类型有独立的AI配置文件（行为倾向、技能选择权重、攻击模式）。这种架构让AI行为既有可控的规律性（玩家可以预判和学习），又有足够的多样性（不同敌人有不同个性）。

核心AI类型：
- **攻击型**：优先伤害输出，激进进攻
- **防御型**：优先存活，保存实力
- **支援型**：优先治疗和增益队友
- **控制型**：优先使用控制技能

每种AI类型有不同的决策权重，组合出丰富的敌人行为模式。

## Player Fantasy

玩家在AI系统中的核心感受是**"我的对手是聪明的"**——敌人不是随机行动的木桩，而是有逻辑、有风格、有威胁的存在。

**AI的可学习性**带来的是**预判的乐趣**。当玩家发现"这个斧兵总是优先攻击我的弓手"时，当玩家学会"在弓手旁边放一个骑士吸引仇恨"时——这种**观察、总结、制定对策**的循环是魂系战斗哲学的核心。不是数值碾压敌人，而是**理解敌人的逻辑并找到克制方法**。

**AI的行为多样性**带来的是**每场战斗的新鲜感**。当玩家遇到一个"总是先放治疗的法师"和一个"总是冲脸的狂战士"时，战术应对完全不同——这种**敌人性格**让战斗不再重复。

**AI的配合威胁**带来的是**编队决策的重量感**。当玩家发现"如果我不先击杀那个牧师，他就会一直治疗"时，战斗从"依次击杀"变成"优先集火"——这种**战术优先级排序**是SRPG策略深度的核心。

**具体锚点：**
- 观察敌人行为模式后，成功预判并反制 → "我了解这个敌人"的成就感
- 用战术引诱敌人进入陷阱（一波带走） → 智谋碾压的爽感
- 敌人AI做出"聪明"的选择（如占领高地） → 遭遇真正对手的紧张感
- Boss的AI阶段切换（转入狂暴状态） → 战斗节奏变化的刺激感

**参考游戏**：三角战略的AI可学习性、XCOM的敌人行为预判、皇骑的敌人多样性

## Detailed Design

### Core Rules

**C.1 AI行为树架构**

AI决策采用分层行为树，分为三级：

**Level 1 - 战术决策（每回合开始时执行一次）：**
- 评估战场局势（敌人分布、高低差、元素地形）
- 确定本回合主要目标（集火目标、需击杀目标）
- 选择行动模式（进攻、防守、控制）

**Level 2 - 策略决策（每个可行动单位执行一次）：**
- 根据行动模式选择具体技能
- 评估技能命中率和预期收益
- 确定攻击/治疗/移动优先级

**Level 3 - 执行决策（技能/移动确定后执行）：**
- 选择最优位置（考虑地形、高低差）
- 选择最优目标（考虑克制、仇恨）
- 执行行动

**C.2 AI类型定义**

| AI类型 | 行动倾向 | 决策权重 |
|--------|---------|---------|
| **攻击型** | 优先造成伤害 | 伤害输出权重×1.5 |
| **防御型** | 优先保证存活 | 存活率权重×1.5 |
| **支援型** | 优先治疗/增益队友 | 治疗/增益权重×1.5 |
| **控制型** | 优先使用控制技能 | 控制效果权重×1.5 |
| **均衡型** | 各维度平衡 | 无特别倾斜 |

**C.3 仇恨系统**

仇恨值决定AI目标的优先级：

```
threat_score = base_threat + damage_dealt_recent + skill_threat + position_threat
// base_threat: 基础仇恨（目标当前生命值的倒数）
// damage_dealt_recent: 最近造成伤害的累计值
// skill_threat: 使用技能的威胁度（治疗>增益>控制>伤害）
// position_threat: 位置威胁（占高位>占低位的敌人）
```

**仇恨更新规则：**
- 每次造成伤害：仇恨+伤害量的10%
- 每次受到伤害：仇恨-受伤量的5%
- 治疗队友：仇恨+治疗量的20%
- 施放增益：仇恨+固定威胁值（10点）

**C.4 目标选择算法**

```
target_selection_priority:
    1. 如果存在仇恨值≥threshold的敌人，选择最高仇恨目标
    2. 如果无高仇恨目标，选择可击杀目标中血量最低
    3. 如果无可击杀目标，选择血量最低的目标
    4. 如果存在可克制的目标，优先选择（×1.5权重）
```

**C.5 技能选择算法**

```
skill_selection:
    1. 评估每个可用技能的预期收益（伤害/治疗/控制效果）
    2. 考虑技能冷却状态（仅选择可用的技能）
    3. 考虑技能消耗（mana/资源）
    4. 考虑当前位置对技能效果的影响（范围、命中率）
    5. 根据AI类型权重调整选择
    6. 选择综合得分最高的技能
```

**C.6 移动位置算法**

```
position_selection:
    1. 获取当前可移动范围内的所有空格
    2. 评估每个位置对战斗的影响：
        - 到目标距离
        - 高低差（是否占高位）
        - 是否在元素地形上（如油渍）
        - 是否在友军治疗范围内
    3. 根据AI类型加权评分
    4. 选择得分最高的位置
```

**C.7 Boss AI特殊行为**

Boss拥有特殊的AI行为模式：

- **阶段切换**：Boss在特定血量阈值切换行为模式（例：50%血量转入狂暴）
- **狂暴状态**：攻击伤害+30%，攻击速度+20%
- **技能强化**：进入新阶段后解锁新技能
- **AOE倾向**：Boss优先使用大范围技能

## Formulas

### D.1 仇恨值计算

```
threat_score = base_threat + damage_dealt_recent + skill_threat + position_threat
```

**变量定义:**

| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| base_threat | bt | float | [0.0, 1.0] | 基础仇恨=100/当前HP |
| damage_dealt_recent | ddr | int | [0, +∞) | 最近造成伤害累计值的10% |
| skill_threat | st | int | {0, 5, 10, 15, 20} | 技能威胁度（无=0, 伤害=5, 控制=10, 增益=15, 治疗=20） |
| position_threat | pt | int | {0, 5, 10} | 位置威胁（低位=0, 平地=5, 高位=10） |
| threat_score | ts | float | [0, +∞) | 最终仇恨值 |

**输出范围:** 0到正无穷，通常在0-200之间

**示例:**
- 100HP弓手，被造成50伤害后：`bt=100/100=1.0, ddr=50×0.1=5.0, pt=10(高位)=10.0` → `ts=16.0`

---

### D.2 技能预期收益

```
skill_expected_value = base_damage × hit_probability × kill_bonus × type_multiplier
```

**变量定义:**

| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| base_damage | bd | int | [技能基础伤害] | 技能的基础伤害值 |
| hit_probability | hp | float | [0.0, 1.0] | 预估命中率 |
| kill_bonus | kb | float | {1.0, 1.5} | 可击杀目标加成（可击杀=1.5，否则=1.0） |
| type_multiplier | tm | float | {1.0, 1.2, 1.5} | AI类型加权（攻击型=1.2, 控制型=1.5等） |
| skill_expected_value | sev | float | [0, +∞) | 技能预期收益 |

**输出范围:** 0到正无穷

**示例:**
- 基础伤害100，命中率80%，攻击型AI，可击杀目标：`sev = 100 × 0.8 × 1.5 × 1.2 = 144`

---

### D.3 位置评分

```
position_score = distance_score × height_score × element_score × support_score
```

**变量定义:**

| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| distance_score | ds | float | [0.0, 1.0] | 距离评分（越近越高） |
| height_score | hs | float | {0.8, 1.0, 1.2} | 高低差评分（高位=1.2, 平地=1.0, 低位=0.8） |
| element_score | es | float | {0.5, 1.0} | 元素地形评分（危险元素=0.5, 普通=1.0） |
| support_score | ss | float | [0.8, 1.2] | 支援评分（治疗范围=1.2，否则=1.0） |
| position_score | ps | float | [0.0, +∞) | 最终位置评分 |

**输出范围:** 0到正无穷

**示例:**
- 中等距离(0.7)，高位(1.2)，不在危险元素上(1.0)，在治疗范围(1.2)：`ps = 0.7 × 1.2 × 1.0 × 1.2 = 1.008`

---

### D.4 AI决策延迟（模拟思考时间）

```
ai_think_time = base_time + complexity_penalty
```

**变量定义:**

| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| base_time | bt | float | 0.5秒 | 基础决策时间 |
| complexity_penalty | cp | float | [0.0, 2.0] | 复杂度惩罚（基于可选行动数量） |
| ai_think_time | att | float | [0.5, 2.5]秒 | AI总思考时间 |

**输出范围:** 0.5秒到2.5秒

**示例:**
- 简单局面（3个选项）：`0.5 + 0.2 = 0.7秒`
- 复杂局面（10个选项）：`0.5 + 1.0 = 1.5秒`

## Edge Cases

### E.1 仇恨目标死亡

**场景**: 当前仇恨目标被击杀，AI需要重新选择目标

**处理**:
- 立即重新执行目标选择算法
- 优先选择仇恨值次高的目标
- 如果无仇恨目标，选择血量最低的可攻击目标
- 切换目标不应产生明显延迟

### E.2 所有友军无法行动

**场景**: AI队伍中所有单位被封印/无法行动

**处理**:
- 跳过该AI队伍的行动回合
- 立即进入结算阶段
- 记录"AI队伍全灭"状态

### E.3 技能冷却中无可用技能

**场景**: 当前单位所有技能都在冷却中

**处理**:
- AI选择基础攻击（无冷却技能）
- 如果基础攻击也不可用，选择待机（不消耗行动）
- 不选择会造成负收益的技能（如对自己造成伤害的技能）

### E.4 移动范围内无可用位置

**场景**: 当前单位无法移动到任何安全位置

**处理**:
- AI评估是否留在原地
- 如果原地更安全，选择待机
- 如果需要移动以获得技能射程，选择最近的可用位置

### E.5 Boss血量进入新阶段

**场景**: Boss受到足够伤害，血量进入阶段阈值

**处理**:
- 立即触发阶段切换动画和音效
- AI行为模式切换（新技能解锁、新行为权重）
- UI显示"Boss进入狂暴状态"提示
- 玩家获得一次免费行动机会（熟悉新阶段）

### E.6 AI无法击杀任何目标

**场景**: AI单位所有可攻击目标都无法击杀（血量太高）

**处理**:
- AI评估是否消耗技能尝试击杀
- 如果有足够资源（MP/技能），使用高消耗高伤害技能
- 如果没有，选择待机或移动到更有利位置
- 记录"无法击杀"状态，用于后续战术调整

### E.7 元素地形影响AI决策

**场景**: AI单位周围存在元素地形（油渍、水洼）

**处理**:
- AI评估元素地形的影响（对自己有利/有害/无关）
- 如果油渍在敌方，AI倾向于引诱敌人到油渍区
- 如果油渍在自己脚下，AI倾向于引诱敌人远离或不使用火系技能
- AI的element_score考虑元素地形的影响

### E.8 仇恨系统被刻意操控

**场景**: 玩家刻意使用低伤害技能控制仇恨

**处理**:
- 仇恨计算考虑技能威胁度（治疗>增益>控制>伤害）
- 单纯控制仇恨不足以保持仇恨，必须配合实际伤害
- 防御型AI对高仇恨目标的倾向性更高

## Dependencies

### Upstream Dependencies

| System | Dependency Type | Interface Contract |
|--------|---------------|-------------------|
| **属性与成长系统** | 硬依赖 | AI角色使用与玩家相同的属性系统；属性值影响AI的攻击力、防御力、血量等 |
| **职业系统** | 硬依赖 | AI角色使用与玩家相同的职业系统；职业决定AI的技能和属性加成 |
| **战术机制** | 硬依赖 | AI读取战场地形（高低差、元素地形）用于决策 |
| **技能系统** | 硬依赖 | AI使用技能系统定义的技能；技能冷却、消耗由技能系统管理 |
| **装备系统** | 硬依赖 | AI装备影响属性和技能效果 |

### Downstream Dependents

| System | Dependency Type | Interface Contract |
|--------|---------------|-------------------|
| **回合制模式** | 数据消费者 | 回合制模式驱动AI行动顺序；AI输出行动指令给回合制模式执行 |
| **战斗系统** | 数据消费者 | AI的攻击/治疗/移动指令由战斗系统执行 |
| **关卡设计** | 数据消费者 | 关卡设计使用AI配置文件定义敌人行为 |
| **难度系统** | 受影响方 | 难度倍率修改AI威胁权重和决策参数（见 difficulty-system.md） |

### Dependency Details

**战术机制 → AI系统**:
- AI调用 `get_terrain_info(position)` 获取地形信息（类型、高度、元素状态）
- AI调用 `get_height_advantage(attacker_pos, defender_pos)` 获取高低差修正
- AI调用 `get_elemental_interaction(position)` 获取元素交互状态

**AI系统 → 回合制模式**:
- AI输出 `ai_decision = {unit_id, action_type, target_id, position, skill_id}`
- 回合制模式接收AI决策并执行
- AI决策顺序由速度序列决定

**AI系统 → 战斗系统**:
- AI决策触发战斗系统执行对应行动（攻击/治疗/移动/使用技能）
- 战斗系统反馈结果（伤害值、治疗值、命中/闪避）给AI用于更新仇恨

**技能系统 → AI系统**:
- 技能系统提供 `get_available_skills(unit_id)` 返回可用技能列表
- AI调用 `can_use_skill(unit_id, skill_id)` 检查技能是否可用（冷却、消耗）

## Tuning Knobs

### AI Behavior Parameters

| Knob | Default | Safe Range | Affected Gameplay | Notes |
|------|---------|-----------|-----------------|-------|
| **攻击型伤害权重** | ×1.5 | ×1.2~×2.0 | 攻击型AI的伤害决策倾向 | 提高使攻击型AI更激进 |
| **防御型存活权重** | ×1.5 | ×1.2~×2.0 | 防御型AI的存活决策倾向 | 提高使防御型AI更保守 |
| **支援型治疗权重** | ×1.5 | ×1.2~×2.0 | 支援型AI的治疗决策倾向 | 提高使支援型AI更专注治疗 |
| **控制型控制权重** | ×1.5 | ×1.2~×2.0 | 控制型AI的控制决策倾向 | 提高使控制型AI更倾向控制技能 |

### Threat System Parameters

| Knob | Default | Safe Range | Affected Gameplay | Notes |
|------|---------|-----------|-----------------|-------|
| **仇恨基础值计算** | 100/HP | 50/HP~150/HP | 仇恨基准线 | 提高使低血量目标更容易成为目标 |
| **伤害仇恨转化率** | 10% | 5%~20% | 伤害对仇恨的影响 | 提高使输出角色更容易被集火 |
| **治疗仇恨权重** | 20% | 10%~30% | 治疗对仇恨的影响 | 提高使奶妈更容易被集火 |
| **位置威胁基础分** | 10 | 5~15 | 高位的仇恨加成 | 提高使高位占据更有威胁感 |

### Skill Selection Parameters

| Knob | Default | Safe Range | Affected Gameplay | Notes |
|------|---------|-----------|-----------------|-------|

### AI Difficulty Parameters

| Knob | Default | Safe Range | Affected Gameplay | Notes |
|------|---------|-----------|-----------------|-------|
| **AI基础命中率修正** | +0% | -10%~+10% | AI命中整体修正 | 正值使AI更难 |

### Suggested First Tuning Session

建议首轮调参优先级（按影响力排序）:
1. AI基础命中率修正（+0%）— 直接影响AI难度
2. 可击杀加成倍率（×1.5）— 影响AI集火策略
3. Boss狂暴伤害加成（+30%）— Boss战体验核心

## Visual/Audio Requirements

### Visual Requirements

**AI决策可视化**:
- AI"思考"时显示轻微的等待动画（不打断玩家）
- AI选择目标时，目标头上显示瞄准标记
- AI选择位置时，显示移动路径预览

**Boss阶段切换**:
- Boss进入新阶段时有独特的视觉变化（颜色、光效变化）

## Open Questions

### OQ.1 AI行为的学习曲线

**问题**: AI行为是否应该有"学习"机制，让敌人随着战斗次数增加变得更聪明？
- 当前设计：固定AI行为配置，无学习机制
- 备选方案：AI每次失败后微调行为参数

**待确认**: 学习机制增加重复可玩性但也增加实现复杂度。建议MVP阶段无学习机制，Alpha阶段再考虑。

### OQ.2 仇恨系统的可见性

**问题**: 仇恨系统是否应对玩家可见？
- 当前设计：仇恨系统为内部机制，玩家只能通过AI行动间接感知
- 备选方案：显示AI仇恨目标指示器

**待确认**: 仇恨可见让玩家更容易预判，但可能减少"发现敌人逻辑"的乐趣。建议先做不可见，通过AI行为暗示。

### OQ.3 AI难度的调节方式

**问题**: 如何让玩家调节AI难度？
- 当前设计：通过AI命中率/回避率修正调整
- 备选方案：AI决策深度调整（困难AI考虑更多因素）

**待确认**: 建议用简单修正值控制，让不同难度有不同的数值表现。

### OQ.4 AI队友的行为

**问题**: 玩家队伍的AI队友如何行动？
- 当前设计：自动战斗模式由玩家开启，AI自动操作我方单位
- 备选方案：我方AI使用与敌方不同的行为树

**待确认**: 我方AI可以使用简化版行为树，优先执行玩家设定的战术指令（如"集火"、"治疗"）。
