# 职业系统

> **Status**: In Design
> **Author**: user + agents
> **Last Updated**: 2026-04-16
> **Implements Pillar**: 系统互锁——职业解锁驱动玩家决策，是核心循环的关键

## Overview

职业系统是玩家角色成长的核心框架——每一次转职都是对角色Build方向的一次确认。

玩家不是"选择职业"，而是通过"我已经培养了足够的属性 + 我在这条路上积累了经验"来解锁新职业。职业不是标签，而是玩家投入的方向性证明。当一个角色从"战士"转向"剑圣"时，这不是换个名字——这是玩家明确选择了"我走力量/剑道路线"的宣言。

**核心机制:**
- **双重解锁条件**: 属性门槛（由属性与成长系统决定）+ 职业经验（战斗中累积）
- **三层职业**: 基础职业（一周目起始）→ 进阶职业（一周目可解锁）→ 特殊职业（多周目成就兑换）
- **转职代价**: 每次转职保留属性值和潜质，但重新开始积累职业经验
- **职业分支**: 同一角色可通往多个职业但资源有限，选择即放弃

**职业分层设计:**
- **基础职业**: 战士、法师、弓手、盗贼、牧师、骑士等（一周目起始可用）
- **进阶职业**: 基础职业的高级形态（如剑圣、咒术师、神射手等）
- **特殊职业**: 独特职业，更复杂技能树，通过多周目成就点兑换

## Player Fantasy

每一次转职都是玩家对角色道路的庄严宣告。当属性达标 + 职业经验达标，两个进度条同时变绿时——玩家感到"我准备好了，我宣告这个角色的道路"。这不是换个标签，而是"我的投入在这一刻得到了证明"。

转职保留属性但重置职业经验——这意味着玩家做了一个有风险的方向性决定。选择剑圣，就是放弃弓手的积累。职业不是标签，是玩家对角色未来的赌注。

深度专精意味着"誓言"有重量——你选择了这条路，就意味着放弃了其他可能性。这让职业选择从机械的数字运算变成了真正的角色扮演时刻。

**具体锚点:**
- 属性门槛 + 职业经验双达标 → 两个进度条同时变绿 → "我准备好了"的宣告感
- 转职确认界面强调"这是你的选择" → 强化决策的仪式感
- 转职后获得新技能/外观变化 → "蜕变"的视觉证明
- 选择剑圣 = 放弃弓手积累 → 有重量的选择感

**C.2 职业解锁判定**

```
CAN_UNLOCK(class_id, character_id) =
    (get_attribute_value(character_id, primary_attr[class_id]) >= threshold_primary[class_id])
    AND (get_attribute_value(character_id, secondary_attr[class_id]) >= threshold_secondary[class_id])
    AND (get_class_experience(character_id, class_id) >= experience_required[class_id])
```

---

**C.3 职业经验获取**

```
class_exp_gained = floor(damage_dealt × 0.02) + kill_bonus + battle_bonus
// kill_bonus = is_kill ? 10 : 0
// battle_bonus = 20 (胜败均有)
```

**C.4 转职流程**

1. 打开职业界面 → 系统显示可解锁职业 + 双重进度条（属性/经验）
2. 选择目标职业 → 系统验证 CAN_UNLOCK
3. 显示转职确认界面（强调"这是你的选择"）
4. 玩家确认后执行：
   - 属性V和潜质P完全保留
   - 新职业经验重置为0（其他职业经验保留）
   - 新职业加成立即生效
   - 触发门槛奖励检查

### States and Transitions

**角色职业状态机：**

```
enum CharacterClassState {
    NONE,               // 初始（立即进入BASIC_ACTIVE）
    BASIC_ACTIVE,       // 基础职业已激活
    ADVANCED_UNLOCKED,  // 进阶职业已解锁（可转职）
    ADVANCED_ACTIVE,    // 进阶职业已激活
    SPECIAL_UNLOCKED,   // 特殊职业已解锁（可转职）
    SPECIAL_ACTIVE      // 特殊职业已激活（终态）
}
```

**状态转换规则：**

| 当前状态 | 触发条件 | 目标状态 | 操作 |
|---------|---------|---------|------|
| NONE | 角色创建 | BASIC_ACTIVE | 初始化为基础职业（默认战士，侦察兵等需剧情或玩家选择解锁） |
| BASIC_ACTIVE | 门槛达标且经验≥500 | ADVANCED_UNLOCKED | 解锁进阶选项 |
| ADVANCED_UNLOCKED | 玩家选择转职 | ADVANCED_ACTIVE | 执行转职流程 |
| ADVANCED_UNLOCKED | 玩家保持 | BASIC_ACTIVE | 记录决策 |
| ADVANCED_ACTIVE | 门槛达标且成就点≥2000 | SPECIAL_UNLOCKED | 解锁特殊选项 |
| ADVANCED_ACTIVE | 成就点不足 | ADVANCED_ACTIVE | 更新UI进度 |
| SPECIAL_UNLOCKED | 玩家选择转职 | SPECIAL_ACTIVE | 执行转职流程 |
| SPECIAL_ACTIVE | — | — | 终态，不可再转 |

### Interactions with Other Systems

**技能系统：**

- **从本系统获取**：`get_class_id()`、`get_class_skill_tree()`
- **提供给本系统**：转职时触发技能解锁/锁定事件
- **接口**：`class_changed(character_id, old_class, new_class)`、`class_level_up(character_id, new_level)`

**装备系统：**

- **从本系统获取**：`get_class_bonus()` 职业加成表
- **提供给本系统**：职业变更后重新计算装备可行性
- **属性计算顺序**：`final_attr = base_value + class_bonus + equipment_bonus + barrier_bonus`

**战斗系统：**

- **从本系统获取**：职业基础数据（移动距离、攻击范围）
- **提供给本系统**：`report_damage_dealt()` → 触发职业经验计算
- **接口**：`apply_class_exp(character_id, class_id, exp)`

## Formulas

### D.1 职业解锁判定公式

```
CAN_UNLOCK(class_id, character_id) =
    (V_pa >= θ_p) AND (V_sa >= θ_s) AND (E_c >= E_req)
```

**变量定义:**

| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| class_id | — | enum | 见职业表 | 目标职业ID |
| character_id | — | enum | 角色列表 | 目标角色ID |
| V_pa | v_pa | int | [0, 999] | 角色主属性当前值 |
| V_sa | v_sa | int | [0, 999] | 角色副属性当前值 |
| θ_p | theta_p | int | 见职业表 | 该职业主属性门槛 |
| θ_s | theta_s | int | 见职业表 | 该职业副属性门槛 |
| E_c | e_c | int | [0, CAP] | 角色当前在该职业的职业经验 |
| E_req | theta_e | int | {0, 500, 2000} | 该职业所需职业经验门槛 |
| CAN_UNLOCK | result | bool | {TRUE, FALSE} | 是否满足全部解锁条件 |

**职业门槛表:**

| 职业ID | θ_p (主属性) | θ_s (副属性) | E_req (经验) | 适用条件 |
|---------|------------|------------|------------|---------|
| BASIC_* (所有基础职业) | 无 | 无 | 无 | 默认解锁，无需判定 |
| ADV_SWORDMASTER | 50 (STR) | 40 (AGI) | 500 | 一周目可解锁 |
| ADV_BATTLEMAGE | 50 (INT) | 40 (STR) | 500 | 一周目可解锁 |
| ADV_MARKSMAN | 50 (AGI) | 40 (STR) | 500 | 一周目可解锁 |
| ADV_ASSASSIN | 50 (AGI) | 40 (LUK) | 500 | 一周目可解锁 |
| ADV_HIGHCLERIC | 50 (CHA) | 40 (INT) | 500 | 一周目可解锁 |
| ADV_PALADIN | 50 (CON) | 40 (CHA) | 500 | 一周目可解锁 |
| SPC_* (特殊职业) | — | — | — | 由多周目成就系统判定，非本公式范围 |

**输出范围:** `TRUE` 或 `FALSE`

**示例:**
- 角色STR=52，AGI=45，职业经验=600 → `CAN_UNLOCK(ADV_SWORDMASTER) = TRUE`（可解锁剑圣）
- 角色INT=48，STR=42，职业经验=500 → `CAN_UNLOCK(ADV_BATTLEMAGE) = FALSE`（主属性差2点）

---

### D.2 职业经验获取公式

```
class_exp_gained = min(floor(damage_dealt × 0.02), DMG_CAP) + kill_bonus + battle_bonus
// DMG_CAP = 500 (单次伤害经验获取上限)
// kill_bonus = 10 (击杀) / 0 (非击杀)
// battle_bonus = 20 (无论胜败)
```

**变量定义:**

| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| damage_dealt | d | int | [0, +∞) | 本次造成的伤害量 |
| DMG_CAP | d_cap | int | 500 | 单次伤害经验获取上限 |
| kill_bonus | k | int | {0, 10} | 击杀奖励 |
| battle_bonus | b | int | 20 | 战斗参与奖励 |
| class_exp_gained | exp | int | [20, 530] | 本次获得的总职业经验 |

**约束条件:**
- `damage_dealt` 仅计入对敌方造成的有效伤害
- 同一敌人被多名角色击杀，只有最后一击者获得 `kill_bonus`
- `battle_bonus` 每场战斗结算时触发，无论胜败

**输出范围:** 最小20，最大530

**示例:**
- 300伤害+击杀+战斗获胜：`exp = min(floor(300×0.02), 500) + 10 + 20 = 36`
- 800伤害（超上限）+未击杀+战斗获胜：`exp = min(floor(800×0.02), 500) + 0 + 20 = 36`
- 0伤害+未击杀+战斗获胜：`exp = 20`

---

### D.3 职业等级公式

```
class_level = floor(E_c / CAP_c) + 1
// CAP_c: 基础职业=1000，进阶/特殊职业=2000
```

**变量定义:**

| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| E_c | e_c | int | [0, +∞) | 累计职业经验 |
| CAP_c | cap | int | {1000, 2000} | 该职业经验上限 |
| class_level | L | int | [1, +∞) | 当前职业等级，无上限 |

**职业经验上限表:**

| 职业层级 | CAP_c | 等级2所需 | 等级3所需 |
|---------|-------|---------|---------|
| 基础职业 | 1000 | 1000 | 2000 |
| 进阶职业 | 2000 | 2000 | 4000 |
| 特殊职业 | 2000 | 2000 | 4000 |

**示例:**
- 战士经验0：`L = 1`
- 战士经验1000：`L = 2`
- 剑圣经验5500：`L = 3`

## Edge Cases

### E.1 属性达标但经验未达标

**场景**: 玩家主/副属性满足进阶职业门槛，但职业经验不足500

**处理**:
- `CAN_UNLOCK` 返回 `FALSE`
- UI显示双进度条：属性进度条绿色（达标），经验进度条黄色（未达标）
- 禁止确认转职按钮（置灰）
- 提示文本："再获得 [X] 职业经验即可解锁"

### E.2 经验达标但属性未达标

**场景**: 玩家职业经验≥500，但主/副属性之一不满足进阶职业门槛

**处理**:
- `CAN_UNLOCK` 返回 `FALSE`
- UI显示双进度条：经验进度条绿色（达标），属性进度条红色（未达标）
- 禁止确认转职按钮（置灰）
- 提示文本："[属性名] 还需 +[X] 点即可解锁"

### E.3 转职后属性因装备/buff下降至门槛以下

**场景**: 玩家成功转职后，由于装备卸下或buff消退，属性降低至低于当前职业的门槛

**处理**:
- **不撤销已解锁的职业**
- 角色保持当前职业状态，正常战斗和升级
- UI显示警告："当前[属性名]低于职业门槛，属性加成暂时失效"
- 当再次满足门槛时，职业加成自动恢复

### E.4 玩家达到特殊职业条件但选择不转职

**场景**: 玩家满足SPC_*解锁条件，但主动选择保持当前进阶职业

**处理**:
- 不自动触发转职
- 记录玩家决策：`class_choice_recorded = TRUE`
- 特殊职业解锁状态保持，下次满足条件时仍可解锁
- UI显示"已记录您的选择"，不强制推送转职提示

### E.5 特殊职业转职后的终态处理

**场景**: 玩家从进阶职业转入特殊职业（SPC_*）

**处理**:
- 状态机进入 `SPECIAL_ACTIVE`（终态）
- 特殊职业界面禁用所有其他转职选项
- UI显示"已达职业巅峰"提示
- 不再显示职业解锁进度条

### E.6 多周目成就点数不足以解锁特殊职业

**场景**: 玩家满足属性和经验条件，但成就点数不足

**处理**:
- `CAN_UNLOCK` 对特殊职业额外检查：`achievement_points >= SPC_COST`
- UI成就点数进度条显示红色
- 提示文本："成就点数不足，还需 [X] 点"
- 战斗结算时成就点数独立累积，不影响职业经验

### E.7 多个职业同时满足解锁条件

**场景**: 一个角色同时满足多个进阶职业的解锁条件（如同时满足剑圣和圣骑士）

**处理**:
- UI职业选择界面高亮所有可解锁职业
- 玩家必须明确选择其中一个
- 选择后其他职业进度条变为灰色（已放弃路径）
- 其他职业的职业经验保留（如弓手经验保留，即使选择了剑圣）

### E.8 零职业经验获取的战斗

**场景**: 角色造成0点伤害（如完全未命中、伤害为0的技能）

**处理**:
- `damage_dealt = 0` 时，公式D.2：`exp = min(floor(0×0.02), 500) + 0 + 20 = 20`
- 即使0伤害，仍获得 `battle_bonus = 20`（战斗参与奖励）
- `kill_bonus = 0`（未击杀）

### E.9 超过经验上限的溢出处理

**场景**: 角色职业经验超过当前职业上限（如进阶职业经验溢出到特殊职业）

**处理**:
- 职业经验**不跨职业转移**
- 当前职业经验达到上限后不再增长，直到转职
- 转职后新职业经验从0开始计算

### E.10 基础职业间的切换

**场景**: 玩家是否可以在基础职业间切换（如从战士切换到法师）

**处理**:
- **不允许跨基础职业切换**（战士→法师需要通过进阶职业路径）
- 基础职业是角色的初始路径声明，不可逆转
- 同一角色只能拥有一个当前活跃职业

### E.11 成就系统兑换失败

**场景**: 玩家用成就点数兑换特殊职业后后悔

**处理**:
- **成就点数不可退还**
- 兑换结果不可逆
- 这是多周目设计的一部分——选择即永久承诺

## Dependencies

### Upstream Dependencies

| System | Dependency Type | Interface Contract |
|--------|---------------|-------------------|
| **属性与成长系统** | 硬依赖 | 本系统依赖其属性值（V）、潜质（P）、属性上限；职业解锁门槛基于属性值，属性加成计算基于基础属性 |
| **成就系统** | 硬依赖（仅特殊职业） | 特殊职业需成就点数兑换；成就点数独立于职业经验累积 |
| **存档系统** | 硬依赖 | 职业状态、职业经验、已解锁职业列表、玩家选择记录均需持久化 |

### Downstream Dependents

| System | Dependency Type | Interface Contract |
|--------|---------------|-------------------|
| **技能系统** | 数据提供方 | 本系统提供 `get_class_id()`、`get_class_skill_tree()`；技能系统需知道当前职业才能解锁对应技能 |
| **装备系统** | 数据提供方 | 本系统提供 `get_class_bonus()` 职业加成表；装备系统据此计算最终属性 `final_attr = base_value + class_bonus + equipment_bonus + barrier_bonus` |
| **战斗系统** | 数据消费者 | 本系统接收 `report_damage_dealt()` 触发职业经验计算；战斗系统需调用 `apply_class_exp()` |
| **AI系统** | 数据消费者 | AI角色同样遵循职业系统规则；AI转职逻辑与玩家一致 |
| **羁绊系统** | 无直接依赖 | 羁绊系统不直接依赖职业系统；但羁绊buff可能影响属性从而间接影响职业解锁 |
| **回合制模式** | 无直接依赖 | 回合制模式不直接依赖职业系统；但职业影响AI行为模式 |
| **UI系统** | 数据消费者 | UI系统渲染职业界面、解锁进度条、转职确认框；依赖本系统提供的数据结构 |

### Dependency Details

**属性与成长系统 → 职业系统**:
- 输入：`get_attribute_value(character_id, attr_id)` → 当前属性值
- 输入：`get_attribute_potential(character_id, attr_id)` → 潜质等级
- 输出：职业解锁判定结果
- 约束：职业系统不直接修改属性值，只读取和呈现

**战斗系统 ↔ 职业系统**:
- 战斗系统调用 `report_damage_dealt(character_id, damage, is_kill, battle_result)`
- 职业系统内部计算职业经验，调用 `apply_class_exp(character_id, class_id, exp)`
- 战斗系统不需知道职业经验的内部计算规则

**技能系统 ← 职业系统**:
- 职业系统触发 `class_changed(character_id, old_class_id, new_class_id)` 信号
- 技能系统监听该信号，执行技能解锁/锁定逻辑
- 职业系统提供 `get_class_skill_tree(class_id)` 返回该职业的技能列表

**装备系统 ← 职业系统**:
- 装备系统调用 `get_class_bonus(character_id)` 获取当前职业的固定加成
- 最终属性计算顺序：`final_attr = base_value + class_bonus + equipment_bonus + barrier_bonus`
- 职业变更后装备可行性需重新验证（如某些装备有职业限制）

## Tuning Knobs

### Core Balance Parameters

| Knob | Default | Safe Range | Affected Gameplay | Notes |
|------|---------|-----------|-----------------|-------|
| **进阶职业主属性门槛 (θ_p)** | 50 | 40-60 | 进阶职业解锁速度 | 提高门槛延长养成周期，降低门槛加速解锁 |
| **进阶职业副属性门槛 (θ_s)** | 40 | 30-50 | 进阶职业解锁速度 | 通常低于主属性门槛10点 |
| **进阶职业经验门槛 (E_req)** | 500 | 300-800 | 进阶职业解锁速度 | 与属性门槛共同控制解锁节奏 |
| **基础职业经验上限 (CAP_basic)** | 1000 | — | 基础职业升级周期 | 1000=等级2需1000exp |
| **进阶职业经验上限 (CAP_adv)** | 2000 | — | 进阶职业升级周期 | 2000=等级2需2000exp |
| **特殊职业成就点成本 (SPC_COST)** | 2000-3000 | 1500-4000 | 多周目价值感 | 帝王3000点，龙骑/夜使者2000点 |

### Class Experience Formula Parameters

| Knob | Default | Safe Range | Affected Gameplay | Notes |
|------|---------|-----------|-----------------|-------|
| **伤害经验系数** | 0.02 | 0.01-0.05 | 职业经验获取速度 | 降低使战斗经验贡献减小，升高使伤害输出更重要 |
| **单次伤害经验上限 (DMG_CAP)** | 500 | 300-800 | 大伤害输出的经验收益上限 | 防止超伤害量碾压获取过多经验 |
| **击杀奖励 (kill_bonus)** | 10 | 5-20 | 击杀导向 | 提高使击杀更重要，降低使战斗参与更重要 |
| **战斗参与奖励 (battle_bonus)** | 20 | 10-30 | 战斗参与感 | 即使0伤害也能获得基准经验 |

| Knob | Default | Safe Range | Affected Gameplay | Notes |
|------|---------|-----------|-----------------|-------|

| **转职确认延迟** | 即时 | 0.5-2s | 决策仪式感 | 延迟可增强"宣告"感，但过高会造成挫败 |

### Suggested First Tuning Session

建议首轮调参优先级（按影响力排序）:
1. 进阶职业经验门槛（E_req=500）— 最直接影响节奏
2. 伤害经验系数（0.02）— 影响战斗与养成的关联感
3. 进阶职业属性门槛（θ_p=50, θ_s=40）— 控制角色build方向验证时机

## UI Requirements

### 职业界面（Class Screen）

**布局结构**:
- 左侧：角色列表（当前队伍角色）
- 中央：角色详细信息（属性、职业信息、经验条）
- 右侧/下方：可解锁职业列表（显示所有进阶/特殊职业选项）

**双进度条组件**:
- 每个可解锁职业显示两个进度条
- 属性进度条：显示"当前值/门槛值"，配合对应属性图标
- 经验进度条：显示"当前经验/所需经验"，配合星星图标
- 颜色状态：灰色（未开始）、黄色（进行中）、绿色（达标）
- 数字标签：显示具体数值如"STR 52/50"

**职业信息面板**:
- 当前职业名称 + 图标 + 等级
- 职业加成详情（显示+STR、+AGI等具体数值）
- 已解锁技能列表预览

**可解锁职业面板**:
- 未解锁：灰色 + 显示解锁条件
- 已解锁：绿色边框 + "可转职"标签 + 转职按钮
- 转职按钮：仅在双进度条均达标时启用

### 转职确认界面（Promotion Confirmation）

**触发时机**: 玩家点击"转职"按钮后

**界面元素**:
- 遮罩背景（半透明黑色）
- 中央确认框
- 标题文字（如"选择即誓言"）
- 当前职业 → 目标职业的箭头/对比展示
- 属性变化预览（不变，仅展示）
- "确认转职"按钮（主要）
- "我再想想"按钮（次要）

**交互约束**:
- 确认后执行转职流程
- 取消返回职业界面
- 键盘支持：Enter确认，Esc取消

### 职业解锁提示（Class Unlock Toast）

**触发时机**: 职业解锁条件首次满足时

**显示方式**:
- 屏幕顶部弹出Toast通知
- 显示"职业解锁！"+ 职业图标 + 职业名称
- 自动消失（3秒）或点击消失

### 特殊职业兑换界面（SPC Exchange UI）

**触发时机**: 满足特殊职业条件后点击"兑换"时

**界面元素**:
- 成就点数余额显示
- 目标特殊职业 + 所需点数
- 确认/取消按钮
- 警告文本："成就点数兑换后不可退还"

**约束**:
- 成就点数不足时，兑换按钮禁用
- 确认后成就点数扣除，结果不可逆

## Acceptance Criteria

### AC.1 职业状态机

- [ ] **AC.1.1** 角色创建时自动进入 `BASIC_ACTIVE` 状态，初始化为战士职业
- [ ] **AC.1.2** 满足进阶职业条件时，状态转换为 `ADVANCED_UNLOCKED`
- [ ] **AC.1.3** 玩家确认转职后，状态从 `ADVANCED_UNLOCKED` 转换为 `ADVANCED_ACTIVE`
- [ ] **AC.1.4** 进阶职业激活后，满足成就点条件时状态转换为 `SPECIAL_UNLOCKED`
- [ ] **AC.1.5** 特殊职业激活后，状态进入 `SPECIAL_ACTIVE`（终态），不可再转职

### AC.2 职业解锁判定

- [ ] **AC.2.1** `CAN_UNLOCK` 函数正确判断三条件：主属性≥门槛 AND 副属性≥门槛 AND 职业经验≥门槛
- [ ] **AC.2.2** 基础职业默认解锁，无需任何条件判定
- [ ] **AC.2.3** 特殊职业额外检查成就点数是否足够
- [ ] **AC.2.4** 不满足解锁条件时，转职按钮禁用且显示具体缺少条件

### AC.3 职业经验获取

- [ ] **AC.3.1** 每次造成伤害时触发 `report_damage_dealt()`，正确计算职业经验
- [ ] **AC.3.2** 经验公式：`exp = min(floor(damage×0.02), 500) + kill_bonus + battle_bonus`
- [ ] **AC.3.3** 击杀时额外+10点，未击杀+0点
- [ ] **AC.3.4** 每场战斗结算时，无论胜败都+20点
- [ ] **AC.3.5** 0伤害战斗仍获得20点（战斗参与奖励）

### AC.4 转职流程

- [ ] **AC.4.1** 转职后属性V和潜质P完全保留
- [ ] **AC.4.2** 转职后新职业经验重置为0
- [ ] **AC.4.3** 转职后其他职业经验（如有）保留
- [ ] **AC.4.4** 转职后新职业加成立即生效
- [ ] **AC.4.5** 转职触发门槛奖励检查
- [ ] **AC.4.6** 转职触发 `class_changed` 信号，通知技能系统更新技能列表

### AC.5 职业属性加成

- [ ] **AC.5.1** 职业加成表正确应用：`final_attr = base_value + class_bonus + equipment_bonus + barrier_bonus`
- [ ] **AC.5.2** 职业变更后，装备可行性正确重新验证
- [ ] **AC.5.3** 属性降低至门槛以下时，职业加成暂时失效（不撤销职业状态）

### AC.6 职业等级计算

- [ ] **AC.6.1** 职业等级公式：`level = floor(E_c / CAP_c) + 1`
- [ ] **AC.6.2** 基础职业：0-999=等级1，1000=等级2
- [ ] **AC.6.3** 进阶职业：0-1999=等级1，2000=等级2

### AC.7 存档与恢复

- [ ] **AC.7.1** 职业状态、职业经验、已解锁职业列表正确保存到存档
- [ ] **AC.7.2** 玩家选择不转职的决策被记录
- [ ] **AC.7.3** 读档后职业状态正确恢复

### AC.8 UI显示

- [ ] **AC.8.1** 职业界面显示双进度条：属性进度（当前/门槛）和经验进度（当前/门槛）
- [ ] **AC.8.2** 双达标时两个进度条同时变绿
- [ ] **AC.8.3** 转职确认界面强调"这是你的选择"的仪式感
- [ ] **AC.8.4** 转职后获得新技能/外观变化有明确的视觉反馈

## Open Questions

### OQ.1 职业外观变化的实现粒度

**问题**: 转职后角色外观变化的具体程度？
- 是否仅改变光效/光环？
- 还是需要换装（武器、护甲变化）？
- 特殊职业是否有独特外观模型？

**待确认**: 美术团队规模与实现成本决定了外观变化的粒度。建议MVP阶段仅做光效/图标变化，Alpha阶段再扩展换装系统。

### OQ.2 基础职业间的初始选择

**问题**: 玩家在创建角色时是否需要主动选择基础职业？
- 当前设计：角色创建时自动初始化为战士
- 备选方案：玩家主动选择6个基础职业之一作为起始

**待确认**: 这影响角色创建界面的复杂度。建议MVP阶段使用自动初始化战士，后续可扩展为选择界面。

### OQ.3 职业经验获取是否应区分伤害来源

**问题**: 职业经验公式D.2中，是否应区分物理伤害和魔法伤害？
- 当前设计：统一公式，不区分来源
- 备选方案：物理系职业从物理伤害获取更多经验，魔法系职业反之

**待确认**: 这会让职业分化更明显，但增加公式复杂度。建议当前简单公式足够，必要时可通过职业加成表实现差异化。

### OQ.4 AI角色的职业选择策略

**问题**: AI队友/敌方角色如何选择职业路径？
- 是否让AI自动选择最优职业？
- 还是AI也有人为设定的固定职业路线？

**待确认**: 这影响AI系统的设计复杂度。建议MVP阶段AI使用预设职业路线，Alpha阶段再实现智能职业选择。

### OQ.5 多周目继承规则

**问题**: 二周目开始时，职业系统继承什么？
- 选项A：仅继承成就点数
- 选项B：继承角色基础属性和潜质，但不继承职业状态
- 选项C：完全继承（包括职业状态）

**待确认**: 多周目设计的一部分，影响重复游玩价值感。建议选项B——保留成长积累但不保留职业状态，让玩家在每周目重新做职业决策。
