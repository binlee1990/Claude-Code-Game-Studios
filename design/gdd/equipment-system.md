# 装备系统

> **Status**: In Design
> **Author**: user + agents
> **Last Updated**: 2026-04-16
> **Implements Pillar**: 系统互锁——装备驱动核心循环

## Overview

装备系统是角色战斗力的核心来源之一——它决定了"我穿什么"和"我有多强"。

游戏采用**词缀+强化+分解+套装**的装备框架：
- **词缀系统**：每件装备有1-4条随机词缀，基于品质等级决定词缀数量和数值范围
- **强化系统**：消耗金币和材料提升装备基础属性，有安全区和风险区
- **分解系统**：装备可分解为养成资源（强化材料、稀有材料）
- **套装系统**：集齐特定套装部件激活套装效果

装备品质等级：白 → 绿 → 蓝 → 紫 → 金

这种设计让装备既有随机性带来的惊喜感（词缀掉落），又有养成带来的满足感（强化提升），还有收集带来的策略感（套装效果）。

## Player Fantasy

玩家在装备系统中的核心感受是**"我的装备在变强"**——每一次强化成功、每一次词缀爆发、每一次集齐套装，都是角色战斗力的质的飞跃。

**强化成功**带来的是**投入的确认感**。当玩家看着+10的紫色装备发出金色光芒时，当保护符在强化失败时保住装备时——这种**我的决策保护了我的投入**的成就感是装备系统的核心情感。

**词缀爆发**带来的是**随机性的惊喜感**。当一件蓝装竟然出了两条满值词缀时，当金色装备第四条词缀是想要的属性时——这种**意外之喜**的兴奋是暗黑like装备系统的精髓。

**套装收集**带来的是**收集的满足感**。当2件套效果激活、4件套效果激活时，当玩家意识到"就差一件就齐了"时——这种**目标驱动的收集欲**是装备系统的持久动力。

**具体锚点：**
- +10强化成功，金光四射 → 投入得到回报的爽感
- 蓝装分解出了稀有材料 → 意外收获的惊喜
- 终于凑齐了套装，套装效果激活 → 收集完成的满足感
- 保护符保住+12装备 → 风险规避的如释重负

**参考游戏**：暗黑破坏神的词缀和套装系统、曹操传的装备强化系统

## Detailed Design

### Core Rules

**C.1 装备分类**

装备分为6个部位：

| 部位 | 占用位置 | 装备类型 |
|------|---------|---------|
| 武器 | 右手 | 剑、枪、斧、弓、法杖、拳套 |
| 护甲 | 身体 | 轻甲、中甲、重甲 |
| 头盔 | 头部 | 头盔、帽子、冠冕 |
| 护腿 | 腿部 | 护腿、裙子 |
| 护靴 | 脚部 | 靴子、鞋子 |
| 饰品 | 特殊 | 戒指、项链、护符 |

**C.2 装备品质**

| 品质 | 颜色 | 词缀数量 | 基础属性范围 | 强化等级上限 |
|------|------|---------|------------|------------|
| 白 | 白色 | 1条 | 低 | +5 |
| 绿 | 绿色 | 2条 | 中 | +10 |
| 蓝 | 蓝色 | 3条 | 高 | +15 |
| 紫 | 紫色 | 4条 | 很高 | +20 |
| 金 | 金色 | 4条+套装 | 极高 | +25 |

**C.3 词缀系统**

词缀类型（按属性分类）：

| 词缀类型 | 属性加成 | 备注 |
|---------|---------|------|
| 攻击类 | +STR、+AGI、+INT、+暴击率 | 数值随机 |
| 防御类 | +CON、+防御、+异常抵抗 | 数值随机 |
| 生存类 | +HP上限、+生命恢复 | 数值随机 |
| 特殊类 | +移动力、+攻击范围、+技能效果 | 稀有 |

词缀数值范围（基于品质）：

| 品质 | 攻击词缀范围 | 防御词缀范围 |
|------|------------|------------|
| 白 | +1~+3 | +1~+3 |
| 绿 | +2~+6 | +2~+6 |
| 蓝 | +4~+12 | +4~+12 |
| 紫 | +8~+20 | +8~+20 |
| 金 | +12~+30 | +12~+30 |

**C.4 强化系统**

> **设计考量（D-03）**: 安全区（+1~+5）100% 成功不构成玩家决策——每件装备都会自动强化到 +5。决策仅从 +6 开始。安全区的价值在于让玩家无需思考低阶强化，直接进入有意义的策略区间。如需引入安全区决策感，可将 +4/+5 改为 95%/90% 成功率（低成本风险），或让安全区仅到 +3。

强化消耗（基于强化等级）：

| 强化等级 | 金币成本 | 材料数量 | 成功率 |
|---------|---------|---------|--------|
| +1~+5 | 100×等级 | 5×等级 | 100% |
| +6 | 1000 | 30 | 70% |
| +7 | 1200 | 36 | 60% |
| +8 | 1500 | 45 | 50% |
| +9 | 2000 | 60 | 40% |
| +10 | 3000 | 90 | 30% |
| +11~+15 | 逐级递增 | 逐级递增 | 逐级递减 |
| +16~+20 | 更高 | 更高 | 更低 |
| +21~+25 | 极高 | 极高 | 极低 |

强化失败处理：
- +6以上失败：装备降级5级（如+8失败→+3）
- 有保护符：装备保持当前等级，保护符-1

**C.5 套装系统**

套装分组：

| 套装名称 | 部件数量 | 2件套效果 | 4件套效果 |
|---------|---------|----------|----------|
| 战士之力 | 4件（武器+护甲+护腿+靴子） | STR+10，HP+100 | 攻击时有20%概率造成双倍伤害 |
| 法师智慧 | 4件（武器+护甲+头盔+饰品） | INT+10，MP+50 | 技能伤害+15% |
| 弓手精准 | 4件（武器+护甲+护腿+饰品） | AGI+10，暴击率+5% | 暴击伤害+30% |
| 骑士荣耀 | 4件（武器+护甲+头盔+护腿） | CON+10，防御+20 | 受到攻击时反弹10%伤害 |

**C.6 装备分解**

分解产出：

| 装备品质 | 分解产出 |
|---------|---------|
| 白 | 基础材料×2 |
| 绿 | 基础材料×5 |
| 蓝 | 基础材料×10，稀有材料×1（20%概率） |
| 紫 | 基础材料×20，稀有材料×2（50%概率） |
| 金 | 基础材料×50，稀有材料×5（100%概率） |

**C.7 装备获取**

装备主要获取渠道：
- 战斗掉落（普通敌人、Boss）
- 商店购买（消耗金币）
- 任务奖励
- 锻造打造（消耗材料）

## Formulas

### D.1 最终属性计算

```
final_attribute = base_value + class_bonus + equipment_bonus + barrier_bonus
```

**变量定义:**

| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| base_value | bv | int | [0, 999] | 基础属性值（属性与成长系统） |
| class_bonus | cb | int | [0, +) | 职业加成（职业系统） |
| equipment_bonus | eb | int | [0, +) | 装备加成（所有装备词缀加成总和） |
| barrier_bonus | bb | int | [0, +) | 壁障突破奖励（属性与成长系统） |
| final_attribute | fa | int | [0, +) | 最终生效的属性值 |

**输出范围:** 所有加成的总和，无理论上限

**示例:**
- 基础力量100，职业加成+15，装备加成+30，壁障奖励+10：`final = 100 + 15 + 30 + 10 = 155`

---

### D.2 词缀数值计算

```
affix_value = random_range(min_affix, max_affix) × quality_multiplier
```

**变量定义:**

| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| min_affix | min | int | [1, 100] | 词缀最小值（固定） |
| max_affix | max | int | [min, 100] | 词缀最大值（固定） |
| quality_multiplier | qm | float | {1.0, 1.5, 2.0, 3.0, 5.0} | 品质系数（白/绿/蓝/紫/金） |
| affix_value | av | int | [1, 500] | 最终词缀值 |

**输出范围:** 1到500

**示例:**
- 蓝色装备攻击词缀范围4-12：`random_range(4, 12) × 2.0 = 8 ~ 24`

---

### D.3 强化成功率

风险区（+6以上）成功率由 C.4 表格直接定义，不使用简单线性公式（因为实际成功率在 +6 处有显著断崖式下降，不符合线性衰减）。

**C.4 表格成功率（权威来源）：**

| 强化等级 | 成功率 |
|---------|--------|
| +1~+5 | 100% |
| +6 | 70% |
| +7 | 60% |
| +8 | 50% |
| +9 | 40% |
| +10 | 30% |
| +11~+25 | 逐级递减 |

**输出范围:** +6 起从 70% 逐级递减至极低值

---

### D.4 强化降级幅度

```
new_enhancement_level = max(0, current_level - downgrade_penalty)
```

**变量定义:**

| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| current_level | cl | int | [6, 25] | 强化失败前的等级 |
| downgrade_penalty | dp | int | 5 | 降级幅度 |
| new_enhancement_level | nel | int | [0, 20] | 降级后的等级 |

**输出范围:** 0到20（+5以上失败不会降到0以下）

**示例:**
- +10强化失败：`new = max(0, 10-5) = +5`
- +8强化失败：`new = max(0, 8-5) = +3`

## Edge Cases

### E.1 强化达到上限

**场景**: 装备强化达到当前品质的上限等级

**处理**:
- 强化按钮禁用
- UI显示："已达到最高强化等级"
- 玩家需要提升装备品质（如蓝装→紫装）才能继续强化

### E.2 强化失败无保护符

**场景**: +6以上强化失败，背包中无保护符

**处理**:
- 装备降级5级
- 材料不返还
- UI显示强化失败特效和结果

### E.3 保护符耗尽

**场景**: 保护符为0时选择强化

**处理**:
- 强化界面显示警告："保护符为0，强化风险极高"
- 玩家仍可选择强化（确认对话框）
- 强化失败后装备降级

### E.4 套装效果叠加

**场景**: 同一套装多件装备的效果

**处理**:
- 2件套效果：装备2件后自动激活
- 4件套效果：装备4件后自动激活
- 套装效果不叠加（同一套装只有一个4件套效果）
- 不同套装效果可以叠加

### E.5 装备分解与强化材料返还

**场景**: 装备分解时的返还规则

**处理**:
- 分解不返还强化材料（强化投入已经产生价值）
- 分解仅产出新材料
- 已强化的装备分解时，强化等级不影响分解产出

### E.6 装备掉落重复

**场景**: 掉落已拥有的装备

**处理**:
- 重复装备可选择：卖出获得金币，或分解获得材料
- 建议添加装备图鉴，记录已掉落的装备

### E.7 词缀冲突

**场景**: 两件装备词缀产生冲突效果

**处理**:
- 词缀效果独立计算，不冲突
- 如有冲突，按描述优先级执行
- 玩家需要理解词缀组合的实际效果

## Dependencies

### Upstream Dependencies

| System | Dependency Type | Interface Contract |
|--------|---------------|-------------------|
| **属性与成长系统** | 硬依赖 | 装备加成计入最终属性；属性与成长系统提供壁障突破奖励 |
| **资源经济** | 硬依赖 | 强化消耗金币和材料；分解产出金币和材料；保护符由资源经济管理 |
| **职业系统** | 软依赖 | 职业可能影响可装备的武器类型 |

### Downstream Dependents

| System | Dependency Type | Interface Contract |
|--------|---------------|-------------------|
| **战斗系统** | 数据消费者 | 战斗系统读取装备属性计算最终属性；装备词缀影响战斗数值 |
| **基地系统** | 数据消费者 | 基地提供装备锻造功能 |
| **羁绊系统** | 无直接依赖 | 羁绊buff可能影响装备选择 |
| **HP系统** | 数据消费者 | HP 公式调用 `EquipmentComponent.get_stat_bonus("hp")` 聚合 AffixType.HP 词缀与 4 件套 hp 加成 |

### Suggested First Tuning Session

建议首轮调参优先级（按影响力排序）:
1. 强化安全区（+1~+5）— 玩家安全感核心

## Visual/Audio Requirements

### Visual Requirements

**装备品质显示**:
- 装备图标边框颜色对应品质（白/绿/蓝/紫/金）
- 强化等级用数字显示在装备图标上（+10等）
- 套装装备有特殊边框标识

**强化特效**:
- 强化成功：金色光芒从装备中心向外扩散
- 强化失败：红色碎裂特效
- 保护符生效：蓝色护盾包裹装备的特效

**套装效果激活**:
- 2件套激活时：套装图标发光
- 4件套激活时：更强的光效+套装名称显示

### Audio Requirements

**装备操作音效**:
- 装备穿戴：金属碰撞音效
- 装备卸下：金属解除音效
- 强化成功："铛！"清脆金属声
- 强化失败："咔嚓"破碎声
- 保护符生效：魔法护盾展开音效
- 分解装备：熔炉燃烧音效

## UI Requirements

### 装备信息界面

**装备详情面板**:
- 位置：角色面板内或弹出框
- 显示内容：
  - 装备名称和品质（颜色标识）
  - 强化等级
  - 基础属性
  - 词缀列表（数值和类型）
  - 套装信息（如果是大套装）
  - 估价（卖出价格）

### 强化界面

**强化面板**:
- 位置：装备详情面板内或独立界面
- 显示内容：
  - 当前强化等级
  - 目标强化等级选择
  - 消耗预览（金币+材料）
  - 成功率显示
  - 保护符选项（勾选框）
- 强化按钮状态：
  - 资源充足：可用（高亮）
  - 资源不足：禁用（置灰）+ 显示缺少数量
  - 保护符为0且+6以上：警告色

**强化结果弹窗**:
- 成功：金光扩散+"强化成功！"
- 失败（无保护符）：碎裂特效+"强化失败，降至+XX"
- 失败（有保护符）：护盾光效+"保护符生效！"

### 套装查看界面

**套装图鉴**:
- 位置：菜单或图鉴界面
- 显示内容：
  - 所有已知套装列表
  - 每个套装的收集进度（X/4件）
  - 已激活的套装效果预览
  - 缺少的部件提示

## Acceptance Criteria

### AC.1 装备系统

- [ ] **AC.1.1** 装备正确分为6个部位，每个部位只能装备一件
- [ ] **AC.1.2** 装备品质正确影响词缀数量（白1条-金4条+套装）
- [ ] **AC.1.3** 装备基础属性根据品质正确生成

### AC.2 强化系统

- [ ] **AC.2.1** +1~+5强化100%成功
- [ ] **AC.2.2** +6以上强化有失败概率，成功率随等级递减
- [ ] **AC.2.3** 强化失败时无保护符装备降级5级
- [ ] **AC.2.4** 有保护符时强化失败不降级，保护符消耗1个

## Open Questions

### OQ.1 装备交易是否限制

**问题**: 玩家之间是否可以交易装备？
- 当前设计：纯单机，无交易功能
- 备选方案：玩家之间可以交易非绑定装备

**待确认**: 纯单机无需考虑交易。建议MVP阶段无交易。

### OQ.2 装备是否需要耐久度

**问题**: 装备是否需要有耐久度机制？
- 当前设计：装备无耐久度，简化系统
- 备选方案：装备有耐久度，需要维修

**待确认**: 耐久度增加策略维度但也增加复杂度。建议MVP阶段无耐久度。

### OQ.3 装备词缀池大小

**问题**: 词缀池应该有多大？
- 当前设计：基础攻击/防御词缀+少量特殊词缀
- 备选方案：更大的词缀池，增加随机多样性

**待确认**: MVP阶段建议小词缀池，确保核心体验。
