# 美术风格系统

> **Status**: Designed (pending review)
> **Author**: binlee1990 + agents
> **Last Updated**: 2026-04-22
> **Implements Pillar**: 系统互锁（视觉一致性保障所有系统联动的可读性）

## Overview

美术风格系统是 SRPG 所有视觉呈现的基础设施层，定义并强制执行统一的视觉管线规范，确保游戏内所有渲染输出符合 "HD-2D 中国风" 的视觉一致性要求。

该系统不直接与玩家交互。玩家通过战斗可读性、地图空间感、UI 沉浸感间接体验美术风格的效果。视觉方向的创意定义由 `design/art/art-bible.md` 承担；本 GDD 描述美术风格作为**引擎子系统的行为规则**——包括渲染管线配置、资源规格约束、氛围状态机驱动、性能降级策略，以及与其他系统的视觉集成接口。

核心职责：
1. **渲染管线管理**：配置并维护 HD-2D 渲染管线（3D 场景 + 像素角色 + 后处理），确保视觉输出符合 art-bible 定义的三层视觉原则
2. **氛围状态驱动**：根据游戏状态（探索/战斗/叙事抉择）切换光照、粒子、色调参数，由氛围状态机统一控制
3. **资源规格强制**：定义并校验角色精灵、场景资源、UI 元素的像素规格和格式标准，防止不一致资产进入构建
4. **性能降级**：在帧预算不足时按优先级降级视觉效果，保证 60FPS 目标

## Player Fantasy

玩家不应"注意到"美术风格系统——他们应该沉浸在一个**可游历的中国山水长卷**中，在每一次战术决策时获得清晰的空间信息，在每一段叙事时刻感受到氛围的自然流转。

美术风格系统赋能的玩家体验：
- **战术可读性幻想**：战场上，地形高度差、元素交互区域、敌我阵型一目了然。玩家感受"我能看清局势并做出明智决策"，而非"画面信息太多看不懂"
- **氛围沉浸幻想**：从探索的宁静山水到战斗的水墨浸染，再到叙事抉择的朱砂聚焦，玩家感受"世界随我的行动自然呼吸"
- **东方美学身份幻想**：HD-2D 中国风不是装饰，而是身份——这是属于中国叙事的 SRPG，水墨留白给策略思考提供视觉呼吸，像素精密度给养成系统提供辨识度

美术风格系统成功时，玩家不会评价"画面好看"，而是评价"这个游戏很有氛围"或"战场信息很清晰"——系统隐形，体验可见。

## Detailed Design

### Core Rules

**规则 1 — 统一渲染管线**

所有游戏场景必须通过 HD-2D 渲染管线输出，管线层级如下：

```
3D 场景层（WorldEnvironment + DirectionalLight3D）
  ↓
3D 地形/建筑（GridMap 或 MeshInstance3D）
  ↓
2D 像素角色层（SubViewport → CanvasTexture 方案，或 Sprite3D）
  ↓
后处理栈（Environment Resource）:
  1. 体积光（VolumetricFog）
  2. 暗角（Vignette）
  3. 色调映射（ColorCorrection / ColorAdjustment）
  4. 辉光（Glow）
  ↓
UI 层（CanvasLayer，独立于场景渲染）
```

Godot 4.6.2 中，管线由 `WorldEnvironment` 节点驱动，`Environment` Resource 控制后处理参数。角色像素精灵使用 `Sprite3D`（保持 3D 空间中的 z-sorting）或 `SubViewport` + `BackBufferCopy` 方案（需 ADR 决定具体方案）。

**规则 2 — 氛围状态机驱动**

渲染管线的所有动态参数由氛围状态机统一驱动。状态机定义见 art-bible 2.4 节，三态模型：

| 状态 | 光照目标 | 色调偏移 | 粒子系统 | 过渡时间 |
|------|----------|----------|----------|----------|
| Explore | 5500K，低对比 | 青墨基调 | 尘埃/水汽（低密度） | — |
| Combat | +200-400K 暖偏移 | 赭墨基调 + 环境残留色 | 打击/轨迹/受击（中密度） | 0.8s ease_in_out |
| Narrative | 聚光效果，非焦点 -60% | 路线色偏移 30-50% | 信念光点/路线确认 | 0.5s ease_in_out |

状态切换最小间隔 0.5s，同一帧只能处于一种状态。叙事抉择可从 Explore 或 Combat 进入。

**规则 3 — 资源规格强制**

所有视觉资源必须满足以下规格才能进入构建：

| 资源类型 | 格式 | 像素规格 | 色板约束 |
|----------|------|----------|----------|
| 角色精灵（主角/重要NPC） | PNG | 32×32 ~ 48×48 | 水墨五色 + 元素色 |
| 角色精灵（普通NPC） | PNG | 16×16 ~ 24×24 | 水墨五色 |
| 战斗单位精灵 | PNG | 32×32 | 水墨五色 + 元素色 |
| UI 图标角色 | PNG | 16×16 | UI 色板（art-bible 4.4） |
| 场景纹理 | WebP/ASTC | 2 的幂次，最大 2048×2048 | 场景色调范围 |
| 3D 模型 | glTF 2.0 | ≤5000 三角面/角色 | — |
| UI 元素 | PNG | 整数像素对齐 | UI 色板 |

命名规范：`[category]_[name]_[variant].png`（如 `char_hero_warrior_idle.png`）。

**规则 4 — 性能降级策略**

帧预算 16.6ms（60FPS）。当帧时间持续超过 14ms 时，按以下优先级降级：

| 优先级 | 效果 | 降级行为 |
|--------|------|----------|
| 1 | 体积光 | 降至环境遮蔽探针 |
| 2 | 战斗粒子 | 降低密度至 50%，禁用次要粒子 |
| 3 | 叙事粒子 | 路线确认爆发降低 30% |
| 4 | 情绪色彩渗透 | 完全关闭 |

UI 层和色盲安全标识**永不降级**。

**规则 5 — 色盲安全强制**

所有语义色彩标识必须配合唯一形状备份（art-bible 4.5）。视觉系统必须提供"色盲模拟"预览模式（开发期验证用），使用 WCAG 2.1 AA 级对比度标准。

### States and Transitions

氛围状态机的完整状态转换图：

```
Explore ←──0.8s──→ Combat
   ↑                   ↑
   │                   │
   └─→ Narrative ──────┘
       (可从两个状态进入)
```

| 从 → 到 | 触发条件 | 过渡动画 | 参数变化 |
|----------|----------|----------|----------|
| Explore → Combat | 进入战斗 | 0.8s ease_in_out | 前 0.3s 色调偏移，后 0.5s 光照插值 |
| Combat → Explore | 战斗结束 | 0.8s ease_in_out | 反向插值 |
| Explore → Narrative | 接近叙事抉择点 | 0.5s ease_in_out | 聚光 + 路线色偏移 |
| Combat → Narrative | 战斗中叙事触发 | 0.5s ease_in_out | 战斗参数保持底线，叠加叙事参数 |
| Narrative → Explore | 选择完成 | 0.5s ease_in_out | 回归青墨基调 |

**约束**：状态切换最小间隔 0.5s，防止快速往返闪烁。

### Interactions with Other Systems

| 下游系统 | 消费的数据 | 接口方向 | 说明 |
|----------|------------|----------|------|
| **视角与地图**（camera-map-system） | 3D 场景配置、网格尺寸、高度差渲染参数 | 美术风格 → 视角地图 | 场景渲染管线为地图系统提供视觉底座 |
| **UI 系统**（ui-system） | UI 色板、按钮状态色、色盲形状标识 | 美术风格 → UI | UI 层引用 art-bible 4.4 色板，不自行定义颜色 |
| **战术机制**（tactical-mechanism） | 元素色彩绑定（五行色）、元素视觉效果 | 美术风格 → 战术 | 火/水/风/土/金的命中特效、轨迹残留由色彩系统提供 |
| **回合制模式**（turn-based-mode） | 氛围状态（Explore/Combat） | 回合制 → 美术风格 | 回合制系统通知氛围状态机切换 |
| **叙事/信念系统** | 路线色彩标识、叙事粒子 | 美术风格 → 叙事 | 三路线色（朱红/墨黑/青绿）统一由色彩系统提供 |
| **战斗结算**（battle-settlement） | 胜利粒子效果参数 | 战斗结算 → 美术风格 | 结算触发胜利粒子，密度 50-80/秒 |
| **AI 系统** | 无直接交互 | — | AI 不消费视觉参数 |

## Formulas

**F1 — 帧预算分配**

```
frame_budget = 1000 / target_fps = 1000 / 60 = 16.6ms
```

| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| target_fps | fps | int | 60（固定） | 目标帧率 |
| frame_budget | fb | float | 16.6ms | 每帧可用时间 |
| degradation_threshold | dt | float | 14.0ms | 降级触发阈值 |
| visual_budget | vb | float | fb - logic_budget | 渲染管线可用时间 |

**输出范围**：visual_budget ≈ 10-12ms（扣除逻辑/物理开销后）

**F2 — 氛围过渡插值**

```
transition_progress = ease_in_out(t / duration)
```

其中 `ease_in_out(t)` 定义为：

```
if t < 0.5:
    return 2 * t * t
else:
    return 1 - pow(-2 * t + 2, 2) / 2
```

| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| t | t | float | 0.0 ~ duration | 当前过渡经过时间 |
| duration | d | float | 0.5s 或 0.8s | 过渡总时长（见状态转换表） |
| transition_progress | p | float | 0.0 ~ 1.0 | 过渡进度 |

**输出范围**：0.0（起始状态）到 1.0（目标状态），非线性曲线避免"滤镜闪烁"

**F3 — 色温偏移**

```
combat_color_temp = base_color_temp + warm_offset
```

```
base_color_temp = 5500K
warm_offset = clamp(scene_offset, 200, 400)
```

| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| base_color_temp | T_base | int | 5500K | 探索阶段基准色温 |
| warm_offset | ΔT | int | 200 ~ 400K | 战斗暖偏移量 |
| scene_offset | ΔT_scene | int | 200 ~ 400K | 场景特定偏移（森林200，洞穴400） |

**F4 — 聚光强度衰减（叙事状态）**

```
focus_intensity = base_light * (1.0 - dim_factor)
```

```
base_light = 1.0（归一化）
dim_factor = 0.6 ~ 0.7（非焦点区域）
```

| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| base_light | L_base | float | 1.0 | 基准光照强度（归一化） |
| dim_factor | D | float | 0.6 ~ 0.7 | 非焦点区域衰减比例 |
| focus_intensity | L_focus | float | 0.3 ~ 0.4 | 非焦点区域最终强度 |

**F5 — 粒子密度降级**

```
degraded_density = base_density * degradation_factor
```

| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| base_density | ρ_base | int | 见 art-bible 2.2 | 各粒子类型的基础密度 |
| degradation_factor | k | float | 0.5 ~ 1.0 | 降级系数（1.0=全开，0.5=半量） |

**降级系数映射**：

| 降级级别 | k 值 | 触发条件 |
|----------|------|----------|
| 0（全开） | 1.0 | 帧时间 < 14ms |
| 1（轻度） | 0.8 | 帧时间 ≥ 14ms，关闭体积光 |
| 2（中度） | 0.5 | 帧时间 ≥ 14ms 持续 3 帧，关闭体积光 + 粒子减半 |
| 3（重度） | 0.3 | 帧时间 ≥ 14ms 持续 10 帧，仅保留核心粒子 |

## Edge Cases

- **If 氛围过渡被新状态中断**（如 Explore→Combat 过渡中触发 Narrative）：立即将当前插值进度作为起点，以新目标状态重新开始过渡。不回滚已完成的参数变化。原因是避免视觉跳变，连续过渡比"先回退再前进"更自然。

- **If 帧时间在降级后恢复**（帧时间回落到 <14ms 持续 30 帧）：以 0.5s 渐变恢复上一降级级别，不直接跳回全开。逐级恢复（L3→L2→L1→L0），每级间隔 30 帧。原因是防止帧率在阈值附近震荡导致视觉效果反复闪烁。

- **If 资源不满足规格约束**（如角色精灵尺寸为 30×30 而非 32×32）：资源导入管线拒绝该资源并记录警告日志，游戏运行时使用占位符（纯色方块 + 尺寸标签）。原因是宁可显示明显的占位符也不显示错误尺寸的资源，防止对齐和渲染问题。

- **If 同时触发两个氛围状态切换**（如战斗结束时立即触发叙事抉择）：按 Combat→Explore→Narrative 顺序排队，不跳过中间状态。Explore 过渡使用缩短的 0.3s 版本（正常 0.8s），然后进入 Narrative。原因是状态机不允许跨态跳转，但可以加速中间态。

- **If 色盲模式下两个不同语义使用了相同亮度**（如危险橙红和灼烧暗橙在去色后亮度接近）：色盲安全验证必须在提交前检查所有色彩对的亮度差。If 亮度差 < 15%，强制调整其中一个的亮度（保持色相不变），或替换为完全不同的辅助形状。原因是亮度不足时形状备份是唯一区分手段。

- **If 体积光在低配硬件上完全不可用**（Godot 4.6 VolumetricFog 需要 GLES3+ / D3D12）：回退到预计算的体积光贴图（baked lightmap）+ 2D 径向渐变叠加。性能成本约 0.5ms，视觉保真度降低但保留基础氛围感。

- **If 场景中像素角色与 3D 环境的 z-sorting 冲突**（角色被地形遮挡或浮空）：使用 Godot 的 `Sprite3D.texture_filter = nearest` + 调整 `offset` 偏移值。如果 Sprite3D 方案不可行，回退到 SubViewport + BackBufferCopy 方案。具体方案由 ADR 决定。

## Dependencies

### 上游依赖（本系统依赖）

无。美术风格系统是 Foundation 层的根节点，不依赖任何其他游戏系统。

外部依赖：
- **art-bible.md**（`design/art/art-bible.md`）：视觉方向权威文档，本系统消费其定义的所有参数（光照值、色板、粒子规格）。art-bible 变更时，本系统必须同步更新对应参数。
- **Godot 4.6.2 引擎**：渲染管线能力受引擎版本约束。具体引擎 API 选择（Sprite3D vs SubViewport 等）由 ADR 决定，不在本 GDD 范围内。

### 下游依赖（其他系统依赖本系统）

| 系统 | 依赖类型 | 消费接口 | 依赖强度 |
|------|----------|----------|----------|
| **视角与地图** | 3D 场景渲染配置、网格规格 | 场景管线参数、高度差渲染设置 | 硬依赖 |
| **UI 系统** | UI 色板、色盲标识 | art-bible 4.4 色板、4.5 形状备份规范 | 硬依赖 |
| **战术机制** | 元素色彩绑定 | 五行色定义（art-bible 4.3）、元素视觉效果 | 硬依赖 |
| **回合制模式** | 氛围状态切换 | 氛围状态机 API（set_atmosphere_state） | 硬依赖 |
| **叙事/信念系统** | 路线色彩标识 | 三路线色值、叙事粒子参数 | 硬依赖 |
| **战斗结算** | 胜利粒子效果 | 粒子密度/持续时间参数 | 软依赖（降级后仍可正常结算） |
| **AI 系统** | 无 | — | 无依赖 |

### 双向一致性要求

- 视角与地图 GDD 中应注明"场景渲染管线由美术风格系统提供"
- UI 系统 GDD 中应注明"UI 色板引用自美术风格系统，不自行定义颜色"
- 战术机制 GDD 中应注明"元素色彩绑定引用自美术风格系统"

## Tuning Knobs

| Knob ID | 名称 | 类型 | 默认值 | 安全范围 | 极端行为 | 影响系统 |
|---------|------|------|--------|----------|----------|----------|
| `TK-ART-01` | 战斗色温偏移 | int | 300K | 200 ~ 400K | 过高=战场泛黄；过低=无明显战斗感 | 氛围系统 |
| `TK-ART-02` | 战斗过渡时长 | float | 0.8s | 0.3 ~ 2.0s | 过短=突兀切换；过长=拖沓感 | 氛围状态机 |
| `TK-ART-03` | 叙事过渡时长 | float | 0.5s | 0.2 ~ 1.5s | 过短=聚焦不充分；过长=等待焦躁 | 氛围状态机 |
| `TK-ART-04` | 状态切换最小间隔 | float | 0.5s | 0.1 ~ 1.0s | 过短=快速闪烁；过长=响应迟钝 | 氛围状态机 |
| `TK-ART-05` | 降级触发阈值 | float | 14.0ms | 12.0 ~ 15.0ms | 过低=频繁降级；过高=卡顿后才降级 | 性能降级 |
| `TK-ART-06` | 降级恢复等待帧数 | int | 30 帧 | 10 ~ 60 帧 | 过少=振荡；过多=恢复迟缓 | 性能降级 |
| `TK-ART-07` | 叙事非焦点衰减 | float | 0.65 | 0.4 ~ 0.8 | 过高=背景太暗看不清；过低=聚焦感弱 | 叙事氛围 |
| `TK-ART-08` | 暗角强度基准 | float | 0.2 | 0.0 ~ 0.5 | 过高=视野受限；过低=空间感弱 | 后处理 |
| `TK-ART-09` | 体积光浓度基准 | float | 0.2 | 0.1 ~ 0.4 | 过高=模糊；过低=无空气感 | 后处理 |
| `TK-ART-10` | 胜利粒子密度 | int | 65/秒 | 30 ~ 100/秒 | 过高=GPU 压力；过低=奖励感弱 | 战斗结算 |
| `TK-ART-11` | 降级系数（中度） | float | 0.5 | 0.3 ~ 0.7 | 过低=视觉空洞；过高=降级不明显 | 性能降级 |

**交互注意**：
- `TK-ART-02` 和 `TK-ART-03` 受 `TK-ART-04` 约束——过渡时长不能小于最小间隔
- `TK-ART-05` 越高，`TK-ART-06` 的值需要相应增大（阈值宽松时恢复应更保守）
- `TK-ART-08` 和 `TK-ART-09` 同时增大可能导致对比度不足——视觉层次变平

## Visual/Audio Requirements

本系统不直接产出视觉/音频资产，而是定义视觉管线的渲染约束。具体视觉方向由 `design/art/art-bible.md` 承担。

渲染管线必须满足的视觉标准：
- 像素角色在 3D 场景中的 nearest-neighbor 缩放，不允许模糊插值
- 后处理栈（体积光/暗角/辉光/色调映射）的参数范围由 art-bible 第二章约束
- 场景帧间一致性：同一场景连续帧中，相同像素的颜色差异不得超过 5%（防止色彩抖动）

音频与美术风格系统无直接交互。音频氛围由音频系统独立管理。

## UI Requirements

本系统不直接包含 UI 界面。但美术风格系统为 UI 系统提供以下约束：

- UI 必须使用 art-bible 4.4 定义的 UI 色板，不自行定义颜色
- UI 图标中的形状标识必须符合 art-bible 4.5 色盲安全规范
- UI 层（CanvasLayer）独立于场景渲染管线，不受性能降级影响
- UI 边框风格统一使用 1px 墨线描边（art-bible 3.3.1）

## Acceptance Criteria

**GIVEN** 游戏运行于目标硬件（PC / Steam 最低配置），**WHEN** 进入任意场景，**THEN** 帧率稳定 ≥60FPS，无帧时间超过 16.6ms 的持续情况（允许单帧抖动）。

**GIVEN** 游戏处于 Explore 状态，**WHEN** 触发战斗进入 Combat 状态，**THEN** 氛围在 0.8s（±0.1s）内完成过渡，色温偏移 200-400K，色调从青墨转为赭墨。

**GIVEN** 游戏处于 Combat 状态，**WHEN** 战斗结束返回 Explore 状态，**THEN** 氛围在 0.8s（±0.1s）内完成反向过渡，所有参数回到 Explore 基准值。

**GIVEN** 游戏处于任意氛围状态，**WHEN** 接近叙事抉择点进入 Narrative 状态，**THEN** 非焦点区域光照强度降低 60-70%，路线色偏移 30-50%，过渡 0.5s（±0.1s）。

**GIVEN** 帧时间持续超过 14ms 达 3 帧以上，**WHEN** 性能降级系统激活，**THEN** 按优先级依次降级（体积光→粒子→叙事粒子→情绪色彩），UI 层和色盲标识不受影响。

**GIVEN** 性能降级已激活，**WHEN** 帧时间恢复 <14ms 持续 30 帧，**THEN** 逐级恢复视觉效果，每级间隔 30 帧，不直接跳回全开。

**GIVEN** 任意视觉资源（精灵/纹理/模型），**WHEN** 尝试导入到构建中，**THEN** 管线校验像素规格、格式、色板合规性。不合规资源被拒绝，运行时显示占位符。

**GIVEN** 游戏使用色盲模拟滤镜（Protanopia/Deuteranopia/Tritanopia/Achromatopsia），**WHEN** 检查所有语义色彩组合，**THEN** 每对色彩均可通过形状备份独立区分，无纯颜色依赖。

**GIVEN** UI 层显示，**WHEN** 检查所有文字与背景的色彩对比度，**THEN** 普通文字对比度 ≥ 4.5:1，大文字 ≥ 3:1，图形元素 ≥ 3:1（WCAG 2.1 AA 级）。

**GIVEN** 两个氛围状态切换在 0.5s 内相继触发，**WHEN** 状态机处理切换请求，**THEN** 第二次切换被排队，不跳过中间状态，不产生闪烁。

## Open Questions

- **OQ-1**：像素角色的渲染方案（Sprite3D vs SubViewport + BackBufferCopy）需要 ADR 决定。影响 z-sorting 行为和性能开销。
- **OQ-2**：资源规格校验是在 Godot 导入时自动执行（自定义导入插件）还是独立的构建前检查脚本？影响开发工作流。
- **OQ-3**：最低目标硬件配置尚未确定。性能降级的 L3 级别（重度降级）是否需要支持集成显卡？影响降级策略的最终底线。
