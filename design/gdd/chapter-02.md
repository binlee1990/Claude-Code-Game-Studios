## 1. Overview

第二章"义路歧途"是玩家在完成第一章守关任务后踏入更广阔势力格局的第一个章节。
本章共三场战斗，叙事核心是：主角队伍追查叛逃的飞骑营（R3·陈朗所在部队），
途中卷入地方豪强与流民军的冲突，被迫在"平定秩序"与"庇护弱者"之间做出立场选择。
设计目标有三：其一，使三路线信念值（仁/义/智）在本章首次出现可感知的差异（+15 分以上的分歧）；
其二，引入一场带新机制的关卡（保护 NPC 的护送战）；其三，将 Boss 战难度正式提升至
`enemy_stat_multiplier 1.15–1.30`，并在最终战中首次引入限制回合数压力。
本章结束后玩家会进入基地系统初步功能，完成第一次果子投资决策，为第三章的职业分化奠基。

## 2. Player Fantasy

### 目标情感（MDA Aesthetics）

**主导情感：叙事（Narrative）+ 挑战（Challenge）**
**次要情感：发现（Discovery）**

玩家在本章应感受到以下三层体验：

**层一：选择有重量**
第一次发现"同一个事件，选不同立场，战场和队友都会不同"。
不是抽象的"路线A/B"，而是具体到：选择庇护流民，当前战局多一个可控 NPC 但守护压力更大；
选择配合豪强，战术更简洁但战后有角色对主角产生信念值反弹。
玩家第一次感到"我的选择正在塑造这支队伍的性格"。

**层二：战术维度扩展**
Ch.1 教会了"高低差 + 元素交互 + 兵种克制"三层基础。Ch.2 的护送战第一次引入**动态目标**：
需要保护的 NPC 会移动，战场不再只有"我方 vs 敌方"，增加了保护代价的计算压力。
玩家完成护送战后应有"原来 SRPG 不只是歼灭战"的认知升级。

**层三：养成的第一次分叉**
本章战后结算首次出现果子选择界面（力量果子 vs 敏捷果子 vs 智力果子 各一份，
只能选两种），玩家第一次感到"我在主动塑造角色的成长方向"。
这与信念值路线选择产生共振：走仁路线的玩家往往倾向投智力，走义路线的倾向力量——但没有对错。

### SDT 验证

| 心理需求 | 满足机制 |
|---------|---------|
| **自主感（Autonomy）** | 立场选择影响第二战的敌方构成和可用 NPC；果子二选三均为有效选项 |
| **胜任感（Competence）** | 护送战引入前有 NPC 避险提示，失败惩罚为退场剧情（可招回），不是 Game Over |
| **关联感（Relatedness）** | R3·陈朗在本章重新登场（变为可能的对手或盟友），玩家对 Ch.1 队友的羁绊被激活 |

**失败条件**：我方所有战斗单位全部退场。

| R1 | 玩家（后备） | basic_cleric | 神官 | 可投入战场 |
| R4 | 玩家（后备） | basic_mage | 法师 | 可投入战场 |
| E1 | 敌方 | basic_knight | 豪强家丁队长（精英） | 驱散分支时为敌；收留分支时为中立 |
| E2 | 敌方 | basic_warrior | 家丁×2 | — |
| E3 | 敌方 | basic_archer | 哨兵弓手 | 高台位置，高低差优势 |

**开场叙事选择（信念值节点 B2-N1）**：

**新机制实验**：本战为 Ch.2 教学战，不引入新机制，重点在于立场叙事的重量感。

### 3.4 新机制：NPC 保护（ch.2-2A 专用）

**核心设计意图**：让玩家第一次体验"动态目标"——我的决策不只是攻击敌人，
还要在队伍机动和保护平民之间权衡。

**NPC 行为规则（王秀 AI）**：
1. 每回合（王秀的速度序列位置）自动向安全区执行 A* 最短路径移动，每次移动 1 格。
2. 若安全区方向被敌方单位占据（直线距离 ≤ 2 格），王秀停止移动该回合（"畏缩"）。
3. 若我方单位占据王秀相邻 4 格之一，触发**护卫姿态**（见下）。

**护卫姿态规则**：
- 触发条件：我方单位在王秀的上/下/左/右相邻格（对角线不算）。
- 效果：该回合王秀不移动；王秀受到的伤害按护卫公式分摊至相邻我方单位。
- 伤害分摊比（`guard_transfer_ratio`）：默认 30%。
- 一个我方单位只能护卫一个 NPC（一对一）；若多个我方单位相邻，取速度序列最靠前的单位触发。
- 护卫姿态优先级低于攻击行动：我方单位本回合仍可正常行动（移动+攻击），护卫分摊在行动后结算。

**退场剧情触发**：
- 王秀 HP 降至 0：播放王秀退场剧情动画，王秀从地图移除；记录 `wang_xiu_departed = true`（影响后续叙事）。
- 不弹出 Game Over；战斗继续（玩家仍可通过歼灭追兵赢得战斗）。

### 3.5 Ch.2-2B：镇压追击（branch_variant = suppression）

**场景**：开阔平原，豪强骑兵与主角联合追击流民残部，对方为散兵撤退状态。
**地图尺寸**：18×18，地形含平原（grass）、破损栅栏（obstacle）、水沟（water）。
**胜利条件**：击败所有流民残部（包括逃跑单位到达地图边缘也算歼灭一个目标）。
**失败条件**：我方全员退场；或超过 4 名流民成功逃离地图边缘（构成"镇压失败"结算）。

**出场单位**：

| 单位 ID | 阵营 | 职业 | 备注 |

| P1, P2, R1/R4 | 玩家 | — | 最多带 3 名 |

### 3.6 Ch.2-3：飞骑营对决（chapter_02_finale）

**场景**：古道要隘，R3·陈朗带领飞骑营残部据守，主角队伍正面交锋。
**地图尺寸**：20×20，地形含高台（highland）、断桥（水 + obstacle 组合）、营地建筑（obstacle）。

**失败条件**：我方全员退场。

**出场单位**：

| 单位 ID | 阵营 | 职业 | 备注 |
|--------|------|------|-----|
| P1, P2 + 玩家自选 1 名后备 | 玩家 | — | 共 3 名 |
| BOSS·陈朗 | 敌方 | boss_knight | HP=130，三阶段；阶段阈值 0.65 / 0.30 |
| E10–E12 | 敌方护卫 | basic_knight×2 + basic_archer×1 | enemy_stat_multiplier = 1.30 |
| REINFORCE-A/B | 援军（第 12 回合刷新） | basic_knight×2 | 从地图右侧刷新，永久追击 |

**Boss·陈朗三阶段行为**：

| 阶段 | HP 区间 | 行为变化 | 阶段名 |
|-----|---------|---------|------|
| 阶段一 | 100%–65% | 标准攻击 + 回避姿态（每 3 回合） | 飞骑先锋 |
| 阶段二 | 64%–30% | 解锁"突刺冲锋"技能（直线 3 格穿透）；速度+4 | 狂骑突阵 |
| 阶段三 | 29%–0% | 全属性+15%；援军提前触发（第 10 回合刷新）；解锁"飞骑绝技"（全体伤害小范围） | 孤注一掷 |

**检查点规则（继承总纲 Boss 设计七原则）**：
- 阶段一完成（陈朗 HP ≤ 65%）时自动存检查点；已削减血量保留 15%（`checkpoint_retained_hp_ratio = 0.15`）。
- 阶段二完成（陈朗 HP ≤ 30%）时再次存检查点；保留 15%。
- 失败后从最近检查点重开（不重置整场）。

**限时压力规则**：
- 第 12 回合开始时（默认）：地图右侧刷新"飞骑营援军"2 名基础骑士，永久追击，不影响胜利条件。
- Boss 进入阶段三时，若当前回合 < 10，援军刷新提前至第 10 回合。
- 援军设计目的：强迫玩家在"清场小兵"与"集中击 Boss"之间决策，测试速度序列制掌握程度。

**战斗前信念值节点 B2-N3**（对话选项）：
- 「喊话劝降」→ 仁+6 / 义-3；陈朗额外对话台词（叙事丰富度+）
- 「直接接战」→ 义+6 / 仁-6；战斗开始时陈朗速度+2（被激怒状态）

### 3.7 章节结算：果子二选三

### 3.8 信念值分支节点规则

**判断时机**：每场战斗结算后、进入下一战前。
**判断方式**：读取 `belief_values.ren / belief_values.yi / belief_values.zhi` 三个累计值。
**第一次分叉阈值（Ch.2-1 后）**：

```
if (yi - max(ren, zhi)) >= 5:
    → branch_variant = "suppression"  # 触发 2B
else:
    → branch_variant = "mercy"        # 触发 2A（默认）
```

**设计意图**：阈值故意设置为 5（低门槛），确保 Ch.1 中小幅倾向义路线的玩家走 2B；
信念值均衡的玩家默认走 2A（仁/智路线更具教学价值）。
双平局（yi = max，ren = zhi）时，默认走 2A。

**信念值上限约束**：单一属性累计值 ≤ 100，≥ 0。超出范围的增减量自动截断。

## 4. Formulas

### 4.1 敌方数值缩放公式

以 Ch.1 finale（`enemy_stat_multiplier = 1.0`）为基准，Ch.2 各战的敌方属性倍率为：

```
enemy_stat = base_stat × enemy_stat_multiplier × new_game_plus_multiplier
```

| 战斗 | enemy_stat_multiplier | 说明 |
|-----|----------------------|-----|
| Ch.2-1 | 1.10 | 比 Ch.1 finale 高一档；玩家仍有适应空间 |
| Ch.2-2A/B | 1.15 | 引入新机制的同时保持数值适中 |
| Ch.2-3（护卫） | 1.30 | Boss 战，高于 Ch.2-2；低于设计极限 1.50 |

> `new_game_plus_multiplier` 由难度系统提供，第一周目固定为 1.0。

**敌方 HP 衍生公式**（继承总纲属性系统）：

```
enemy_hp = base_hp + (vitality × 8) × enemy_stat_multiplier
```

其中 `base_hp` 为单位职业基础值，`vitality` 为体力属性值。

**示例**（Ch.2-3 Gate Commander 基础 str=25，agi=14，vita=20，base_hp=60）：

```
enemy_hp = (60 + 20×8) × 1.30 = 220 × 1.30 = 286 ≈ 130（设计值取整，见 Boss 设计）
```

> 注：Boss HP 由关卡设计师手动校准至目标 TTK，公式用于普通单位推导；Boss 取手动值。

### 4.2 护卫姿态伤害分摊公式

```
guard_damage = incoming_damage × guard_transfer_ratio
npc_actual_damage = incoming_damage × (1 - guard_transfer_ratio)
guardian_actual_damage += guard_damage
```

**变量定义**：
- `incoming_damage`：敌方对王秀的原始伤害值（防御减免后）
- `guard_transfer_ratio`：护卫姿态伤害转移比例，默认 0.30，范围 [0.15, 0.50]
- `npc_actual_damage`：王秀实际承受伤害（扣减 HP 使用此值）
- `guardian_actual_damage`：护卫单位额外承受的伤害（叠加至该单位受伤计算）

**示例**（敌方对王秀造成 20 点伤害，guard_transfer_ratio = 0.30）：

```
guard_damage = 20 × 0.30 = 6
npc_actual_damage = 20 × 0.70 = 14（王秀扣 14 HP）
guardian 扣 6 HP（护卫单位额外受伤）
```

### 4.3 信念值累计平衡验证

本章各节点的信念值增减量汇总（以全程选"仁路线"为例）：

| B2-N2A（王秀存活） | +12 | +12 | 0 | 0 |
| B2-N3（喊话劝降） | +6 / -3 | +6 | -3 | 0 |
| **本章仁路线合计** | | **+28** | **-6** | **+5** |

以全程选"义路线"为例：

| 节点 | 操作 | 仁 | 义 | 智 |
|-----|------|---|---|---|
| B2-N1（驱散流民） | -5 / +8 / +3 | -5 | +8 | +3 |

### 4.4 援军速度值标准

目标：第 12 回合刷新的援军，不得在刷新后 2 回合内到达地图中线（x=10）。
地图宽 20 格，援军刷新于 x=18，速度序列中每回合最多移动 `move` 格。

```
turns_to_midline = (18 - 10) / move_per_turn = 8 / move_per_turn
```

要求 `turns_to_midline ≥ 2`，即 `move_per_turn ≤ 4`。
援军骑士 `move = 3`（基础骑士标准），满足条件（8÷3 ≈ 2.67 回合）。

### 4.5 分叉阈值公式

```
if (yi - max(ren, zhi)) >= BRANCH_THRESHOLD:
    branch = suppression
else:
    branch = mercy
```

- `BRANCH_THRESHOLD`：默认 5，安全范围 [3, 10]（gate knob）
- 阈值 < 3 导致几乎所有玩家走 2B（削弱 2A 的教学价值）
- 阈值 > 10 导致仅极端义路线玩家才能触发 2B（削弱分支感）

## 5. Edge Cases

### 5.1 玩家从 Ch.1 不同分支进入 Ch.2 的场景

| Ch.1 存档状态 | Ch.2 入场处理 |
|-------------|-------------|
| Ch.1 三战全通（标准路径） | 正常加载 chapter_02_act_a，R3 以 `departed/chapter_2_exit` 状态进入 Ch.2 剧情 |
| 存档来自 Ch.1 中间（chapter_01_crossroads 完成，finale 未完成） | 不应出现（Ch.2 解锁条件：`chapter_01_complete = true`）；若存档损坏，强制跳回 Ch.1 finale |
| Ch.1 所有可用单位均已退场（仅靠 P1 通关） | Ch.2 入场时后备名单只有 R1/R4；2A/2B 的编队约束自动降低为"携带 2 名"（不强制 3 名） |
| 信念值已有倾向（Ch.1 玩家做了仁/义/智选择） | 直接叠加，无需重置；分叉阈值公式读取累计值 |

### 5.2 王秀在到达安全区前 Boss 被击败

- Ch.2-2A 中不存在 Boss，该情形不适用。
- 若玩家歼灭所有追兵（胜利条件 2 触发）但王秀仍在路途中：视为王秀存活，仁+12 照常结算；不再继续移动王秀至安全区（战斗已结束）。

### 5.3 王秀 HP = 0 的结算顺序

1. 敌方攻击命中王秀，伤害计算后 HP ≤ 0。
2. 若护卫姿态激活，先扣减护卫单位 HP，再以减免后值判断王秀 HP；若减免后仍 ≤ 0，继续下一步。
3. 触发退场剧情动画（约 3 秒）；王秀从地图移除。
4. 设置 `wang_xiu_departed = true`，记录至存档。
5. 战斗继续，玩家只剩歼灭追兵路线（胜利条件 1 已不可达）。
6. 战斗胜利结算时，触发 仁-8（不与仁+12 叠加，只取其一）。

### 5.4 双平局信念值分叉

- 情形：`yi = max(ren, zhi)` 且 `yi - max(ren, zhi) = 0 < 5`
- 处理：默认走 2A（mercy 路线），不走 2B。
- 日志记录：`belief_branch = "mercy_default"` 以区分"主动选择 2A"与"平局默认 2A"。

### 5.5 R3·陈朗羁绊特殊对话

- 若玩家在 Ch.1 与 R3 建立了 B 级及以上羁绊（`bond_level >= B`）：
  - Ch.2-3 开场对话新增一条专属台词，不影响信念值选项。
  - 若选择「喊话劝降」，陈朗有 20% 概率（固定 RNG，seed 绑定存档）触发特殊反应台词（叙事深度+）。
- 实现依赖 `bond-system` 提供的 `get_bond_level(unit_id)` 接口。

### 5.6 信念值数值溢出

- 单一属性值区间：`[0, 100]`，超出自动截断（clamp）。
- 示例：若仁值已为 95，此时触发 仁+10，实际结算仁+5（截断至 100）。
- 负溢出同理：仁值为 3，触发 仁-8，实际结算 仁-3（截断至 0）。

### 5.7 跳过教学提示

- 若玩家关闭了新机制提示（设置项）：护卫姿态提示弹窗不出现，但机制正常生效。
- 不存在"跳过后护卫姿态无法触发"的 bug 风险（提示与逻辑解耦）。

### 5.8 果子选择界面未交互（玩家关闭窗口）

- 果子界面为强制弹窗（不可 Esc 关闭），直至玩家完成两种选择并点击确认。
- 若存档在选择前中断（crash），重开存档后再次弹出界面（`fruit_selection_done` 标志未置真）。

### 5.9 Ch.2-2B 限时失败（流民逃离超过 4 名）

- 触发"镇压失败"结算：
  - 不是 Game Over，但给予义-5（追杀不利，被豪强嘲讽）；
  - 战斗仍可通过后续补击获胜；若我方也全员退场，才触发 Game Over。
- 设计意图：给部分玩家提供"虽然失利但不死局"的容错空间。

## 7. Tuning Knobs

| 参数名 | 当前值 | 安全范围 | 类别 | 影响的游戏感受 |

## 8. Acceptance Criteria

### AC-CH2-001：分叉路由（功能性）
- [ ] 给定存档：Ch.2-1 结算后 yi=15, ren=5, zhi=8（义领先 ≥5），系统路由至 chapter_02_act_b（branch_variant = suppression）。
- [ ] 给定存档：Ch.2-1 结算后 yi=10, ren=12, zhi=8（仁领先），系统路由至 chapter_02_act_b（branch_variant = mercy）。
- [ ] 给定存档：三值相等（yi=ren=zhi=10），系统默认路由至 mercy，`belief_branch = "mercy_default"` 写入存档。

### AC-CH2-002：王秀 AI 移动（功能性）
- [ ] 模拟 5 回合：每回合王秀的位置向安全区（左上 3×3）方向移动至少 1 格（无障碍情形）。
- [ ] 模拟阻塞情形：敌方单位距王秀 ≤2 格时，王秀该回合不移动（畏缩生效）。

### AC-CH2-003：护卫姿态伤害分摊（功能性）
- [ ] 我方单位 P1 占据王秀正右方；敌方对王秀造成 20 点伤害；验证王秀扣 14 HP，P1 扣 6 HP（guard_transfer_ratio = 0.30）。
- [ ] 王秀无相邻我方单位时；敌方造成 20 点伤害；验证王秀扣 20 HP（无分摊）。

### AC-CH2-004：王秀退场（功能性）
- [ ] 王秀 HP 降至 0 时，触发退场剧情动画；`wang_xiu_departed = true` 写入存档；战斗不结束（可继续作战）。
- [ ] 王秀退场后，战斗结算时信念值结算为 仁-8（不是 仁+12）。

### AC-CH2-005：Boss 援军刷新（功能性）
- [ ] Ch.2-3 普通路径：第 12 回合开始时，地图右侧出现 2 名 basic_knight 援军。
- [ ] Boss 阶段三提前路径：Boss HP ≤ 30% 时当前回合为第 8 回合，援军在第 10 回合刷新（提前 2 回合）。
- [ ] 援军出现后，胜利条件仍为"击败 Boss"（不需要同时击败援军）。

### AC-CH2-006：检查点保留（功能性）
- [ ] 陈朗 HP 降至 65% 以下时，系统保存检查点；Boss HP 以 `65% × 0.15 = ~9.75%` 基准重开（≈ 检查点 HP 保留 15%）。
- [ ] 从检查点重开时，玩家己方单位 HP/状态与检查点一致（不重置为战斗开始值）。

### AC-CH2-007：信念值边界（功能性）
- [ ] 仁值为 95，触发 仁+10，最终仁值 = 100（不超出上限）。
- [ ] 义值为 3，触发 义-8，最终义值 = 0（不低于下限）。

### AC-CH2-008：果子结算界面（功能性）
- [ ] Ch.2-3 胜利后，结算屏出现果子三选二界面，显示 fruit_str / fruit_agi / fruit_int 各 1 份。
- [ ] 玩家选择 2 种后点击确认；对应 fruit 写入库存；第三种不进入库存。
- [ ] 若存档中断后重载，界面重新弹出（`fruit_selection_done = false` 检测）。

### AC-CH2-009：体验性验收（Playtest）
- [ ] 首次进行护送战（Ch.2-2A）的玩家，在战后问卷中 ≥70% 表示"感到王秀需要保护的压力"。
- [ ] Boss 战中，≥60% 的首次尝试玩家会在援军刷新后调整策略（转向集中击 Boss 或分兵）。
- [ ] 章节通关后，仁/义/智最大差值 ≥ 15（存档数据验证，可编写统计脚本）。
