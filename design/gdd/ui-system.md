# UI系统

> **Status**: In Design
> **Author**: user + agents
> **Last Updated**: 2026-04-16
> **Implements Pillar**: 系统互锁——UI是玩家与所有系统的交互界面

## Overview

UI系统是玩家与所有系统的交互界面——它决定了玩家如何"看到"和"操作"游戏。

UI系统涵盖所有游戏界面的呈现和交互：
- **战斗UI**：HP/MP条、技能栏、行动顺序、战斗信息
- **角色UI**：角色状态、属性、装备
- **菜单UI**：存档/读档、系统设置
- **HUD**：实时显示的资源、快捷信息

UI遵循HD-2D中国风美术风格，与游戏美术风格保持一致。

## Player Fantasy

玩家在UI系统中的核心感受是**"我能清晰地看到和操作一切"**——UI是游戏对玩家的"回应"，好的UI让玩家感到"一切尽在掌控"。

**信息的清晰传达**带来的是**决策的自信**。当HP条、MP条、技能CD、行动顺序都一目了然时，玩家能做出准确的判断——这种**信息的透明**是策略游戏UI的核心价值。

**视觉的美感**带来的是**沉浸的延续**。当UI与中国风美术风格完美融合，当每一次点击都有恰当的视觉反馈——这种**细节的精致**让游戏从头到尾都保持审美体验。

**具体锚点：**
- 战斗时，HP条跳动显示伤害，MP条减少显示消耗 → 战斗的紧张感
- 资源不足时，UI变红提示 → 决策的警示
- 升级时，UI闪烁金光 → 成长的成就感

## Detailed Design

### Core Rules

**C.1 UI层级结构**

| UI层级 | 描述 | 示例 |
|-------|------|------|
| 底层 | 地图和战场 | 2.5D战场、角色、敌人 |
| 中层 | 战斗信息 | 行动顺序条、范围显示 |
| 上层 | 操作界面 | 技能栏、角色面板 |
| 顶层 | 临时提示 | 伤害飘字、Buff图标 |

**C.2 战斗UI组件**

| 组件 | 位置 | 功能 |
|------|------|------|
| 行动顺序条 | 屏幕右侧 | 显示所有单位行动顺序 |
| 技能栏 | 屏幕下方 | 显示可用技能 |
| 角色状态面板 | 屏幕左侧 | 当前行动单位的HP/MP/状态 |
| 移动/攻击范围 | 战场覆盖层 | 显示可移动/攻击范围 |

**C.3 HUD资源显示**

| 位置 | 显示内容 |
|------|---------|
| 屏幕顶部 | 金币、当前关卡、暂停按钮 |
| 屏幕左上 | 主要资源图标（果子、保护符等） |

**C.4 菜单系统**

| 菜单 | 功能 |
|------|------|
| 角色菜单 | 查看/管理角色状态、装备、技能 |
| 背包菜单 | 查看/使用道具、装备 |
| 存档菜单 | 保存/加载游戏 |
| 设置菜单 | 音量、画质、按键设置 |

## Dependencies

### Upstream Dependencies

UI系统依赖所有其他系统提供的数据显示：

| System | Data Provided |
|--------|---------------|
| 属性与成长系统 | 属性值、等级、HP/MP上限 |
| 职业系统 | 职业名称、职业技能 |
| 技能系统 | 技能列表、技能冷却、MP消耗 |
| 装备系统 | 装备信息、词缀、强化等级 |
| 资源经济 | 金币、材料、果子、保护符 |
| 回合制模式 | 行动顺序、当前行动单位 |
| 战斗系统 | 伤害数值、战斗状态 |
| 战术机制 | 地形信息、高低差、元素状态 |
