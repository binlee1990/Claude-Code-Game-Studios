# 回合制模式

> **Status**: In Design
> **Author**: user + agents
> **Last Updated**: 2026-04-16
> **Implements Pillar**: 系统互锁——回合制模式是核心玩法本身

## Overview

回合制模式是游戏核心玩法的载体——它决定了"如何战斗"。

游戏采用**速度序列制**的回合制框架：
- 所有单位（玩家和敌人）根据速度(AGI)属性排序行动
- 速度越高，行动越快，越先行动
- 可视化行动顺序条显示所有单位的行动顺序
- 支持**自动战斗**和**加速模式**便捷功能

每个单位每回合可以：
- 移动（消耗移动力）
- 使用技能/攻击（消耗MP/触发冷却）
- 待机（不消耗资源，保留行动）

速度序列制让"速度投资"有明确回报——高速单位可以在敌方行动前连续行动多次，创造战术优势。

## Player Fantasy

玩家在回合制系统中的核心感受是**"我的决策在正确时机发挥作用"**——不是简单地"选最强的招式"，而是思考"我现在该做什么，谁能先行动"。

**速度序列**带来的是**策略的时机感**。当玩家看到自己的高速弓手在敌人行动前连续射出三箭时，当玩家意识到"如果我把速度拉满，这个Boss永远没有还手之力"时——这种**我掌控了战斗节奏**的满足感是速度序列制独有的。

**行动顺序可视化**带来的是**全局的掌控感**。当玩家看着行动条，思考"这一轮我能不能在那个敌人行动前击杀他"时，当玩家决定"我要不要跳过这轮行动来保留技能"时——这种**预见和计划**的乐趣是回合制策略的核心。

**自动战斗和加速**带来的是**farm的便捷感**。当玩家在刷经验时开启自动战斗，看着角色自动配配合击败敌人时——这种**我的队伍已经成为高效机器**的成就感是养成乐趣的延伸。

**具体锚点：**
- 高速弓手在敌人行动前连续三连击 → 速度优势的爽感
- 看着行动条，精准计算能否在敌人前击杀 → 策略预判的满足感
- 开启自动战斗，角色自动配合Farm → 养成成果的便利感
- "我需要提升速度，让这个角色先手" → 属性投资的方向感

**参考游戏**：FFT的速度序列制、皇骑的行动顺序条

## Detailed Design

### Core Rules

**C.1 速度序列系统**

行动顺序由AGI（敏捷）属性决定：

```
turn_order = units.sort_by(agi_value, descending)
```

- AGI越高，行动越早
- 同AGI时，随机决定顺序（保证公平）
- 行动完成后，重新排序（基于剩余AGI或固定AGI）

**每回合行动规则：**
- 每个单位每回合有1次行动机会
- 行动可以是：移动、攻击/技能、待机
- 行动消耗后，该单位本回合不再行动

**C.2 移动系统**

移动消耗：
- 基础移动力：5格/回合
- 地形影响：沙地消耗2倍移动力，泥沼降低AGI
- 障碍物不可通过

移动规则：
- 移动可以选择路径（不走斜线）
- 移动后可以立即攻击/使用技能
- 移动可以中断（停在任意位置）

**C.3 行动系统**

**主动行动类型：**

| 行动类型 | 消耗 | 说明 |
|---------|------|------|
| 普通攻击 | 无MP，触发冷却 | 基于武器的基础攻击 |
| 技能攻击 | MP+冷却 | 消耗MP，使用职业技能或普通技能 |
| 治疗技能 | MP+冷却 | 消耗MP，治疗己方单位 |
| 待机 | 无消耗 | 保留行动，下一回合速度不变 |

**MP（魔法值）系统：**
- MP上限由属性和装备决定
- MP每回合恢复：MP上限×10%
- 战斗开始时MP=上限的100%

**C.4 战斗流程**

```
combat_flow:
    1. 战斗开始 → 初始化双方单位
    2. 生成行动顺序（按AGI排序）
    3. 循环直到战斗结束：
        a. 当前行动单位执行行动
        b. 检查战斗结束条件
        c. 下一行动单位
    4. 战斗结束 → 结算奖励
```

**战斗结束条件：**
- 胜利：所有敌方单位血量=0
- 失败：所有己方单位血量=0
- 中断：玩家选择撤退（需有撤退技能/道具）

**C.5 自动战斗模式**

自动战斗开启后，AI接管己方单位行动：

**自动战斗规则：**
- AI自动选择目标（基于仇恨系统）
- AI自动选择技能（基于伤害收益最大化）
- AI自动决定移动（基于最优位置）
- 玩家可以随时关闭自动战斗

**C.6 加速模式**

加速模式加速战斗动画和AI决策：

| 模式 | 动画速度 | AI决策延迟 |
|------|---------|----------|
| 正常 | 1x | 0.5-1.5秒 |
| 加速 | 2x | 0.2-0.5秒 |
| 最速 | 3x | 0秒（无延迟） |

## Formulas

### D.1 行动顺序排序

```
turn_order_index = units.sort_by(agi_value, descending)
```

**变量定义:**

| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| agi_value | av | int | [0, 999] | 单位敏捷属性值 |
| turn_order_index | toi | int | [1, N] | 行动顺序（1=最先行动） |

**输出范围:** 1到单位总数N

**示例:**
- 队伍：AGI100, AGI80, AGI60, AGI40 → 顺序：100→80→60→40

---

### D.2 MP恢复

```
mp_recovered = max_mp × recovery_rate
max_mp = base_mp + equipment_mp_bonus
```

**变量定义:**

| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| base_mp | bm | int | [基础MP] | 角色基础MP上限 |
| equipment_mp_bonus | emb | int | [装备加成] | 装备提供的MP加成 |
| max_mp | mm | int | [总MP上限] | 总MP上限 |
| recovery_rate | rr | float | 0.1 | 每回合恢复率（10%） |
| mp_recovered | mr | int | [恢复量] | 每回合恢复的MP |

**输出范围:** 最小0，最大max_mp的10%

**示例:**
- max_mp=100，每回合恢复：`mr = 100 × 0.1 = 10`

---

### D.3 移动力消耗

```
movement_cost = terrain_cost × base_movement
```

**变量定义:**

| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| base_movement | bm | int | 5 | 基础移动力（格） |
| terrain_cost | tc | float | {1.0, 2.0} | 地形消耗系数（普通=1.0，沙地=2.0） |
| movement_cost | mc | int | [消耗移动力] | 实际消耗的移动力 |

**输出范围:** 1到基础移动力的2倍

**示例:**
- 进入沙地：`mc = 5 × 2.0 = 10`（消耗全部移动力）

## Edge Cases

### E.1 全部单位同时AGI相同

**场景**: 多个单位AGI完全相同

**处理**:
- 相同AGI的单位随机排列顺序
- 保证公平性（不固定谁先谁后）

### E.2 单位在行动中被击杀

**场景**: 单位A正在行动时，单位B击杀了单位A

**处理**:
- 单位A的行动立即中断
- 击杀者获得击杀奖励
- 如果单位A是玩家操作的，该玩家输入中断

### E.3 MP耗尽

**场景**: 单位MP耗尽，无法使用技能

**处理**:
- 需要MP的技能显示"MP不足"
- 单位只能使用普通攻击或待机
- 待机后下回合MP继续恢复

### E.4 所有单位无法行动

**场景**: 所有单位都被封印/无法行动

**处理**:
- 战斗自动结束（平局）
- 玩家不获得胜利奖励，也不获得失败惩罚
- 记录战斗结果为"无结果"

### E.5 自动战斗时玩家道具不足

**场景**: 自动战斗时需要使用道具，但道具耗尽

**处理**:
- 自动战斗跳过该行动
- 不中断自动战斗流程
- 玩家可以手动补充道具后继续

### E.6 加速模式下动画异常

**场景**: 加速模式导致动画播放异常

**处理**:
- 加速模式跳过所有动画，仅显示结果
- 伤害数字仍然显示
- 保障战斗结果正确

## Dependencies

### Upstream Dependencies

| System | Dependency Type | Interface Contract |
|--------|---------------|-------------------|
| **AI系统** | 硬依赖 | AI系统提供敌方单位行动决策；回合制调用AI获取行动指令 |
| **战术机制** | 硬依赖 | 回合制使用战术机制的地形、移动规则、高低差计算 |
| **技能系统** | 硬依赖 | 回合制调用技能系统执行技能；技能冷却和MP消耗由技能系统管理 |
| **装备系统** | 软依赖 | 装备影响属性（AGI、MP等），间接影响回合制 |

### Downstream Dependents

| System | Dependency Type | Interface Contract |
|--------|---------------|-------------------|
| **战斗系统** | 数据消费者 | 回合制调用战斗系统执行伤害计算、状态更新等 |
| **角色管理** | 数据消费者 | 回合制涉及角色HP、MP、状态管理 |
| **HP系统** | 数据消费者 | 回合制在 register_unit 时调用 `unit.get_max_hp()` 派生玩家 max_hp（敌人仍走 battle_definition + difficulty 倍率） |

## Tuning Knobs

### Speed Sequence Parameters

| Knob | Default | Safe Range | Affected Gameplay | Notes |
|------|---------|-----------|-----------------|-------|
| **基础移动力** | 5格 | 4-6格 | 战场机动性 | 提高加快节奏，降低增加位置重要性 |
| **MP恢复率** | 10%/回合 | 5%-20% | MP管理 | 提高使技能使用更频繁 |
| **初始MP比例** | 100% | 80%-100% | 第一回合资源 | 100%保证第一回合资源充足 |

### Auto Battle Parameters

| Knob | Default | Safe Range | Affected Gameplay | Notes |
|------|---------|-----------|-----------------|-------|
| **AI决策延迟（正常）** | 0.5-1.5秒 | 0.3-2.0秒 | 战斗节奏 | 提高使AI感觉更像人，降低加快节奏 |
| **AI决策延迟（加速）** | 0.2-0.5秒 | 0.1-1.0秒 | 战斗节奏 | — |
| **AI决策延迟（最速）** | 0秒 | 0秒 | 最快节奏 | 无延迟 |

### Suggested First Tuning Session

建议首轮调参优先级（按影响力排序）:
1. 基础移动力（5格）— 战场机动性核心
2. MP恢复率（10%/回合）— 技能使用频率核心
3. AI决策延迟（0.5-1.5秒）— 战斗节奏感

## Visual/Audio Requirements

### Visual Requirements

**行动顺序显示**:
- 屏幕侧面或顶部显示行动顺序条
- 显示所有单位的图标和顺序位置
- 当前行动单位高亮显示
- 已行动单位显示灰色/完成标记

**当前行动单位标识**:
- 当前行动单位周围有高亮边框
- 显示单位名称和当前HP/MP

**行动结果反馈**:
- 伤害数字飘出显示
- 治疗数字飘出显示
- 状态变化（buff/debuff）有图标显示

### Audio Requirements

**回合开始音效**:
- 当前行动单位开始时：轻微的"滴"确认音
- 行动选择时：每个选项有对应的选择音效

**行动完成音效**:
- 普通攻击：武器碰撞音效
- 技能释放：技能特有的音效
- 移动完成：脚步落地音效
- 待机：轻微的"唰"确认音

**战斗结束音效**:
- 胜利：胜利号角/鼓声
- 失败：低沉的失败音效

## UI Requirements

### 战斗界面布局

**行动顺序条**:
- 位置：屏幕右侧或顶部
- 内容：从上到下/从左到右显示所有单位图标
- 当前行动单位：金色边框高亮
- 已行动单位：灰色遮罩
- 即将行动单位：正常显示

**当前单位信息面板**:
- 位置：屏幕左侧
- 内容：当前行动单位的HP条、MP条、头像
- 行动选项按钮：攻击、技能、道具、待机

### 行动选择界面

**行动选项面板**:
- 位置：屏幕下方或弹出框
- 选项：移动（显示移动范围）、攻击（显示攻击范围）、技能（显示技能列表）、道具、待机
- 状态显示：每个选项旁边显示消耗和冷却状态

**技能选择面板**:
- 显示当前可用技能列表
- 每个技能显示：图标、名称、MP消耗、冷却状态
- 不可用技能显示禁用状态

### 自动战斗/加速控制

**控制栏**:
- 位置：屏幕右下角
- 内容：
  - 自动战斗开关（ON/OFF）
  - 加速模式按钮（1x / 2x / 3x）
  - 跳过按钮（跳过当前回合）

## Open Questions

### OQ.1 行动顺序是否使用随机因素

**问题**: 除了AGI，是否还有其他因素影响行动顺序？
- 当前设计：纯AGI决定，同AGI时随机
- 备选方案：AGI+少量随机因素

**待确认**: 纯AGI决定让速度投资最有价值。建议MVP阶段纯AGI。

### OQ.2 移动后是否可以取消

**问题**: 玩家在移动过程中是否可以取消移动？
- 当前设计：可以取消，选择其他行动
- 备选方案：移动必须完成

**待确认**: 可取消增加玩家灵活性。建议MVP阶段支持取消。

### OQ.3 自动战斗的智能程度

**问题**: 自动战斗是否需要考虑战术优先级（如集火）？
- 当前设计：AI自动选择目标
- 备选方案：AI自动选择+玩家设定集火目标

**待确认**: 基础自动战斗已经足够。建议MVP用简单AI，Alpha再扩展。
