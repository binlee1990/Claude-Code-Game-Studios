# Boss战系统

> **Status**: Designed (pending review)
> **Author**: binlee1990 + agents
> **Last Updated**: 2026-04-22
> **Implements Pillar**: 系统互锁（Boss战综合考验所有养成和战术系统）

## Overview

Boss战系统定义多阶段 Boss 战的设计规则、检查点机制和失败恢复策略。Boss 是叙事节点和养成验证的综合体——每个 Boss 检验玩家的属性分配、职业选择、技能搭配、装备配置和战术运用。系统融合魂系试错哲学（可学习的行动模式+检查点）和 SRPG 策略深度（多解设计），以资源消耗作为最终兜底机制。

## Player Fantasy

- **学习的快感**：Boss 有可学习的行动模式——第一次遇到新招式会惊讶，第二次能预判，第三次能反制。"我变强了"不是数值上的，而是理解上的
- **阶段转换的仪式感**：Boss 血量下降到阈值时切换阶段，改变招式组合和战术需求，保持紧张感
- **失败不是终点**：检查点机制让"再试一次"的成本很低，但"回去练级"也是合理选择

## Detailed Design

### Core Rules

**规则 1 — Boss 类型分类**

| 类型 | 目的 | 阶段数 | 难度 | 机制教学 |
|------|------|--------|------|---------|
| 教学 Boss | 引入核心机制 | 1-2 | 低 | 明确提示 |
| 叙事 Boss | 推动剧情 | 2 | 中 | 弱提示 |
| 能力测试 Boss | 检验养成 | 2-3 | 中高 | 无提示 |
| 巅峰 Boss | 综合考验 | 3+ | 高 | 无提示 |
| 隐藏 Boss | 极限挑战 | 多阶段 | 极高 | 无提示 |

**规则 2 — 多阶段检查点**

Boss 分多个阶段，每阶段切换时自动保存检查点：
- 失败后从当前阶段重开
- 已削减 Boss 血量保留 10-20%
- 首次失败后显示 Boss 当前阶段的行动模式提示
- 已学会的模式提示永久保留

**规则 3 — 失败恢复策略**

| 失败次数 | 恢复选项 | 说明 |
|----------|----------|------|
| 1-2 次 | 从检查点重试 | 直接重开当前阶段 |
| 3-5 次 | 资源消耗再试 | 消耗 10% 士气/补给，但保留所有经验和掉落 |
| 5+ 次 | 回去练级 | 建议玩家先提升等级/装备，不强制 |
| 任意 | 直接重试 | 始终可以从检查点免费重试（不消耗资源） |

资源消耗是**可选**的增强选项，不是强制的付费墙。免费重试始终可用。

**规则 4 — Boss 七原则**

1. 可学习的行动模式——Boss 招式有明确的视觉前兆
2. 阶段切换改变战术需求——非纯数值提升
3. 多解设计——没有唯一正确解
4. 检查点机制——失败成本可控
5. 部分失败恢复——不从头开始
6. 难度曲线双维度——数值+策略同步管理
7. Boss 作为叙事节点——每场 Boss 战推进故事

**规则 5 — Boss 行动模式**

Boss 在每阶段有 2-4 种行动模式：

| 模式元素 | 说明 |
|----------|------|
| 前兆动画 | 0.5-1.0 秒视觉提示，告知即将使用的招式 |
| 范围指示 | 攻击前显示影响区域（红色高亮） |
| 元素属性 | 招式绑定元素，可用对应克制策略 |
| 冷却周期 | 同一招式有最小使用间隔（2-3 回合） |
| 阶段切换 | 血量低于阈值时强制切换（血量 50%/25%） |

### States and Transitions

```
Boss_Intro → Phase_1 → [HP≤50%] → Phase_2 → [HP≤25%] → Phase_3 → Boss_Defeated
                 ↑          ↓                      ↓
                 └── 检查点 ←─── 失败重试 ←────────┘
```

| 状态 | 行为 |
|------|------|
| Boss_Intro | 播放 Boss 登场动画，显示名称和阶段数 |
| Phase_N | Boss 执行当前阶段的行动模式轮转 |
| Boss_Defeated | 播放胜利动画，进入战斗结算 |

### Interactions with Other Systems

| 系统 | 接口 | 方向 | 说明 |
|------|------|------|------|
| AI 系统 | Boss 专属 AI 策略 | Boss→AI | Boss 使用独立 AI 策略，非普通敌人 AI |
| 战术机制 | 元素克制、高低差 | 双向 | Boss 招式受战术机制约束 |
| 战斗结算 | 经验/掉落 | 结算→Boss | Boss 击败后的奖励计算 |
| 难度系统 | 难度倍率 | 难度→Boss | 敌人属性缩放应用于 Boss |
| 叙事系统 | Boss 剧情 | 叙事→Boss | Boss 作为叙事节点 |
| 存档系统 | 检查点保存 | Boss→存档 | 阶段切换时保存检查点 |

## Formulas

**F1 — 阶段切换阈值**

```
phase_trigger = boss_max_hp × phase_threshold[N]
```

| 阶段 N | threshold | 说明 |
|--------|-----------|------|
| Phase 1→2 | 0.50 (50% HP) | 第一次阶段切换 |
| Phase 2→3 | 0.25 (25% HP) | 第二次阶段切换（如有） |

**F2 — 失败后血量保留**

```
retained_hp = phase_start_hp × 0.15
```

保留当前阶段起始血量的 15%，避免无限消耗。

**F3 — 资源消耗兜底**

```
resource_cost = base_resource × (1 + attempts × 0.02)
```

| 变量 | 符号 | 类型 | 范围 | 说明 |
|------|------|------|------|------|
| base_resource | R_base | float | 士气/补给基值 | 基础消耗 |
| attempts | N | int | 3+ | 失败次数（前2次免费） |

## Edge Cases

- **If Boss 在阶段切换动画中被击杀**：跳过切换动画，直接进入 Boss_Defeated 状态。
- **If 单次攻击跨越多个阶段阈值**（如 HP 从 55% 直接降到 20%）：Boss 按顺序依次触发所有跨越的阶段转换，每个阶段的行动模式切换和新机制立即生效一次，然后进入最终对应阶段。Boss 不会跳过任何阶段的机制。
- **If 玩家在检查点处选择"回去练级"**：检查点数据保留。玩家可以在基地强化后返回，从检查点继续。
- **If Boss 的元素属性与玩家所有角色均被克制**：Boss 设计必须至少有一个非克制弱点（如地形利用、特定策略），确保任何阵容都有理论解法。
- **If 检查点存档损坏**：从 Boss 战开头重新开始，不保留任何阶段进度。

## Dependencies

### 上游
- **AI 系统**：Boss 使用独立 AI 策略
- **战斗结算**：Boss 击败后的奖励
- **难度系统**：难度倍率应用于 Boss

### 下游
- **叙事系统**：Boss 作为叙事节点
- **存档系统**：检查点保存
- **音效系统**：Boss 专属音乐

## Tuning Knobs

| Knob ID | 名称 | 默认值 | 安全范围 | 极端行为 |
|---------|------|--------|----------|----------|
| `TK-BOSS-01` | 阶段切换血量% | 50%/25% | 30-70% | 过高=太早切换；过低=前一阶段太长 |
| `TK-BOSS-02` | 血量保留% | 15% | 5-25% | 过高=无失败感；过低=重复劳动 |
| `TK-BOSS-03` | 前兆动画时长 | 0.7s | 0.3-1.5s | 过短=无法反应；过长=太简单 |
| `TK-BOSS-04` | 免费重试次数 | 2 | 0~5 | 过多=无压力；过少=挫败 |
| `TK-BOSS-05` | 模式提示触发 | 失败1次后 | 1~3次 | 过早=失去发现感；过晚=卡关 |

## Visual/Audio Requirements

- Boss 登场：全屏暗化 + Boss 名称显示 + 专属登场动画
- 阶段切换：屏幕震动 + Boss 外观变化 + 新招式预览
- Boss 击败：慢动作 + 胜利粒子 + 专属胜利音乐
- 每阶段 Boss 有视觉差异（光效/颜色/体型变化）

## UI Requirements

- Boss 血条显示在屏幕顶部，标注阶段划分
- 前兆动画时显示招式范围提示（红色/橙色区域）
- 失败后显示恢复选项菜单（重试/资源消耗/回去练级）
- 模式提示以图文形式显示（非纯文字）

## Acceptance Criteria

**GIVEN** Boss 进入 Phase 2（HP≤50%），**WHEN** 阶段切换，**THEN** 保存检查点，Boss 切换行动模式，不恢复 HP。

**GIVEN** 玩家在 Boss Phase 2 失败，**WHEN** 选择重试，**THEN** 从 Phase 2 检查点重开，Boss 保留 15% Phase 2 起始 HP。

**GIVEN** 玩家首次在 Boss Phase 1 失败，**WHEN** 重试开始，**THEN** 显示 Phase 1 的行动模式提示。

**GIVEN** Boss 招式前兆动画开始，**WHEN** 0.7 秒内玩家观察，**THEN** 攻击范围可视化，玩家有时间做出战术反应。

**GIVEN** 任意难度倍率，**WHEN** Boss 属性计算，**THEN** 基础 HP 和攻击力乘以难度倍率。

## Open Questions

- **OQ-1**：隐藏 Boss 的解锁条件需要与叙事系统和多周目系统对齐
- **OQ-2**：Boss 专属 AI 策略的复杂度上限（不影响帧率的决策树深度）
- **OQ-3**：多阶段 Boss 在 16× 难度下的数值平衡（HP×16 是否导致战斗过长）
