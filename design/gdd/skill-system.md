# 技能系统

> **Status**: In Design
> **Author**: user + agents
> **Last Updated**: 2026-04-16
> **Implements Pillar**: 系统互锁——技能是玩家表现的直接体现

## Overview

技能系统是玩家战斗表现的直接工具——它决定了"我能做什么"和"我有多强"。

游戏采用**熟练度驱动+位阶进阶**的技能成长框架：
- **熟练度**：每次战斗获得，用于提升技能等级；技能越强升级所需熟练度越高
- **位阶系统**：技能有位阶上限（初级/中级/高级/大师），达到当前位阶极限后需通过进阶考核才能继续提升
- **每10级特性选择**：技能在10/20/30级时触发特性选择，从2-3个特性中选择一个永久强化技能风格

技能分类：
- **普通技能**：所有职业可学习
- **职业技能**：特定职业专属
- **主动技能**：消耗资源（MP/冷却）主动释放
- **被动技能**：常驻生效，无需操作

这种设计让技能成长既有"肝"的积累感（熟练度驱动），又有"质变"的惊喜感（位阶进阶和特性选择）。

## Player Fantasy

玩家在技能系统中的核心感受是**"我的投入在变强"**——每一次战斗积累的熟练度都在让技能变得更强，每一次特性选择都在塑造独特的战斗风格。

**熟练度积累**带来的是**线性的成长感**。当玩家看着技能经验条一点点增长，当"升级！"的提示出现时——这种**稳定可预期的进步**是SRPG养成乐趣的核心。不是随机抽奖，而是**每一分努力都有回报**。

**位阶进阶**带来的是**质变的爽感**。当技能从"中级"升到"高级"，当玩家解锁了新的技能特效时——这种**突破性成长**的瞬间是游戏中最具成就感的时刻之一。

**特性选择**带来的是**Build的个性化**。当火球术在10级可以选择"范围+1"或"伤害+20%"时，当玩家选择了一个代表自己战斗风格的特性时——这种**选择塑造角色**的参与感是RPG最重要的情感连接。

**具体锚点：**
- 技能升级时看到数值变化 → 投入回报的满足感
- 位阶进阶时看到新的技能特效解锁 → 质变的爽感
- 10/20/30级特性选择时认真权衡 → Build决策的重量感
- 用自己培养的特色技能击败Boss → "这是我的角色"的归属感

**参考游戏**：兰斯9的熟练度系统（肝的满足感）、轮回乐园的技能特性选择（Build塑造感）

## Detailed Design

### Core Rules

**C.1 技能分类**

技能按两个维度分类：

**按来源：**

| 类别 | 描述 | 示例 |
|------|------|------|
| 普通技能 | 所有职业可学习 | 防御、攻击强化 |
| 职业技能 | 特定职业专属 | 剑圣的剑气、法师的冰墙 |

**按使用方式：**

| 类别 | 描述 | 消耗 |
|------|------|------|
| 主动技能 | 主动释放 | MP、冷却时间 |
| 被动技能 | 常驻生效 | 无 |

**C.2 技能数据结构**

每个技能有以下属性：

```
Skill {
    skill_id: string
    name: string
    type: enum {普通, 职业}
    usage: enum {主动, 被动}
    rank: enum {初级, 中级, 高级, 大师}
    level: int [1-99]
    proficiency: int [0, max_proficiency]
    max_proficiency: int // 升级所需熟练度
    mp_cost: int
    cooldown: int // 回合数
    base_damage: int
    effects: Effect[]
    unlock_condition: Condition
    rank_up_condition: Condition
    traits: Trait[] // 已选择的特性
}
```

**C.3 熟练度系统**

熟练度获取规则：
- 每场战斗结算时，根据技能使用次数和效果获得熟练度
- `proficiency_gained = base_proficiency × (1 + synergy_bonus)`
- 协同攻击、连击等配合行为获得额外熟练度加成

熟练度升级规则：
- `proficiency >= max_proficiency` 时技能升级
- 升级后：`level += 1`，`proficiency = 0`，`max_proficiency` 更新
- `max_proficiency` 随等级增长：`max_proficiency = base_cost × (level ^ 1.5)`

**C.4 位阶系统**

技能位阶定义：

| 位阶 | 最高等级 | 解锁条件 |
|------|---------|---------|
| 初级 | 10级 | 默认解锁 |
| 中级 | 20级 | 初级达到10级 + 隐藏属性达标 |
| 高级 | 30级 | 中级达到20级 + 隐藏属性达标 |
| 大师 | 99级 | 高级达到30级 + 完成进阶考核 |

位阶进阶规则：
- 位阶升级后，技能等级保持，但解锁新的技能特效
- 每次位阶进阶需要满足特定条件（属性门槛或考核）

**C.5 特性选择系统**

技能在特定等级触发特性选择（10/20/30级）：

从2-3个特性中选择1个永久生效：

**10级特性示例（火球术）：**
- 特性A："范围+1" — 攻击范围+1格
- 特性B："伤害+20%" — 基础伤害×1.2
- 特性C："燃烧+1回合" — 燃烧持续时间+1回合

**20级特性示例（火球术）：**
- 特性A："连环爆炸" — 有30%概率触发二次小范围爆炸
- 特性B："火抗穿透" — 目标火抗无效
- 特性C："燃烧扩散" — 燃烧可以扩散到相邻格子

**30级特性示例（火球术）：**
- 特性A："陨石天降" — 终极技能，召唤陨石造成大范围高伤害
- 特性B："炎神化身" — 被动，火焰技能伤害+50%，自身获得火焰护盾

**C.7 技能学习规则**

- 特殊职业技能：特殊职业专属，需满足特殊条件

## Formulas

### D.1 熟练度获取

```
proficiency_gained = base_proficiency × (1 + synergy_bonus + talent_bonus)
```

**变量定义:**

| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| base_proficiency | bp | int | [10, 50] | 基础熟练度（由技能等级决定） |
| synergy_bonus | sb | float | [0.0, 0.5] | 协同加成（连击/协同攻击） |
| talent_bonus | tb | float | [0.0, 0.3] | 天赋加成（角色属性与技能的匹配度） |
| proficiency_gained | pg | int | [10, 75] | 本次获得的总熟练度 |

**输出范围:** 最小10，最大约75

**示例:**
- 基础熟练度30，协同加成0.2，无天赋加成：`pg = 30 × (1 + 0.2) = 36`
- 基础熟练度50，协同加成0.2，天赋加成0.3：`pg = 50 × (1 + 0.2 + 0.3) = 75`

---

### D.2 升级所需熟练度

```
max_proficiency = base_cost × (level ^ 1.5)
```

**变量定义:**

| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| base_cost | bc | int | [100, 500] | 基础成本（由技能类型决定） |
| level | lv | int | [1, 99] | 当前等级 |
| max_proficiency | mp | int | [100, 500000] | 升级所需熟练度 |

**输出范围:** 最小100（1级），最大约500,000（高等级）

**示例:**
- 火球术1级（基础成本200）：`mp = 200 × (1 ^ 1.5) = 200`
- 火球术10级：`mp = 200 × (10 ^ 1.5) = 200 × 31.62 = 6324`
- 火球术20级：`mp = 200 × (20 ^ 1.5) = 200 × 89.44 = 17888`

---

### D.3 技能伤害计算

```
final_damage = base_damage × level_multiplier × trait_multiplier × (1 + attribute_bonus)
```

**变量定义:**

| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| base_damage | bd | int | [技能基础伤害] | 技能基础伤害值 |
| level_multiplier | lm | float | [1.0, 10.0] | 等级倍率 = 1 + (level-1) × 0.1 |
| trait_multiplier | tm | float | {1.0, 1.2, 1.3...} | 特性加成（选择的特性效果） |
| attribute_bonus | ab | float | [0.0, +∞) | 属性加成（智力/力量等属性的百分比加成） |
| final_damage | fd | int | [最终伤害] | 技能实际造成的伤害 |

**输出范围:** 基础伤害的1倍到10倍以上

**示例:**
- 火球术Lv10，基础伤害100，智力加成0.5，无特性加成：`fd = 100 × (1 + 9×0.1) × 1.0 × (1 + 0.5) = 100 × 1.9 × 1.5 = 285`

---

### D.4 特性效果计算

```
trait_effect = base_value × trait_modifier
```

**特性效果表:**

| 特性 | 效果 | 计算方式 |
|------|------|---------|
| 范围+1 | 攻击范围+1格 | 基础范围+1 |
| 伤害+20% | 基础伤害×1.2 | 乘法 |
| 燃烧+1回合 | 燃烧持续时间+1 | 加法 |
| 连环爆炸 | 30%概率触发 | 独立概率 |
| 伤害+50% | 基础伤害×1.5 | 乘法 |

## Edge Cases

### E.1 技能达到位阶上限

**场景**: 技能等级达到当前位阶上限，但未满足进阶条件

**处理**:
- 技能无法继续升级，直到满足进阶条件
- UI显示："技能已达[位阶]上限，需满足条件解锁下一位阶"
- 进阶条件不满足时，显示具体缺少条件

### E.2 特性选择时放弃

**场景**: 玩家在特性选择界面选择"稍后再说"

**处理**:
- 特性选择延后，不中断当前游戏
- 特性选择界面可在训练场重新打开
- 未选择的特性不会自动选择，必须玩家主动决定
- 等待期间的技能暂时无该等级特性加成

### E.3 技能冷却中无法行动

**场景**: 玩家回合开始时，目标技能仍在冷却

**处理**:
- 该技能在冷却栏显示剩余回合数
- 玩家需选择其他技能或普通攻击
- 冷却中的技能无法选中

### E.4 MP不足使用技能

**场景**: 玩家选择技能但MP不足

**处理**:
- 技能按钮显示"MP不足"
- 无法选中该技能
- UI显示"还需X点MP"

### E.5 位阶进阶考核失败

**场景**: 玩家参加技能位阶进阶考核但未通过

**处理**:
- 考核失败后，技能保持当前状态
- 可在一定时间后重新参加考核
- 考核消耗游戏内资源（建议）

### E.6 职业技能在转职后丢失

**场景**: 角色从原职业转职到新职业

**处理**:
- 原职业的职业技能保留，但无法继续升级
- 新职业的职业技能在转职后解锁
- 已学习的被动技能如果有新职业版本，需重新学习

### E.7 技能与特性冲突

**场景**: 两个已选特性在某些情况下产生冲突

**处理**:
- 特性效果独立计算，不冲突
- 如有冲突，按描述优先级执行（技能描述中注明的优先级）
- 玩家需要理解特性组合的效果

### E.8 技能熟练度溢出

**场景**: 战斗结算时获得大量熟练度，超过升级所需

**处理**:
- 溢出部分转入下一级：`proficiency = overflow`
- 可能触发连续升级（每次升级都播放升级特效）
- 连续升级可以一次处理多个等级

## Dependencies

### Upstream Dependencies

| System | Dependency Type | Interface Contract |
|--------|---------------|-------------------|
| **属性与成长系统** | 硬依赖 | 技能伤害受属性加成影响；位阶进阶条件包含属性门槛 |
| **职业系统** | 硬依赖 | 职业技能由职业系统解锁；职业变更触发职业技能重新计算 |
| **资源经济** | 硬依赖 | 技能学习消耗金币；位阶进阶考核可能消耗资源 |

### Downstream Dependents

| System | Dependency Type | Interface Contract |
|--------|---------------|-------------------|
| **战斗系统** | 数据消费者 | 技能系统提供技能数据给战斗系统执行；战斗系统反馈技能使用结果用于熟练度计算 |
| **回合制模式** | 无直接依赖 | 回合制提供技能选择界面，但不直接处理技能逻辑 |
| **羁绊系统** | 数据消费者 | 羁绊系统提供协同攻击等触发技能熟练度加成的事件 |

## Tuning Knobs

### Proficiency System Parameters

| Knob | Default | Safe Range | Affected Gameplay | Notes |
|------|---------|-----------|-----------------|-------|
| **基础熟练度获取** | 10-50 | 5-100 | 技能升级速度 | 提高加速升级，降低拉长养成周期 |
| **协同攻击熟练度加成** | +20% | +10%~+50% | 配合行为激励 | 提高使配合更有价值 |
| **天赋熟练度加成** | +10%~+30% | +5%~+50% | 属性匹配激励 | 与属性和技能的匹配度挂钩 |
| **升级熟练度系数** | 1.5 | 1.2~2.0 | 升级难度曲线 | 提高使后期升级更慢 |

| Knob | Default | Safe Range | Affected Gameplay | Notes |

### Suggested First Tuning Session

建议首轮调参优先级（按影响力排序）:
1. 基础熟练度获取（10-50）— 技能升级速度核心
2. 等级伤害倍率（每级+10%）— 技能成长感核心
3. 升级熟练度系数（1.5）— 后期升级难度曲线

## Visual/Audio Requirements

### Visual Requirements

**技能释放特效**:
- 每种技能有独特的视觉特效（火焰、冰霜、雷电等）
- 技能等级提升时，特效有细微增强（如光芒更亮、范围更大）
- 位阶进阶时，特效有显著变化（如火球变成火海）

**技能升级特效**:
- 升级时播放金色光芒环绕角色
- 特性选择时播放独特的选择动画

**特性选择界面**:
- 每个特性选项有独立的图标和特效预览
- 选中的特性有高亮边框

### Audio Requirements

**技能释放音效**:
- 每种技能类型有独特的音效（火焰"呼呼"、冰霜"喀喀"、雷电"滋滋"）
- 技能等级越高，音效略有增强（更丰富、更强烈）

**技能升级音效**:
- 升级时播放清脆的"叮~"提升音
- 位阶进阶时播放更庄重的进阶音效

**特性选择音效**:
- 特性选项出现时有轻微的"嗡"声
- 确认选择时有"唰"的确认音效

## UI Requirements

### 技能信息面板

**技能详情显示**:
- 位置：角色状态面板内或单独技能界面
- 显示内容：
  - 技能名称和图标
  - 当前等级和位阶
  - 熟练度进度条（当前/升级所需）
  - 技能效果描述
  - MP消耗和冷却时间
  - 已选择的特性列表

### 技能选择界面

**战斗中的技能栏**:
- 位置：屏幕底部或侧面
- 显示：当前可用的主动技能图标
- 状态显示：
  - 冷却中：灰色+倒计时
  - MP不足：红色+MP数值
  - 可用：正常显示

### 特性选择界面

**触发时机**: 技能达到10/20/30级时

**界面元素**:
- 遮罩背景（半透明黑色）
- 中央选择框
- 当前技能信息显示
- 2-3个特性选项（图标+名称+效果描述）
- "确认选择"按钮
- "稍后再说"按钮

**交互约束**:
- 选择后确认执行
- 取消可稍后重新选择

### 技能升级提示

**升级通知**:
- 技能升级时显示浮动提示："[技能名]升至Lv.X！"

## Open Questions

### OQ.1 被动技能的处理方式

**问题**: 被动技能是否应该在战斗外也生效？
- 当前设计：被动技能常驻生效，包括战斗外
- 备选方案：被动技能仅在战斗中和特定场景生效

**待确认**: 被动技能在战斗外生效增加养成感，但实现复杂度增加。建议MVP阶段被动技能常驻生效。

### OQ.2 职业技能的解锁时机

**问题**: 职业技能是在转职时立即解锁，还是需要额外学习？
- 当前设计：转职后自动解锁
- 备选方案：转职后还需消耗资源学习

**待确认**: 自动解锁简化转职流程，学习额外消耗增加资源决策点。建议先用自动解锁。

### OQ.3 技能冷却的显示方式

**问题**: 技能冷却应该如何显示？
- 当前设计：显示剩余回合数
- 备选方案：显示具体回合数+图标填充动画

**待确认**: 回合数显示简单直接，图标动画更直观但实现成本更高。建议MVP用回合数。
