# 多周目系统

> **Status**: Designed (pending review)
> **Author**: binlee1990 + agents
> **Last Updated**: 2026-04-22
> **Implements Pillar**: 系统互锁（成就点数跨周目积累，难度倍率消耗点数，解锁内容驱动重玩）

## Overview

多周目系统管理游戏周目间的数据传承。每周目结束（通关或玩家主动开始新周目）时结算成就点数，玩家在新周目启动时用累积的成就点数兑换功能开关（自由属性分配、特殊职业、难度倍率等）。每个新周目是"定制化的新体验"——保留核心养成成果，解锁新玩法维度。

## Player Fantasy

- **传承的满足感**：一周目的努力不会白费——成就点数、解锁的职业、了解的故事都带入新周目
- **定制化挑战**：二周目不是我再做一遍，而是"这次我要用不同的方式玩"——更高的难度、更自由的属性分配、新的职业
- **完整的故事**：三路线需要三个周目才能全部体验，每个周目发现新内容

## Detailed Design

### Core Rules

**规则 1 — 成就点数系统**

| 成就类别 | 示例 | 点数 |
|----------|------|------|
| 战斗成就 | 无伤通关、全灭敌人、3 回合内胜利 | 10~50 |
| 叙事成就 | 完成路线、解锁隐藏剧情、达成 S 级羁绊 | 20~80 |
| 收集成就 | 收集全部装备、完成全部支线、发现全部隐藏区域 | 15~60 |
| 挑战成就 | 4× 难度通关、全角色满级、无角色退场 | 50~100 |
| 隐藏成就 | 特殊条件触发 | 30~100 |

成就点数跨周目永久累积，不随新周目重置。

**规则 2 — 新周目开关**

| 开关类别 | 示例 | 成就点消耗 | 说明 |
|----------|------|-----------|------|
| 属性成长 | 自由分配开关 | 200 | 开启后属性按潜质增长但玩家自选分配方向 |
| 职业解锁 | 特殊职业可用 | 2000~3000 | 每个特殊职业独立解锁（龙骑士/暗夜使者=2000，帝王=3000） |
| 难度倍率 | 2×/4×/8×/16× | 100~1000 | 见难度系统 GDD |
| 内容解锁 | 隐藏副本/隐藏角色 | 100~200 | 特殊关卡和角色 |
| 便利功能 | 快速战斗、跳过对话 | 50~100 | 减少重复体验的摩擦 |

**规则 3 — 周目间数据传承**

| 保留 | 重置 |
|------|------|
| 成就点数（永久累积） | 队伍等级和经验 |
| 已解锁的特殊职业 | 装备和背包 |
| 已完成的路线记录（用于叙事参考） | 羁绊等级 |
| 成就达成记录 | 基地升级状态 |
| 游戏设置 | 章节进度 |
| 难度倍率选择 | 信念值（重新开始累积） |

**规则 4 — 隐藏内容解锁条件**

部分内容需要特定组合才能解锁：

| 内容 | 条件 | 说明 |
|------|------|------|
| 隐藏 Boss | 二周目 + 特定信念组合 | 需要故意培养非主导信念 |
| 隐藏结局 | 三条路线全部通关 | 第四个结局在全部完成后解锁 |
| 特殊职业"策士" | 智力≥80 + 完成全部支线 | 高难度解锁条件 |
| 隐藏角色 | 特定路线 + 特定羁绊 S 级 | 路线和羁绊双重条件 |

### States and Transitions

```
Active_Playthrough ──[通关/手动选择]──> NG+ Setup
                                          ↓
                                    Select Switches (消耗成就点数)
                                          ↓
                                    New Playthrough (保留传承数据)
```

### Interactions with Other Systems

| 系统 | 接口 | 方向 | 说明 |
|------|------|------|------|
| 叙事系统 | 路线完成记录 | 叙事→多周目 | 记录已完成的路线 |
| 存档系统 | 成就点数持久化 | 多周目→存档 | 成就点数跨周目保存 |
| 难度系统 | 倍率选择 | 多周目→难度 | 新周目选择难度倍率 |
| 职业系统 | 特殊职业解锁 | 多周目→职业 | 成就点数兑换职业 |
| 属性系统 | 自由分配开关 | 多周目→属性 | 解锁自由属性分配 |
| 基地系统 | 基地重置 | 多周目→基地 | 新周目基地从 Lv1 开始 |

## Formulas

**F1 — 单周目预期成就点数**

```
expected_ng_points = sum(all_achievements_in_route)
```

| 周目 | 预期点数 | 主要来源 |
|------|---------|---------|
| 一周目 | ~400 | 首次体验成就 + 路线完成 |
| 二周目 | ~300 | 战斗/收集成就（部分重复成就不可再次获取） |
| 三周目 | ~250 | 剩余成就 + 隐藏成就 |

**F2 — 解锁消费平衡**

```
total_unlock_cost = 200 (自由分配) + 300 (特殊职业) + 300 (难度) + 200 (内容) = 1000
```

设计目标：完成 3 个周目后累积约 950 点，刚好够解锁大部分内容，但不能全部解锁——"选择即放弃"原则贯穿始终。

## Edge Cases

- **If 成就点数不足以选择任何开关**：可以 0 消耗开始新周目（1× 难度，无额外开关），确保不会卡住。
- **If 玩家在周目中途主动开始新周目**：未完成的成就不计入，已获得的成就点保留。当前周目进度丢失。
- **If 重复成就（同一成就在一周目和二周目都满足条件）**：不重复计算点数。每个成就只能获得一次。
- **If 成就点数溢出**（累积超过解锁所有内容的总成本）：多余的点数无消耗对象，显示为"荣誉点数"（纯展示用）。

## Dependencies

### 上游
- **战斗结算**：战斗成就的触发
- **角色管理**：角色相关成就

### 下游
- **难度系统**：成就点数兑换难度倍率
- **职业系统**：特殊职业解锁
- **属性系统**：自由分配开关
- **存档系统**：成就点数跨周目持久化

## Tuning Knobs

| Knob ID | 名称 | 默认值 | 安全范围 | 极端行为 |
|---------|------|--------|----------|----------|
| `TK-NG-01` | 自由分配消耗 | 200 | 100~400 | 过低=太容易；过高=不值得 |
| `TK-NG-02` | 单周目预期点数 | ~400 | 200~600 | 过少=买不起东西；过多=无选择压力 |

## Visual/Audio Requirements

- 周目结算画面：滚动显示成就列表和点数
- 新周目启动：开关选择界面 + 成就点数余额动画
- 隐藏内容解锁时：特殊视觉/音效提示

## UI Requirements

- 成就列表：分类显示 + 已完成/未完成标记
- NG+ 设置界面：开关选择 + 点数余额 + 消耗预览
- 周目统计：已玩周目数、已完成路线、总成就点数

## Acceptance Criteria

**GIVEN** 玩家完成一周目获得 400 成就点数，**WHEN** 启动二周目，**THEN** 可选择消耗点数兑换开关（自由分配 200 + 特殊职业 150 = 350），剩余 50 点。

**GIVEN** 成就"无伤通关"已在一周目获得，**WHEN** 二周目再次满足条件，**THEN** 不重复获得成就点数。

**GIVEN** 保留数据包含成就点数和已解锁职业，**WHEN** 新周目开始，**THEN** 成就点数传承，已解锁特殊职业可用，队伍等级/装备/羁绊重置。

**GIVEN** 三条路线全部通关，**WHEN** 检查隐藏结局条件，**THEN** 隐藏结局解锁。

## Open Questions

- **OQ-1**：是否允许"跳过周目"——直接从多周目菜单购买内容而非实际通关？
- **OQ-2**：成就点数溢出后的"荣誉点数"是否有展示用途（排行榜/成就徽章）？
- **OQ-3**：多周目是否允许继承部分装备（如一周目的套装）？当前设计：不继承。
