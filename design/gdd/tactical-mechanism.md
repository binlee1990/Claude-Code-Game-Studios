# 战术机制

> **Status**: In Design
> **Author**: user + agents
> **Last Updated**: 2026-04-16
> **Implements Pillar**: 系统互锁——战术机制让位置/编队/克制产生涌现式互动

## Overview

战术机制是战斗的空间与策略基础——它决定了"在哪里战斗"和"用什么打"能产生优势。

游戏采用**三层叠加的战术系统**：

- **克制三角**（兵种/武器类型间的基础克制关系）：剑克斧、斧克枪、枪克剑——让编队有意义
- **元素交互**（地形中的元素可相互作用）：火烧油产生范围伤害、水导电产生连锁伤害、风助火使火焰扩散——让地形有意义
- **高低差**（位置高度影响战斗数值）：高处攻击低处获得射程+和命中+的加成——让位置有意义

三层各自简洁，但叠加后产生涌现式策略深度。玩家需要同时考虑"我用什么打"（武器类型）、"我在哪里打"（地形元素+高度）、"我派谁打"（兵种配合），而不是单一最优解。这种多维度决策是SRPG区别于其他战棋游戏的核心乐趣。

战场采用**2.5D斜45度方形网格**（15×15～25×25格），支持视角旋转和高低差地形。

## Player Fantasy

玩家在战术系统中的核心感受是**"我的每一个战术决策都在被验证"**——不是简单地"选最强的单位"，而是思考"用什么打、在哪里打、怎么配合"。

**克制三角**带来的是**编队的意义感**。当玩家把剑兵排在前面准备克制枪兵，把弓手放在后面准备狙击被剑兵克制的斧兵时——这种"我在思考战术"的乐趣是SRPG独有的。不是数值碾压，而是**智力的博弈**。

**元素交互**带来的是**地形的惊喜感**。当玩家发现"这片油渍可以用火点燃，范围伤害能波及三个敌人"时，当玩家故意把敌人引到水边再用水系技能导电时——这种**利用环境创造战术优势**的发现感是玩家最爱的"啊哈时刻"。

**高低差**带来的是**位置的价值感**。当玩家发现把弓手放在高处能获得射程加成，当玩家决定"我需要先占领那个高地"时——这种**空间本身成为战略资源**的设计让战场变成了立体的棋盘。

**具体锚点：**
- 用弓手在高地狙击，一箭三连（穿透击杀）→ 高低差+武器克制的组合爽感
- 把敌人引到油渍区，一把火烧死四个 → 元素交互的连锁惊喜
- 故意用枪兵引出敌人的剑兵，再用剑兵反克 → 克制三角的预判博弈感
- "我需要先占领那个高台"成为战斗首要目标 → 位置决策的重量感

**参考游戏**：火纹的兵种克制、三角战略的元素交互、皇骑的高低位系统

## Detailed Design

### Core Rules

**C.1 克制三角**

兵种/武器类型间的基础克制关系（剑>斧>枪>剑）：

| 攻击方武器 | 被克制方 | 克制效果 |
|-----------|---------|---------|
| 剑 | 枪 | 伤害×1.5 |
| 枪 | 斧 | 伤害×1.5 |
| 斧 | 剑 | 伤害×1.5 |
| 弓 | — | 无克制（远程优势） |
| 魔法 | — | 无克制（元素优势） |
| 拳/爪 | — | 无克制（速度优势） |

**克制判定规则：**
- 克制效果：伤害×1.5（50%增伤）
- 被克制方不承受额外惩罚（仅攻击方获得克制加成）
- 同时受属性碾压判定影响（见属性与成长系统）
- 克制与碾压可以叠加

**C.2 元素交互系统**

战场上存在可交互的元素地形，共4种元素（战斗层面的四元素，与世界观五行的关系见注）：

| 元素 | 地形示例 | 可交互行为 | 交互结果 |
|------|---------|-----------|---------|
| 火 | 油渍、干燥草地 | 接触火源 | 燃烧：范围伤害，持续2回合 |
| 水 | 水洼、河流 | 接触电系攻击 | 导电：连锁伤害，跳转到相邻单位 |
| 风 | 风口、风柱 | 火系接触 | 助燃：火焰范围扩大 |
| 土 | 沙地、泥沼 | 接触水系 | 泥沼：移动速度降低50%，持续1回合 |

**元素交互规则：**
- 燃烧范围：初始3×3格，助燃后5×5格
- 导电连锁：最多跳转3个单位，每个跳转伤害递减20%
- 泥沼减速：AGI临时-50%，下回合开始时恢复
- 元素地形在交互后消耗（油渍燃烧后消失，水洼蒸发后消失）

**注 — 四元素 vs 五行世界观**: 战斗系统使用火/水/风/土四元素进行战术交互。世界观的五行体系（火/水/风/土/金）中，金元素仅作为叙事/文化设定存在，不参与战斗元素克制。这保持了五行文化的完整性，同时避免在战术层面引入不对称的第五元素。

**C.3 高低差系统**

战场高度分为3个层级：低地(0)、平地(1)、高地(2)

| 攻击方向 | 射程修正 | 命中修正 |
|---------|---------|---------|
| 高→低 | +1格 | +10% |
| 低→高 | -1格 | -10% |
| 平→平 | ±0 | ±0 |

**高低差判定规则：**
- 高度差每1级，射程+1/-1格
- 高度差每1级，命中+10%/-10%
- 远程攻击（弓/魔法）同样受高低差影响
- 高度差不影响伤害数值，只影响射程和命中

**C.4 地形类型**

| 地形类型 | 效果 | 备注 |
|---------|------|------|
| 普通 | 无特殊效果 | — |
| 草地 | 可点燃 | 接触火元素后变为燃烧地形 |
| 水洼 | 可蒸发/导电 | 接触火元素后蒸发，接触电元素后导电 |
| 沙地 | 移动成本+1 | 进入沙地消耗2点移动力 |
| 泥沼 | AGI-50% | 接触水元素后变为泥沼 |
| 高地 | 高度=2 | 提供高位优势 |
| 障碍物 | 阻挡移动和远程 | 不可通行 |

**C.5 战场尺寸**

- 标准尺寸：15×15格
- 大型尺寸：20×20格
- 超大型尺寸：25×25格
- 每格为方形，1格=1个单位的标准占据面积

**距离度量**: 战场上所有距离计算使用**曼哈顿距离**（|Δx| + |Δy|）。包括但不限于：视野范围、羁绊生效范围、技能射程、移动消耗。

## Formulas

### D.1 克制伤害倍率

```
final_damage_multiplier = base_multiplier × crush_multiplier
// base_multiplier = 1.0 (无克制)
// base_multiplier = 1.5 (攻击方克制被攻击方)
// crush_multiplier = 1.0 (无碾压) 或 1.5 (碾压触发)
// 最终伤害 = 基础伤害 × final_damage_multiplier
```

**变量定义:**

| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| base_multiplier | bm | float | {1.0, 1.5} | 克制基础倍率（1.0=无克制，1.5=克制） |
| crush_multiplier | cm | float | {1.0, 1.5} | 碾压倍率（1.0=无碾压，1.5=碾压触发） |
| final_damage_multiplier | fm | float | {1.0, 1.5, 2.25} | 最终伤害倍率 |

**输出范围:** 1.0（无任何加成）到 2.25（克制+碾压同时触发）

**叠乘上限**: 所有伤害倍率来源（克制、碾压、羁绊组合技等）的乘积上限为 **×3.0**。当乘积超过 ×3.0 时，截断为 ×3.0。此上限确保即使在极限叠加场景下，单次攻击不会一击秒杀同等级敌人。

**示例:**
- 剑攻击枪（克制）：`1.5 × 1.0 = 1.5` → 150%伤害
- 剑攻击枪（克制+碾压）：`1.5 × 1.5 = 2.25` → 225%伤害
- 无克制无碾压：`1.0 × 1.0 = 1.0` → 100%伤害

### D.2 元素燃烧伤害

```
burn_damage = base_skill_damage × 0.3
burn_range_initial = 3×3
burn_range_enhanced = 5×5
burn_duration = 2 turns
```

**变量定义:**

| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| base_skill_damage | bd | int | [技能基础伤害] | 触发燃烧的技能基础伤害 |
| burn_damage | b | int | [伤害的30%] | 每回合燃烧伤害 |
| burn_range_initial | br_i | int | 3 | 初始燃烧范围（格） |
| burn_range_enhanced | br_e | int | 5 | 风助燃后燃烧范围（格） |
| burn_duration | dur | int | 2 | 燃烧持续回合数 |

**输出范围:** 燃烧伤害=基础伤害×30%，持续2回合

**示例:**
- 火球术基础伤害100，触发燃烧：`burn_damage = 30`/回合，持续2回合

---

### D.3 导电连锁伤害

```
chain_damage_1 = base_damage × 1.0
chain_damage_2 = chain_damage_1 × 0.8
chain_damage_3 = chain_damage_2 × 0.8
max_chain_targets = 3
```

**变量定义:**

| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| base_damage | bd | int | [技能基础伤害] | 触发导电的技能基础伤害 |
| chain_damage_1 | c1 | int | 100% | 第1跳转目标受到伤害 |
| chain_damage_2 | c2 | int | 80% | 第2跳转目标受到伤害 |
| chain_damage_3 | c3 | int | 64% | 第3跳转目标受到伤害 |
| max_chain_targets | mct | int | 3 | 最大跳转目标数 |

**输出范围:** 连锁伤害最多3个目标，伤害逐级递减20%

**示例:**
- 雷电术基础伤害80，触发导电：第1目标80伤害，第2目标64伤害，第3目标51伤害

---

### D.4 高低差射程修正

```
effective_range = base_range + height_diff
height_diff = attacker_height - defender_height
```

**变量定义:**

| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| base_range | br | int | [基础射程] | 技能/武器的基础射程 |
| attacker_height | ah | int | {0, 1, 2} | 攻击方所处高度 |
| defender_height | dh | int | {0, 1, 2} | 防守方所处高度 |
| height_diff | hd | int | {-2, -1, 0, 1, 2} | 高度差 |
| effective_range | er | int | [修正后射程] | 修正后的实际射程 |

**输出范围:** 射程修正范围 [-2, +2] 格

**示例:**
- 基础射程3格的弓，高处攻击低处：`3 + (2-0) = 5` 格
- 基础射程3格的弓，低处攻击高处：`3 + (0-2) = 1` 格

---

### D.5 高低差命中修正

```
hit_modifier = height_diff × 10%
height_diff = attacker_height - defender_height
```

**变量定义:**

| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| height_diff | hd | int | {-2, -1, 0, 1, 2} | 高度差 |
| hit_modifier | hm | float | {-20%, -10%, 0%, +10%, +20%} | 命中修正 |

**输出范围:** 命中修正 [-20%, +20%]

**示例:**
- 高处攻击低处：`hit_modifier = (2-0) × 10% = +20%`
- 低处攻击高处：`hit_modifier = (0-2) × 10% = -20%`

## Edge Cases

### E.1 克制与碾压同时触发

**场景**: 攻击方同时满足克制条件和属性碾压条件

**处理**:
- 克制倍率（×1.5）和碾压倍率（×1.5）**叠加计算**
- `final_damage = base_damage × 1.5 × 1.5 = base_damage × 2.25`
- UI显示双重特效（克制光效+碾压光效）

### E.2 元素交互与地形消失

**场景**: 油渍燃烧后，水洼蒸发后

**处理**:
- 燃烧/蒸发后，原有元素地形变为普通地形
- 效果持续回合结束后地形恢复为普通地形（而非恢复原元素）
- 地形状态改变不触发二次交互

### E.3 导电链路过载

**场景**: 导电跳转时，周围没有其他单位

**处理**:
- 导电中断，不产生连锁
- 单体目标受到100%伤害（等同于不触发导电）
- 不消耗额外技能效果

### E.4 高低差为0时的射程

**场景**: 平地攻击平地，或高度差为0

**处理**:
- `effective_range = base_range + 0 = base_range`
- 命中修正为0%
- 视为普通攻击，无任何修正

### E.5 跨越障碍物的高差攻击

**场景**: 高处攻击低处，但中间有障碍物

**处理**:
- 障碍物阻挡，攻击无效
- 高低差修正不补偿障碍物阻挡
- 玩家需要绕开障碍物或占据更高位置

### E.6 泥沼效果与AGI的关系

**场景**: 低AGI角色进入泥沼 vs 高AGI角色进入泥沼

**处理**:
- 泥沼固定使AGI临时-50%，与角色基础AGI无关
- 高AGI角色进入泥沼后，AGI仍然降低50%
- 回合结束时（或其他触发条件）恢复正常AGI

### E.7 燃烧伤害与防御

**场景**: 燃烧伤害是否受防御/抗性影响

**处理**:
- 燃烧伤害为基础伤害的30%，受目标防御力影响
- 计算方式：`burn_damage = floor(base × 0.3 × (100 / (100 + target_defense)))`
- 燃烧不属于技能伤害，属于持续伤害

### E.8 连锁伤害与克制叠加

**场景**: 雷电术触发导电链，同时攻击方克制被攻击方

**处理**:
- 连锁伤害的每一跳都独立判定克制
- 如果跳转目标是枪兵且攻击方是剑，则该跳享受克制加成
- 连锁伤害的递减（100%→80%→64%）先计算，再叠加克制

## Dependencies

### Upstream Dependencies

| System | Dependency Type | Interface Contract |
|--------|---------------|-------------------|
| **属性与成长系统** | 硬依赖 | 本系统读取属性值用于高低差命中计算；属性碾压（公式D.4）与本系统克制叠加 |
| **战斗系统** | 硬依赖 | 本系统的克制倍率、元素交互、高低差修正输入到战斗系统的伤害计算 |

### Downstream Dependents

| System | Dependency Type | Interface Contract |
|--------|---------------|-------------------|
| **战斗系统** | 数据消费者 | 本系统提供克制倍率（×1.5）、元素交互结果（燃烧/导电等）、高低差修正（射程/命中） |
| **AI系统** | 数据消费者 | AI使用本系统的地形信息（高低差、元素地形）进行决策 |
| **回合制模式** | 无直接依赖 | 回合制提供战场，但不直接影响本系统 |
| **关卡设计** | 数据消费者 | 关卡设计需要遵循本系统的地形类型和战场尺寸规范 |

### Dependency Details

**属性与成长系统 → 战术系统**:
- 属性与成长系统提供属性值，战术系统读取用于判定碾压条件
- 公式D.1中的`crush_multiplier`依赖属性与成长系统的公式D.4（属性碾压公式）

**战术系统 → 战斗系统**:
- 战斗系统调用 `get_triangle_modifier(attacker_weapon, defender_weapon)` 获取克制倍率
- 调用 `get_height_modifier(attacker_height, defender_height)` 获取高低差修正
- 调用 `apply_element_interaction(element_type, position)` 触发元素交互效果
- 调用 `get_burn_damage(base_damage)` 获取燃烧伤害

**战术系统 → AI系统**:
- AI系统调用 `get_terrain_info(position)` 获取地形类型、高度、元素状态
- AI决策时考虑高低差优势（优先占据高位）和元素地形利用（引导敌人到油渍区）

|------|---------|-----------|-----------------|-------|
| **克制伤害倍率** | ×1.5 | ×1.25~×2.0 | 克制关系强度 | 提高使克制更重要，降低使数值碾压更容易 |

### Suggested First Tuning Session

## Visual/Audio Requirements

### Visual Requirements

**战场地形显示**:
- 不同地形类型有明确的视觉区分（颜色、纹理）
- 高地有明显的抬升视觉效果（阴影、光照差异）
- 元素地形（油渍、水洼）有动态元素特效
- 燃烧区域有火焰粒子效果

**克制/元素交互特效**:
- 克制命中时：攻击方武器发出对应颜色光芒
- 碾压触发时：额外的冲击波特效
- 元素交互触发时：
  - 燃烧：橙红色火焰扩散特效
  - 导电：蓝色电弧连锁特效
  - 助燃：火焰范围扩大动画
  - 泥沼：地面变为深色沼泽

**战斗UI地形提示**:
- 移动范围显示时，泥沼/沙地等减速地形用不同颜色标注
- 攻击范围显示时，高地/低地的射程修正数值浮显

**元素交互音效**:
- 燃烧触发："轰"火焰爆发音效
- 导电连锁："滋滋"电弧跳跃音效
- 助燃扩散：火焰扩散音效

- 克制命中：武器碰撞+额外"斩杀"音效

## Open Questions

### OQ.1 克制三角是否需要扩展

**问题**: 当前克制三角仅覆盖剑/枪/斧三类，是否需要扩展更多武器类型？
- 当前设计：仅剑>斧>枪>剑
- 备选方案：加入弓/魔法/拳套的克制关系

**待确认**: 扩展克制关系会增加策略深度但也增加复杂度。建议MVP阶段仅保留剑/枪/斧三角，弓/魔法保持独立无克制地位。

### OQ.2 元素地形的生成规则

**问题**: 战场上的元素地形是预设的还是随机生成？
- 当前设计：部分预设（如剧情战斗）、部分随机（如某些关卡）
- 备选方案：完全预设

**待确认**: 随机元素增加重复可玩性，但预设元素让玩家可以规划战术。建议采用混合模式——核心关卡预设，支线关卡随机。

### OQ.3 高地数量的关卡设计规范

**问题**: 每个关卡应该有多少比例的高地？
- 当前设计：未定义
- 备选方案：10-20%格为高地

**待确认**: 高地比例影响战斗节奏。建议先定义一个初始比例，在测试中调整。

### OQ.4 元素交互的伤害是否享受属性加成

**问题**: 燃烧伤害和导电伤害是否受攻击方属性加成？
- 当前设计：燃烧基础伤害×30%，导电无基础伤害
- 备选方案：导电享受智力/力量加成

**待确认**: 这影响元素系技能的设计方向。建议MVP阶段燃烧享受技能加成，导电享受智力加成。

### OQ.5 地形的可破坏性

**问题**: 地形是否可以被破坏（如砍倒树木、填平水洼）？
- 当前设计：地形不可破坏
- 备选方案：某些地形可被特定技能破坏

**待确认**: 地形可破坏增加策略选择但也增加实现复杂度。建议MVP阶段地形不可破坏。
