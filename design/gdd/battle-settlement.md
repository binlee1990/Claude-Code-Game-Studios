# 战斗结算

> **Status**: In Design
> **Author**: user + agents
> **Last Updated**: 2026-04-16
> **Implements Pillar**: 系统互锁——战斗结算完成奖励闭环

## Overview

战斗结算是战斗结束后资源奖励的处理系统——它决定了"赢了之后得到什么"。

战斗结算提供完整的奖励闭环：
- **经验值**：角色升级所需
- **金币**：基础货币
- **材料**：装备强化和分解的消耗
- **果子**：属性潜质提升的稀缺道具
- **成就点数**：多周目兑换的核心资源

战斗结算根据战斗结果（胜利/失败）、战斗表现（伤害量、击杀数）和敌人难度（Boss/普通）计算奖励，确保资源获取与玩家投入成正比。

## Player Fantasy

玩家在战斗结算中的核心感受是**"我的战斗得到了回报"**——每一次胜利都带来积累，每一份投入都看到回报。

**奖励结算**带来的是**成就确认感**。当战斗胜利，屏幕上的金币哗哗增加，经验值跳动，果子出现在奖励列表时——这种**我的决策和操作产生了价值**的满足感是SRPG最核心的反馈循环。

**战斗表现挂钩**带来的是**高超操作的价值感**。当玩家用技巧以弱胜强，看到奖励列表比预期更多时——这种**我比预期做得更好**的自我认可感是游戏的深层动力。

**具体锚点：**
- 胜利结算画面，金币+100，经验+50，果子×2 → 战斗回报的满足感
- Boss击杀，稀有材料出现 → 挑战成功的惊喜
- 战斗评价（完美/优秀/普通/失败） → 表现认可的成就感

**参考游戏**：主流SRPG的战后奖励系统

## Detailed Design

### Core Rules

**C.1 结算触发条件**

战斗结算在以下条件满足时触发：
- **胜利**：所有敌方单位HP=0
- **失败**：所有己方单位HP=0
- **撤退**：玩家主动撤退（需有撤退手段）

**失败结算**：
- 不获得经验值
- 不获得金币
- 可能获得少量材料（冒险奖励）
- 不触发成就

**C.2 经验值结算**

经验值分配规则：
- 总经验值 = 敌方总经验值
- 经验值平均分配给所有存活的己方单位
- 经验值自动累加到角色经验条，达到上限时触发升级

敌方单位经验值表：

| 敌人类型 | 经验值 |
|---------|-------|
| 普通敌人 | 50 |
| 精英敌人 | 150 |
| 困难敌人 | 300 |
| Boss | 1000 |

**C.3 金币结算**

金币结算公式（由资源经济系统提供）：
- `gold = base_reward + floor(damage × 0.1) + kill_bonus`
- 金币由资源经济系统管理

**C.4 材料结算**

材料掉落：
- 普通敌人：10%掉落装备
- 精英敌人：50%掉落装备
- Boss：100%掉落装备
- 额外材料掉落：由资源经济系统计算

**C.5 战斗评价**

战斗结束后，根据战斗表现给出评价：

| 评价 | 条件 | 奖励加成 |
|------|------|---------|
| 完美 | 无人死亡，0伤通关 | +50%经验 |
| 优秀 | 无人死亡 | +20%经验 |
| 普通 | 有单位死亡 | 正常经验 |
| 失败 | 己方全灭 | 0经验 |

**C.6 职业经验结算**

职业经验计算（由职业系统提供）：
- `class_exp_gained = min(floor(damage × 0.02), 500) + kill_bonus + battle_bonus`
- 由职业系统计算和管理

## Formulas

### D.1 经验值分配

```
经验的单位获得量 = 总经验值 / 存活己方单位数
```

**变量定义:**

| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| 总经验值 | te | int | [敌人经验总和] | 击败所有敌人获得的总经验 |
| 存活己方单位数 | sa | int | [1, 4] | 战斗结束时存活的己方单位数 |
| 每个单位获得量 | epu | int | [分配结果] | 每个存活单位获得的经验 |

**示例:**
- 总经验1000，存活3个单位：`epu = 1000 / 3 = 333`

---

### D.2 战斗评价加成

```
final_exp = base_exp × (1 + evaluation_bonus)
```

**变量定义:**

| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| base_exp | be | int | [基础经验] | 战斗基础奖励经验 |
| evaluation_bonus | eb | float | {0.0, 0.2, 0.5} | 评价加成（普通=0, 优秀=0.2, 完美=0.5） |
| final_exp | fe | int | [最终经验] | 应用评价后的最终经验 |

**示例:**
- 基础经验500，完美评价：`fe = 500 × (1 + 0.5) = 750`

## Edge Cases

### E.1 战斗失败结算

**场景**: 所有己方单位全灭

**处理**:
- 不获得经验值
- 不获得金币
- 获得少量冒险奖励（少量材料）
- 不触发成就
- 可以选择"消耗资源再挑战"或"回去练级"

### E.2 只有一个单位存活

**场景**: 战斗结束时只有一个己方单位存活

**处理**:
- 经验值全部给该单位
- 金币正常分配（单人多拿）
- 战斗评价按实际情况判定

### E.3 Boss战胜利但有单位死亡

**场景**: Boss战胜利，但有己方单位死亡

**处理**:
- 仍然获得Boss奖励
- 但战斗评价为"普通"（有死亡）
- 经验加成+20%，而非完美+50%

### E.4 经验溢出

**场景**: 获得经验后，角色经验超过升级所需

**处理**:
- 溢出经验转入下一级
- 可能触发连续升级
- 连续升级时播放多次升级特效

### E.5 撤退结算

**场景**: 玩家主动撤退

**处理**:
- 视为战斗失败
- 不获得经验
- 不获得金币
- 可获得少量冒险奖励

## Dependencies

### Upstream Dependencies

| System | Dependency Type | Interface Contract |
|--------|---------------|-------------------|
| **回合制模式** | 硬依赖 | 战斗结算在回合制模式战斗结束后触发；战斗结果（胜败）传入结算系统 |
| **资源经济** | 硬依赖 | 结算系统调用资源经济计算金币和材料掉落；资源经济系统管理资源数据 |
| **职业系统** | 硬依赖 | 结算系统调用职业系统计算职业经验；职业系统提供职业经验数据 |

### Downstream Dependents

| System | Dependency Type | Interface Contract |
|--------|---------------|-------------------|
| **UI系统** | 数据消费者 | 结算系统输出奖励数据给UI系统显示；UI显示结算面板 |
| **羁绊系统** | 数据消费者 | 羁绊系统需要知道战斗结果来更新好感度 |

## Open Questions

### OQ.1 失败后是否需要惩罚机制

**问题**: 战斗失败是否应该有额外惩罚？
- 当前设计：失败不获得经验，但有少量冒险奖励
- 备选方案：失败扣除部分资源/降低士气

**待确认**: 惩罚机制增加紧张感但也可能造成挫败感。建议MVP用简单机制。
