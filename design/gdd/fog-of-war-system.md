# 战争迷雾系统

> **Status**: Designed (pending review)
> **Author**: binlee1990 + agents
> **Last Updated**: 2026-04-22
> **Implements Pillar**: 系统互锁（迷雾与侦察兵职业、战术地形互锁）

## Overview

战争迷雾系统在特定关卡类型中限制玩家视野，隐藏未探索区域的敌人、地形和物品。大部分关卡信息全开，仅在夜战、伏击、侦察等特殊关卡类型中启用。迷雾作为"特殊机制"丰富关卡多样性，与侦察兵类职业和探索维度信念值产生互锁。

## Player Fantasy

- **信息不确定的紧张感**：不知道前方有什么，每一步都可能遭遇伏击或发现宝藏
- **侦察的价值**：侦察兵职业在迷雾关卡中价值暴增，"我的职业选择真的有策略意义"
- **揭开迷雾的满足感**：每推进一步，地图揭示更多信息，从黑暗中拼凑出完整战局

## Detailed Design

### Core Rules

**规则 1 — 启用条件**

迷雾仅在以下关卡类型中启用：

| 关卡类型 | 迷雾密度 | 视野范围 | 说明 |
|----------|---------|---------|------|
| 夜战 | 重度 | 3 格 | 全图迷雾，模拟夜间视线受限 |
| 伏击战 | 中度 | 5 格 | 玩家初始位置附近可见，远处迷雾 |
| 侦察任务 | 轻度 | 7 格 | 大范围可见，局部区域迷雾 |
| 普通关卡 | 无 | 全图 | 大部分关卡无迷雾 |

**规则 2 — 视野计算**

每个单位的视野范围由基础视野 + 属性修正 + 职业修正决定：

```
vision_range = base_vision + agility_modifier + class_modifier
```

| 修正来源 | 值 | 说明 |
|----------|-----|------|
| 基础视野 | 3 格 | 所有单位的基础 |
| 敏捷修正 | +0~2 格 | 敏捷≥60 +1, ≥80 +2 |
| 侦察兵职业 | +3 格 | 侦察兵专属加成 |
| 高处修正 | +1 格 | 站在高处增加视野 |
| 火焰/光源 | +2 格 | 靠近火把/光源的区域临时可见 |

**规则 3 — 迷雾揭示规则**

- 单位移动后，以新位置为中心、视野范围为半径的区域变为已探索
- 已探索区域永久可见（不会重新遮蔽）
- 敌人在迷雾中不显示，进入视野后立即显示
- 地形特征（高低差、水域）在迷雾中不可见

**规则 4 — 敌人在迷雾中的行为**

- 敌人 AI 在迷雾中正常执行策略，不受迷雾影响（敌人"知道"地图布局）
- 迷雾中的敌人攻击不可见目标时，命中率 -20%（盲射惩罚）
- 敌人进入玩家视野时触发"发现"动画和音效

### States and Transitions

每个地图格子有三种状态：

```
Unknown → Explored → Visible
                         ↓ (离开视野范围)
                    Explored (地形可见，敌人不显示)
```

### Interactions with Other Systems

| 系统 | 接口 | 方向 | 说明 |
|------|------|------|------|
| 战术机制 | 高低差视野修正 | 战术→迷雾 | 高处增加视野范围 |
| AI 系统 | 敌人视野和盲射惩罚 | 迷雾→AI | 迷雾中敌人命中降低 |
| 职业系统 | 侦察兵视野加成 | 职业→迷雾 | 侦察兵专属视野优势 |
| 属性系统 | 敏捷视野修正 | 属性→迷雾 | 敏捷影响视野范围 |
| 美术风格 | 迷雾视觉效果 | 迷雾→美术 | 迷雾覆盖层的渲染 |

## Formulas

**F1 — 视野范围计算**

```
vision_range = BASE_VISION + floor(agility / 60) + class_bonus + height_bonus
```

| 变量 | 值 | 说明 |
|------|-----|------|
| BASE_VISION | 3 | 基础视野格数 |
| agility / 60 | 0~2 | 敏捷修正 |
| class_bonus | 0 或 3 | 侦察兵 +3 |
| height_bonus | 0 或 1 | 高处 +1 |

**F2 — 盲射惩罚**

```
hit_rate = base_hit_rate × (1.0 - BLIND_FIRE_PENALTY)
```

BLIND_FIRE_PENALTY = 0.20（迷雾中攻击不可见目标命中 -20%）

## Edge Cases

- **If 所有玩家单位都被击败但迷雾未完全揭开**：战斗失败，下次重试迷雾重新覆盖（检查点不保留迷雾状态）。
- **If 侦察兵在迷雾中阵亡**：失去侦察兵的视野加成，剩余单位视野缩小，增加策略压力。
- **If 火焰技能在迷雾中使用**：火焰范围 +2 格临时视野，持续 2 回合后熄灭。

## Dependencies

### 上游
- **AI 系统**：敌人 AI 行为
- **战术机制**：地形和视野修正

### 下游
- **职业系统**：侦察兵视野加成
- **美术风格**：迷雾渲染效果

## Tuning Knobs

| Knob ID | 名称 | 默认值 | 安全范围 | 极端行为 |
|---------|------|--------|----------|----------|
| `TK-FOG-01` | 基础视野 | 3格 | 2~5 | 过小=太受限；过大=迷雾无意义 |
| `TK-FOG-02` | 侦察兵加成 | +3格 | +1~+5 | 过小=职业无特色；过大=侦察兵必选 |
| `TK-FOG-03` | 盲射惩罚 | -20% | -10%~-40% | 过低=迷雾无影响；过高=无法战斗 |

## Visual/Audio Requirements

- 迷雾使用半透明暗色覆盖层，已探索区域稍亮，当前视野区域最亮
- 迷雾边缘使用水墨渐变效果（呼应美术风格）
- 敌人从迷雾中出现时有"浮现"动画
- 迷雾中的脚步声和环境音效加强（听觉补偿视觉缺失）

## UI Requirements

- 迷雾关卡开始时显示特殊提示"此关卡存在战争迷雾"
- 小地图显示已探索/未探索区域的分界
- 侦察兵单位有视野范围指示器

## Acceptance Criteria

**GIVEN** 夜战关卡，**WHEN** 单位移动到新位置，**THEN** 以新位置为中心 3 格（基础视野）内的区域变为已探索，敌人立即显示。

**GIVEN** 侦察兵单位在迷雾关卡中，**WHEN** 计算视野范围，**THEN** 视野 = 3（基础）+ 3（职业）= 6 格。

**GIVEN** 敌人在迷雾中攻击不可见目标，**WHEN** 计算命中率，**THEN** 基础命中率 × 0.80（-20% 盲射惩罚）。

**GIVEN** 迷雾关卡中火焰技能施放，**WHEN** 技能生效，**THEN** 火焰范围内 +2 格临时可见，持续 2 回合。

## Open Questions
