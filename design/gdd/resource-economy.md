# 资源经济系统

> **Status**: In Design
> **Author**: user + agents
> **Last Updated**: 2026-04-16
> **Implements Pillar**: 系统互锁——双层资源驱动核心决策

## Overview

资源经济系统是游戏内所有流通货币和材料的供给-消耗框架，决定了玩家"能做什么"和"不能做什么"。

游戏采用**双层资源架构**：
- **普通资源层**（充裕）：金币、基础材料、经验值、熟练度——保证日常养成不焦虑
- **稀有资源层**（稀缺）：果子（提升属性潜质）、稀有材料（装备附魔/高级强化）、保护符（防止强化损毁）、壁障突破资源（突破属性上限）、成就点数（多周目兑换）——创造核心策略取舍

两层资源消耗渠道分离，避免"一种资源争夺所有系统"。普通资源用于基础强化和日常消耗，稀有资源用于关键成长节点和装备质变。这种分层让玩家在日常循环中保持流畅体验，同时在重要决策点感受到稀缺资源分配的重量。

普通资源的获取与战斗表现挂钩（战斗奖励），稀有资源通过挑战性内容获取（Boss掉落、特殊任务），确保资源获取与玩家投入成正比。

## Player Fantasy

玩家在资源系统中的核心感受是**"我的投入正在积累"**——每一次战斗都在积累资源，每一份资源都是对未来Build的投资。

**普通资源层**带来的是**流畅的日常感**：玩家不会因为"缺钱"而卡关，不会因为"没经验"而无法升级。资源获取是线性的、可预期的、稳定的。玩家感受到"稳定进步"，而不是"焦虑等待"。

**稀有资源层**带来的是**策略性的重量感**：当玩家面对一颗稀有的果子时，决策是沉重的——"给谁？走什么Build路线？"这种"资源有限，选择即放弃"的紧张感是魂系哲学的体现。保护符的稀缺让强化有风险，壁障突破资源的稀有让成长有阶段性质变。

**具体锚点：**
- 战斗结算时看到金币和材料哗哗入账 → 积累感的满足
- 背包里躺着最后一颗珍贵果子，思考给谁 → 决策的重量
- 用保护符保住+10装备时的如释重负 → 风险规避的成就感
- 突破壁障后属性开始快速增长→稀缺投入得到回报的爽感
- 成就点数跨周目累积，看到积累显示"5000点" → 多周目投入的长期价值确认

**参考游戏**：暗黑破坏神的材料管理系统（史诗感）、曹操传的果子系统（稀缺感）

## Detailed Design

### Core Rules

**C.1 资源分类与数据结构**

游戏内资源分为两层五大类：

**普通资源层（充裕）：**

| 资源ID | 名称 | 获取来源 | 主要消耗渠道 | 堆叠上限 |
|--------|------|---------|------------|---------|
| RES_GOLD | 金币 | 战斗奖励、出售装备、任务奖励 | 装备购买、基础强化、基地升级 | 9,999,999 |
| RES_BASIC_MATERIAL | 基础材料 | 战斗掉落、探索、分解白色装备 | 装备基础强化、基地升级 | 9,999 |
| RES_EXP | 经验值 | 战斗 | 角色升级（由升级系统自动消耗） | N/A（不囤积） |
| RES_PROFICIENCY | 熟练度 | 战斗 | 技能升级（由技能系统自动消耗） | N/A（不囤积） |

**稀有资源层（稀缺）：**

| 资源ID | 名称 | 获取来源 | 主要消耗渠道 | 堆叠上限 |
|--------|------|---------|------------|---------|
| RES_FRUIT_* | 果子（9种） | Boss掉落、特殊任务、装备分解 | 提升属性潜质（属性与成长系统） | 99 |
| RES_RARE_MATERIAL | 稀有材料 | 高难关卡、隐藏副本 | 装备附魔/高级强化 | 999 |
| RES_PROTECT_SYMBOL | 保护符 | 极稀有掉落 | 防止装备强化损毁 | 99 |
| RES_BARRIER_RESOURCE | 壁障突破资源 | Boss/特殊挑战 | 属性壁障突破（属性与成长系统） | 99 |
| RES_ACHIEVEMENT | 成就点数 | 成就达成 | 多周目开关兑换（多周目系统） | 无上限 |

**果子分类（9种）：**

| 资源ID | 名称 | 对应属性 |
|--------|------|---------|
| RES_FRUIT_STR | 力量果 | 力量(STR) |
| RES_FRUIT_AGI | 敏捷果 | 敏捷(AGI) |
| RES_FRUIT_CON | 体力果 | 体力(CON) |
| RES_FRUIT_INT | 智力果 | 智力(INT) |
| RES_FRUIT_CHA | 魅力果 | 魅力(CHA) |
| RES_FRUIT_LUK | 幸运果 | 幸运(LUK) |
| RES_FRUIT_WIL | 意志果 | 意志力(WIL) |
| RES_FRUIT_RES | 抵抗果 | 异常抵抗(RES) |
| RES_FRUIT_SOU | 灵魂果 | 灵魂强度(SOU) |

**C.2 资源获取规则**

普通资源获取与战斗表现挂钩：

| 资源类型 | 获取公式/规则 |
|---------|-------------|
| 金币 | `gold = base_reward + floor(damage_dealt × 0.1) + kill_bonus` |
| 基础材料 | `materials = enemy_tier × random(1-3)` |
| 经验值 | 由属性与成长系统的升级机制管理，不在此系统囤积 |
| 熟练度 | 由技能系统的熟练度机制管理，不在此系统囤积 |

稀有资源获取规则：

| 资源类型 | 获取条件 |
|---------|---------|
| 果子 | Boss战必定掉落1-2个；普通战斗5%概率掉落；装备分解稀有概率获得 |
| 稀有材料 | 困难/地狱难度战斗10%掉落；隐藏副本boss 100%掉落1个 |
| 保护符 | 地狱难度战斗2%掉落；特殊成就奖励 |
| 壁障突破资源 | 完成壁障突破挑战后返还50%；Boss战5%额外掉落 |
| 成就点数 | 完成成就后自动获得，点数由成就定义决定 |

**C.3 资源消耗规则**

普通资源消耗：

| 资源类型 | 消耗场景 | 消耗公式 |
|---------|---------|---------|
| 金币 | 装备购买 | 装备标价 |
| 金币 | 装备基础强化 | `cost = base_cost × (current_level + 1)` |
| 金币 | 基地功能区升级 | 由基地系统定义 |
| 基础材料 | 装备基础强化 | `materials = 5 × target_level` |
| 经验值 | 角色升级 | 自动消耗（由属性与成长系统处理） |
| 熟练度 | 技能升级 | 自动消耗（由技能系统处理） |

稀有资源消耗：

| 资源类型 | 消耗场景 | 消耗规则 |
|---------|---------|---------|
| 果子 | 提升属性潜质 | 消耗1个对应属性果子，潜质+1级 |
| 稀有材料 | 装备附魔 | 消耗3个，重置1条随机词缀 |
| 稀有材料 | 装备高级强化(+6以上) | 消耗数量=目标等级×2 |
| 保护符 | 防止强化损毁 | 消耗1个，100%保护装备不损毁 |
| 壁障突破资源 | 属性壁障突破 | 消耗1个触发壁障突破挑战 |

**C.4 资源上限与溢出处理**

- 达到堆叠上限后，超出部分**不再获取**（溢出丢弃）
- UI显示警告：当某资源接近上限（如>90%）时，提示"资源接近上限，建议使用"
- 成就点数**无上限**，跨周目累积

**C.5 资源稀有度感知**

为帮助玩家识别资源价值，UI层需体现资源稀有度：
- 普通资源：白/灰底色图标
- 稀有资源：金/紫底色图标 + 边框发光

## Formulas

### D.1 金币获取公式

```
gold_gained = base_reward + floor(damage_dealt × 0.1) + kill_bonus
```

**变量定义:**

| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| base_reward | b | int | [50, 200] | 关卡基础奖励（取决于关卡难度） |
| damage_dealt | d | int | [0, +∞) | 本次战斗造成的总伤害 |
| kill_bonus | k | int | {0, 20} | 击杀奖励（击杀boss时=20，普通敌人=0） |
| gold_gained | g | int | [50, 2000] | 本次战斗获得的金币 |

**输出范围:** 最小50（0伤害0击杀），最大~2000（高伤害+击杀boss）

**示例:**
- 普通关卡基础50 + 300伤害 + 0击杀 = `50 + floor(300×0.1) + 0 = 80`
- Boss关卡基础100 + 800伤害 + 击杀boss = `100 + floor(800×0.1) + 20 = 200`

---

### D.2 基础材料获取公式

```
materials_gained = enemy_tier × random_in_range(1, 3)
```

**变量定义:**

| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| enemy_tier | t | int | {1, 2, 3, 4} | 敌人等级（普通=1, 精英=2, 困难=3, Boss=4） |
| random_in_range(1,3) | r | int | {1, 2, 3} | 随机数1-3 |
| materials_gained | m | int | [1, 12] | 本次获得的基础材料数量 |

**输出范围:** 最小1（普通敌人），最大12（Boss掉落3个×4级）

**示例:**
- 击败普通敌人(Tier1)：`1 × 2 = 2` 材料
- 击败困难敌人(Tier3)：`3 × 3 = 9` 材料
- 击败Boss(Tier4)：`4 × 3 = 12` 材料

---

### D.3 果子掉落概率

```
fruit_drop_check = random() < fruit_drop_rate
fruit_quantity = random_in_range(1, 2)
```

**变量定义:**

| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| fruit_drop_rate | p | float | [0.0, 1.0] | 果子掉落概率（普通战斗=0.05, Boss战=1.0） |
| random() | r | float | [0.0, 1.0) | 随机数[0,1) |
| fruit_quantity | q | int | {1, 2} | 掉落数量 |

**果子类选取规则:**
- 优先掉落与本次战斗中表现最好的属性对应的果子
- 若无法判定，随机选择

**输出范围:** 0个（不掉落）或1-2个

**示例:**
- 普通战斗：`random()=0.03 < 0.05 = TRUE`，掉落1个果子
- Boss战：`random()=0.50 < 1.0 = TRUE`，掉落2个果子

---

### D.4 强化成本公式

**安全区（+1~+5）线性公式：**

```
enhancement_cost_gold = base_cost × (current_enhance_level + 1)
enhancement_cost_materials = 5 × target_enhance_level
```

**变量定义:**

| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| base_cost | c | int | 100 | 强化基础金币成本 |
| current_enhance_level | cl | int | [0, 4] | 当前强化等级（安全区） |
| target_enhance_level | tl | int | [1, 5] | 目标强化等级（安全区） |
| enhancement_cost_gold | cg | int | [100, 500] | 强化所需金币 |
| enhancement_cost_materials | cm | int | [5, 25] | 强化所需基础材料 |

**输出范围:** 安全区金币[100, 500]，材料[5, 25]

**安全区规则:** +1~+5必定成功，不消耗保护符

**风险区规则:** +6以上成本和成功率由装备系统 C.4 表格定义，本公式不适用

**示例:**
- +1强化：金币 `100 × (0+1) = 100`，材料 `5 × 1 = 5`
- +5强化：金币 `100 × (4+1) = 500`，材料 `5 × 5 = 25`
- +6强化：见装备系统 C.4 表格（金币1000，材料30）

## Edge Cases

### E.1 资源达到堆叠上限

**场景**: 玩家持有某资源达到堆叠上限后，再次获取该资源

**处理**:
- 超出部分直接丢弃，不进入背包
- UI显示飘字提示："[资源名]已达上限（+X溢出）"
- 战斗结算界面显示实际获得量与最大堆叠量的对比

### E.2 强化失败且无保护符

**场景**: 玩家在+6以上强化失败，且背包中没有保护符

**处理**:
- 装备降级至+5（或更低，视具体失败阶段）
- 部分强化材料返还（返还50%的材料消耗）
- 返还少量金币（不返还）
- 记录本次强化失败日志

### E.3 强化失败且有保护符

**场景**: 玩家在+6以上强化失败，但背包中有保护符

**处理**:
- 保护符自动消耗（减1）
- 装备保持当前强化等级不变（不降级）
- 材料不返还
- UI显示"保护符生效，装备安全"

### E.4 果子使用在已达S级潜质

**场景**: 玩家对潜质已达S级的属性使用对应果子

**处理**:
- 果子不消耗
- UI显示提示："[属性]潜质已达最高等级，无法再使用[果子名]"
- 果子保留在背包

### E.5 果子使用在已达壁障

**场景**: 玩家对已达当前壁障上限的属性使用果子

**处理**:
- 不可使用果子（与属性与成长系统一致：壁障上限属性禁用果子）
- 果子不消耗，静默拒绝
- 提示玩家："属性已达壁障上限，需先突破壁障才能使用果子"

### E.6 壁障突破资源不足

**场景**: 玩家属性达到壁障阈值，但没有壁障突破资源

**处理**:
- UI显示"壁障突破资源不足"
- 不触发突破挑战
- 属性值继续增长到壁障上限后静默停止（不溢出）
- 提示玩家通过Boss战等渠道获取突破资源

### E.7 保护符耗尽时的强化决策

**场景**: 玩家试图对高风险装备强化，但保护符为0

**处理**:
- UI强化界面显示警告："保护符为0，强化风险极高"
- 玩家仍可选择强化（但不推荐）
- 确认对话框增加风险提示："当前保护符：0，+7强化失败将降至+4，是否继续？"

### E.8 多周目成就点数溢出

**场景**: 成就点数在多个周目后累积到非常大

**处理**:
- 成就点数无上限，允许无限累积
- UI使用千位分隔符显示（如"1,234,567"）
- 多周目开关消耗成就点数，当前余额清晰显示

### E.9 资源不足时的装备购买

**场景**: 玩家金币不足购买装备

**处理**:
- 购买按钮禁用（置灰）
- 显示"金币不足，还需X金"
- 给出可替代的低成本选项（如果有）

### E.10 Boss战掉落与普通战斗掉落同时触发

**场景**: 一场战斗既是Boss战又满足稀有掉落条件

**处理**:
- 金币奖励取Boss关卡的基础奖励（取高）
- 果子掉落按Boss掉落表必定掉落
- 稀有材料掉落单独计算
- 所有掉落结果合并后在结算界面一次性显示

## Dependencies

### Upstream Dependencies

| System | Dependency Type | Interface Contract |
|--------|---------------|-------------------|
| **属性与成长系统** | 硬依赖 | 本系统提供果子（潜质提升道具）和壁障突破资源；果子消耗和突破资源由本系统管理 |
| **战斗系统** | 硬依赖 | 本系统从战斗系统获取战斗数据（伤害量、击杀状态、敌人等级）用于计算资源奖励 |
| **成就系统** | 硬依赖 | 本系统提供成就点数的获取接口；成就点数由成就系统定义和管理 |
| **存档系统** | 硬依赖 | 所有资源数据（持有量、累计获取量）需持久化保存 |

### Downstream Dependents

| System | Dependency Type | Interface Contract |
|--------|---------------|-------------------|
| **装备系统** | 数据消费者 | 本系统提供金币、基础材料、稀有材料、保护符的消耗接口；强化公式由本系统定义 |
| **基地系统** | 数据消费者 | 本系统提供基地升级所需的金币和基础材料消耗接口 |
| **技能系统** | 无直接依赖 | 熟练度由战斗系统直接管理，不经过本系统 |
| **羁绊系统** | 无直接依赖 | 羁绊道具可能需要本系统的资源作为消耗材料（待定） |

### Dependency Details

**战斗系统 → 资源系统**:
- 战斗系统调用 `calculate_rewards(combat_result)` 触发奖励计算
- 输入：战斗结果（胜败）、造成的伤害、击杀信息、敌人等级
- 输出：资源奖励列表（金币、材料、道具）
- 本系统内部计算各资源数量

**资源系统 → 属性与成长系统**:
- 资源系统提供果子消耗接口：`consume_fruit(character_id, fruit_type)`
- 资源系统提供壁障突破资源消耗接口：`consume_barrier_resource()`
- 果子效果由属性与成长系统处理（潜质+1）

**装备系统 → 资源系统**:
- 装备系统调用 `request_enhancement(item_id, target_level)` 查询强化成本
- 调用 `pay_enhancement_cost(gold, materials, use_protect_symbol)` 支付强化消耗
- 强化结果（成功/失败）由装备系统判定，本系统处理材料消耗逻辑

**基地系统 → 资源系统**:
- 基地系统调用 `request_upgrade(zone_id)` 查询升级成本
- 调用 `pay_upgrade_cost(gold, materials)` 支付升级消耗

**成就系统 ← 资源系统**:
- 资源系统不主动推送成就点数
- 成就系统通过监听战斗完成事件，自行判定成就完成并发放点数

## Tuning Knobs

### Core Resource Parameters

| Knob | Default | Safe Range | Affected Gameplay | Notes |
|------|---------|-----------|-----------------|-------|

## Open Questions

### OQ.1 稀有材料是否需要分类

**问题**: 稀有材料是否需要像果子一样按属性分类，还是统一为一种稀有材料？
- 当前设计：统一稀有材料池，不分类
- 备选方案：按装备类型分类（防具材料、武器材料、饰品材料）

**待确认**: 分类会增加策略深度但也增加复杂度。建议MVP阶段使用统一稀有材料，Alpha阶段再扩展分类。

### OQ.2 保护符的获取途径

**问题**: 除了地狱难度2%掉落，是否应有其他稳定获取途径？
- 当前设计：仅地狱难度掉落
- 备选方案：成就奖励、任务奖励、基地产出

**待确认**: 保护符过少会使高阶强化过于焦虑。建议保留稀有掉落为主，添加少量稳定产出（如每周签到奖励1个）。

### OQ.3 资源是否需要时间衰减机制

**问题**: 某些资源（如buff药水、临时增益）是否需要时间衰减？
- 当前设计：不设时间衰减，保持简单
- 备选方案：稀有资源可设置过期时间

**待确认**: 时间衰减增加策略压力但也增加紧迫感。建议MVP不设衰减，简化资源系统复杂度。

### OQ.4 BOSS掉落是否需要随机性

**问题**: Boss战资源掉落是否需要完全确定性？
- 当前设计：部分随机（果子1-2个随机）
- 备选方案：完全确定（如每个Boss固定掉落2个果子）

**待确认**: 完全确定让玩家可规划，但减少惊喜感。建议保留少量随机（1-2个浮动）保持乐趣。

### OQ.5 果子掉落的属性判定

**问题**: 果子掉落时，属性优先级的判定逻辑？
- 当前设计：与本次战斗表现最好的属性对应
- 备选方案：完全随机 / 队伍中最低潜质属性对应

**待确认**: 这影响玩家Farm果子时的策略。建议"最好属性"方案，引导玩家均衡发展。
