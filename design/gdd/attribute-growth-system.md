# 属性与成长系统

> **Status**: Designed
> **Author**: user + agents
> **Last Updated**: 2026-04-16
> **Implements Pillar**: 系统互锁——属性与成长驱动所有下游决策
> **Creative Director Review (CD-GDD-ALIGN)**: SKIPPED — Solo mode

## Overview

属性与成长系统是整个游戏的数据基础层，定义了所有角色的核心数值框架。

**数据结构:**
- **普通属性**（5维）: 力量、敏捷、体力、智力、魅力
- **隐藏属性**（4维）: 幸运、意志力、异常抵抗、灵魂强度
- **属性潜质**: 每个属性有独立潜质值，决定每级成长量，与属性值完全分离

**核心机制:**
- **果子系统**: 消耗果子提升属性潜质，间接影响长期成长
- **壁障突破**: 属性达到特定阈值时需稀缺资源/挑战突破，解锁新成长空间
- **门槛奖励**: 属性达到特定门槛触发额外奖励（如力量50解锁重击）
- **属性碾压**: 属性差距超过阈值时触发碾压效果

**下游接口:**
该系统向所有下游系统提供只读属性数据，是职业系统（门槛）、技能系统（加成计算）、装备系统（需求/强化）、战斗系统（伤害计算）、AI系统（属性引用）的基础数据源。

## Player Fantasy

每一次升级带来的数值增长让人感到"我变强了"——这不是数字的变化，而是玩家感受到自己的角色在变强。

SRPG的核心乐趣之一就是Build优化。当玩家看到属性差距产生碾压效果时，当玩家用果子终于把某个属性潜质拉高时，当角色突破壁障获得新成长空间时——这些都是"我精心规划的养成路线正在生效"的证据。

**具体锚点:**
- 看到角色从"刮痧"到"秒杀"的伤害变化 → 属性碾压带来的爽感
- 用有限果子选择"这周目先把力量拉满" → 策略性取舍的满足感
- 力量刚好达到50解锁重击 → 门槛奖励带来的惊喜感
- 突破壁障时看到新成长空间解锁 → 稀缺资源投入的回报确认

**参考游戏**: 曹操传的"练果"系统、皇骑的角色成长反馈

## Detailed Design

### Core Rules

**C.1 属性结构**

- 每个角色有9个基础属性，分为两类：
  - **普通属性（5维）**: 力量(STR)、敏捷(AGI)、体力(CON)、智力(INT)、魅力(CHA)
  - **隐藏属性（4维）**: 幸运(LUK)、意志力(WIL)、异常抵抗(RES)、灵魂强度(SOU)
- 每个属性有两个独立值：
  - **属性值 (V)**: 当前原始数值，范围 [0, 999]
  - **属性潜质 (P)**: 决定每级成长量，范围 [E, D, C, B, A, S] 对应 [1, 2, 3, 4, 5, 6]
- 每级有效成长 = P（由潜质决定）
- 第一周目确定成长，无RNG

**C.2 升级流程**

- 每次升级，每个属性：`V_new = V_old + P_current`
- 潜质不因升级改变（仅果子改变潜质）
- 角色最大等级由章节系统设定（见回合制模式），最大属性值为999

**C.3 果子系统**

- 果子是消耗性道具，永久提升一个属性的潜质+1等级
- 每种果子按属性分类，效果固定：+1潜质等级
- 果子分类：力量果、敏捷果、体力果、智力果、魅力果、幸运果、意志果、抵抗果、灵魂果
- 潜质等级：E(1) → D(2) → C(3) → B(4) → A(5) → S(6)；最高为S
- 已达S级潜质的属性无法使用果子
- 已达当前阶段壁障上限的属性无法使用果子
- 果子获取来源由资源经济系统定义，本系统仅消费

**C.4 壁障突破系统**

- 壁障是每个属性的阶段性成长上限
- 壁障阈值全属性统一：第1阶段上限50，第2阶段上限100，第3阶段上限150；MVP无第4阶段
- 突破壁障需挑战关卡：
  - **触发条件**：属性V达到阈值且壁障未突破
  - **挑战类型**：独立挑战关卡（非普通战斗）
  - **成功率**：100%（曹操传模式，确定性）
  - **消耗**：进入挑战时扣除资源（资源类型和数量见资源经济）
  - **成功结果**：永久突破壁障，移除该阶段成长上限，属性可继续增长
- 壁障突破后永久生效（不重置）
- 突破前属性无法超过壁障上限，超出部分静默丢弃

**C.5 门槛奖励**

- 当任意普通属性（STR/AGI/CON/INT/CHA）首次达到阈值（50/100/150）时，触发门槛奖励事件
- 门槛奖励由下游系统定义（职业系统/技能系统）：
  - **属性达到50**：触发奖励事件（如解锁新技能节点或触发角色事件）
  - **属性达到100**：更强奖励，解锁高级天赋分支
  - **属性达到150**： capstone奖励，标志该属性轴的精通
- 隐藏属性（LUK/WIL/RES/SOU）在MVP不触发门槛奖励
- 门槛奖励每人每阈值仅触发一次（永久解锁）
- 奖励事件为阻塞式（叙事/UI暂停），确保玩家注意到

**C.6 属性碾压**

- 在战斗结算过程中，伤害计算前检查碾压条件：
  - **碾压条件**：`|攻击方V - 防御方V| > 30`（基于该攻击类型的对应属性）
  - **碾压方向**：攻击方V > 防御方V + 30 → 攻击方碾压；反之防御方碾压攻击方
  - **碾压效果**：伤害倍率 **1.5倍**；防御方该次攻击防御力降低20%
- 仅检杢该攻击类型对应的单个属性轴
- 碾压不叠加，取最高相关属性轴
- 碾压不适用于治疗、增益、非伤害行为

### States and Transitions

**角色属性状态机**

| 状态 | 进入条件 | 退出条件 | 关键属性 |
|------|---------|---------|---------|
| **INIT** | 角色首次创建（新游戏） | 任意属性V > 0 | 所有V = 10（起始基准），所有P = E（潜质基准） |
| **GROWING** | INIT或升级后 | 属性达到当前壁障上限 | 每级V按P增长；上限为壁障阈值 |
| **BARRIER_BLOCKED** | V达到壁障阈值但壁障未突破 | 壁障挑战完成 | V冻结在阈值；超出增长丢弃直至壁障突破 |
| **BARRIER_BROKEN** | 壁障挑战成功完成 | 从不（永久） | 壁障上限移除；V继续增长；下一壁障阈值生效 |
| **POTENTIAL_RAISED** | 角色使用果子 | 从不（永久） | 目标属性P提升+1等级 |
| **POTENTIAL_MAXED** | 某属性P达到S级(6) | 从不 | 该属性无法再使用果子 |
| **THRESHOLD_UNLOCKED** | 任意普通属性V首次达到50/100/150 | 从不（一次性） | 下游奖励事件触发；解锁状态保存 |
| **CAP_VALUE_REACHED** | 任意属性V达到999 | 从不 | V硬性封顶999；超出来源忽略 |
| **CRUSH_VULNERABLE** | 战斗中防御方相关属性V比攻击方低30+ | 战斗结束 | 防御方受到1.5倍伤害+20%防御降低 |
| **CRUSH_IMMUNE** | 战斗中防御方相关属性V比攻击方高30+，或该次攻击已应用碾压 | 战斗结束 | 攻击方不受碾压；该攻击同轴不再检查 |

### Interactions with Other Systems

**职业系统**

- **从本系统获取**：创建时和升级检查点的普通属性值（STR/AGI/CON/INT/CHA）
- **提供给本系统**：职业解锁阈值（如"力量≥50可转职战士"）；职业属性加成（作为V修正量应用于成长计算后）
- **接口**：`get_attribute_value(character_id, attribute_name) → int`；`get_attribute_potential(character_id, attribute_name) → int (1-6)`
- **依赖方向**：职业系统依赖本系统；本系统不依赖职业系统

**技能系统**

- **从本系统获取**：属性值和潜质等级用于技能威力缩放计算
- **提供给本系统**：各属性轴的技能伤害系数；`on_threshold_unlocked(character_id, attribute, threshold)` 事件用于技能解锁触发
- **接口**：`get_skill_attribute_scaling(skill_id) → attribute_name`；战斗结算时调用本系统碾压检查
- **依赖方向**：技能系统依赖本系统；本系统不依赖技能系统

**装备系统**

- **从本系统获取**：角色属性值用于装备需求检查（如"需要力量≥30"）
- **提供给本系统**：装备加成值（作为V修正量应用于基础成长计算后）
- **接口**：`get_equipment_bonus(character_id, attribute_name) → int`；`check_equipment_requirement(character_id, equipment_id) → bool`
- **依赖方向**：装备系统依赖本系统；本系统不依赖装备系统

**战斗系统**

- **从本系统获取**：战斗时所有属性值（V）用于碾压条件判断；所有属性潜质（P）用于战斗奖励后成长计算
- **提供给本系统**：战斗结果数据（获得经验值、是否升级）触发成长应用流程；碾压事件需记录并展示给玩家
- **接口**：战斗系统调用 `evaluate_crush(attacker_id, defender_id, attribute_name) → CrushResult`；战斗结束时调用 `apply_level_up_growth(character_id)`
- **依赖方向**：战斗系统依赖本系统；本系统不依赖战斗系统

**AI系统**

- **从本系统获取**：所有属性值用于AI决策（如目标选择：优先低防御；威胁评估基于角色属性配置）
- **提供给本系统**：无（AI是属性数据的消费者）
- **接口**：`get_character_attributes(character_id) → AttributeSnapshot {V: dict, P: dict}`；无写回
- **依赖方向**：AI系统依赖本系统；本系统不依赖AI系统

## Formulas

### D.1 每级属性成长公式

```
V_new = V_old + P_current
```

**变量定义:**
| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| V_old | v | int | [0, 999] | 升级前属性值 |
| P_current | p | int | [1, 6] | 当前潜质等级（E=1, D=2, C=3, B=4, A=5, S=6） |
| V_new | v' | int | [0, 999] | 升级后属性值，受壁障上限和999硬顶约束 |

**约束条件:**
- 突破壁障前：`V_new ≤ min(V_old + P_current, BARRIER_LIMIT)`
- 突破壁障后：`V_new ≤ min(V_old + P_current, 999)`
- `V_new = 999` 时硬性封顶，超出部分丢弃

**输出范围:** 正常游戏下，每次升级 [1, 6] 点属性（由潜质决定）

**示例:**
- 角色力量V=45，潜质P=C(3)，升级后V=45+3=48
- 角色敏捷V=98，潜质P=A(5)，升级后V=98+5=103（若壁障已突破）
- 角色体力V=149，潜质P=B(4)，升至壁障上限50/100/150前：V=149+4=153
- 角色智力V=997，潜质P=S(6)，升级后V=999（硬顶，丢弃4点）

### D.2 果子使用公式

```
P_new = P_old + 1
条件: P_old < S(6) 且 属性V未达当前阶段壁障上限
```

**变量定义:**
| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| P_old | p₀ | int | [1, 5] | 使用前潜质等级（E~B级） |
| P_new | p₁ | int | [2, 6] | 使用后潜质等级（+1），最高S(6) |
| V_current | v | int | [0, 999] | 使用果子时该属性当前值 |
| BARRIER_LIMIT | b | int | {50, 100, 150} | 当前阶段壁障阈值 |

**限制条件:**
- `P_old = S(6)` 时：不可使用果子，结果为 `P_new = P_old`（无变化）
- `V_current ≥ BARRIER_LIMIT` 且壁障未突破时：不可使用果子
- 潜质映射：E=1, D=2, C=3, B=4, A=5, S=6

**输出范围:** 潜质等级 +1（若可使用时）

**示例:**
- 力量P=D(2)，使用力量果后P=C(3)
- 敏捷P=B(4)，使用敏捷果后P=A(5)
- 体力P=S(6)，使用体力果后仍为S(6)（已达上限，无变化）
- 智力V=100（第二壁障未破），即使P=A(5)，不可使用果子

### D.3 壁障突破判定

```
CAN_BREAK = (V ≥ BARRIER_THRESHOLD) AND (BARRIER_STATE = UNBROKEN)
BREAK_RESULT: BARRIER_STATE = BROKEN，壁障上限移除
```

**变量定义:**
| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| V | v | int | [0, 999] | 属性当前值 |
| BARRIER_THRESHOLD | b | int | {50, 100, 150} | 壁障触发阈值（全属性统一） |
| BARRIER_STATE | s | enum | {UNBROKEN, BROKEN} | 该阶段壁障状态 |
| BARRIER_STAGE | stage | int | {1, 2, 3} | 当前壁障阶段 |

**壁障阈值表:**
| 阶段 | 阈值 | 突破前上限 | 突破后状态 |
|------|------|-----------|-----------|
| Stage 1 | 50 | V ≤ 50 | 突破后移除上限 |
| Stage 2 | 100 | V ≤ 100 | 突破后移除上限 |
| Stage 3 | 150 | V ≤ 150 | 突破后移除上限（MVP） |

**突破流程:**
1. 检查 `CAN_BREAK` 条件
2. 条件满足则进入挑战（曹操传模式：100%成功率）
3. 挑战完成：`BARRIER_STATE = BROKEN`，属性可超过该阈值继续增长
4. 突破后 `V` 超出部分保留（如V=52突破1阶壁障，52保留）

**输出范围:** 布尔判定 + 永久状态变更

**示例:**
- 力量V=50，壁障1未突破 → `CAN_BREAK = TRUE`，可触发突破
- 力量V=51，壁障1已突破，壁障2未突破 → `CAN_BREAK = FALSE`（需V≥100）
- 体力V=150，壁障3未突破 → `CAN_BREAK = TRUE`，可触发第3阶段突破

### D.4 属性碾压公式

```
碾压条件 = (|V_attacker - V_defender| > 30)

碾压方向:
  - 攻击方碾压: V_attacker > V_defender + 30
  - 防御方碾压: V_defender > V_attacker + 30

碾压效果（仅适用于伤害行为）:
  - 伤害倍率 = 1.5
  - 防御降低 = 20% (该次攻击)
```

**变量定义:**
| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| V_attacker | vₐ | int | [0, 999] | 攻击方该轴属性值 |
| V_defender | v_d | int | [0, 999] | 防御方该轴属性值 |
| 属性差 | Δv | int | [-999, 999] | `vₐ - v_d` |
| 碾压判定 | M | bool | {TRUE, FALSE} | `|Δv| > 30` 时为真 |
| 伤害倍率 | mult | float | {1.0, 1.5} | 正常=1.0，碾压=1.5 |
| 防御降低率 | def_mult | float | {1.0, 0.8} | 正常=1.0，碾压时=0.8 |

**碾压伤害计算:**
```
damage_final = base_damage × mult × def_mult
```

**限制条件:**
- 仅适用于伤害行为（治疗、增益、非伤害技能不触发）
- 仅检查该攻击类型对应的单个属性轴
- 碾压不叠加，同一次攻击取最高相关属性轴
- 碾压方向由属性差决定

**输出范围:**
- 伤害倍率：1.0 或 1.5
- 防御倍率：1.0 或 0.8

**示例:**
- 攻击方力量V=80，防御方力量V=45 → Δv=35>30，攻击方碾压 → 伤害×1.5，防御×0.8
- 攻击方敏捷V=60，防御方敏捷V=35 → Δv=25≤30，无碾压，伤害×1.0，防御×1.0
- 攻击方智力V=90，防御方智力V=62 → Δv=28≤30，无碾压
- 攻击方体力V=100，防御方体力V=40 → Δv=60>30，攻击方碾压（体力轴）

### D.5 门槛奖励触发公式

```
TRIGGER = (attribute_type ∈ {STR, AGI, CON, INT, CHA})
          AND (V ≥ THRESHOLD)
          AND (first_reach = TRUE)
```

**变量定义:**
| 变量 | 符号 | 类型 | 范围 | 描述 |
|------|------|------|------|------|
| attribute_type | t | enum | {STR, AGI, CON, INT, CHA, LUK, WIL, RES, SOU} | 属性类型 |
| V | v | int | [0, 999] | 属性当前值 |
| THRESHOLD | θ | int | {50, 100, 150} | 门槛值 |
| first_reach | f | bool | {TRUE, FALSE} | 该角色该阈值是否首次达到 |
| 奖励状态 | reward_state | enum | {LOCKED, UNLOCKED} | 奖励是否已解锁 |

**门槛阈值表:**
| 阈值 | 奖励类型 | 隐藏属性是否触发 |
|------|---------|---------------|
| 50 | 初级奖励（如解锁新技能节点） | 否 |
| 100 | 中级奖励（如解锁高级天赋分支） | 否 |
| 150 | Capstone奖励（如该属性轴精通标志） | 否 |

**触发条件（同时满足）:**
1. 属性为普通属性（STR/AGI/CON/INT/CHA）
2. V ≥ 门槛值（50/100/150）
3. 该角色该阈值首次达到（`first_reach = TRUE`）

**限制条件:**
- 隐藏属性（LUK/WIL/RES/SOU）在MVP不触发门槛奖励
- 每人每阈值仅触发一次（`first_reach` 置为 FALSE）
- 奖励事件为阻塞式（UI暂停，确保玩家注意）

**输出范围:** 布尔判定 + 一次性事件触发

**示例:**
- 角色力量V首次达到50 → `TRIGGER = TRUE`，解锁重击技能节点
- 角色智力V第二次达到50（之前已触发） → `TRIGGER = FALSE`（不重复触发）
- 角色幸运V达到80 → `TRIGGER = FALSE`（隐藏属性不触发）
- 角色敏捷V达到100 → `TRIGGER = TRUE`，解锁高级天赋分支

## Edge Cases

### E.1 零点与最大值边界

- **If V = 0 且角色升级**: V_new = 0 + P ≥ 1，属性从正值开始增长。[理由：起始基准V=10，但任意原因导致V=0后，升级仍按公式正常增长]
- **If V = 999 并再次升级**: V_new = 999（硬顶），超出的增长量静默丢弃，无报错。[理由：公式D.1明确999为硬顶]
- **If P = E (1) 角色长期不获得潜质提升**: 每级仅+1成长，极慢成长曲线。[理由：这是最低潜质的设计意图，非异常]
- **If P = S (6) 并使用果子**: P_new = S (6），无变化，果子效果被静默吸收。[理由：公式D.2条件限制P_old < S(6)]

### E.2 壁障边界

- **If V 刚好等于壁障阈值（50/100/150）且壁障未突破**: CAN_BREAK = TRUE，可立即触发突破挑战；挑战成功后V超出部分保留（如V=50突破1阶，50保留）。[理由：公式D.3条件为V ≥ BARRIER_THRESHOLD]
- **If V = 壁障阈值 且 壁障未突破 时触发升级**: V_new = min(V_old + P, BARRIER_LIMIT) = BARRIER_LIMIT，增长被壁障静默截断，角色停留在阈值但突破挑战可主动触发。[理由：公式D.1约束"突破壁障前 V_new ≤ min(V_old + P, BARRIER_LIMIT)"]
- **If V 刚好突破壁障时的值恰好是下一壁障阈值**: V_old=99，壁障1已破，壁障2未破，升级+P=1后V=100。CAN_BREAK = TRUE（第2阶段壁障触发），突破后V=100保留。[理由：第2阶段阈值100，V=100满足触发条件]
- **If 连续壁障均未突破，V已超出下一阈值**: 不可能发生——V被壁障1截断在50，无法达到壁障2的100。
- **If P很高（如S=6）且在壁障前升级导致瞬间越过阈值**: V_old=48, P=6, 壁障1未破。升级计算V_new=48+6=54>50，应用壁障截断后V=50（阈值），截断丢弃4点。同时触发CAN_BREAK=TRUE（突破挑战可立即触发）。[理由：壁障截断于阈值后，threshold条件仍满足]
- **If 突破壁障后，该次升级的增长量超出部分是否保留**: 突破成功后，V_new = V_old + P（如V_old=50, P=6, V_new=56），超出部分完全保留。[理由：公式D.3"突破后V超出部分保留"]

### E.3 碾压边界

- **If |Δv| = 30（刚好等于30）**: 碾压条件不触发（|Δv| > 30为真需要严格大于），正常伤害计算。[理由：公式D.4条件为严格不等式]
- **If |Δv| = 31（刚好超过30）**: 碾压触发。M = TRUE，伤害×1.5，防御×0.8。[理由：严格大于30的下边界]
- **If 多轴属性同时满足碾压条件（攻击方A比防御方D在STR高35、AGI高40）**: 仅取最高相关属性轴，不叠加。[理由：公式D.4"碾压不叠加，取最高相关属性轴"]
- **If 攻击方碾压防御方，但防御方另一属性轴高于攻击方30+（反向碾压）**: 仅当前攻击轴有效，防御方不可反压攻击方——碾压是单向针对该次攻击的攻防关系。[理由：碾压方向由该次攻击的攻方V和防方V的差决定]
- **If 碾压发生在治疗行为上**: 不触发碾压效果（碾压不适用于治疗、增益、非伤害行为）。[理由：公式D.4限制条件]
- **If V_attacker = V_defender = 50**: |Δv| = 0，无碾压，正常伤害。[理由：|Δv| ≤ 30]
- **If 碾压判定时防御方装备了增加属性的装备**: V_defender取含装备加成后的最终值（装备加成为V修正量在基础成长后应用）。[理由：装备系统接口在战斗碾压检查前合并]
- **If 防御方被碾压后，该次攻击对另一属性轴的伤害是否仍受碾压影响**: 碾压效果（伤害×1.5，防御×0.8）作用于该次攻击的完整结算，该次攻击所有伤害子项均受此倍率影响。[理由：碾压效果作用于攻击结算，不是单轴]

### E.4 果子使用边界

- **If P = S (6) 时使用对应属性果子**: P_new = S (6），无任何变化；系统静默拒绝，不消耗果子（若已消耗则果子被退回）。[理由：公式D.2条件P_old < S(6)，实现应为"检查通过后才消耗果子"]
- **If V已达当前阶段壁障上限（50/100/150）且壁障未突破时使用果子**: 不可使用果子，静默拒绝（若果子已消耗则退回）。[理由：公式D.2条件"属性V未达当前阶段壁障上限"]
- **If P = B (4)，连续使用3个对应属性果**: 第1次P=5(A)，第2次P=6(S)，第3次无变化（已达S）。[理由：每次使用独立检查]
- **If 角色同时满足"未达壁障上限"和"潜质已达S"**: 潜质已达S时即使未达壁障也不可使用果子（果子条件为P<S且V未达壁障上限，两者需同时满足）。[理由：公式D.2两个条件AND关系]
- **If 果子作用在V=0的属性上**: V=0不受果子直接影响（果子提升P），下次升级时P增加导致V增长更大。属性V=0不影响果子使用条件。[理由：果子改变P不改变V]
- **If 果子作用在隐藏属性（LUK/WIL/RES/SOU）上**: 潜质提升正常生效，但隐藏属性不触发门槛奖励（门槛奖励条件要求attribute_type ∈ {STR, AGI, CON, INT, CHA}）。[理由：门槛奖励触发条件公式D.5]

### E.5 门槛奖励边界

- **If V刚好踩线达到门槛值（V=50/100/150首次达到）**: TRIGGER = TRUE，门槛奖励立即触发（阻塞式UI暂停）。[理由：公式D.5条件V ≥ THRESHOLD，first_reach = TRUE]
- **If V从低于门槛直接越过门槛（如V=47，升级后V=50，P=3）**: 奖励触发，触发时机为该次升级结算后。[理由：首次达到即触发，阈值以升级后V为准]
- **If V从48升级到50触发壁障截断（壁障未破，P=6）**: V最终=50（截断），first_reach=TRUE，门槛奖励触发；之后可主动触发壁障突破挑战。[理由：截断后V=50，threshold条件满足]
- **If 同一角色、同一属性、同一阈值多次达到50（例：降维后重升）**: first_reach已=FALSE，不重复触发奖励。[理由：公式D.5"first_reach = FALSE"永久化]
- **If 多属性同时达到各自门槛（力量50，敏捷50，体力50）**: 每个属性独立检查，每个满足条件的属性均触发各自奖励事件，均为first_reach=TRUE。[理由：门槛奖励以属性为单位独立追踪]
- **If 隐藏属性达到50/100/150**: 不触发门槛奖励（TRIGGER=FALSE）。[理由：公式D.5明确hidden attributes不触发]
- **If 门槛奖励触发时，该属性正被壁障锁定在阈值（如V=50壁障未破）**: 奖励正常触发（奖励触发与壁障状态独立），之后角色可选择突破壁障继续增长。[理由：门槛奖励条件与壁障状态独立]

### E.6 溢出与下溢情况

- **If V增长导致超过999（如V=997，P=S=6）**: V_new = 999，超出部分(+4)静默丢弃；P保持不变。[理由：公式D.1 999硬顶约束]
- **If 经验值溢出（极大数值）**: 升级按每次计算，每次应用D.1公式，不会因经验值数值大而跳过验证。[理由：经验值系统和属性成长系统为串行调用]
- **If 连续快速升级导致瞬间穿越多个壁障**: 每次升级单独判定壁障截断——若壁障1未破则停在50；突破后下次升级继续；不可能在单次升级中跳过多个未破壁障。[理由：单次V_new计算只应用当前壁障上限]
- **If 属性值因Bug变为负数**: V ∈ [0, 999]为数据不变式；若出现负值，战斗系统取max(V, 0)处理；溢出告警记录，由数据修复机制处理。[理由：防御性编程]
- **If P因Bug变为0或负数**: P范围应为[1, 6]；若出现异常值，视为P=1（最低）处理，告警记录。[理由：防御性编程]
- **If Δv计算结果溢出（极端情况：两角色属性分别为0和999）**: Δv = -999，在int范围内；|Δv| = 999 > 30，碾压触发，伤害×1.5。[理由：int范围足够大]

## Tuning Knobs

### 可调参数表

| 调参项 | 默认值 | 安全范围 | 影响系统 | 超出范围后果 |
|--------|--------|---------|---------|------------|
| 起始属性值 V | 10 | 1~20 | 早期游戏难度 | 过低导致早期战斗简单/过高导致早期碾压 |
| 起始潜质 P | E (1) | E~C | 早期成长速度 | 过高导致成长太快/过低导致成长体验差 |
| 每级最大成长 | 6 (S潜质) | 1~10 | 满级角色强度 | 过高导致数值膨胀/过低导致养成无意义 |
| 壁障阈值 1 | 50 | 20~100 | 突破节奏 | 过低导致频繁突破/过高导致突破太稀有 |
| 壁障阈值 2 | 100 | 50~200 | 中期突破节奏 | 同上 |
| 壁障阈值 3 | 150 | 100~300 | 后期突破节奏 | 同上 |
| 碾压触发差值 | 30 | 20~50 | 碾压触发频率 | 过低导致碾压常见/过高导致碾压稀有 |
| 碾压伤害倍率 | 1.5x | 1.2~2.0 | 碾压严重程度 | 过低导致碾压无意义/过高导致数值崩溃 |
| 碾压防御降低 | 20% | 10%~30% | 碾压对脆皮影响 | 过低导致碾压不痛/过高导致脆皮无法存活 |
| 属性值上限 | 999 | 500~9999 | 养成天花板 | 过低导致养成终点早/过高导致数值膨胀 |
| 潜质等级数 | 6 (E~S) | 4~10 | 果子价值 | 过多导致果子效果微/过少导致果子太珍贵 |
| 门槛奖励阈值 1 | 50 | 20~100 | 早期奖励节奏 | 过低导致奖励无意义/过高导致奖励难触发 |
| 门槛奖励阈值 2 | 100 | 50~200 | 中期奖励节奏 | 同上 |
| 门槛奖励阈值 3 | 150 | 100~300 | 后期奖励节奏 | 同上 |

### 调参优先级

| 优先级 | 调参项 | 理由 |
|--------|--------|------|
| **P0** | 壁障阈值、属性上限 | 这些影响数值膨胀速度和养成天花板 |
| **P1** | 碾压触发差值/倍率 | 战斗体验的核心杠杆 |
| **P2** | 起始值、潜质等级数 | 影响早期体验和果子价值 |
| **P3** | 门槛奖励阈值 | 影响奖励节奏，优先级最低 |

### 调参交互

- 降低壁障阈值 + 降低属性上限 = 更快的突破节奏但更短的养成天花板
- 提高碾压触发差值 + 提高碾压伤害倍率 = 更少触发但更严重的碾压
- 提高潜质等级数 + 降低果子获取量 = 更长的养成曲线

## Visual/Audio Requirements

属性与成长系统为数据层系统，无直接视觉/音效表现。玩家通过以下方式感知系统效果：
- **属性面板**: 展示角色9属性当前值(V)、潜质等级(P)、下一级成长量
- **成长数字**: 升级时属性数字增长动画（+N 飘字）
- **果子特效**: 使用果子时的属性值变化动画
- **壁障突破特效**: 突破成功时的光效和属性增长动画
- **碾压提示**: 战斗结算时显示碾压触发图标和伤害倍率
